# Quick Start для Claude

Минимальный контекст для начала работы.

## SSOT — единый источник истины

Каждый формальный артефакт имеет стандарт — SSOT-документ.

**Правило:** Если видишь `**SSOT:** [файл](путь)` — **ОБЯЗАТЕЛЬНО** прочитай перед выполнением.

**Исключения (без SSOT):** черновики, `/CLAUDE.md`, `/README.md`, `/CONTRIBUTING.md`.

Подробнее: [ssot.md](./ssot.md)

## Артефакты системы

| Слой | Артефакт | Назначение |
|------|----------|------------|
| Базовый | Инструкция | Описание процесса |
| Базовый | Скрипт | Автоматизация валидации |
| Автоматизация | Скилл | Команда для типовой операции |
| Контекст | Rule | Автозагружаемое правило |
| Автономность | Агент | Помощник для Task tool |
| Структура | README | Описание папки |
| Временные | Черновик | Исследования и планирование |

Подробнее: [artifacts.md](./artifacts.md)

## Скиллы вместо команд

| Действие | ЗАПРЕЩЕНО | ОБЯЗАТЕЛЬНО |
|----------|-----------|-------------|
| Создать папку | `mkdir`, Write README | `/structure-create` |
| Изменить папку | `rm -rf`, Edit README | `/structure-modify` |
| Создать инструкцию | Write вручную | `/instruction-create` |
| Создать скилл | Write вручную | `/skill-create` |
| Изменить стандарт | Edit напрямую | `/migration-create` |

## Версионирование

Документы имеют `standard-version` в frontmatter:

```yaml
---
standard-version: v1.2
---
```

При изменении стандарта — **ОБЯЗАТЕЛЬНА** миграция через `/migration-create` (9 шагов).
