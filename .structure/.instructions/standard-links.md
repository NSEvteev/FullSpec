## 1. Типы ссылок

| Тип | Описание | Пример |
|-----|----------|--------|
| Внутренние | На файлы/папки проекта | `[README](/src/README.md)` |
| Внешние | На внешние ресурсы | `[Docs](https://example.com)` |
| Якорные | На секцию документа | `[Шаги](#шаги)` |

### Внутренние ссылки

```markdown
[Название](путь/к/файлу.md)
[Название](путь/к/папке/)
```

### Внешние ссылки

```markdown
[Название](https://example.com)
```

**Правила:**
- Предпочитать HTTPS
- Без trailing slash
- Ссылаться на конкретную страницу

## 2. Абсолютные vs относительные

| Тип | Формат | Когда использовать |
|-----|--------|-------------------|
| Абсолютные | `/путь/от/корня` | Между разными папками проекта |
| Относительные | `./файл` или `../папка` | Внутри одной папки или рядом |

### Правила выбора

```
Та же папка         → ./файл.md
Соседняя папка      → ../папка/файл.md
Далёкая папка       → /абсолютный/путь.md
```

### Примеры

```markdown
<!-- Относительные -->
[format.md](./format.md)
[README родителя](../README.md)

<!-- Абсолютные -->
[CLAUDE.md](/CLAUDE.md)
[src/README.md](/src/README.md)
```

## 3. Ссылки на папки

### Формат

```markdown
[Название](путь/к/папке/)
```

**Важно:** trailing slash `/` в конце пути.

### С указанием README

```markdown
[Название](путь/к/папке/README.md)
```

### Примеры

```markdown
[skills/](./skills/)
[skills/README.md](./skills/README.md)
[/src/](/src/)
```

## 4. Ссылки на файлы

### Формат

```markdown
[Название](путь/к/файлу.ext)
```

### Расширения

| Тип файла | Расширение |
|-----------|------------|
| Markdown | `.md` |
| YAML | `.yaml`, `.yml` |
| JSON | `.json` |
| Код | `.py`, `.ts`, `.go` |

## 5. Якорные ссылки

### Формат

```markdown
<!-- В том же файле -->
[Название секции](#название-секции)

<!-- В другом файле -->
[Название](путь/к/файлу.md#якорь)
```

### Правила формирования якоря

| Действие | Пример |
|----------|--------|
| Lowercase | `Шаги` → `шаги` |
| Пробелы → дефисы | `Формат описания` → `формат-описания` |
| Удалить спецсимволы | `1. Введение` → `1-введение` |
| Сохранить цифры | `API v2` → `api-v2` |

### Примеры

```markdown
[См. Шаги](#шаги)
[Формат секции](./format.md#формат-секции)
[Скиллы в CLAUDE.md](/CLAUDE.md#скиллы-14)
```

## 6. Ссылки в SSOT структуры

В `/.structure/README.md` используется специальный формат для ссылок на папки.

### Формат

```markdown
### 🔗 [папка/](../папка/README.md)

**Краткое назначение.**

Описание содержимого.
```

### Правила

| Элемент | Правило |
|---------|---------|
| Иконка | `🔗` — папка имеет README |
| Название | `папка/` с trailing slash |
| Ссылка | На `README.md` папки |
| Заголовок | `###` (h3) |

### Исключение

Если папка — текущий документ:

```markdown
### 🔗 .structure/ (этот файл)
```

## 7. Ссылки в frontmatter

```yaml
standard: .instructions/standard-instruction.md
index: .structure/.instructions/README.md
```

**Правила:**
- БЕЗ квадратных скобок
- БЕЗ ведущего `/`
- Относительный путь от корня репозитория

## 8. Валидация ссылок

### Что проверяется

| Проверка | Описание |
|----------|----------|
| Существование цели | Файл/папка по ссылке существует |
| Корректность якоря | Заголовок с таким якорем есть в документе |
| Формат пути | Соответствует правилам (см. выше) |

### Что НЕ проверяется

- Внешние ссылки (по умолчанию)
- Ссылки в code blocks
- Ссылки в комментариях

### Типы проверок

**Проверка существования:**
```
Ссылка: [README](/src/README.md)
    │
    ├── Файл существует? → ✅ Валидно
    └── Не существует?   → ❌ Битая ссылка
```

**Проверка якорей:**
```
Ссылка: [Шаги](./workflow.md#шаги)
    │
    ├── Файл существует? → НЕТ → ❌ Битая ссылка
    └── Якорь существует?
        ├── ДА → ✅ Валидно
        └── НЕТ → ⚠️ Битый якорь
```

### Битые ссылки

| Тип | Описание | Пример |
|-----|----------|--------|
| Несуществующий файл | Цель удалена/перемещена | `[old](/deleted.md)` |
| Битый якорь | Заголовок изменён | `[x](#old-heading)` |
| Неверный путь | Опечатка в пути | `[x](/scr/file.md)` |

### Формат пометки битых ссылок

```markdown
<!-- До -->
[Удалённый файл](/old/path.md)

<!-- После (через /links-delete) -->
[Удалённый файл](/old/path.md) <!-- BROKEN: файл удалён -->
```

### Вывод /links-validate

```
Проверка ссылок...

❌ /src/README.md:15
   [старый API](/docs/api-v1.md)
   → файл не существует

⚠️ /specs/README.md:42
   [Шаги](#шаги-воркфлоу)
   → якорь не найден (есть: #шаги, #воркфлоу)

✅ Проверено: 156 ссылок
❌ Ошибок: 1
⚠️ Предупреждений: 1
```
