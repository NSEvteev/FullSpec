## 1. Типы ссылок

| Тип | Описание | Пример |
|-----|----------|--------|
| Внутренние | На файлы/папки проекта | `[README](/src/README.md)` |
| Внешние | На внешние ресурсы | `[Docs](https://example.com)` |
| Якорные | На секцию документа | `[Шаги](#шаги)` |

### Внутренние ссылки

```markdown
[Название](путь/к/файлу.md)
[Название](путь/к/папке/)
```

### Внешние ссылки

```markdown
[Название](https://example.com)
```

**Правила:**
- Предпочитать HTTPS
- Без trailing slash
- Ссылаться на конкретную страницу

## 2. Абсолютные vs относительные

| Тип | Формат | Когда использовать |
|-----|--------|-------------------|
| Абсолютные | `/путь/от/корня` | Между разными папками проекта |
| Относительные | `./файл` или `../папка` | Внутри одной папки или рядом |

### Правила выбора

```
Та же папка         → ./файл.md
Соседняя папка      → ../папка/файл.md
Далёкая папка       → /абсолютный/путь.md
```

### Примеры

```markdown
<!-- Относительные -->
[format.md](./format.md)
[README родителя](../README.md)

<!-- Абсолютные -->
[CLAUDE.md](/CLAUDE.md)
[src/README.md](/src/README.md)
```

## 3. Ссылки на папки

### Формат (предпочтительно)

```markdown
[Название](путь/к/папке/README.md)
```

**Предпочтительно:** ссылка на README.md папки.

### Альтернативный формат

```markdown
[Название](путь/к/папке/)
```

Допустим trailing slash `/` в конце пути.

### Примеры

```markdown
[skills/README.md](./skills/README.md)  <!-- предпочтительно -->
[skills/](./skills/)                     <!-- допустимо -->
[/src/](/src/)
```

## 4. Ссылки на файлы

### Формат

```markdown
[Название](путь/к/файлу.ext)
```

### Расширения

| Тип файла | Расширение |
|-----------|------------|
| Markdown | `.md` |
| YAML | `.yaml`, `.yml` |
| JSON | `.json` |
| Код | `.py`, `.ts`, `.go` |

## 5. Якорные ссылки

### Формат

```markdown
<!-- В том же файле -->
[Название секции](#название-секции)

<!-- В другом файле -->
[Название](путь/к/файлу.md#якорь)
```

### Правила формирования якоря

| Действие | Пример |
|----------|--------|
| Lowercase | `Шаги` → `шаги` |
| Пробелы → дефисы | `Формат описания` → `формат-описания` |
| Удалить спецсимволы | `1. Введение` → `1-введение` |
| Сохранить цифры | `API v2` → `api-v2` |

### Примеры

```markdown
[См. Шаги](#шаги)
[Формат секции](./format.md#формат-секции)
[Скиллы в CLAUDE.md](/CLAUDE.md#скиллы-14)
```

## 6. Ссылки в SSOT структуры

В `/.structure/README.md` используется специальный формат для ссылок на папки.

### Формат

```markdown
### 🔗 [папка/](../папка/README.md)

**Краткое назначение.**

Описание содержимого.
```

### Правила

| Элемент | Правило |
|---------|---------|
| Иконка | `🔗` — папка имеет README |
| Название | `папка/` с trailing slash |
| Ссылка | На `README.md` папки |
| Заголовок | `###` (h3) |

### Исключение

Если папка — текущий документ:

```markdown
### 🔗 .structure/ (этот файл)
```

## 7. Ссылки в frontmatter

```yaml
standard: .instructions/standard-instruction.md
index: .structure/.instructions/README.md
```

**Правила:**
- БЕЗ квадратных скобок
- БЕЗ ведущего `/`
- Относительный путь от корня репозитория
