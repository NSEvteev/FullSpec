## Принцип SSOT

> **README.md создаётся ВМЕСТЕ с папкой.** Папка без README не существует.

Это гарантирует, что структура всегда документирована в момент создания.

**Шаблон:** [template-readme.md](./template-readme.md)

---

## Шаги

### Шаг 1: Создать папку и README (одновременно)

1. Создать папку в файловой системе
2. **Сразу** создать `README.md` по [шаблону](./template-readme.md)

### Шаг 2: Добавить секцию в "Корневые папки"

Добавить описание папки в секцию "1. Корневые папки" по формату [format.md](./format.md).

**Расположение:** в алфавитном порядке среди папок.

### Шаг 3: Добавить в Оглавление

Добавить ссылку в секцию "Оглавление" в начале файла.

```markdown
- [{папка}/](#-{папка})
```

### Шаг 4: Добавить в "Дерево папок"

Добавить папку в ASCII-дерево в секции "3. Дерево папок".

**Правила:**
- Соблюдать алфавитный порядок
- Добавить комментарий с назначением
- Если есть подпапки — добавить их тоже

### Шаг 5: Обновить связанные документы

1. **CLAUDE.md** — если папка важна для Claude Code
2. **/.instructions/structure.md** — если нужны локальные инструкции
3. **/.instructions/coverage.md** — если планируются инструкции

### Шаг 6: Валидация

Проверить согласно [validation.md](./validation.md):
- Ссылки корректны
- Дерево соответствует файловой системе
- Оглавление синхронизировано

## Примеры

### Добавление папки docs/

**Шаг 2:** Секция в "Корневые папки"

```markdown
### 🔗 [docs/](../docs/README.md)

**Документация проекта.**

Архитектурные схемы (`architecture/`), руководства пользователя (`guides/`), API документация (`api/`).
```

**Шаг 3:** Ссылка в Оглавление

```markdown
- [docs/](#-docs)
```

**Шаг 4:** Запись в дереве

```
├── docs/                               # Документация проекта
│   ├── architecture/                   #   Архитектурные схемы
│   ├── guides/                         #   Руководства
│   └── api/                            #   API документация
```
