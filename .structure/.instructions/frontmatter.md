# Frontmatter

SSOT правил оформления frontmatter для всех `.md` файлов проекта.

**Полезные ссылки:**
- [Инструкции для .structure](./README.md)
- [Структура проекта](../README.md)

---

## Обязательные поля

| Поле | Описание | Пример |
|------|----------|--------|
| `description` | Краткое описание документа (одно предложение) | `SSOT структуры проекта` |
| `standard` | Путь к стандарту оформления | `/.structure/.instructions/readme-standard.md` |

---

## Правила

**description:**
- Одно предложение, используется для индексации и поиска
- Содержит **ключевые слова** для поиска (frontmatter, README, валидация)
- Указывает **область применения** (для .md файлов, для specs/)
- Описывает **что делает**, а не что это

**standard:**
- Путь к инструкции, по которой создан документ

---

## Примеры description

**Хорошо:**
- `SSOT правил оформления frontmatter для .md файлов проекта`
- `Стандарт оформления README — формат и шаблон`
- `Правила валидации ссылок между документами`

**Плохо:**
- `Документация` — нет ключевых слов
- `Инструкция` — не описывает область
- `Правила` — не понятно, для чего

---

## Пример

```yaml
---
description: SSOT структуры проекта
standard: /.structure/.instructions/readme-standard.md
---
```
