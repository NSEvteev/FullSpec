## Типы обновлений

| Тип | Описание | Пример |
|-----|----------|--------|
| Переименование | Изменение имени папки | `utils/` → `helpers/` |

| Комбинированный | Оба действия | `src/utils/` → `shared/helpers/` |

## Переименование

### Шаг 1: Найти все ссылки на папку

```bash
# Поиск всех упоминаний
grep -r "old-name/" .
```

**Где искать:**
- `/.structure/README.md` — SSOT структуры
- `CLAUDE.md` — точка входа
- `**/README.md` — README папок
- `.instructions/**` — инструкции
- `specs/**` — спецификации

### Шаг 2: Переименовать папку в файловой системе

```bash
mv old-name/ new-name/
```

### Шаг 3: Обновить /.structure/README.md

1. **Секция папки** — изменить название и ссылку
2. **Оглавление** — обновить ссылку
3. **Дерево папок** — обновить название

### Шаг 4: Обновить README папки

Обновить заголовок и ссылку на ответственность (если изменился уровень).

### Шаг 5: Использовать /links-update

```
/links-update old-name/ → new-name/
```

### Шаг 6: Валидация

Проверить согласно [validation-structure.md](./validation-structure.md).

## Перемещение

### Шаг 1: Найти все ссылки на папку

Аналогично переименованию.

### Шаг 2: Переместить папку в файловой системе

```bash
mv src/utils/ shared/utils/
```

### Шаг 3: Обновить /.structure/README.md

1. **Секция старого родителя** — удалить упоминание подпапки
2. **Секция нового родителя** — добавить упоминание подпапки
3. **Дерево папок** — переместить ветку

### Шаг 4: Обновить README перемещённой папки

**Обязательно:** обновить ссылку на ответственность по новому уровню.

| Было | Стало | Ссылка на ответственность |
|------|-------|---------------------------|
| `/src/utils/` (уровень 2) | `/shared/utils/` (уровень 2) | `[Разделение ответственности при работе с shared](../README.md)` |
| `/utils/` (уровень 1) | `/shared/utils/` (уровень 2) | `[Разделение ответственности при работе с shared](../README.md)` |

### Шаг 5: Обновить README дочерних папок

Если у перемещённой папки есть подпапки — их ссылки на ответственность тоже могут измениться.

### Шаг 6: Использовать /links-update

```
/links-update src/utils/ → shared/utils/
```

### Шаг 7: Валидация

Проверить согласно [validation-structure.md](./validation-structure.md).

## Примеры

### Переименование: utils/ → helpers/

**Шаг 2:** Переименование
```bash
mv shared/utils/ shared/helpers/
```

**Шаг 3:** Обновление /.structure/README.md
```markdown
# Было
├── shared/
│   └── utils/                    #   Утилиты

# Стало
├── shared/
│   └── helpers/                  #   Хелперы
```

**Шаг 5:** Обновление ссылок
```
/links-update shared/utils/ → shared/helpers/
```

### Перемещение: src/common/ → shared/libs/

**Шаг 2:** Перемещение
```bash
mv src/common/ shared/libs/
```

**Шаг 4:** Обновление README
```markdown
# shared/libs/README.md

# /shared/libs/ — Общие библиотеки

**Полезные ссылки:**
- [Разделение ответственности при работе с shared](../README.md)
```

**Шаг 6:** Обновление ссылок
```
/links-update src/common/ → shared/libs/
```
