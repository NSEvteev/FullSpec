Правила оформления описаний папок в SSOT структуры проекта.

## Правила

### Формат секции папки

Каждая папка в секции "Корневые папки" описывается по шаблону:

```markdown
### 🔗 [.{папка}/](../{папка}/README.md)

**{Краткое назначение — 1-3 слова}.**

{Описание содержимого: подпапки и файлы через запятую, с пояснениями в скобках.}
```

### Ссылка на README

- **Обязательно:** каждая папка ссылается на свой README.md
- **Формат:** `[.{папка}/](../{папка}/README.md)`
- **Исключение:** если README нет, ссылка ведёт на саму папку

### Иконка ссылки

- Используется эмодзи `🔗` перед названием папки
- Показывает, что папка имеет собственный README

### Описание содержимого

| Элемент | Формат |
|---------|--------|
| Подпапка с инструкциями | `Стандарты X (\`.instructions/\`)` |
| Подпапка | `{назначение} (\`{имя}/\`)` |
| Файлы | `{назначение} (\`{pattern}\`)` |

### Дерево папок

Секция "Дерево папок" содержит ASCII-дерево всех папок:

```
/
├── .{папка1}/                       # Комментарий
│   ├── {подпапка}/                  #   Комментарий (с отступом)
│   └── {файл}                       #   Комментарий
└── {папка2}/                        # Комментарий
```

**Правила:**
- Комментарии выровнены по столбцу
- Подпапки с отступом 2 пробела в комментарии
- TODO помечается как `(TODO)` в комментарии

## Примеры

### Папка с инструкциями и подпапками

**Контекст:** описание папки platform/

```markdown
### 🔗 [platform/](../platform/README.md)

**Общая инфраструктура.**

Стандарты инфраструктуры (`.instructions/`), Docker-конфигурации (`docker/`), API Gateway — Traefik/Nginx (`gateway/`), Kubernetes манифесты (`k8s/`), мониторинг — Grafana дашборды, Loki логи, Prometheus метрики (`monitoring/`), операционные runbooks (`runbooks/`), инфраструктурные скрипты деплоя и бэкапа (`scripts/`).
```

### Папка без подпапок

**Контекст:** описание папки .structure/

```markdown
### 🔗 .structure/

**SSOT структуры проекта.**

Единый источник истины о структуре папок проекта — этот файл.
```

### Дерево папок

**Контекст:** фрагмент дерева

```
├── platform/                            # Общая инфраструктура
│   ├── .instructions/                   #   Стандарты инфраструктуры (TODO)
│   ├── docker/                          #   Docker конфигурации
│   └── k8s/                             #   Kubernetes манифесты
```
