## Ссылка на контекст

Единый формат для README папок и файлов инструкций.

**Формат строки:**

```markdown
> [{текст}]({ссылка})
```

---

### В README папок

Каждый README.md папки содержит ссылку на контекст ответственности.

| Уровень | Текст ссылки | Ссылается на |
|---------|--------------|--------------|
| 1 (корневой) | Разделение ответственности всех папок проекта | `/.structure/README.md` |
| 2 | Разделение ответственности при работе с {родитель} | `../README.md` |
| 3+ | {Контекст работы} в {родитель} | `../README.md` |

**Примеры:**

```markdown
<!-- Уровень 1: /config/ -->
> [Разделение ответственности всех папок проекта](/.structure/README.md)

<!-- Уровень 2: /.claude/skills/ -->
> [Разделение ответственности при работе с .claude](../README.md)

<!-- Уровень 3: /.claude/skills/spec-create/ -->
> [Работа со скиллами в .claude](../README.md)

<!-- Уровень 3: /platform/monitoring/grafana/ -->
> [Мониторинг в platform](../README.md)
```

**Правило формирования текста:**

| Уровень | Шаблон текста |
|---------|---------------|
| 1 | `Разделение ответственности всех папок проекта` (фиксированный) |
| 2 | `Разделение ответственности при работе с {имя_родителя}` |
| 3+ | `{Назначение_текущей_папки} в {имя_родителя}` |

---

### В файлах инструкций

Каждый файл инструкции содержит ссылку на родительский README.

| Тип файла | Текст ссылки | Ссылается на |
|-----------|--------------|--------------|
| README.md папки инструкций | Разделение ответственности при работе с {область} | `../README.md` |
| Файл инструкции | Инструкции {область} | `./README.md` |
| Файл в подпапке | {Назначение подпапки} в {область} | `./README.md` |

**Примеры:**

```markdown
<!-- README папки инструкций: /.structure/.instructions/README.md -->
> [Разделение ответственности при работе с .structure](../README.md)

<!-- Файл инструкции: /.structure/.instructions/format.md -->
> [Инструкции по работе со структурой проекта](./README.md)

<!-- README подпапки: /.structure/.instructions/links/README.md -->
> [Инструкции по работе со структурой проекта](../README.md)

<!-- Файл в подпапке: /.structure/.instructions/links/format.md -->
> [Инструкции по работе со ссылками](./README.md)
```

**Правило формирования текста:**

| Тип | Шаблон текста |
|-----|---------------|
| README папки инструкций | `Разделение ответственности при работе с {область}` |
| Файл инструкции | `Инструкции по работе с {что_делает_область}` |
| README подпапки | `Инструкции по работе с {что_делает_родитель}` |
| Файл в подпапке | `Инструкции по работе с {что_делает_подпапка}` |

### Правило развёрнутого текста

Текст ссылки должен быть понятен без контекста файла.

| Плохо | Хорошо |
|-------|--------|
| `Инструкции .structure` | `Инструкции по работе со структурой проекта` |
| `Инструкции .instructions` | `Инструкции по работе с инструкциями` |
| `Инструкции specs` | `Инструкции по работе со спецификациями` |
| `Работа со ссылками в .structure` | `Инструкции по работе со ссылками` |
| `Работа со скиллами в .claude` | `Инструкции по работе со скиллами` |

**Шаблон:** `Инструкции по работе с {чем}` или `Инструкции по работе со {чем}`.

## Формат README папок

### Формат секции папки

Каждая папка в секции "Корневые папки" описывается по шаблону:

```markdown
### 🔗 [.{папка}/](../{папка}/README.md)

**{Краткое назначение — 1-3 слова}.**

{Описание содержимого: подпапки и файлы через запятую, с пояснениями в скобках.}
```

### Ссылка на README

- **Обязательно:** каждая папка ссылается на свой README.md
- **Формат:** `[.{папка}/](../{папка}/README.md)`
- **Исключение:** если README нет, ссылка ведёт на саму папку

### Иконка ссылки

- Используется эмодзи `🔗` перед названием папки
- Показывает, что папка имеет собственный README

### Описание содержимого

| Элемент | Формат |
|---------|--------|
| Подпапка с инструкциями | `Стандарты X (\`.instructions/\`)` |
| Подпапка | `{назначение} (\`{имя}/\`)` |
| Файлы | `{назначение} (\`{pattern}\`)` |

### Дерево папок

Секция "Дерево папок" содержит ASCII-дерево всех папок:

```
/
├── .{папка1}/                       # Комментарий
│   ├── {подпапка}/                  #   Комментарий (с отступом)
│   └── {файл}                       #   Комментарий
└── {папка2}/                        # Комментарий
```

**Правила:**
- Комментарии выровнены по столбцу
- Подпапки с отступом 2 пробела в комментарии
- TODO помечается как `(TODO)` в комментарии

## Примеры

### Папка с инструкциями и подпапками

**Контекст:** описание папки platform/

```markdown
### 🔗 [platform/](../platform/README.md)

**Общая инфраструктура.**

Стандарты инфраструктуры (`.instructions/`), Docker-конфигурации (`docker/`), API Gateway — Traefik/Nginx (`gateway/`), Kubernetes манифесты (`k8s/`), мониторинг — Grafana дашборды, Loki логи, Prometheus метрики (`monitoring/`), операционные runbooks (`runbooks/`), инфраструктурные скрипты деплоя и бэкапа (`scripts/`).
```

### Папка без подпапок

**Контекст:** описание папки .structure/

```markdown
### 🔗 .structure/

**SSOT структуры проекта.**

Единый источник истины о структуре папок проекта — этот файл.
```

### Дерево папок

**Контекст:** фрагмент дерева

```
├── platform/                            # Общая инфраструктура
│   ├── .instructions/                   #   Стандарты инфраструктуры (TODO)
│   ├── docker/                          #   Docker конфигурации
│   └── k8s/                             #   Kubernetes манифесты
```
