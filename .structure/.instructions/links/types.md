## Внутренние ссылки

### На файлы

```markdown
[Название](путь/к/файлу.md)
```

| Цель | Пример |
|------|--------|
| README папки | `[src/README.md](/src/README.md)` |
| Инструкция | `[format.md](./format.md)` |
| Спецификация | `[ADR-001](/specs/services/auth/adr/001-jwt.md)` |

### На папки

```markdown
[Название](путь/к/папке/)
```

| Цель | Пример |
|------|--------|
| Корневая папка | `[/src/](/src/)` |
| Подпапка | `[skills/](./skills/)` |

### На ответственность

Специальный тип ссылки в README папок:

```markdown
> [Разделение ответственности всех папок проекта](/.structure/README.md)
```

См. [format.md](../format.md#ссылка-на-ответственность-в-readme-папки)

## Внешние ссылки

### Формат

```markdown
[Название](https://example.com)
```

### Правила

| Правило | Описание |
|---------|----------|
| HTTPS | Предпочитать HTTPS над HTTP |
| Без trailing slash | `https://example.com` (не `https://example.com/`) |
| Конкретные страницы | Ссылаться на конкретную страницу, не на главную |

### Примеры

```markdown
<!-- Документация -->
[Kubernetes Docs](https://kubernetes.io/docs/)

<!-- GitHub -->
[Issue #123](https://github.com/org/repo/issues/123)

<!-- API Reference -->
[OpenAPI Spec](https://spec.openapis.org/oas/v3.1.0)
```

## Якорные ссылки

### В том же документе

```markdown
[Название секции](#название-секции)
```

**Правила формирования якоря:**
- Lowercase
- Пробелы → дефисы
- Удаление спецсимволов

| Заголовок | Якорь |
|-----------|-------|
| `## Шаги` | `#шаги` |
| `## 1. Формат описания` | `#1-формат-описания` |
| `### Ссылка на README` | `#ссылка-на-readme` |

### В другом документе

```markdown
[Название](путь/к/файлу.md#якорь)
```

```markdown
[Шаги создания](./workflow-create.md#шаги)
[Формат секции](../format.md#формат-секции-папки)
```
