## Абсолютные vs относительные

| Тип | Формат | Когда использовать |
|-----|--------|-------------------|
| Абсолютные | `/путь/от/корня` | Между разными папками проекта |
| Относительные | `./файл` или `../папка` | Внутри одной папки или рядом |

### Правила выбора

```
Та же папка         → ./файл.md
Соседняя папка      → ../папка/файл.md
Далёкая папка       → /абсолютный/путь.md
```

### Примеры

```markdown
<!-- В .structure/.instructions/links/ -->

<!-- Относительные (внутри .structure/.instructions/) -->
[format.md](./format.md)
[README родителя](../README.md)

<!-- Абсолютные (к другим папкам) -->
[CLAUDE.md](/CLAUDE.md)
[src/README.md](/src/README.md)
```

## Ссылки на папки

### Формат

```markdown
[Название](путь/к/папке/)
```

**Важно:** trailing slash `/` в конце пути.

### С указанием README

```markdown
[Название](путь/к/папке/README.md)
```

### Примеры

```markdown
<!-- На папку -->
[skills/](./skills/)

<!-- На README папки -->
[skills/README.md](./skills/README.md)

<!-- Абсолютный путь -->
[/src/](/src/)
```

## Ссылки на файлы

### Формат

```markdown
[Название](путь/к/файлу.ext)
```

### Расширения

| Тип файла | Расширение |
|-----------|------------|
| Markdown | `.md` |
| YAML | `.yaml`, `.yml` |
| JSON | `.json` |
| Код | `.py`, `.ts`, `.go` и др. |

### Примеры

```markdown
[Инструкция](./format.md)
[Конфиг](../../../config/production.yaml)
[Схема](/shared/contracts/openapi/auth.yaml)
```

## Ссылки с якорями

### Формат

```markdown
[Название](путь/к/файлу.md#якорь)
```

### Правила формирования якоря

| Действие | Пример |
|----------|--------|
| Lowercase | `Шаги` → `шаги` |
| Пробелы → дефисы | `Формат описания` → `формат-описания` |
| Удалить спецсимволы | `1. Введение` → `1-введение` |
| Сохранить цифры | `API v2` → `api-v2` |

### Примеры

```markdown
<!-- В том же файле -->
[См. Шаги](#шаги)

<!-- В другом файле -->
[Формат секции](./format.md#формат-секции-папки)

<!-- Абсолютный с якорем -->
[Скиллы в CLAUDE.md](/CLAUDE.md#скиллы-14)
```

## Особые форматы

### Ссылка на ответственность

В README папок:

```markdown

```

См. [.structure/format.md](../format.md#ссылка-на-ответственность-в-readme-папки)

### Ссылка на контекст

В шапке инструкций (единый формат):

```markdown
> [Инструкции по работе со структурой проекта](./README.md)
```

См. [.structure/format.md](../format.md#ссылка-на-контекст)
