## Шаги

### Шаг 1: Проверить зависимости

Найти все ссылки на удаляемую папку:

1. **/.structure/README.md** — основной SSOT
2. **CLAUDE.md** — если папка упоминается
3. **Другие README** — если есть ссылки
4. **Инструкции** — если папка регулируется

### Шаг 2: Удалить секцию из "Корневые папки"

Удалить полностью секцию папки из "1. Корневые папки".

### Шаг 3: Удалить из Оглавления

Удалить ссылку из секции "Оглавление".

### Шаг 4: Удалить из "Дерева папок"

Удалить папку и все её подпапки из ASCII-дерева.

### Шаг 5: Обновить связанные документы

1. **CLAUDE.md** — удалить упоминания
2. **/.instructions/structure.md** — удалить из списка локальных инструкций
3. **/.instructions/coverage.md** — удалить из таблицы покрытия
4. **Другие README** — обновить или пометить битые ссылки

### Шаг 6: Использовать /links-delete

Вызвать скилл для пометки битых ссылок:

```
/links-delete {папка}/
```

### Шаг 7: Удалить папку из файловой системы

**Только после обновления всех документов** удалить папку.

### Шаг 8: Валидация

Проверить согласно [validation.md](./validation.md):
- Нет битых ссылок
- Дерево соответствует файловой системе
- Оглавление синхронизировано

## Примеры

### Удаление папки legacy/

**Шаг 1:** Поиск ссылок

```bash
# Найти все упоминания
grep -r "legacy/" .
```

**Шаг 2-4:** Удаление из /.structure/README.md

Удалить:
- Секцию `### 🔗 [legacy/](...)`
- Ссылку `- [legacy/](#-legacy)` из Оглавления
- Записи в дереве папок

**Шаг 6:** Пометка битых ссылок

```
/links-delete legacy/
```
