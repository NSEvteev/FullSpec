## Когда валидировать

- После создания/редактирования markdown-документов
- После переименования/перемещения файлов или папок
- После изменения заголовков (меняются якоря)
- Перед коммитом изменений в документацию
- Периодически — для поиска накопившихся ошибок

## Шаги валидации

### Шаг 1: Существование цели

**Правило:** Каждая внутренняя ссылка должна вести на существующий файл/папку.

**Проверка:**
```
Для каждой ссылки [текст](путь):
├── путь начинается с http/https? → пропустить (внешняя)
├── путь начинается с #? → проверить якорь в текущем файле
└── иначе:
    ├── разрешить путь относительно текущего файла
    └── проверить существование файла/папки
```

**Ошибки:**
| Код | Описание |
|-----|----------|
| `E001` | Файл не существует |
| `E002` | Папка не существует |

---

### Шаг 2: Формат пути

**Правило:** Выбор абсолютного/относительного пути должен соответствовать стандарту.

**Проверка:**
```
Для каждой ссылки [текст](путь):
├── путь внешний? → пропустить
├── цель в той же папке? → должен быть ./файл.md
├── цель в соседней папке? → должен быть ../папка/файл.md
└── цель далеко? → допустим /абсолютный/путь.md
```

**Предупреждения:**
| Код | Описание |
|-----|----------|
| `W001` | Абсолютный путь для файла в той же папке |
| `W002` | Относительный путь с длинной цепочкой `../../../` |

---

### Шаг 3: Якорные ссылки

**Правило:** Якорь должен существовать и соответствовать формату.

**Проверка:**
```
Для каждой ссылки с якорем [текст](путь#якорь):
├── найти целевой файл
├── извлечь все заголовки из файла
├── преобразовать заголовки в якоря:
│   ├── lowercase
│   ├── пробелы → дефисы
│   ├── удалить спецсимволы (кроме дефисов и цифр)
│   └── кириллица остаётся
└── проверить: якорь ∈ список_якорей?
```

**Ошибки:**
| Код | Описание |
|-----|----------|
| `E003` | Якорь не найден в целевом файле |

**Предупреждения:**
| Код | Описание |
|-----|----------|
| `W003` | Похожий якорь существует (опечатка?) |

---

### Шаг 4: Ссылки на папки

**Правило:** Ссылки на папки должны иметь trailing slash или вести на README.md.

**Проверка:**
```
Для каждой ссылки [текст](путь):
├── путь ведёт на папку?
│   ├── заканчивается на /? → ✅ OK
│   ├── заканчивается на /README.md? → ✅ OK
│   └── иначе → ❌ ошибка
└── путь ведёт на файл? → пропустить
```

**Ошибки:**
| Код | Описание |
|-----|----------|
| `E004` | Ссылка на папку без trailing slash |

---

### Шаг 5: Ссылки в frontmatter

**Правило:** Поля `standard` и `index` в frontmatter должны быть путями без `[]` и без ведущего `/`.

**Проверка:**
```
Для каждого .md файла:
├── извлечь frontmatter (между ---)
├── для полей standard, index:
│   ├── содержит []? → ❌ ошибка
│   ├── начинается с /? → ❌ ошибка
│   └── файл существует? → проверить
└── формат корректен? → ✅ OK
```

**Ошибки:**
| Код | Описание |
|-----|----------|
| `E005` | Квадратные скобки в frontmatter-ссылке |
| `E006` | Ведущий `/` в frontmatter-ссылке |
| `E007` | Файл из frontmatter не существует |

---

### Шаг 6: Ссылки в SSOT

**Правило:** В `/.structure/README.md` ссылки на папки должны соответствовать формату.

**Проверка:**
```
Только для /.structure/README.md:
├── найти все строки с ### 🔗
├── для каждой:
│   ├── формат: ### 🔗 [папка/](путь/README.md)
│   ├── название содержит trailing slash?
│   ├── путь ведёт на README.md?
│   └── папка существует?
└── все проверки пройдены? → ✅ OK
```

**Ошибки:**
| Код | Описание |
|-----|----------|
| `E008` | Неверный формат ссылки в SSOT |
| `E009` | Ссылка в SSOT не на README.md |

---

### Шаг 7: Ссылки на скиллы

**Правило:** Ссылки на скиллы в инструкциях должны вести на существующие скиллы.

**Проверка:**
```
Для каждого .md файла в .instructions:
├── найти все ссылки вида [/skill-name](путь/SKILL.md)
├── для каждой ссылки:
│   ├── путь ведёт в /.claude/skills/?
│   ├── папка скилла существует?
│   ├── SKILL.md существует?
│   └── имя скилла в тексте совпадает с папкой?
└── все проверки пройдены? → ✅ OK
```

**Ошибки:**
| Код | Описание |
|-----|----------|
| `E010` | Папка скилла не существует |
| `E011` | SKILL.md не найден в папке скилла |

**Предупреждения:**
| Код | Описание |
|-----|----------|
| `W004` | Имя скилла в тексте не совпадает с папкой |

---

### Шаг 8: Ссылки в скиллах

**Правило:** Ссылки на SSOT-инструкции в скиллах должны вести на существующие файлы.

**Проверка:**
```
Для каждого SKILL.md в /.claude/skills/:
├── найти строку **SSOT:** [ссылка](путь)
├── для каждой ссылки:
│   ├── путь существует?
│   ├── если путь в .instructions — файл не помечен DELETE_?
│   └── якорь (если есть) существует?
└── все проверки пройдены? → ✅ OK
```

**Ошибки:**
| Код | Описание |
|-----|----------|
| `E012` | SSOT-инструкция не существует |
| `E013` | SSOT-инструкция помечена DELETE_ |

**Предупреждения:**
| Код | Описание |
|-----|----------|
| `W005` | Скилл ссылается на переименованную/перемещённую инструкцию |

## Чек-лист

### Базовые проверки

- [ ] Все внутренние ссылки ведут на существующие файлы
- [ ] Все якоря ведут на существующие заголовки
- [ ] Нет ссылок на удалённые/перемещённые файлы

### Формат путей

- [ ] Файлы в той же папке: `./файл.md`
- [ ] Файлы в соседней папке: `../папка/файл.md`
- [ ] Далёкие файлы: `/абсолютный/путь.md`

### Специальные случаи

- [ ] Ссылки на папки ведут на README.md (предпочтительно) или заканчиваются на `/`
- [ ] Frontmatter-ссылки без `[]` и без ведущего `/`
- [ ] SSOT-ссылки в формате `### 🔗 [папка/](../папка/README.md)`
- [ ] Ссылки на скиллы ведут на существующие `/.claude/skills/*/SKILL.md`
- [ ] SSOT-ссылки в скиллах ведут на существующие инструкции (не DELETE_)

## Типичные ошибки

| Ошибка | Пример | Исправление |
|--------|--------|-------------|
| Файл не существует | `[API](/docs/api.md)` | Проверить путь, создать файл |
| Битый якорь | `[Шаги](#шаги-работы)` | Проверить заголовок в целевом файле |
| Нет trailing slash | `[src](./src)` | `[src](./src/)` |
| `[]` в frontmatter | `standard: [file.md]` | `standard: file.md` |
| `/` в frontmatter | `index: /path/README.md` | `index: path/README.md` |
| Длинная цепочка `../` | `[x](../../../file.md)` | Использовать `/абсолютный/путь.md` |
| Скилл не существует | `[/example-skill](…/example-skill/SKILL.md)` | Удалить ссылку или создать скилл |
| SSOT в скилле устарел | `SSOT: [old.md](DELETE_old.md)` | Обновить ссылку или пометить скилл DELETE_ |

## Скрипты

| Скрипт | Назначение | Инструкция |
|--------|------------|------------|
| [validate-links.py](./.scripts/validate-links.py) | Автоматическая проверка ссылок (E001-E013, W001-W005) | Этот документ |
| [update-skill-refs.py](./.scripts/update-skill-refs.py) | Обновление SSOT-ссылок в скиллах | [modify-structure.md](./modify-structure.md) |

**Использование:**
```bash
# Проверить все ссылки в проекте
python .structure/.instructions/.scripts/validate-links.py

# Проверить конкретный файл
python .structure/.instructions/.scripts/validate-links.py --path docs/README.md

# Проверить папку
python .structure/.instructions/.scripts/validate-links.py --path .structure/

# JSON-вывод для автоматизации
python .structure/.instructions/.scripts/validate-links.py --json

# Обновить ссылки в скиллах после переименования инструкции
python .structure/.instructions/.scripts/update-skill-refs.py docs/api docs/endpoints
```
