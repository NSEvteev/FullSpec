## 1. Обновление скрипта

### Когда обновлять

- Изменились требования
- Найден баг
- Добавляется новый функционал
- Изменился API зависимостей

### Шаги обновления

```
1. Найти скрипт
   └── {область}/.instructions/.scripts/{name}.py

2. Обновить код
   └── Внести изменения

3. Обновить docstring
   ├── Использование (если изменился интерфейс)
   ├── Примеры (если изменились)
   └── Возвращает (если изменились коды)

4. Протестировать
   ├── Старые сценарии работают
   └── Новые сценарии работают

5. Валидировать
   └── python validate-script.py {name}.py

6. Обновить документацию
   └── README если изменилось описание
```

### Обратная совместимость

**Сохранять:**
- Существующие аргументы командной строки
- Формат вывода (если используется программно)
- Exit codes

**Можно изменять:**
- Внутреннюю реализацию
- Добавлять новые опциональные аргументы
- Улучшать сообщения об ошибках

## 2. Рефакторинг скрипта

### Когда рефакторить

- Скрипт > 300 строк
- Дублирование кода
- Сложная логика без разделения
- Отсутствуют разделители секций

### Паттерны рефакторинга

| Проблема | Решение |
|----------|---------|
| Длинный скрипт | Разделить на функции |
| Дублирование | Вынести в общую функцию |
| Сложный main() | Вынести логику в отдельные функции |
| Много констант | Сгруппировать в секцию "Константы" |

### Шаги рефакторинга

```
1. Добавить разделители секций
   └── # ===... Название секции ...===

2. Сгруппировать константы
   └── В начало файла после импортов

3. Выделить общие функции
   └── find_repo_root и т.п.

4. Разделить логику
   └── Одна функция — одна задача

5. Протестировать
   └── Все сценарии работают

6. Валидировать
   └── Формат соответствует стандарту
```

## 3. Удаление скрипта

### Когда удалять

- Функционал больше не нужен
- Скрипт заменён другим
- Скрипт перемещён

### Шаги удаления

```
1. Найти зависимости
   └── grep -r "{script-name}" --include="*.md" --include="*.py"

2. Обновить зависимости
   ├── Удалить вызовы скрипта
   ├── Заменить на альтернативу
   └── Обновить документацию

3. Удалить из README
   └── Убрать из секции "Скрипты"

4. Удалить файл
   └── git rm {path/to/script.py}

5. Закоммитить
   └── "chore: удалён {script-name}.py — {причина}"
```

## 4. Поиск зависимостей

### Паттерны поиска

```bash
# Поиск в markdown
grep -r "script-name" --include="*.md"

# Поиск в Python
grep -r "script-name" --include="*.py"

# Поиск в скиллах
grep -r "script-name" .claude/skills/

# Поиск вызовов
grep -r "python.*script-name" --include="*.md"
```

### Типичные места зависимостей

| Место | Что искать |
|-------|------------|
| README.md | Секция "Скрипты" |
| SKILL.md | Шаги воркфлоу |
| Другие скрипты | subprocess вызовы |
| CI/CD | Пайплайны |

## 5. Чек-лист

```markdown
## Чек-лист изменения скрипта

### Обновление
- [ ] Внесены изменения в код
- [ ] Обновлён docstring (если нужно)
- [ ] Протестированы старые сценарии
- [ ] Протестированы новые сценарии
- [ ] Пройдена валидация
- [ ] Обновлён README (если нужно)

### Рефакторинг
- [ ] Добавлены разделители секций
- [ ] Константы сгруппированы
- [ ] Функции разделены по задачам
- [ ] Нет дублирования кода
- [ ] Все тесты проходят

### Удаление
- [ ] Найдены все зависимости
- [ ] Зависимости обновлены/удалены
- [ ] Скрипт удалён из README
- [ ] Файл удалён
- [ ] Создан коммит с причиной удаления
```
