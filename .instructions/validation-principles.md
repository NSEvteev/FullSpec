## Когда валидировать

**Обязательно:**
- При code review
- После рефакторинга кода
- Перед коммитом изменений

**Опционально:**
- При аудите кодовой базы
- При онбординге нового разработчика

## Шаги

### Шаг 1: Проверить KISS

**SSOT:** [standard-principles.md § 1](./standard-principles.md#1-kiss)

**Что искать:**

| Паттерн | Проблема |
|---------|----------|
| Вложенные lambda | Переусложнение |
| Цепочки map/filter/reduce | Трудно читать |
| Однострочники > 80 символов | Непонятный код |

**Команда поиска:**
```bash
grep -n "lambda.*lambda" {file}.py
grep -n "map.*filter\|filter.*map" {file}.py
```

### Шаг 2: Проверить DRY

**SSOT:** [standard-principles.md § 2](./standard-principles.md#2-dry)

**Что искать:**

| Паттерн | Проблема |
|---------|----------|
| Копипаста кода | Дублирование |
| Одинаковые сообщения об ошибках | Повторение |
| Похожие функции | Можно объединить |

**Проверка:**
```
1. Сравнить функции на похожую логику
2. Найти повторяющиеся блоки > 3 строк
3. Проверить сообщения об ошибках на дубли
```

### Шаг 3: Проверить YAGNI

**SSOT:** [standard-principles.md § 3](./standard-principles.md#3-yagni)

**Что искать:**

| Паттерн | Проблема |
|---------|----------|
| Неиспользуемые аргументы | Лишний код |
| Пустые функции | "На будущее" |
| Закомментированный код | Мёртвый код |

**Команда поиска:**
```bash
# Неиспользуемые переменные (требует pylint)
pylint --disable=all --enable=W0612 {file}.py

# Закомментированный код
grep -n "^#.*def \|^#.*class " {file}.py
```

### Шаг 4: Проверить зависимости

**SSOT:** [standard-principles.md § 8](./standard-principles.md#8-минимизация-зависимостей)

**Что искать:**

| Import | Статус |
|--------|--------|
| `import pandas` | ❌ Недопустимо |
| `import requests` | ❌ Недопустимо |
| `import click` | ❌ Недопустимо |
| `import yaml` | ✅ Допустимо |
| `import argparse` | ✅ Допустимо (stdlib) |

**Команда поиска:**
```bash
grep -n "^import \|^from " {file}.py | grep -v "^import sys\|^import os\|^import re\|^import json\|^import argparse\|^from pathlib\|^from typing"
```

### Шаг 5: Проверить обработку ошибок

**SSOT:** [standard-principles.md § 7](./standard-principles.md#7-обработка-ошибок)

**Что искать:**

| Паттерн | Проблема |
|---------|----------|
| `except:` | Голый except |
| `except Exception:` | Слишком широкий |
| `pass` в except | Игнорирование ошибки |
| `print(` в except | Должен быть stderr |

**Команда поиска:**
```bash
grep -n "except:" {file}.py
grep -n "except Exception:" {file}.py
grep -A1 "except" {file}.py | grep "pass"
```

### Шаг 6: Проверить документацию

**SSOT:** [standard-principles.md § 6](./standard-principles.md#6-читаемость-и-документация)

**Что искать:**

| Элемент | Требование |
|---------|------------|
| Функции | Docstring обязателен |
| Аргументы | Type hints |
| Переменные | Осмысленные имена |

**Команда поиска:**
```bash
# Функции без docstring
grep -B1 "def " {file}.py | grep -A1 "def " | grep -v '"""'
```

## Чек-лист

### KISS
- [ ] Нет вложенных lambda
- [ ] Нет длинных цепочек map/filter
- [ ] Однострочники читаемы

### DRY
- [ ] Нет дублирования кода
- [ ] Общие паттерны вынесены в функции
- [ ] Сообщения об ошибках не повторяются

### YAGNI
- [ ] Нет неиспользуемых аргументов
- [ ] Нет пустых функций
- [ ] Нет закомментированного кода

### Зависимости
- [ ] Только stdlib + PyYAML
- [ ] Нет pandas, requests, click

### Обработка ошибок
- [ ] Нет голого `except:`
- [ ] Ошибки в stderr
- [ ] Нет `pass` в except

### Документация
- [ ] Docstring для всех функций
- [ ] Type hints для аргументов
- [ ] Осмысленные имена переменных

## Типичные ошибки

| Ошибка | Код | Принцип | Решение |
|--------|-----|---------|---------|
| Вложенные lambda | P001 | KISS | Развернуть в цикл |
| Дублирование кода | P002 | DRY | Вынести в функцию |
| Неиспользуемый код | P003 | YAGNI | Удалить |
| Тяжёлая зависимость | P004 | Минимум зависимостей | Заменить на stdlib |
| Голый `except:` | P005 | Обработка ошибок | Указать тип исключения |
| Нет docstring | P006 | Читаемость | Добавить документацию |
| Неочевидное поведение | P007 | Наименьшее удивление | Переименовать/рефакторить |
| Модуль делает много | P008 | SOLID (S) | Разделить ответственность |
