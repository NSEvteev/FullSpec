## Шаги

### Шаг 1: Получить путь

Получить путь к инструкции из параметров вызова.

**Формат пути:** `{папка}/{название}.md` относительно `/.claude/.instructions/`

**Примеры:**
- `git/commits.md` → `/.claude/.instructions/git/commits.md`
- `src/api/versioning.md` → `/.claude/.instructions/src/api/versioning.md`

---

### Шаг 2: Fail-fast проверки

Проверки согласно [validation.md](./validation.md):

1. **Формат пути:** только латиница, дефисы, слэши
2. **Название файла:** kebab-case, без пробелов
3. **Допустимая папка:** проверить по [structure.md](./structure.md)
4. **Расширение:** `.md`

**При ошибке:** остановить выполнение, вывести сообщение.

---

### Шаг 3: Проверить существование

```bash
# Проверка существования файла
if [ -f "/.claude/.instructions/{путь}" ]; then
    echo "ОШИБКА: Инструкция уже существует"
    exit 1
fi
```

**При существовании:** предложить `/instruction-update` вместо создания.

---

### Шаг 3.5: Проверить существующие SSOT

Перед описанием правил проверить:
1. Существует ли SSOT для этой темы?
2. Если да — ссылаться, не дублировать

**Формат:** `{Название темы}: [SSOT]({путь}).`

**Подробнее:** [relations.md](./relations.md#ссылка-на-ssot-не-дублировать)

---

### Шаг 4: Сгенерировать содержимое

Использовать шаблон [instruction.md](/.claude/templates/instructions/instruction.md).

**Определить тип** согласно [types.md](./types.md):
- `standard` — стандарты качества (КАК делать)
- `project` — специфика проекта (ЧТО есть)

**Заполнить frontmatter** согласно [relations.md](./relations.md):
```yaml
---
type: {standard | project}
description: {описание}
governed-by: {папка}/README.md   # путь к README папки
related:
  - {связанные инструкции}       # пути относительно /.claude/.instructions/
---
```

---

### Шаг 5: Создать файл инструкции

```bash
# Создать директорию если не существует
mkdir -p "/.claude/.instructions/{папка}"

# Записать содержимое
Write → /.claude/.instructions/{путь}
```

---

### Шаг 6: Создать/обновить README папки

**Правила:** [/.claude/.instructions/.structure/readme-index.md](/.claude/.instructions/.structure/readme-index.md)

**Если README папки не существует:**
1. Создать папку по [principles.md](/.claude/.instructions/.structure/principles.md)
2. Создать README.md по правилам [readme-index.md](/.claude/.instructions/.structure/readme-index.md)
3. Использовать шаблон [template-readme.md](/.structure/.instructions/template-readme.md)

**Если README папки существует:**
- Добавить новую секцию для инструкции
- Обновить таблицу оглавления
- Обновить дерево файлов

**Путь:** `/.claude/.instructions/{папка}/README.md`

**Важно:** Соблюдать правило "1 уровень" — подпапки показывать только как ссылку на их README.md.

---

### Шаг 7: Обновить главный README

Обновить статус инструкции в [/.claude/.instructions/README.md](/.claude/.instructions/README.md).

Согласно [statuses.md](./statuses.md):
- Добавить инструкцию в соответствующую секцию
- Установить статус: `Добавить` или `Заполнить под проект`

---

### Шаг 8: Ревью

**Проверить:**
1. Frontmatter корректен
2. Структура соответствует шаблону
3. Ссылки валидны
4. Нет дублирования с существующими инструкциями

**Спросить пользователя:** "Содержимое корректно? [Y/n]"

---

### Шаг 9: Синхронизировать ссылки

Вызвать скилл: `/links-update`

**Что делает:**
- Находит документы, которые могут ссылаться на новую инструкцию
- Предлагает добавить ссылки

---

### Шаг 10: Обновить контекст

Вызвать скилл: `/context-update`

**Что делает:**
- Распространяет контекст по графу связанных документов
- Обновляет связи в related инструкциях

---

### Шаг 11: Проверить соответствие проекта

Вызвать скилл: `/instruction-update`

**Что делает:**
- Проверяет файлы проекта на соответствие новой инструкции
- Формирует список несоответствий (если есть)

---

### Шаг 12: Обновить связанные скиллы

**ОБЯЗАТЕЛЬНО.** Обновить скиллы, которые:
1. Работают с той же областью
2. Ссылаются на похожие инструкции
3. Указаны в `related` секции

---

### Шаг 13: Анализ и предложение новых скиллов

**Проанализировать:**
- Нужен ли скилл для автоматизации правил из инструкции?
- Есть ли повторяющиеся действия, которые можно автоматизировать?

**Если нужен:** предложить `/skill-create`

---
