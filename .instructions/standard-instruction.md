## 1. Назначение

**Инструкции = ТОЛЬКО стандарты (КАК делать).**

Они НЕ описывают структуру папок — это делает README.

**Когда создавать инструкцию:**
- Есть повторяющийся паттерн (> 2 случаев)
- Нужно стандартизировать подход
- Правила применимы к разным частям проекта

**Когда НЕ создавать инструкцию:**
- Описание структуры папок (→ README)
- Разовое решение
- Информация специфична для одного файла (→ комментарий в коде)

## 2. Расположение

### Структура

```
{область}/.instructions/{name}.md
```

### Централизованные инструкции

| Область | Путь | Что описывает |
|---------|------|---------------|
| Инструкции | `/.instructions/` | Как писать инструкции |
| Скиллы | `/.claude/.instructions/skills/` | Как писать скиллы |
| Specs | `/specs/.instructions/` | Как писать specs |
| Структура | `/.structure/.instructions/` | Как управлять структурой |

### Локальные инструкции (зеркала)

| Папка проекта | Локальные инструкции |
|---------------|---------------------|
| `/src/` | `/src/.instructions/` |
| `/platform/` | `/platform/.instructions/` |
| `/tests/` | `/tests/.instructions/` |
| `/shared/` | `/shared/.instructions/` |
| `/config/` | `/config/.instructions/` |

### Правила именования файлов

| Правило | Пример ✅ | Пример ❌ |
|---------|----------|----------|
| Только латиница | `design.md` | `дизайн.md` |
| Kebab-case | `error-handling.md` | `error_handling.md` |
| Нижний регистр | `api-v2.md` | `API-V2.md` |
| Без пробелов | `code-style.md` | `code style.md` |

## 3. Типы инструкций

### Обзор типов

| Тип | Назначение | При инициализации |
|-----|------------|-------------------|
| **standard** | Требования, КАК делать | Использовать as-is |
| **project** | Специфика, ЧТО есть | Заполнить под проект |

### standard — стандарты

**Характеристики:**
- Описывают КАК делать
- Универсальны для разных проектов
- Можно переиспользовать без изменений
- Содержат правила и паттерны

**Примеры:** правила API, формат коммитов, стиль кода

### project — специфика проекта

**Характеристики:**
- Описывают ЧТО есть в проекте
- Специфичны для данного проекта
- Требуют заполнения при инициализации
- Содержат конфигурации и структуры

**Примеры:** окружения, деплой, контракты

### Как определить тип

**Вопрос:** Можно ли использовать эту инструкцию в другом проекте без изменений?

| Ответ | Тип |
|-------|-----|
| Да, это универсальные правила | `standard` |
| Нет, это специфика этого проекта | `project` |

## 4. Формат файла

### Структура файла

```markdown
---
{frontmatter}
---

# {Заголовок}

{Краткое описание — 1-2 предложения.}

**Полезные ссылки:**
- [Ссылка на README](./README.md)

## Оглавление

- [Секция 1](#секция-1)
- [Секция 2](#секция-2)

---

## Секция 1

{Содержание}

---

## Связанные инструкции

- [file.md](./path) — описание связи
```

### Обязательные элементы

| Элемент | Описание |
|---------|----------|
| Frontmatter | YAML-метаданные |
| Заголовок H1 | `# Название` |
| Описание | 1-2 предложения после заголовка |
| Полезные ссылки | Ссылка на README области |
| Оглавление | Список секций |
| Связанные инструкции | В конце документа |

---

## 5. Frontmatter

### Обязательные поля

```yaml
---
description: {Краткое описание — одно предложение}
standard: .instructions/standard-instruction.md
index: {папка}/.instructions/README.md
---
```

| Поле | Описание | Пример |
|------|----------|--------|
| `description` | Краткое описание | `Правила проектирования API` |
| `standard` | Ссылка на стандарт формата | `.instructions/standard-instruction.md` |
| `index` | Ссылка на README области | `.instructions/README.md` |

### Опциональные поля

```yaml
---
type: standard | project
related:
  - path/to/related.md
---
```

| Поле | Описание |
|------|----------|
| `type` | Тип инструкции (standard/project) |
| `related` | Массив связанных инструкций |

## 6. Секции документа

### Правила

| Правило | Описание |
|---------|----------|
| H2 для секций | `## Название секции` |
| Разделители | `---` между секциями |
| Примеры в секциях | Код в блоках ``` |
| Таблицы | Для сравнений и списков |

### Типичные секции

| Секция | Когда нужна |
|--------|-------------|
| Правила | Всегда — основной контент |
| Примеры | Всегда — иллюстрация правил |
| Скиллы | Если есть связанные скиллы |
| Связанные инструкции | Всегда — в конце |
