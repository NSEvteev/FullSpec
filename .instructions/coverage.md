### Легенда

| Столбец | Описание |
|---------|----------|
| **Инструкции** | `.instructions/` папка и файлы созданы |
| **Заполнение** | Содержимое инструкций заполнено |
| **SSOT-валид** | Инструкции провалидированы как единый источник истины |
| **Скиллы** | Добавлены скиллы для автоматизации |
| **Кросс-тест** | Инструкции и скиллы протестированы совместно |

### Правило вложенности

> **Папка с подпапками** считается "заполненной" только когда ВСЕ её подпапки тоже имеют заполненные инструкции.

**Пример:** `/.claude/` содержит `skills/`, `agents/`. Чтобы отметить ✅ в "Заполнение":
- `/.claude/.instructions/` — заполнена
- `/.claude/.instructions/skills/` — заполнена
- И т.д. для всех подпапок

### Значения

| Значок | Значение |
|--------|----------|
| ✅ | Выполнено |
| 🔄 | В работе (частично) |
| ⬜ | Не выполнено |

## Всеобщая валидация

Периодическая проверка всех инструкций проекта на согласованность.

### Чек-лист валидации

- [ ] Все папки из таблицы существуют в файловой системе
- [ ] Все `.instructions/` содержат README.md
- [ ] Все инструкции имеют корректный frontmatter
- [ ] Все ссылки между инструкциями валидны
- [ ] Все `standard` указывают на существующие файлы
- [ ] Все `related` симметричны (A→B = B→A)
- [ ] Нет дублирования правил между инструкциями
- [ ] Скиллы соответствуют инструкциям

### Когда запускать

| Триггер | Действие |
|---------|----------|
| Добавление папки | Проверить покрытие |
| Создание инструкции | Проверить связи |
| Перед релизом | Полная валидация |
| Раз в неделю | Плановая проверка |
