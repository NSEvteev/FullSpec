## Обзор типов

| Тип | Назначение | При инициализации |
|-----|------------|-------------------|
| **standard** | Требования, КАК делать | Использовать as-is |
| **project** | Специфика, ЧТО есть | Заполнить под проект |

---

## standard — стандарты

**Назначение:** Стандарты качества, применимые к любому проекту.

**Характеристики:**
- Описывают КАК делать
- Универсальны для разных проектов
- Можно переиспользовать без изменений
- Содержат правила и паттерны

**Примеры:**
- src/api/design.md — правила проектирования API
- git/commits.md — формат коммитов
- src/data/errors.md — формат ошибок

---

## project — специфика проекта

**Назначение:** Специфика конкретного проекта.

**Характеристики:**
- Описывают ЧТО есть в проекте
- Специфичны для данного проекта
- Требуют заполнения при инициализации
- Содержат конфигурации и структуры

**Примеры:**
- config/environments.md — окружения проекта
- platform/deployment.md — деплой конкретного проекта
- shared/contracts.md — контракты этого проекта

---

## Как определить тип

**Вопрос:** Можно ли использовать эту инструкцию в другом проекте без изменений?

| Ответ | Тип |
|-------|-----|
| Да, это универсальные правила | standard |
| Нет, это специфика этого проекта | project |

**Где указан тип:**
1. В frontmatter инструкции: `type: standard`
2. В таблице README.md: столбец "Тип"

---

## Примеры

### Пример standard

```yaml
---
type: standard
description: Правила проектирования REST API
standard: instructions/README.md
related:
  - src/api/versioning.md
  - src/data/errors.md
---
```

**Содержимое:** правила URL, HTTP методы, статус-коды.

### Пример project

```yaml
---
type: project
description: Конфигурация окружений проекта
standard: instructions/README.md
related:
  - platform/deployment.md
---
```

**Содержимое:** список окружений (dev, staging, prod), переменные, endpoints.
