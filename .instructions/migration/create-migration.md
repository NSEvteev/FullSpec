## Принципы

> **Миграция выполняется СРАЗУ после изменения стандарта.**

> **Порядок: сначала Уровень 1, затем Уровень 2.**

> **Содержание важнее версии.** Обновить `standard-version` недостаточно — нужно обновить шаги, чек-листы, примеры.

> **⚠️ КРИТИЧНО: `check-version-drift.py` проверяет только ВЕРСИИ в frontmatter, НЕ содержание.** Версии могут совпадать, но содержание не соответствовать стандарту. LLM ОБЯЗАН выполнять сравнение содержания вручную.

> **⚠️ КРИТИЧНО: Шаг 6 (скрипты/скиллы) ОБЯЗАТЕЛЕН.** Нельзя считать миграцию завершённой без обновления скриптов. Если скрипт реализует проверки — он ДОЛЖЕН реализовывать ВСЕ проверки из стандарта.

## Шаги

### Шаг 1: Определить изменённый стандарт

Какой `standard-*.md` был изменён?

```bash
# Показать изменения
git diff --name-only HEAD~1 | grep "standard-"
```

### Шаг 2: Проверить версию стандарта

Убедиться, что версия стандарта увеличена:

```bash
python .instructions/.scripts/bump-standard-version.py --check <стандарт>
```

Если версия не увеличена — увеличить:

```bash
python .instructions/.scripts/bump-standard-version.py <стандарт>
```

### Шаг 3: Найти зависимые файлы

```bash
python .instructions/.scripts/check-version-drift.py <стандарт>
```

**Вывод покажет:**
- Уровень 1: связанные документы (validation, create, modify)
- Уровень 2: экземпляры

### Шаг 4: Обновить Уровень 1

**⚠️ ОБЯЗАТЕЛЬНАЯ ПРОЦЕДУРА — выполнить для КАЖДОГО Level 1 документа:**

#### 4.1. Извлечь секции стандарта

```bash
grep -n "^## " <путь-к-стандарту>
```

Записать список секций (например: §1 Определение, §2 Формат, §3 Примеры...).

#### 4.2. Сравнить с шагами Level 1 документа

Для каждой секции стандарта проверить:
- Есть ли соответствующий шаг в `validation-{object}.md`?
- Есть ли соответствующий шаг в `create-{object}.md`?
- Есть ли соответствующий шаг в `modify-{object}.md`?

**Таблица соответствия (заполнить):**

| Секция стандарта | validation | create | modify |
|------------------|------------|--------|--------|
| §1 ... | Шаг N / ❌ | Шаг N / ❌ | Шаг N / ❌ |
| §2 ... | Шаг N / ❌ | Шаг N / ❌ | Шаг N / ❌ |

#### 4.3. Добавить недостающие шаги

Если секция стандарта не покрыта — добавить шаг:

```
/instruction-modify validation-{object}.md
# → Добавить шаг для проверки секции §N
```

#### 4.4. Обновить frontmatter

```yaml
standard-version: vX.Z  # новая версия
```

**ЗАПРЕЩЕНО:** Обновлять только frontmatter без проверки содержания.

### Шаг 5: Обновить Уровень 2

Для каждого экземпляра из списка `level2`:

```
/modify-{type} <файл>
```

Где `{type}` определяется по стандарту:
- `standard-instruction.md` → `/modify-instruction`
- `standard-skill.md` → `/modify-skill`
- `standard-agent.md` → `/modify-agent`
- и т.д.

**Приоритеты экземпляров:**

| Приоритет | Критерий | Действие |
|-----------|----------|----------|
| Критичный | Используется другими документами | Обновить ОБЯЗАТЕЛЬНО |
| Обычный | Используется в разработке | Обновить при возможности |
| Отложенный | Deprecated или в архиве | Не обновлять |

### Шаг 6: Обновить скрипты и скиллы

**⚠️ ЭТОТ ШАГ ОБЯЗАТЕЛЕН — нельзя пропускать!**

#### 6.1. Найти связанные скрипты

```bash
# Скрипты, упомянутые в Level 1 документах
grep -r "\.py" <путь-к-validation>.md | grep -v "^#"

# Скрипты, ссылающиеся на стандарт
grep -r "standard-{object}.md" **/.scripts/
```

#### 6.2. Проверить реализацию проверок в скриптах

Для каждого найденного скрипта (например, `validate-{object}.py`):

1. Прочитать скрипт
2. Извлечь ERROR_CODES или реализованные проверки
3. Сравнить с секциями стандарта

**Таблица соответствия (заполнить):**

| Секция стандарта | Код ошибки в скрипте | Статус |
|------------------|---------------------|--------|
| §1 ... | E001 / ❌ | ✅ / Добавить |
| §2 ... | E002, E003 / ❌ | ✅ / Добавить |

#### 6.3. Добавить недостающие проверки

```
/script-modify validate-{object}.py
# → Добавить проверку E00N для секции §N стандарта
```

#### 6.4. Проверить связанные скиллы

```bash
grep -r "SSOT:" .claude/skills/ | grep "{object}"
```

Для каждого скилла проверить, что SSOT-ссылка актуальна.

**ЗАПРЕЩЕНО:** Считать миграцию завершённой, если скрипты не проверены/обновлены.

### Шаг 7: Обновить README

Если изменилась суть стандарта:

1. Обновить описание в `README.md` области
2. Обновить ссылки на связанные документы

### Шаг 8: Валидация

```bash
python .instructions/.scripts/check-version-drift.py <стандарт>
```

**Критерий успеха:** 0 расхождений.

Дополнительно: `/migration-validate <стандарт>`

### Шаг 9: Отчёт

Вывести отчёт по шаблону из [standard-migration.md § 4](./standard-migration.md#4-формат-отчёта-о-миграции).

## Чек-лист

### Подготовка
- [ ] Определён изменённый стандарт
- [ ] Версия стандарта увеличена
- [ ] Найдены зависимые файлы

### Уровень 1 (⚠️ СОДЕРЖАНИЕ, не только версия!)
- [ ] Секции стандарта извлечены (`grep "^## "`)
- [ ] Таблица соответствия "секция → шаг" заполнена
- [ ] `validation-{object}.md` — шаги покрывают ВСЕ секции стандарта
- [ ] `create-{object}.md` — шаги покрывают ВСЕ секции стандарта
- [ ] `modify-{object}.md` — шаги покрывают ВСЕ секции стандарта
- [ ] Frontmatter обновлён (`standard-version`)

### Уровень 2
- [ ] Критичные экземпляры обновлены
- [ ] Обычные экземпляры обновлены (при возможности)
- [ ] Отложенные экземпляры отмечены в драфте

### Скрипты и скиллы (⚠️ ОБЯЗАТЕЛЬНО!)
- [ ] Скрипты найдены (`grep` по Level 1 документам)
- [ ] Таблица соответствия "секция → код ошибки" заполнена
- [ ] Скрипты реализуют ВСЕ проверки из стандарта
- [ ] Скиллы проверены (SSOT-ссылки актуальны)

### Завершение
- [ ] README обновлён (если нужно)
- [ ] `check-version-drift.py`: 0 расхождений (версии)
- [ ] Содержание проверено вручную (секции ↔ шаги ↔ скрипты)
- [ ] Отчёт сформирован

## Примеры

### Миграция standard-instruction.md v1.1 → v1.2

**Сценарий:** Добавлена секция "Каскадное обновление" в стандарт.

```bash
# 1. Проверить расхождения версий
python .instructions/.scripts/check-version-drift.py .instructions/standard-instruction.md

# 2. Извлечь секции стандарта
grep -n "^## " .instructions/standard-instruction.md
# Вывод:
# 15: ## 1. Четыре типа инструкций
# 30: ## 2. Структура файла
# 50: ## 3. Каскадное обновление  ← НОВАЯ СЕКЦИЯ
# ...

# 3. Заполнить таблицу соответствия для Level 1
# | Секция | validation | create | modify |
# |--------|------------|--------|--------|
# | §1     | Шаг 1      | Шаг 2  | —      |
# | §2     | Шаг 2      | Шаг 3  | Шаг 1  |
# | §3 NEW | ❌         | ❌     | ❌     | ← ДОБАВИТЬ!

# 4. Обновить Level 1 документы
/instruction-modify .instructions/validation-instruction.md
# → Добавить "Шаг N: Проверить секцию Каскадное обновление"

/instruction-modify .instructions/create-instruction.md
# → Добавить "Шаг N: Создать секцию Каскадное обновление"

# 5. Найти связанные скрипты
grep -r "\.py" .instructions/validation-instruction.md
# Вывод: validate-instruction.py

# 6. Проверить реализацию проверок в скрипте
# Прочитать validate-instruction.py, найти ERROR_CODES
# Сравнить с секциями стандарта:
# | Секция | Код в скрипте |
# |--------|---------------|
# | §1     | I001          |
# | §2     | I002, I003    |
# | §3 NEW | ❌            | ← ДОБАВИТЬ I004!

# 7. Обновить скрипт
/script-modify .instructions/.scripts/validate-instruction.py
# → Добавить проверку I004 для секции "Каскадное обновление"

# 8. Обновить Уровень 2 (все инструкции)
/instruction-modify <файл>
# → Обновить standard-version: v1.2
# → Добавить секцию если требуется

# 9. Валидация
python .instructions/.scripts/check-version-drift.py .instructions/standard-instruction.md
# Ожидаемый результат: 0 расхождений
```

### Частичная миграция

**Сценарий:** 3 экземпляра нельзя обновить сейчас (ожидают ревью).

```bash
# После обновления основных экземпляров:

# 1. Создать драфт
# .claude/drafts/2026-02-02-migration-pending-instruction.md

# Содержание драфта:
# - Обновлено: 47 файлов
# - Отложено: 3 файла (specs/api-v2.md, specs/auth.md, specs/db.md)
# - Причина: ожидают ревью PR #123
# - План: обновить после слияния PR

# 2. Отметить в отчёте
# ⏭️ 3 файла отложены (причина: PR #123, план: см. драфт)
```
