## 1. Когда валидировать

**Обязательно:**
- Перед коммитом новой инструкции
- После существенных изменений
- При code review

**Опционально:**
- В CI/CD pipeline
- При рефакторинге документации

## 2. Что проверяется

### Файл

| Проверка | Обязательно | Описание |
|----------|:-----------:|----------|
| Расширение | ✅ | `.md` |
| Имя файла | ✅ | kebab-case, латиница |
| Расположение | ✅ | В папке `.instructions/` |
| Кодировка | ✅ | UTF-8 |

### Frontmatter

| Проверка | Обязательно | Описание |
|----------|:-----------:|----------|
| Наличие | ✅ | Блок `---` в начале |
| `description` | ✅ | Присутствует, не пустое |
| `standard` | ✅ | Ссылка на стандарт |
| `index` | ✅ | Ссылка на README области |
| `type` | ⚠️ | standard или project |

### Структура

| Проверка | Обязательно | Описание |
|----------|:-----------:|----------|
| Заголовок H1 | ✅ | Единственный `#` |
| Описание | ✅ | Текст после H1 |
| Оглавление | ✅ | `## Оглавление` |
| Связанные | ✅ | `## Связанные инструкции` |

### Ссылки

| Проверка | Обязательно | Описание |
|----------|:-----------:|----------|
| Валидность | ✅ | Файлы существуют |
| Относительные | ⚠️ | Предпочтительны |
| Якоря | ⚠️ | Секции существуют |

## 3. Коды ошибок

| Код | Категория | Описание |
|-----|-----------|----------|
| **I001** | Файл | Неверное расширение (не .md) |
| **I002** | Файл | Неверное имя (не kebab-case) |
| **I003** | Файл | Неверное расположение |
| **I010** | Frontmatter | Отсутствует frontmatter |
| **I011** | Frontmatter | Отсутствует description |
| **I012** | Frontmatter | Отсутствует standard |
| **I013** | Frontmatter | Отсутствует index |
| **I014** | Frontmatter | Неверный type |
| **I020** | Структура | Отсутствует заголовок H1 |
| **I021** | Структура | Несколько заголовков H1 |
| **I022** | Структура | Отсутствует оглавление |
| **I023** | Структура | Отсутствует секция "Связанные" |
| **I030** | Ссылки | Битая ссылка |
| **I031** | Ссылки | Битый якорь |

## 4. Запуск валидации

### Ручная проверка

```bash
python .instructions/.scripts/validate-instruction.py path/to/instruction.md
```

### Проверка всех инструкций

```bash
python .instructions/.scripts/validate-instruction.py --all
```

### JSON вывод

```bash
python .instructions/.scripts/validate-instruction.py --json path/to/instruction.md
```

### Вывод

```
✅ instruction.md — валидация пройдена
❌ broken.md — 3 ошибки:
   I010: Отсутствует frontmatter
   I020: Отсутствует заголовок H1
   I030: Битая ссылка: ./nonexistent.md
```

## 5. Чек-лист

```markdown
## Чек-лист валидации инструкции

### Файл
- [ ] Расширение `.md`
- [ ] Имя в kebab-case
- [ ] Расположение в `.instructions/`

### Frontmatter
- [ ] Присутствует блок `---`
- [ ] Поле `description` заполнено
- [ ] Поле `standard` указывает на стандарт
- [ ] Поле `index` указывает на README

### Структура
- [ ] Один заголовок H1
- [ ] Описание после заголовка
- [ ] Секция "Оглавление"
- [ ] Секция "Связанные инструкции"

### Ссылки
- [ ] Все ссылки валидны
- [ ] Все якоря существуют
```
