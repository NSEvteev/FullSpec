## 1. Когда создавать

**Создавать инструкцию когда:**

| Критерий | Порог |
|----------|-------|
| Повторяемость паттерна | > 2 случаев |
| Сложность правил | > 3 правил |
| Область применения | > 1 файла/компонента |

**НЕ создавать инструкцию когда:**

- Описание структуры папок (→ README)
- Разовое решение (→ комментарий)
- Очевидные правила (→ здравый смысл)
- Уже есть похожая инструкция (→ расширить существующую)

## 2. Подготовка

### 2.1. Определить область

```
Вопросы:
1. Какую проблему решает инструкция?
2. К каким файлам/компонентам применяется?
3. Есть ли связанные инструкции?
```

### 2.2. Выбрать тип

| Вопрос | Ответ | Тип |
|--------|-------|-----|
| Можно использовать в другом проекте? | Да | `standard` |
| Специфично для этого проекта? | Да | `project` |

### 2.3. Выбрать расположение

```
{область}/.instructions/{name}.md
```

| Область | Путь |
|---------|------|
| Общие инструкции | `/.instructions/` |
| Инструкции для src | `/src/.instructions/` |
| Инструкции для specs | `/specs/.instructions/` |

### 2.4. Выбрать имя

- Kebab-case: `error-handling.md`
- Описательное: `api-versioning.md`
- Короткое: `naming.md`

## 3. Шаги создания

```
1. Проверить существование
   └── Нет ли уже инструкции на эту тему?

2. Проверить SSOT
   └── Есть ли SSOT для этой темы? → Ссылаться, не дублировать

3. Создать файл
   └── {область}/.instructions/{name}.md

4. Заполнить frontmatter
   ├── description
   ├── standard
   └── index

5. Написать содержание
   ├── Заголовок H1
   ├── Описание (1-2 предложения)
   ├── Полезные ссылки
   ├── Оглавление
   ├── Основные секции
   └── Связанные инструкции

6. Проверить ссылки
   └── Все ссылки валидны

7. Обновить README области
   └── Добавить инструкцию в список

8. Валидировать
   └── python validate-instruction.py {path}

9. Ревью
   └── Спросить пользователя: "Содержимое корректно? [Y/n]"
```
