## Принципы

> **Создавать инструкцию только при повторяемости.** Паттерн должен встречаться > 2 раз.

> **Не дублировать SSOT.** Если есть источник истины — ссылаться, не копировать.

> **Инструкции = КАК делать, README = ЧТО есть.** Описание структуры папок — это README, не инструкция.

> **Скрипты автоматизируют шаги.** Если шаг требует поиска/фильтрации/трансформации — создать скрипт.

**Создавать инструкцию когда:**

| Критерий | Порог |
|----------|-------|
| Повторяемость паттерна | > 2 случаев |
| Сложность правил | > 3 правил |
| Область применения | > 1 файла/компонента |

**НЕ создавать инструкцию когда:**

- Описание структуры папок (→ README)
- Разовое решение (→ комментарий)
- Очевидные правила (→ здравый смысл)
- Уже есть похожая инструкция (→ расширить существующую)

## Шаги

### Шаг 1: Определить необходимость

```
Вопросы:
1. Какую проблему решает инструкция?
2. К каким файлам/компонентам применяется?
3. Есть ли связанные инструкции?
```

### Шаг 2: Подготовка

**Выбрать тип:**

| Тип | Префикс | Когда |
|-----|---------|-------|
| standard | `standard-` | Формат и правила объекта |
| create | `create-` | Процесс создания |
| modify | `modify-` | Процесс изменения |
| validation | `validation-` | Процесс проверки |

**Выбрать расположение:**

```
{область}/.instructions/{prefix}-{object}.md
```

| Область | Путь |
|---------|------|
| Общие инструкции | `/.instructions/` |
| Скиллы | `/.claude/.instructions/skills/` |
| Структура | `/.structure/.instructions/` |
| Спецификации | `/specs/.instructions/` |

**Выбрать имя:**
- Kebab-case: `error-handling.md`
- Описательное: `api-versioning.md`
- Короткое: `naming.md`

**Проверить порядок создания:**

> **ОБЯЗАТЕЛЬНО** для типов validation, create, modify — проверить наличие предыдущих типов в цепочке.

| Создаём | Требуется сначала |
|---------|-------------------|
| `validation-{object}.md` | `standard-{object}.md` |
| `create-{object}.md` | `standard-{object}.md` + `validation-{object}.md` |
| `modify-{object}.md` | все предыдущие |

**Если предыдущий тип отсутствует:**

1. Спросить пользователя (AskUserQuestion):
   - Создать недостающую инструкцию сначала (рекомендуется)
   - Продолжить без неё (нарушение стандарта)
2. При выборе "Создать" — рекурсивно вызвать `/instruction-create` для недостающего типа

### Шаг 3: Проверить существующие инструкции

> **ОБЯЗАТЕЛЬНО** — убедиться, что нет похожей инструкции.

**Получить список инструкций:**
```bash
python .instructions/.scripts/list-instructions.py
```

**Вывод:** список всех инструкций с описаниями — передать LLM для анализа.

**LLM анализирует:**
1. Есть ли инструкция на эту тему?
2. Есть ли похожая инструкция, которую можно расширить?
3. Есть ли SSOT, на который нужно ссылаться?

**Результаты:**

| Результат | Действие |
|-----------|----------|
| Инструкция уже есть | Спросить пользователя: "Инструкция {name} уже существует. Расширить её?" |
| Похожая инструкция | Спросить пользователя: "Найдена похожая инструкция {name}. Расширить или создать новую?" |
| Есть SSOT | Ссылаться на SSOT, не дублировать |
| Ничего не найдено | Продолжить создание |

### Шаг 4: Создать файл

**Автоматически:**
```bash
python .instructions/.scripts/create-instruction-file.py {name} {type} --area {path}
```

**Примеры:**
```bash
# Создать standard-инструкцию в корневой папке
python .instructions/.scripts/create-instruction-file.py api-versioning standard

# Создать validation-инструкцию в области structure
python .instructions/.scripts/create-instruction-file.py links validation --area .structure/.instructions

# С кастомным описанием
python .instructions/.scripts/create-instruction-file.py error-handling standard --description "Стандарт обработки ошибок в API"
```

**Вручную (если нужно):**
1. Создать файл по шаблону из [standard-instruction.md § 6. Шаблоны](./standard-instruction.md#6-шаблоны)
2. Заполнить frontmatter — SSOT: [standard-frontmatter.md](/.structure/.instructions/standard-frontmatter.md)

### Шаг 5: Заполнить содержание

Содержание зависит от типа инструкции — см. [standard-instruction.md](./standard-instruction.md):
- standard → [§ 2](./standard-instruction.md#2-инструкции-типа-standard)
- create → [§ 3](./standard-instruction.md#3-инструкции-типа-create)
- modify → [§ 4](./standard-instruction.md#4-инструкции-типа-modify)
- validation → [§ 5](./standard-instruction.md#5-инструкции-типа-validation)

### Шаг 6: Создать скрипты (если нужно)

> **Для типов create, modify, validation:** если шаги инструкции требуют автоматизации — создать скрипты.

**Когда создавать скрипт:**
- Шаг требует поиска файлов по критерию
- Шаг требует парсинга/трансформации данных
- Шаг требует валидации

**Процесс:**

#### 6.1. Получить список всех скриптов

```bash
python .instructions/.scripts/parse-docstrings.py
```

**Вывод:** список всех скриптов с описаниями — передать LLM для анализа.

#### 6.2. LLM анализирует

1. Есть ли скрипт с аналогичным функционалом?
2. Есть ли похожий скрипт, который можно доработать?
3. В какой директории находится потенциально подходящий скрипт?

#### 6.3. Если найден похожий скрипт

**Прочитать:**
- Код скрипта
- Связанную инструкцию (из секции "Скрипты")
- Понять контекст использования

**Анализ:**
- Можно ли переиспользовать как есть?
- Нужна ли доработка?
- Нужен ли новый скрипт в своей директории?

#### 6.4. Спросить пользователя

> **ОБЯЗАТЕЛЬНО** использовать `AskUserQuestion` tool с кликабельными опциями.

**Варианты:**
1. Переиспользовать существующий скрипт `{name}` — он уже делает нужное
2. Доработать существующий скрипт `{name}` — добавить функционал
3. Создать новый скрипт в `{область}/.instructions/.scripts/`

#### 6.5. Выполнить действие

| Выбор пользователя | Действие |
|-------------------|----------|
| Переиспользовать | Добавить ссылку в секцию "Скрипты" |
| Доработать | Вызвать `/script-modify {путь}` |
| Создать новый | Вызвать `/script-create` |

**Скиллы для работы со скриптами:**

| Скилл | Назначение |
|-------|------------|
| [/script-create](/.claude/skills/script-create/SKILL.md) | Создание нового скрипта |
| [/script-modify](/.claude/skills/script-modify/SKILL.md) | Обновление, рефакторинг, удаление |
| [/script-validate](/.claude/skills/script-validate/SKILL.md) | Валидация формата скрипта |

### Шаг 7: Обновить README области

> **ОБЯЗАТЕЛЬНО перед валидацией** — добавить инструкцию в README папки `.instructions/`.

Формат README — см. [standard-readme.md § 3](/.structure/.instructions/standard-readme.md#3-readme-папок-инструкций).

**Что обновить:**
- Добавить строку в таблицу инструкций
- Обновить оглавление (если есть)

### Шаг 8: Валидация

```bash
python .instructions/.scripts/validate-instruction.py {path}
```

### Шаг 9: Ревью и создание скилла

> **ОБЯЗАТЕЛЬНО** использовать `AskUserQuestion` tool с кликабельными опциями.

**Вопрос:** "Содержимое инструкции корректно?"

**Опции:**
1. Да, всё корректно
2. Нет, нужны правки

**При выборе "Да, всё корректно":**

Если инструкция типа `create-`, `modify-` или `validation-`:
- **АВТОМАТИЧЕСКИ** вызвать `/skill-create` для этой инструкции
- Перейти к Шагу 10

**При выборе "Нет, нужны правки":**

1. Спросить: "Что нужно исправить?" (свободный ввод)
2. Внести изменения в инструкцию согласно ответу пользователя
3. **Повторить Шаг 9** (цикл до выбора "Да, всё корректно")

### Шаг 10: Отчёт о проделанной работе

```
## Отчёт о создании инструкции

**Создана инструкция:** `{путь к инструкции}`

**Тип:** {standard/create/modify/validation}

**Описание:** {description из frontmatter}

**Содержит:**
- {количество} шагов
- {количество} пунктов чек-листа
- {количество} скриптов

**Связанный скилл:** `/{skill-name}` (для create/modify/validation) или "не требуется" (для standard)

**Валидация:** пройдена ✅
```

## Примеры

### Создание standard-инструкции

```bash
# 1. Определить: нужна инструкция для API-версионирования
# 2. Тип: standard, путь: /src/.instructions/standard-api-versioning.md
# 3. Создать файл по шаблону standard из standard-instruction.md § 6
# 4. Заполнить секции: правила версионирования, примеры
# 5. Скрипты: не нужны (тип standard)
# 6. Обновить /src/.instructions/README.md
# 7. Валидация
python .instructions/.scripts/validate-instruction.py src/.instructions/standard-api-versioning.md
```

### Создание validation-инструкции со скриптом

```bash
# 1. Определить: нужна валидация для API
# 2. Тип: validation, путь: /src/.instructions/validation-api.md
# 3. Создать файл по шаблону validation из standard-instruction.md § 6
# 4. Заполнить: Когда валидировать, Шаги, Типичные ошибки

# 5. Скрипты: шаг "Проверить все эндпоинты" требует автоматизации
python .instructions/.scripts/parse-docstrings.py --search "эндпоинт"
# Не найден → создать validate-api.py
# Добавить в секцию "Скрипты"

# 6. Обновить /src/.instructions/README.md
# 7. Валидация
python .instructions/.scripts/validate-instruction.py src/.instructions/validation-api.md
```
