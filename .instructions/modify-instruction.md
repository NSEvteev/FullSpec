## Обновление

### Шаг 1: Найти инструкцию

```
{область}/.instructions/{name}.md
```

### Шаг 2: Определить зону ответственности

| Путь инструкции | Зона ответственности |
|-----------------|---------------------|
| `src/**/*.md` | `/src/**` |
| `tests/**/*.md` | `/tests/**`, `**/*.test.*` |
| `git/**/*.md` | Git операции |
| `api/**/*.md` | `/src/**/api/**` |

### Шаг 3: Внести изменения

Обновить правила, примеры, ссылки.

### Шаг 4: Проверить файлы проекта

```
1. Найти файлы в зоне
   └── Glob по паттерну зоны

2. Извлечь правила из инструкции
   └── Структурные, форматные, содержательные

3. Проверить файлы
   └── Применить правила к каждому файлу

4. Сформировать список несоответствий
   └── [файл, правило, описание]

5. Решение
   ├── Обновить инструкцию под реальность
   └── Исправить файлы под инструкцию
```

### Шаг 5: Обновить связанные документы

- README области
- Связанные инструкции
- Скиллы

### Шаг 6: Валидация

```bash
python .instructions/.scripts/validate-instruction.py {path}
```

## Деактивация

> **Инструкции НЕ удаляются. Они деактивируются.**

**Когда деактивировать:**
- Инструкция больше не актуальна
- Технология/практика не используется
- Инструкция заменена другой

**Шаги:**

### Шаг 1: Найти все ссылки

Использовать паттерны из секции [Обновление ссылок](#обновление-ссылок).

### Шаг 2: Добавить предупреждение

После frontmatter добавить:

```markdown
> **ВАЖНО:** Эта инструкция не используется в проекте.
>
> Содержимое закомментировано. Для реактивации — раскомментировать.
```

### Шаг 3: Закомментировать содержимое

```markdown
---
description: {оригинальное описание}
standard: .instructions/standard-instruction.md
index: {область}/.instructions/README.md
---

> **ВАЖНО:** Эта инструкция не используется в проекте.
>
> Содержимое закомментировано. Для реактивации — раскомментировать.

<!--
# Оригинальный заголовок

{Оригинальное содержимое}
-->
```

### Шаг 4: Обновить ссылающиеся документы

- Удалить из related
- Пометить в README
- Обновить скиллы

## Миграция

**Когда мигрировать:**
- Переименование файла
- Перемещение в другую папку
- Реструктуризация области

**Шаги:**

### Шаг 1: Найти все ссылки на старый путь

Паттерны из секции [Обновление ссылок](#обновление-ссылок).

### Шаг 2: Создать новый файл

Копировать содержимое в новый путь.

### Шаг 3: Обновить frontmatter

Обновить `index` и другие относительные пути.

### Шаг 4: Обновить все ссылки

Заменить старый путь на новый во всех документах.

### Шаг 5: Удалить старый файл

Только после проверки всех ссылок.

### Шаг 6: Валидация

```bash
python .instructions/.scripts/validate-instruction.py {path}
```

## Обновление ссылок

### Где искать

| Место | Glob-паттерн |
|-------|--------------|
| Инструкции | `**/.instructions/**/*.md` |
| Скиллы | `/.claude/skills/*/SKILL.md` |
| README | `**/README.md` |
| Агенты | `/.claude/agents/*.md` |
| Драфты | `/.claude/drafts/*.md` |

### Типы ссылок

**Markdown-ссылка:**
```
[текст](путь/к/инструкции.md)
```

**Таблица README:**
```
| [имя.md](./путь/имя.md) | описание |
```

### Команды поиска

```bash
# Поиск в markdown
grep -r "instruction-name.md" --include="*.md"

# Поиск в скиллах
grep -r "instruction-name" .claude/skills/
```

### Действия при обнаружении

| Тип ссылки | При деактивации | При миграции |
|------------|-----------------|--------------|
| Markdown | Удалить/пометить | Обновить путь |
| README таблица | Пометить неактивной | Обновить путь |
| Скилл | Добавить пометку | Обновить путь |

## Примеры

### Обновление: добавление правила

```bash
# 1. Найти: /.instructions/standard-api.md
# 2. Зона: /src/**/api/**
# 3. Добавить правило версионирования
# 4. Проверить API-файлы на соответствие
# 5. Обновить README, связанные инструкции
# 6. Валидация
python .instructions/.scripts/validate-instruction.py .instructions/standard-api.md
```

### Деактивация: устаревшая инструкция

```bash
# 1. Найти: /.instructions/standard-legacy-api.md
# 2. Найти ссылки: grep -r "standard-legacy-api.md" --include="*.md"
# 3. Добавить предупреждение, закомментировать содержимое
# 4. Обновить ссылающиеся документы
```

### Миграция: переименование

```bash
# 1. Найти ссылки: grep -r "naming.md" --include="*.md"
# 2. Копировать: naming.md → naming-conventions.md
# 3. Обновить frontmatter
# 4. Заменить ссылки: naming.md → naming-conventions.md
# 5. Удалить naming.md
# 6. Валидация
python .instructions/.scripts/validate-instruction.py .instructions/naming-conventions.md
```
