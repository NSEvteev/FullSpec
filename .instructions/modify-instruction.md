## 1. Обновление инструкции

### Когда обновлять

- Изменились требования проекта
- Найдены неточности
- Добавляются новые правила
- Правила устарели

### Шаги обновления

```
1. Найти инструкцию
   └── {область}/.instructions/{name}.md

2. Определить зону ответственности
   └── Какие файлы проекта затрагивает инструкция?

3. Внести изменения
   └── Обновить правила, примеры

4. Проверить файлы проекта
   └── Соответствуют ли файлы новым правилам?

5. Обновить связанные документы
   ├── README области
   ├── Связанные инструкции (related)
   └── Скиллы

6. Валидировать
   └── python validate-instruction.py {path}
```

### Маппинг зон ответственности

| Путь инструкции | Зона ответственности |
|-----------------|---------------------|
| `src/**/*.md` | `/src/**` |
| `tests/**/*.md` | `/tests/**`, `**/*.test.*` |
| `git/**/*.md` | Git операции |
| `api/**/*.md` | `/src/**/api/**` |

### Проверка соответствия проекта

```
1. Найти файлы в зоне
   └── Glob по паттерну зоны

2. Извлечь правила из инструкции
   └── Структурные, форматные, содержательные

3. Проверить файлы
   └── Применить правила к каждому файлу

4. Сформировать список несоответствий
   └── [файл, правило, описание]

5. Решение
   ├── Обновить инструкцию под реальность
   └── Исправить файлы под инструкцию
```

## 2. Деактивация инструкции

### Принцип

**Инструкции НЕ удаляются. Они деактивируются.**

- Сохраняется история
- Можно реактивировать
- Ссылки остаются валидными (с пометкой)

### Когда деактивировать

- Инструкция больше не актуальна
- Технология/практика не используется
- Инструкция заменена другой

### Шаги деактивации

```
1. Найти инструкцию
   └── {область}/.instructions/{name}.md

2. Найти все ссылки
   └── Использовать паттерны поиска (секция 4)

3. Добавить предупреждение
   └── После frontmatter

4. Закомментировать содержимое
   └── HTML-комментарий <!-- ... -->

5. Обновить ссылающиеся документы
   ├── Удалить из related
   ├── Пометить в README
   └── Обновить скиллы

6. Сохранить файл
   └── Frontmatter остаётся незакомментированным
```

### Формат деактивированной инструкции

```markdown
---
description: {оригинальное описание}
standard: .instructions/standard-instruction.md
index: {область}/.instructions/README.md
---

> **ВАЖНО:** Эта инструкция не используется в проекте.
>
> Содержимое закомментировано. Для реактивации — раскомментировать.

<!--
# Оригинальный заголовок

{Оригинальное содержимое}

## Секция 1
...
-->
```

## 3. Миграция инструкции

### Когда мигрировать

- Переименование файла
- Перемещение в другую папку
- Реструктуризация области

### Шаги миграции

```
1. Найти все ссылки на старый путь
   └── Паттерны поиска (секция 4)

2. Создать новый файл
   └── Копировать содержимое

3. Обновить frontmatter
   └── index, related

4. Обновить все ссылки
   └── Заменить старый путь на новый

5. Удалить старый файл
   └── После проверки всех ссылок

6. Валидировать
   └── Все ссылки работают
```

## 4. Паттерны поиска ссылок

### Где искать

| Место | Glob-паттерн |
|-------|--------------|
| Инструкции | `**/.instructions/**/*.md` |
| Скиллы | `/.claude/skills/*/SKILL.md` |
| README | `**/README.md` |
| Агенты | `/.claude/agents/*.md` |
| Драфты | `/.claude/drafts/*.md` |

### Типы ссылок

**Markdown-ссылка:**
```
[текст](путь/к/инструкции.md)
```

**Frontmatter related:**
```yaml
related:
  - путь/к/инструкции.md
```

**Таблица README:**
```
| [имя.md](./путь/имя.md) | описание |
```

### Команды поиска

```bash
# Поиск в markdown
grep -r "instruction-name.md" --include="*.md"

# Поиск в скиллах
grep -r "instruction-name" .claude/skills/

# Поиск в frontmatter
grep -A 5 "related:" **/*.md | grep "instruction-name"
```

### Действия при обнаружении

| Тип ссылки | При деактивации | При миграции |
|------------|-----------------|--------------|
| Markdown | Удалить/пометить | Обновить путь |
| related | Удалить из массива | Обновить путь |
| README таблица | Пометить неактивной | Обновить путь |
| Скилл | Добавить пометку | Обновить путь |
