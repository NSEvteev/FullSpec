## Требования к полноте описания

### Общие требования
- **Название файла:** Должно отсылаться к кодовой базе
- **Полнота описания:** Достаточная для понимания функционала LLM
- **Перечень компонентов:** Все модули, функции, классы, включенные в ресурс
- **Локальная специфика:** После начала разработки переносится в [📖 Документацию папок](../../../general_docs/glossary.md#документация-папок)
- **Связи с архитектурой:** Обязательные ссылки на файлы в `02_architecture/`

## Правила обратной связи ([📖 Feedback](../../../general_docs/glossary.md#feedback))

**При создании нового ресурса:**
1. Добавить ресурс в `05_resources/[тип]/`
2. Обновить индекс `000_[тип].md`
3. Добавить ссылку на ресурс в соответствующий файл архитектуры
4. Запустить Feedback для связанных дискуссий

**При изменении описания ресурса:**
1. Обновить дату последнего обновления
2. Проверить актуальность ссылок на архитектуру
3. Обновить документацию папок (если ресурс уже реализован)
4. При существенных изменениях — обновить архитектуру

**Чек-лист проверки:**
- [ ] Обновлен индекс (000_[тип].md)
- [ ] Проверены ссылки на архитектуру
- [ ] Обновлены связанные документы
- [ ] Все обязательные секции заполнены
- [ ] Соблюдены требования к полноте описания

## Критерии качества описания

**Минимальный уровень (для статуса draft):**
- Название и назначение
- Основные компоненты
- Связь с архитектурой

**Рабочий уровень (для статуса in_progress):**
- Все обязательные секции заполнены
- Описаны основные характеристики
- Указаны зависимости

**Финальный уровень (для статуса final):**
- Полное соответствие требованиям к полноте
- Все ссылки рабочие
- Актуальность подтверждена реализацией
- Синхронизация с документацией папок

## Частые ошибки

**1. Неполное описание компонентов**
- ❌ "Сервис обрабатывает запросы"
- ✅ "Сервис обрабатывает HTTP-запросы через endpoints: /auth/login, /auth/register, /auth/refresh"

**2. Отсутствие связей с архитектурой**
- ❌ Описание ресурса без ссылок на 02_architecture/
- ✅ Явные ссылки на архитектурные решения

**3. Устаревшая информация**
- ❌ Описание не обновляется при изменении кода
- ✅ Регулярная синхронизация с реализацией
