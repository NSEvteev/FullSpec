## 7. Улучшения workflow скиллов

### Проблемы текущего workflow

| Проблема | Скилл | Решение |
|----------|-------|---------|
| Слишком много шагов | `skill-create` (11+ шагов) | Разбить на фазы |
| Нет отката | Все скиллы | Добавить `--dry-run` |
| Нет batch-операций | `skill-create` | Добавить `--batch` |
| Асимметричность | `issue-execute` → `issue-review` | Нет `--retry` для возврата |
| Блокирующие шаги | `skill-create` Шаг 6 | Добавить `--auto-approve` |

### Конкретные улучшения

1. **Добавить `--dry-run`** во все скиллы, меняющие файлы
2. **Добавить `--auto-approve`** для CI/CD контекста
3. **Добавить `--verbose`** для отладки
4. **Унифицировать exit codes** — 0 (успех), 1 (ошибка валидации), 2 (ошибка выполнения)
5. **Добавить `--json`** для машиночитаемого вывода

### Workflow: параллельное выполнение

```
Текущее (sequential):
skill-create → Шаг 9a (links-update) → Шаг 9b (skill-update)

Оптимизированное (parallel):
skill-create → [Шаг 9a (links-update) || Шаг 9b (skill-update)]
```
