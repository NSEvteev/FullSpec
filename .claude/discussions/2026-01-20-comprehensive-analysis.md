## 🔧 ДОПОЛНИТЕЛЬНО: Расширение doc-* скиллов (2026-01-20)

doc-* скиллы работали только с файлами в `/src/`, что ограничивало их применимость.

### Решение

Расширен маппинг путей для работы с ЛЮБЫМИ файлами проекта, кроме исключённых.

### Изменённые файлы

| Файл | Изменения |
|------|-----------|
| `/.claude/templates/doc-rules.md` | Новый маппинг `/{any}/ → /doc/{any}/`, исключения, конфиг-файлы |
| `/.claude/skills/doc-create/SKILL.md` | Новые примеры, исключения, валидация |
| `/.claude/skills/doc-update/SKILL.md` | Новые примеры, исключения, валидация |
| `/.claude/skills/doc-delete/SKILL.md` | Новые примеры, исключения, валидация |

### Правило маппинга

**КАК БЫЛО:**
```
/src/{service}/{path}.{ext} → /doc/src/{service}/{path}.md
```

**КАК СТАЛО:**
```
/{any-path}/{file}.{ext} → /doc/{any-path}/{file}.md
```

**Новые примеры:**

| Исходный файл | Документация |
|---------------|--------------|
| `/config/settings.yaml` | `/doc/config/settings.md` |
| `/platform/docker/compose.yml` | `/doc/platform/docker/compose.md` |
| `/Makefile` | `/doc/Makefile.md` |

### Исключённые пути

| Путь | Причина |
|------|---------|
| `/doc/**` | Целевая папка документации |
| `/.claude/**` | Уже содержит документацию |
| `/.git/**` | Служебные файлы Git |
| `*.md`, `*.rst`, `*.txt` | Самоописывающиеся файлы |

### Новая поддержка: конфигурационные файлы

| Расширение | Что документировать |
|------------|---------------------|
| `.yaml`, `.yml` | Структура, ключевые поля |
| `.json` | Схема, обязательные поля |
| `.toml`, `.ini` | Секции, параметры |
| `.env` | Переменные окружения (без значений!) |
| `Dockerfile` | Этапы сборки, порты |
| `Makefile` | Targets, переменные |
