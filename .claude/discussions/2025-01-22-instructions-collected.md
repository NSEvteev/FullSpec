# Сводка: Система инструкций

> **Цель документа:** Собрать всю информацию из скиллов instruction-* в одном месте для последующего формирования инструкций в /.claude/instructions/instructions/.

---

## 1. Общие концепции

### 1.1 Расположение инструкций

**Базовая папка:** /.claude/instructions/

**Принцип:** Инструкции зеркалируют структуру проекта:

| Папка проекта | Инструкции |
|---------------|------------|
| /src/ | /.claude/instructions/src/ |
| /platform/ | /.claude/instructions/platform/ |
| /tests/ | /.claude/instructions/tests/ |
| /doc/ | /.claude/instructions/doc/ |
| /shared/ | /.claude/instructions/shared/ |
| /config/ | /.claude/instructions/config/ |

**Внутренние инструкции:**
- /.claude/instructions/git/ - Git workflow
- /.claude/instructions/skills/ - скиллы
- /.claude/instructions/agents/ - агенты
- /.claude/instructions/specs/ - спецификации

**Допустимые папки (стандартные):**
src/, platform/, tests/, doc/, shared/, config/, git/, skills/, agents/, specs/

---

### 1.2 Типы инструкций

| Тип | Назначение | Когда использовать |
|-----|------------|-------------------|
| standard | Требования, КАК делать | Стандарты качества |
| project | Специфика, ЧТО есть | Описывает конкретный проект |

---

### 1.3 Формат названия файлов

**Правила:**
- Только латиница, цифры, дефис
- Расширение .md
- Примеры: design.md, error-handling.md

---

### 1.4 Связь инструкций со скиллами

| Скилл | Назначение |
|-------|------------|
| instruction-create | Создание новой инструкции |
| instruction-update | Проверка файлов на соответствие |
| instruction-delete | Очистка ссылок при удалении |

**Утилиты:** links-update, context-update, environment-check, input-validate

---

### 1.5 Индекс инструкций (README.md)

**Путь:** /.claude/instructions/README.md

| Столбец | Назначение |
|---------|------------|
| Файл | Ссылка на инструкцию |
| Описание | Краткое описание |
| Тип | standard / project |
| Создано | check / unchecked |
| Заполнено | check / unchecked |

**Важно:** Строки НЕ удаляются из индекса при удалении инструкции.

## 2. Из instruction-create (1079 строк)

### 2.1 Воркфлоу создания (13 шагов)

1. Получить путь
2. Fail-fast проверки (БЛОКИРУЮЩИЕ)
3. Проверка существования
4. Сгенерировать содержимое
5. Создать файл
6. Ревью
7. Обновить статусы
8. Синхронизация ссылок (/links-update)
9. Обновить контекст (/context-update)
10. Проверить соответствие проекта (/instruction-update)
11. Обновление связанных скиллов (ОБЯЗАТЕЛЬНО)
12. Анализ и предложение новых скиллов
13. Результат

**Формат вызова:**
/instruction-create <путь> [--dry-run] [--auto]

---

### 2.2 Fail-fast проверки

**Скрипт:** python .claude/scripts/instruction-validate.py {путь}

| Проверка | При ошибке |
|----------|------------|
| Папка существует | Сообщить об ошибке |
| Путь допустим | Предупреждение |
| Расширение .md | Блокировать |

---

### 2.3 Ревью инструкции

**Что проверять:**
- Frontmatter: type, description, related
- Структура: соответствие шаблону
- Содержание: полнота правил, качество примеров

---

### 2.4 Обновление связанных скиллов

**Правило:** Скиллы, связанные с областью инструкции, ДОЛЖНЫ быть обновлены.

| Путь инструкции | Связанные скиллы |
|-----------------|------------------|
| git/issues.md | issue-* |
| git/workflow.md | issue-execute |
| doc/structure.md | doc-* |

## 3. Из instruction-delete (512 строк)

### 3.1 Воркфлоу удаления (8 шагов)

1. Получить путь к инструкции
2. Проверить факт удаления
3. Найти ссылки
4. Сформировать предложения
5. Применить изменения
6. Обновить контекст
7. Проверка по чек-листу
8. Результат

---

### 3.2 Паттерны поиска ссылок

| Паттерн | Тип ссылки | Действие |
|---------|------------|----------|
| Markdown-ссылка | [text](path) | Удалить или пометить |
| Frontmatter related | related: | Удалить из массива |
| Строка таблицы | check marks | Заменить на unchecked |

---

### 3.3 Правила: что удалять, что оставлять

| Что | Действие |
|-----|----------|
| Индекс README.md | Сбросить статусы (НЕ удалять строку!) |
| Связанные инструкции | Удалить из раздела |
| Скиллы | Удалить или пометить *(не создан)* |
| Frontmatter related | Удалить элемент |

---

## 4. Из instruction-update (558 строк)

### 4.1 Воркфлоу обновления (11 шагов)

1. Получить путь к инструкции
2. Определить зону ответственности
3. Найти файлы в зоне
4. Извлечь правила из инструкции
5. Проверить файлы на соответствие
6. Сформировать список несоответствий
7. Добавить раздел TODO в инструкцию
8. Обновить контекст связанных скиллов
9. Проверить связанные тесты
10. Проверка по чек-листу
11. Результат

---

### 4.2 Зоны ответственности

| Путь инструкции | Зона |
|-----------------|------|
| src/api/*.md | /src/**/api/**, /src/**/routes/** |
| src/data/*.md | /src/**/*.ts, /src/**/*.py |
| platform/*.md | /platform/** |
| config/*.md | /config/** |
| git/*.md | /.github/**, .git* |

## 6. Философия инструкций

### 6.1 Инструкции НЕ удаляются

**Принцип:** Инструкция СУЩЕСТВУЕТ и содержит требования для проекта.

### 6.2 Неактивные инструкции

Если не используется:
1. НЕ удалять файл
2. Добавить предупреждение
3. Закомментировать содержимое
4. Сохранить frontmatter

### 6.3 instruction-delete -> instruction-deactivate

Переименовать скилл для деактивации.
