## Инструкция по работе с этим файлом

**При исправлении пункта:**

1. Поставить галочку `[x]` в чекбоксе
2. Удалить описание проблемы из соответствующего раздела
3. Добавить комментарий `<!-- ИСПРАВЛЕНО: дата, что сделано -->`

**Пример:**

```markdown
До:
- [ ] Устранить дублирование в CLAUDE.md
| CLAUDE.md (строки 129-146) | README.md | Структура проекта | SSOT в README.md |

После:
- [x] Устранить дублирование в CLAUDE.md
<!-- ИСПРАВЛЕНО: 2025-01-21, удалена структура из CLAUDE.md, добавлена ссылка на README.md -->
```

**Когда все пункты раздела исправлены:**
- Удалить раздел целиком
- Или заменить на `✅ Раздел полностью исправлен`

## 5. Использование сущностей в других проектах

### Что можно переиспользовать

| Сущность | Готовность | Как использовать |
|----------|------------|------------------|
| Скиллы (37 шт) | ✅ Готовы | Копировать .claude/skills/ |
| Шаблоны (6 шт) | ✅ Готовы | Копировать .claude/templates/ |
| Инструкции standard (45 шт) | ✅ Готовы | Копировать, адаптировать project |
| Скрипты Python (2 шт) | ✅ Готовы | Копировать .claude/scripts/ |
| CLAUDE.md | ⚠️ Частично | Удалить project-специфику |

### Что нужно для переиспользования

1. **Разделить CLAUDE.md** на:
   - `CLAUDE-template.md` — универсальная часть
   - `CLAUDE-project.md` — специфика проекта

2. **Создать инструкцию project-init**:
   ```
   /project-init
   → Копирует .claude/ в новый проект
   → Создаёт CLAUDE.md из шаблона
   → Генерирует instructions/project/* пустыми
   ```

3. **Пометить инструкции**:
   - `type: standard` — копировать as-is
   - `type: project` — заполнить под проект

### Контекстные группы

```
Группа "Разработка сервиса":
├── instructions/src/** (правила)
├── doc/src/{service}/** (документация)
├── src/{service}/** (код)
└── Скиллы: doc-create, test-create

├── instructions/git/** (правила)
├── Скиллы: issue-*, /commit (если добавить)
└── .github/workflows/** (CI)

Группа "Тестирование":
├── instructions/tests/** (правила)
├── tests/** (системные тесты)
├── src/{service}/tests/** (unit тесты)
├── .claude/skills/*/tests.md (тесты скиллов)
└── Скиллы: test-*

Группа "Документация":
├── instructions/doc/** (правила)
├── doc/** (документы)
├── templates/doc-rules.md (SSOT)
└── Скиллы: doc-*, links-*
```

## 7. Улучшения workflow скиллов

### Текущие проблемы

| Проблема | Пример | Решение |
|----------|--------|---------|
| Нет авто-валидации | skill-create не проверяет дубли | Добавить шаг валидации |
| Нет отката | doc-update не откатывает при ошибке | Использовать error-handling.md |
| Нет логирования | Скиллы не пишут в лог | Добавить audit trail |
| Нет dry-run везде | Не все скиллы поддерживают | Стандартизировать |

### Предложения

1. **Добавить pre-conditions во все скиллы**:
   ```markdown
   ## Pre-conditions
   - [ ] Файл {path} существует
   - [ ] Git working tree чистый
   - [ ] Нет конфликтующих процессов
   ```

2. **Стандартизировать post-conditions**:
   ```markdown
   ## Post-conditions
   - [ ] Файл создан/обновлён
   - [ ] Ссылки обновлены
   - [ ] README обновлён
   - [ ] Нет битых ссылок
   ```

3. **Добавить rollback во все скиллы**:
   ```markdown
   ## Rollback
   При ошибке на шаге N:
   1. Восстановить файлы из git stash
   2. Удалить созданные файлы
   3. Вывести ошибку
   ```
