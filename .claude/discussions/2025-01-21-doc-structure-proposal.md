# 1. Обзор (README)

Структура `/specs/`, связь с `/doc/`, принятые решения, граф зависимостей документов.

**Содержание:** структура /specs/, принятые решения, связи между документами, структура /doc/, связь /specs/ ↔ /doc/, обратные ссылки (backlinks).

**Контекст решения:** Рефакторинг системы документации. Выбор между старым (llm_instructions_old) — сложным многоуровневым workflow, и текущим (structure.md) — упрощённым с зеркалированием.

**Инструкция:** [/.claude/instructions/specs/README.md](/.claude/instructions/specs/README.md)

---

# 2. Статусы (statuses.md)

Унифицированная система статусов для всех типов документов в `/specs/`: Discussion, Impact, ADR, Plan.

**Содержание:** все статусы, схема переходов, правила по типам, каскадные проверки, правила REJECTED.

**Инструкция:** [/.claude/instructions/specs/statuses.md](/.claude/instructions/specs/statuses.md)

---

# 3. Workflow (workflow.md)

Полный workflow от идеи до реализации: Discussion → Impact → ADR (параллельно) → Plan → Реализация + Документация.

**Содержание:** 6 фаз, точка синхронизации, полная диаграмма переходов.

**Инструкция:** [/.claude/instructions/specs/workflow.md](/.claude/instructions/specs/workflow.md)

---

# 4. Discussions (discussions.md)

Точка входа для новых идей и изменений. Исследование проблемы, сбор информации, принятие решения.

**Содержание:** расположение, формат, статусы, чек-листы переходов (DRAFT→REVIEW→APPROVED→RUNNING→DONE).

**Инструкция:** [/.claude/instructions/specs/discussions.md](/.claude/instructions/specs/discussions.md)

---

# 5. Impact (impact.md)

Импакт-анализ определяет какие сервисы затронуты изменением. Один Impact может создать несколько ADR.

**Содержание:** расположение, формат, статусы, создание нового сервиса, связь Impact↔ADR, чек-листы переходов.

**Инструкция:** [/.claude/instructions/specs/impact.md](/.claude/instructions/specs/impact.md)

---

# 6. ADR (adr.md)

Architecture Decision Records — фиксация архитектурных решений для конкретного сервиса. Каждый ADR ссылается на родительский Impact.

**Содержание:** расположение, формат, статусы, проверка бизнес-логики, проверка конфликтов, Breaking Changes, чек-листы переходов.

**Инструкция:** [/.claude/instructions/specs/adr.md](/.claude/instructions/specs/adr.md)


# 7. Plans (plans.md)

Декомпозиция ADR на конкретные задачи. План требует согласования с пользователем перед реализацией.

**Содержание:** статусы планов, согласование с пользователем, автоматическое создание GitHub Issues, версионирование планов (v2, v3), критерий готовности, чек-листы переходов.

**Инструкция:** [/.claude/instructions/specs/plans.md](/.claude/instructions/specs/plans.md)

---

# 8. Architecture (architecture.md)

Живой документ — описание текущего состояния сервиса. Каждый раздел ссылается на ADR, который его ввёл.

**Содержание:** расположение, формат документа, правила ссылок на ADR, workflow обновления при ADR → DONE.

**Особенность:** Без статусов (DRAFT/REVIEW/DONE). Обновляется автоматически при завершении ADR.

**Инструкция:** [/.claude/instructions/specs/architecture.md](/.claude/instructions/specs/architecture.md)

---

# 9. Glossary (glossary.md)

Единое место для терминов проекта. Обеспечивает консистентность терминологии во всех документах.

**Содержание:** структура термина, категории, пороги масштабирования, правила ведения, скиллы (`/glossary-candidates`, `/glossary-review`, `/glossary-link`).

**Особенность:** Без статусов. Управляется отдельными скиллами, не входящими в систему `/spec-*`.

**Инструкция:** [/.claude/instructions/specs/glossary.md](/.claude/instructions/specs/glossary.md)

---

# 10. Правила

Правила автоматизации и ограничения для работы с документами `/specs/`.

**Содержание:** три универсальных скилла (`/spec-create`, `/spec-status`, `/spec-update`), служебные скиллы (`/specs-health`, `/specs-sync`, `/specs-index`), hook защиты, шаблоны, запреты (миграции, удаления, архивирования).

**Инструкция:** [/.claude/instructions/specs/rules.md](/.claude/instructions/specs/rules.md)
