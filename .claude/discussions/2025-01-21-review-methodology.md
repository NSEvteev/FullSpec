# Методология ревью папок проекта

> **Статус:** В процессе
> **Ветка:** `refactoring-docs`
> **Дата начала:** 2025-01-21

---

## Цель

Систематический ревью и рефакторинг инструкций для каждой папки проекта:
- Консолидация разрозненных инструкций
- Вынос шаблонов и примеров заполнения шаблонов в отдельные файлы
- Создание структурированных инструкций в `/.claude/instructions/`

---

## Алгоритм ревью папки

### Фаза 1: Сбор

1. [ ] Собрать все инструкции для папки в один файл (Discussion)
2. [ ] Определить scope: какие сущности описываются
3. [ ] Выявить дублирование и противоречия

### Фаза 2: Валидация

4. [ ] Провалидировать все сущности инструкций
5. [ ] Проверить актуальность (соответствие текущему коду)
6. [ ] Отметить устаревшие секции

### Фаза 3: Структурирование

7. [ ] Определить будущие инструкции (таблица навигации)
8. [ ] Вынести шаблоны и примеры заполнения шаблонов в `/.claude/templates/{scope}/`
9. [ ] Заменить inline-шаблоны и их примеры на ссылки в документе
10. [ ] Перенести чек-листы в соответствующие секции

### Фаза 4: Создание инструкций

11. [ ] Создать инструкции через `/instruction-create`
12. [ ] Перенести контент из Discussion в инструкции
13. [ ] Обновить ссылки (через `/links-update`)

### Фаза 5: Завершение

14. [ ] Перевести Discussion в статус DONE
15. [ ] Удалить временные файлы (если есть)
16. [ ] Обновить `/.claude/instructions/README.md`

## Принципы

1. **Один документ = один scope** — не смешивать инструкции для разных папок
2. **Шаблоны отдельно** — всегда выносить в `/.claude/templates/`
3. **Ссылки вместо дублирования** — один источник истины
4. **Инкрементальность** — можно прервать и продолжить
5. **Индексы как источник формата** — после создания реальных файлов заменить примеры в документации на ссылки

---

## Примеры действий

### Замена примера формата на ссылку

**Было (в doc-structure-proposal.md):**
```markdown
## 4.5. Формат README.md

\`\`\`markdown
# Дискуссии

## Все дискуссии

| # | Тема | Статус | Impact | Дата |
...
\`\`\`
```

**Стало:**
```markdown
## 4.5. Формат README.md

> **Индекс:** [/specs/discussions/README.md](/specs/discussions/README.md)
```

**Почему:** Реальный файл `/specs/discussions/README.md` теперь является источником истины для формата. Документация ссылается на него, а не дублирует.
