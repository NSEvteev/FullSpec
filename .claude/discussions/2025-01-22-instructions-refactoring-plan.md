# План: Создание /.claude/instructions/instructions/

## Цель

Создать папку `/.claude/instructions/instructions/` для регламентирования работы с инструкциями:
- Собрать общие концепции из трёх скиллов instruction-*
- Устранить дублирование между скиллами
- Создать единый источник правды (SSOT)
- Вынести шаблоны и примеры в `/.claude/templates/instructions/`

---

## ФАЗА 2: Формирование инструкций

### Целевая структура папки

```
/.claude/instructions/instructions/
├── README.md           # Индекс и навигация
├── workflow.md         # Жизненный цикл: CREATE → UPDATE → DELETE
├── rules.md            # Правила и ограничения
├── structure.md        # Расположение и допустимые папки
├── types.md            # Типы инструкций (standard/project)
└── scripts.md          # Скрипты автоматизации
```

### 2.1 README.md — индекс

**Содержимое:**
- Назначение папки instructions/
- Оглавление с описанием каждого файла
- Таблица скиллов для работы с инструкциями
- Быстрый старт (команды)
- Ссылки на шаблоны

### 2.2 workflow.md — жизненный цикл

**Содержимое:**
- Диаграмма: CREATE → UPDATE → DELETE
- Фаза CREATE:
  - Краткое описание (13 шагов → сводка)
  - Ссылка на полный воркфлоу в instruction-create/SKILL.md
- Фаза UPDATE:
  - Краткое описание (11 шагов → сводка)
  - Ссылка на instruction-update/SKILL.md
- Фаза DELETE:
  - Краткое описание (8 шагов → сводка)
  - Ссылка на instruction-delete/SKILL.md
- Граф зависимостей между скиллами

### 2.3 rules.md — правила и ограничения

**Содержимое:**
- Формат названия файлов (латиница, дефис, .md)
- Обязательные секции инструкции
- Система статусов в README.md (✅/⬜)
- Правила удаления (почему строки не удаляются из индекса)
- Паттерны поиска ссылок при удалении
- Запреты и ограничения

### 2.4 structure.md — расположение

**Содержимое:**
- Базовая папка `/.claude/instructions/`
- Зеркалирование структуры проекта
- Таблица допустимых папок
- Маппинг инструкций на зоны ответственности

### 2.5 types.md — типы инструкций

**Содержимое:**
- Таблица типов (standard/project)
- Как определить тип из README.md
- Когда использовать какой тип
- Примеры каждого типа

### 2.6 scripts.md — скрипты автоматизации

**Содержимое:**
- instruction-validate.py — валидация пути
- instruction-readme-update.py — обновление README.md
- instruction-stats.py — статистика

## ВАЖНО: Философия инструкций

### Инструкции НЕ удаляются

**Принцип:** Инструкция СУЩЕСТВУЕТ и ТОЧНО содержит требования для ДАННОГО проекта. Её ВСЕГДА нужно учитывать.

### Неактивные инструкции

Если инструкция не используется в проекте:

1. **НЕ удалять файл**
2. **Добавить предупреждение:**
   ```markdown
   > **ВАЖНО:** В проекте не используется {функциональность}.
   ```
3. **Закомментировать содержимое:** `<!-- ... -->`
4. **Сохранить frontmatter** (type, description, related)

**Пример:** `shared/i18n.md` — локализация не используется, но инструкция сохранена.

### Скилл instruction-delete

**Статус:** Создан преждевременно.

**Варианты:**
1. Переименовать в `instruction-deactivate` — деактивация инструкции
2. Убрать из системы — редкий случай, делается вручную
3. Оставить для очистки ссылок при деактивации

**Решение:** Требует обсуждения с пользователем.

## РЕШЕНИЕ: instruction-delete → instruction-deactivate

### Переименование

```
instruction-delete → instruction-deactivate
```

### Изменения в скилле

**Убрать:**
- Шаг "Найти ссылки" — инструкция существует, ссылки валидны
- Шаг "Удалить ссылки" — не нужен
- Шаг "Сбросить статусы в README.md" — не нужен (инструкция остаётся)
- Паттерны поиска ссылок — не нужны

**Оставить/Добавить:**
- Шаг "Добавить предупреждение" — `> **ВАЖНО:** В проекте не используется...`
- Шаг "Закомментировать содержимое" — `<!-- ... -->`
- Шаг "Сохранить frontmatter"

### Новый воркфлоу instruction-deactivate

```
1. Получить путь к инструкции
2. Проверить существование файла
3. Прочитать текущее содержимое
4. Добавить предупреждение после frontmatter
5. Закомментировать содержимое (кроме frontmatter и предупреждения)
6. Сохранить файл
7. Результат
```
