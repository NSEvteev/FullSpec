# Драфт: Формат навигации в документах

## Проблема

Текущий формат навигации в файлах инструкций:
```markdown
**Индекс:** [/.instructions/README.md](/.instructions/README.md) | **Папка:** [.structure/.instructions/README.md](./README.md)
```

**Что не так:**
- "Индекс" и "Папка" — непонятные термины
- Линейный формат не показывает иерархию
- Формат должен быть единым для README папок и инструкций

---

## Варианты

> **Требование:** Единый формат для README.md папок И для инструкций.

---

### Вариант 1: Мини-дерево

**Для README папки** (`/.structure/.instructions/README.md`):
```markdown
# Инструкции /.structure/

Индекс инструкций для SSOT структуры проекта.

```
📍 /.structure/.instructions/
   └── README.md ← вы здесь
```
```

**Для инструкции** (`/.structure/.instructions/format.md`):
```markdown
# Формат описания папок

Правила оформления описаний папок.

```
📍 /.structure/.instructions/
   └── format.md ← вы здесь
```
```

| Плюсы | Минусы |
|-------|--------|
| Визуально понятная иерархия | Занимает 3-4 строки |
| Консистентно с деревом в README | Code block может выглядеть громоздко |

---

### Вариант 2: Хлебные крошки

**Для README папки**:
```markdown
# Инструкции /.structure/

Индекс инструкций для SSOT структуры проекта.

📍 `/.structure/.instructions/README.md`
```

**Для инструкции**:
```markdown
# Формат описания папок

Правила оформления описаний папок.

📍 `/.structure/.instructions/` → `format.md`
```

| Плюсы | Минусы |
|-------|--------|
| Компактно (1 строка) | Менее наглядно |
| Легко копировать путь | Стрелка → не очень консистентна |

---

### Вариант 3: Полный путь со ссылкой

**Для README папки**:
```markdown
# Инструкции /.structure/

Индекс инструкций для SSOT структуры проекта.

📍 [/.structure/.instructions/](.)
```

**Для инструкции**:
```markdown
# Формат описания папок

Правила оформления описаний папок.

📍 [/.structure/.instructions/](./README.md) / format.md
```

| Плюсы | Минусы |
|-------|--------|
| Кликабельная ссылка на папку | Смешение ссылки и текста |
| Минимум визуального шума | Не сразу понятно что кликабельно |

---

### Вариант 4: Blockquote (как для README папок)

Использовать тот же формат, что уже есть для README папок проекта.

**Для README папки** (`/.structure/.instructions/README.md`):
```markdown
# Инструкции /.structure/

Индекс инструкций для SSOT структуры проекта.

> [Разделение ответственности при работе с .structure](../README.md)
```

**Для инструкции** (`/.structure/.instructions/format.md`):
```markdown
# Формат описания папок

Правила оформления описаний папок.

> [Инструкции .structure](./README.md)
```

| Плюсы | Минусы |
|-------|--------|
| Полная консистентность с README папок | Не показывает полный путь к файлу |
| Уже знакомый формат | Ссылка только на родителя |
| Компактно (1 строка) | — |
| Кликабельно | — |
