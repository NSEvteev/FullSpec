## Рекомендации по улучшению

### 1. Отсутствует: порядок чтения документов

**Проблема:** 11 SSOT-ссылок без указания приоритета. Новый участник не знает с чего начать.

**Рекомендация:** Добавить секцию "Порядок изучения":
```markdown
## Порядок изучения (для новых участников)

1. **Обязательно:** standard-development-workflow.md — полный цикл
2. **Обязательно:** standard-labels.md — система категоризации
3. **По необходимости:** остальные стандарты (Issues, PR, Release)
```

### 2. Не описано: как обновляются стандарты

**Проблема:** Нет процедуры изменения стандартов.

**Рекомендация:** Добавить секцию "Обновление стандартов":
- Изменение стандарта — через PR
- Review от владельца репозитория
- Уведомление команды после merge
- Версионирование: `Версия стандарта: X.Y` в каждом документе

### 3. Отсутствует: глоссарий терминов

**Проблема:** Термины (Issue, PR, Milestone, Release) используются без определений.

**Рекомендация:** Добавить секцию или ссылку на глоссарий:
| Термин | Определение |
|--------|-------------|
| Issue | Задача, баг или запрос функциональности |
| PR | Pull Request — запрос на слияние кода |
| Milestone | Группировка Issues для спринта/релиза |
| Release | Публикация версии с тегом и changelog |

### 4. Не описано: исключения из правил

**Проблема:** Принципы сформулированы как абсолютные правила, но есть исключения.

**Рекомендация:** Добавить примечания к принципам:
- "Прямые коммиты в main запрещены" → исключение: экстренный hotfix с post-review
- "Squash Merge по умолчанию" → исключение: release branches

### 5. Неясность: что такое "сложная структура JSON" для Branch Protection

**Проблема:** Написано "Web UI допустим для Branch Protection (сложная структура JSON)", но не объяснено почему.

**Рекомендация:** Убрать или пояснить:
- CLI поддерживает настройку Branch Protection через `gh api`
- Web UI удобнее для первоначальной настройки
- CLI для автоматизации/повторения настроек

### 6. Не описано: связь с другими системами

**Проблема:** GitHub может интегрироваться с Jira, Slack, etc. Нет упоминания.

**Рекомендация:** Добавить секцию "Интеграции" или примечание:
- Основная система: GitHub (Issues, Projects)
- Интеграции: настраиваются при необходимости
- Ссылка: GitHub Marketplace → Integrations

### 7. Мелкое: типичные ошибки неполные

**Проблема:** Только 3 ошибки в таблице.

**Рекомендация:** Добавить или сделать ссылку на расширенный troubleshooting:
- `error: cannot lock ref` → другой процесс использует репо
- `Your branch is behind` → `git pull`
- `merge conflict` → разрешить конфликты вручную
