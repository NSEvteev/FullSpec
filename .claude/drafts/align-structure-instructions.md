## Ключевые концепции

### SSOT (Single Source of Truth)

**SSOT** — единственный источник истины. Документ, содержащий эталонное описание.

> **КРИТИЧЕСКОЕ ПРАВИЛО:** Если в описании действия есть ссылка на SSOT — Claude **ОБЯЗАН** прочитать этот SSOT перед выполнением. Нельзя выполнять действие по памяти или предположениям.

**Примеры SSOT:**
- `/.structure/README.md` — SSOT структуры проекта
- `create-structure.md` — SSOT процедуры создания папки
- `standard-readme.md` — SSOT формата README

### Правило использования стандартов

> **ПРАВИЛО:** Стандарт не может существовать изолированно. Каждый стандарт **ДОЛЖЕН** быть включён как минимум в один из процессов:
> - **Валидация** — проверка соответствия стандарту
> - **Создание** — применение стандарта при создании
> - **Модификация** — соблюдение стандарта при изменении

**Связь стандарт → процесс:**

| Стандарт | Валидация | Создание | Модификация |
|----------|-----------|----------|-------------|
| standard-readme.md | validation-structure | create-structure | modify-structure |
| standard-links.md | validation-links | create-structure | modify-structure |
| standard-frontmatter.md | validation-links | create-structure | modify-structure |

Если стандарт не используется ни в одном процессе — он бесполезен и должен быть удалён.

### Иерархия компонентов

```
┌─────────────────────────────────────────────────────────┐
│                    ИНСТРУКЦИИ (SSOT)                    │
│                                                         │
│  ┌──────────┐  ┌───────────┐  ┌─────────┐  ┌─────────┐ │
│  │ Стандарт │  │ Валидация │  │Создание │  │Изменение│ │
│  │(справка) │  │(воркфлоу) │  │(воркфлоу)│  │(воркфлоу)│ │
│  └────┬─────┘  └─────┬─────┘  └────┬────┘  └────┬────┘ │
└───────┼──────────────┼─────────────┼───────────┼───────┘
        │              │             │           │
        │              ▼             ▼           ▼
        │         ┌──────────────────────────────────────┐
        │         │              СКРИПТЫ                 │
        │         │      (автоматизация рутины)          │
        │         └──────────────────────────────────────┘
        │              │             │           │
        │              ▼             ▼           ▼
        │         ┌──────────────────────────────────────┐
        │         │              СКИЛЛЫ                  │
        │         │   (триггер + параметры + → SSOT)     │
        │         └──────────────────────────────────────┘
        │              │             │           │
        ▼              ▼             ▼           ▼
┌─────────────────────────────────────────────────────────┐
│                       RULES                             │
│              (автозагрузка по paths)                    │
│                                                         │
│  - Ссылаются на скиллы (для воркфлоу)                  │
│  - Могут ссылаться на стандарты (справочно)            │
└─────────────────────────────────────────────────────────┘
```

**Принцип:** Каждый уровень ссылается на предыдущий, не дублирует.

| Тип инструкции | Скрипт | Скилл | Rule ссылается на |
|----------------|--------|-------|-------------------|
| Стандарт | — | — | Стандарт напрямую |
| Валидация | ✅ | ✅ | Скилл |
| Создание | ✅ | ✅ | Скилл |
| Изменение | ✅ | ✅ | Скилл |

### Формат скилла (облегчённый)

Скилл **НЕ дублирует** шаги из инструкции. Скилл содержит только:

```markdown
---
name: skill-name
triggers: ...
---

# Название

**SSOT:** [инструкция.md](/путь/к/инструкции.md)

## Формат вызова
/skill-name <параметры>

## Воркфлоу
> ⚠️ **Перед выполнением** прочитать SSOT-инструкцию
→ Выполнить шаги из SSOT.

## Чек-лист
→ См. [инструкция.md#чек-лист](/путь/к/инструкции.md#чек-лист)

## Примеры
```

# Фаза 4: Rules (отложено)

## Концепция

Rules автоматически загружаются при работе с файлами по паттерну `paths`.

**Важно:** Rules ссылаются на **скиллы**, не на инструкции напрямую.

```
Пользователь работает с .structure/
         ↓
Rule автоматически загружается (paths match)
         ↓
Rule: "Используй /structure-create или /structure-modify"
         ↓
Skill читает SSOT-инструкцию и выполняет
         ↓
В конце → /structure-validate, /links-validate
```

## Структура

```
.claude/
├── .instructions/
│   └── rules/                    # Инструкции: как писать rules
│       ├── README.md
│       ├── standard-rules.md
│       ├── create-rules.md
│       └── ...
└── rules/
    └── structure.md              # Rule → скиллы structure-*
```

## Формат rule

```markdown
---
paths:
  - ".structure/**"
  - "**/README.md"
---

# Структура проекта

При создании папки:
→ `/structure-create`

При изменении папки:
→ `/structure-modify`

После любых изменений:
→ `/structure-validate`
→ `/links-validate`
```
