# Фаза 1: Инструкции структуры

## Объект "Структура" и его свойства

```
Объект: Структура проекта
├── Свойство: Папки (folders)
├── Свойство: README
├── Свойство: Ссылки (links)
└── Свойство: Frontmatter
```

> **Примечание:** Дерево и оглавление — части README, не отдельные свойства.
> SSOT (`.structure/README.md`) описывается правилами для README папок проекта.

### 2. Секция "Скиллы и скрипты"

- **Стандарты:** УДАЛИТЬ (не создают инструменты)
- **Валидации:** ОСТАВИТЬ (создают скрипты/скиллы)
- **Создание/Модификация:** ОСТАВИТЬ если есть скрипты

### 3. Формат таблиц "Скиллы" и "Скрипты"

Таблица с 3 колонками вместо таблицы + блок `**Использует:**`:

```markdown

|--------------|------------|------------|
| [ссылка] | описание | [инструкция.md](...) |
```

### 4. Секция "Шаблоны"

**УДАЛИТЬ** из раздела "Ресурсы" — шаблоны не используются, есть только стандарты.

### 5. Секция "Обязательные обновления"

**УДАЛИТЬ** из README индексов — перенести в `standard-readme.md` как раздел 5 "Требования для обновления при изменении структуры".

# Фаза 4: Rules (отложено)

После завершения Фаз 1 и 2 — создать rules и инструкции по их написанию.

## Концепция

**Стандарты (standard-*.md)** = ЧТО (формат, структура, как должно выглядеть)
**Валидации, Создание, Модификация** = КАК (процедуры, действия)

Rule ссылается только на документы "КАК делать". Стандарты читаются из них через секцию "Связанные документы".

## Где что лежит

Rules — инструмент Claude Code (как skills, agents):

| Инструмент | Где лежит | Инструкция как писать |
|------------|-----------|----------------------|
| Skills | `.claude/skills/` | `.claude/.instructions/skills/` |
| Agents | `.claude/agents/` | `.claude/.instructions/agents/` |
| Rules | `.claude/rules/` | `.claude/.instructions/rules/` |

## Структура

```
.claude/
├── .instructions/
│   └── rules/                    # Инструкции: как работать с rules
│       ├── README.md             # Индекс
│       ├── standard-rules.md     # Стандарт: ЧТО (формат, структура rule)
│       ├── validation-rules.md   # Валидация: КАК проверить
│       ├── create-rules.md       # Создание: КАК создать rule
│       └── modify-rules.md       # Модификация: КАК изменить rule
└── rules/
    ├── structure.md              # Rule для структуры → .structure/.instructions/
    └── rules.md                  # Rule для rules → .claude/.instructions/rules/
```

## Формат rule-файла (пример для structure.md)

```markdown
---
paths:
  - "**/README.md"
  - "**/*.md"
  - "!node_modules/**"
---

# Структура проекта

Инструкции: [/.structure/.instructions/](/.structure/.instructions/README.md)

**При создании новой папки:**
```
