# Фаза 1: Инструкции структуры

## Объект "Структура" и его свойства

```
Объект: Структура проекта
├── Свойство: Папки (folders)
├── Свойство: README
├── Свойство: Ссылки (links)
└── Свойство: Frontmatter
```

> **Примечание:** Дерево и оглавление — части README, не отдельные свойства.
> SSOT (`.structure/README.md`) описывается правилами для README папок проекта.


# Фаза 2: Документация по скриптам

## Объект "Инструкции" и свойство "Скрипты"

Скрипты создаются для автоматизации инструкций, поэтому документация по скриптам живёт в папке инструкций, а не структуры.

```
Объект: Инструкции
└── Свойство: Скрипты (.scripts/)
    ├── standard-scripts.md     # ЧТО: для чего скрипты, что в них должно быть
    ├── validation-scripts.md   # КАК проверить: соответствие скрипта заданию
    ├── create-scripts.md       # КАК создать: на основании чего
    └── modify-scripts.md       # КАК изменить: при изменении стандарта
```

**Расположение:** `/.instruction_new/`

## Содержание документов

### standard-scripts.md

- Для чего вводятся скрипты
- Что должно быть в скрипте (структура, комментарии, логирование)
- Связь скрипта с инструкцией (скрипт реализует часть инструкции)
- Именование скриптов

### validation-scripts.md

- Как проверять соответствие скрипта заданию
- Сравнение выхода скрипта с ожидаемым результатом
- Тестирование скриптов

### create-scripts.md

- На основании какой инструкции создаётся скрипт
- Шаги создания скрипта
- Связь с секцией "Скиллы и скрипты" в инструкции

### modify-scripts.md

- Как изменять скрипт при изменении стандарта
- Что делать при удалении стандарта
- Обновление связей

# Фаза 3: Rules (отложено)

После завершения Фаз 1 и 2 — создать rules и инструкции по их написанию.

## Концепция

**Стандарты (standard-*.md)** = ЧТО (формат, структура, как должно выглядеть)
**Валидации, Создание, Модификация** = КАК (процедуры, действия)

Rule ссылается только на документы "КАК делать". Стандарты читаются из них через секцию "Связанные документы".

## Где что лежит

Rules — инструмент Claude Code (как skills, agents):

| Инструмент | Где лежит | Инструкция как писать |
|------------|-----------|----------------------|
| Skills | `.claude/skills/` | `.claude/.instructions/skills/` |
| Agents | `.claude/agents/` | `.claude/.instructions/agents/` |
| Rules | `.claude/rules/` | `.claude/.instructions/rules/` |

## Структура

```
.claude/
├── .instructions/
│   └── rules/                    # Инструкции: как работать с rules
│       ├── README.md             # Индекс
│       ├── standard-rules.md     # Стандарт: ЧТО (формат, структура rule)
│       ├── validation-rules.md   # Валидация: КАК проверить
│       ├── create-rules.md       # Создание: КАК создать rule
│       └── modify-rules.md       # Модификация: КАК изменить rule
└── rules/
    ├── structure.md              # Rule для структуры → .structure/.instructions/
    └── rules.md                  # Rule для rules → .claude/.instructions/rules/
```

## Формат rule-файла (пример для structure.md)

```markdown
---
paths:
  - "**/README.md"
  - "**/*.md"
  - "!node_modules/**"
---

# Структура проекта

Инструкции: [/.structure/.instructions/](/.structure/.instructions/README.md)

**При создании новой папки:**
→ [create-structure.md](/.structure/.instructions/create-structure.md)

**При переименовании/перемещении/удалении папки:**
→ [modify-structure.md](/.structure/.instructions/modify-structure.md)

**При валидации структуры:**
→ [validation-structure.md](/.structure/.instructions/validation-structure.md)

**При валидации ссылок:**
→ [validation-links.md](/.structure/.instructions/validation-links.md)
```
