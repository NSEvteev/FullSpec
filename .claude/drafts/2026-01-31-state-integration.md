## Контекст

После завершения стандарта `standard-state.md` необходимо обновить документацию агентов, чтобы они знали о:
- Файлах состояния других агентов
- Блокировках ресурсов
- Координации при параллельной работе

---

## Содержание

### Файлы для обновления

#### 1. Документация агентов

| Файл | Что добавить |
|------|--------------|
| `.claude/.instructions/agents/standard-agent.md` | Раздел о работе с state: какие файлы читать, как проверять блокировки |
| `.claude/.instructions/agents/validation-agent.md` | Проверка корректности использования state |
| `.claude/.instructions/agents/create-agent.md` | Шаг добавления инструкций по state в новых агентов |
| `.claude/.instructions/agents/modify-agent.md` | Обновление существующих агентов для работы с state |

#### 2. Агент Эми

| Файл | Что добавить |
|------|--------------|
| `.claude/settings.json` | Обновить промпт amy-santiago с инструкциями по работе с state |

Эми должна знать:
- Как проверять блокировки перед работой с файлами
- Как координироваться с другими агентами
- Когда и как использовать `/.claude/state/`

#### 3. Связанные скрипты

После обновления документации — найти и обновить скрипты, которые создают/модифицируют агентов.

#### 4. Связанные скиллы

Скиллы `agent-create`, `agent-modify`, `agent-validate` должны учитывать новые требования к state.

---

### Что агент должен знать о state

1. **Перед началом работы:**
   - Проверить `/.claude/state/locks/` на блокировки файлов, с которыми будет работать
   - Прочитать `/.claude/state/agents-status.json` (если есть) для понимания контекста

2. **Во время работы:**
   - Создать блокировку на файлы, которые модифицирует
   - Обновлять свой статус в `agents-status.json`

3. **После завершения:**
   - Удалить свои блокировки
   - Записать результат (если нужен другим агентам)
