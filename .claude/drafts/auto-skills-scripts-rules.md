# Автоматизация создания скиллов, скриптов и rules

Комплексный план добавления автоматического создания скриптов и скиллов при создании инструкций, а также создание системы rules.

---

## Статус: Драфт

**Дата:** 2026-01-29

---

## Принцип типологии

### Инструкции — 4 типа

| Тип | Файл | Назначение |
|-----|------|------------|
| standard | `standard-{object}.md` | Стандарт формата |
| create | `create-{object}.md` | Процесс создания |
| modify | `modify-{object}.md` | Процесс изменения |
| validation | `validation-{object}.md` | Процесс валидации |

### Скиллы — ТОЛЬКО 3 типа

| Тип | Скилл | SSOT |
|-----|-------|------|
| create | `/{object}-create` | `create-{object}.md` |
| modify | `/{object}-modify` | `modify-{object}.md` |
| validate | `/{object}-validate` | `validation-{object}.md` |

> **Важно:** Для `standard-{object}.md` скилл НЕ создаётся

### Скрипты — без ограничений

Скрипты создаются по необходимости, без привязки к типологии.
Примеры: `validate-*.py`, `create-*.py`, `list-*.py`, `find-*.py`, `generate-*.py`

---

## Проблема

Сейчас при создании инструкции типа `validation-X.md`:
1. Скрипт `validate-X.py` создаётся вручную
2. Скилл `/X-validate` создаётся вручную
3. Нет rules для автоматической загрузки контекста

---

## Целевое состояние

```
Создание инструкции
         ↓
   create-instruction.md
         ↓
┌────────┴────────┐
│  Шаг N: Скрипт  │ → Автопредложение создать скрипт
│  Шаг N+1: Скилл │ → Автопредложение создать скилл
└────────┬────────┘
         ↓
   Rule загружается при работе в .instructions/
         ↓
   "Используй /instruction-create"
```

# Фаза 3: Rules инфраструктура

**Цель:** Создать систему rules с автоматической загрузкой контекста.

## 3.1. Создать инструкции для rules

**Путь:** `/.claude/.instructions/rules/` (НОВАЯ ПАПКА)

```
/.claude/.instructions/rules/
├── README.md              # Индекс
├── standard-rule.md       # Стандарт rules
├── create-rule.md         # Создание rule
├── modify-rule.md         # Изменение rule
├── validation-rule.md     # Валидация rules
└── .scripts/
    ├── validate-rule.py     # Валидация формата rule
    ├── create-rule-file.py  # Создание rule по шаблону
    └── list-rules.py        # Список всех rules
```

**Формат rule:**

```markdown
---
paths:
  - "паттерн/**"
description: Краткое описание
---

# Название

Краткие правила и ссылки на скиллы.
```

## 3.2. Создать скиллы для rules

> Типология: 3 скилла — create, modify, validate

| Скилл | Назначение | SSOT |
|-------|------------|------|
| `/rule-create` | Создание rule | `create-rule.md` |
| `/rule-modify` | Изменение rule | `modify-rule.md` |
| `/rule-validate` | Валидация rule | `validation-rule.md` |

## 3.3. Создать rules для rules

**Путь:** `/.claude/rules/rules.md`

```markdown
---
paths:
  - ".claude/rules/**"
---

# Rules

При создании rule:
→ `/rule-create`

При изменении rule:
→ `/rule-modify`

При валидации rule:
→ `/rule-validate`
```
