## Содержание

### Проблема

При частых изменениях структуры (добавление/удаление файлов) README быстро устаревает. Ручная синхронизация — узкое место.

### Предложенное решение

Pre-commit хук, который проверяет:
- Все файлы папки описаны в README?
- Дерево соответствует файловой системе?
- Нет битых ссылок?

### Реализация

```bash
# .pre-commit-config.yaml
repos:
  - repo: local
    hooks:
      - id: sync-readme
        name: Sync README with file system
        entry: python .structure/.instructions/.scripts/sync-readme.py --check
        language: python
        types: [markdown]
        pass_filenames: false
```

### Скрипт sync-readme.py

**Функционал:**
- `--check` — проверить соответствие (exit 1 если расхождения)
- `--fix` — автоматически обновить дерево и секции
- `--path` — проверить конкретную папку

**Проверки:**
1. Дерево в README соответствует `ls -la`?
2. Все файлы из папки упомянуты в секции "Файлы"?
3. Все подпапки упомянуты в секции "Папки"?
4. Нет файлов/папок в README, которых нет в файловой системе?
