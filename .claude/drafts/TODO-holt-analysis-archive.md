# Архив анализов Captain Holt

## Проблема

1. **Нет истории анализов** — после применения рекомендаций файл в `drafts/` перезаписывается при следующем анализе
2. **Нет способа отследить улучшения** — непонятно, стал ли документ лучше после правок
3. **Нет связи с версиями критериев** — при обновлении критериев старые анализы несопоставимы

## Предлагаемое решение

### 1. Структура архива

```
/.claude/state/holt/
├── archive/                     # Архив анализов
│   ├── 2026-02/                 # По месяцам
│   │   ├── standard-instruction/
│   │   │   ├── 2026-02-01-v1.0.md
│   │   │   └── 2026-02-15-v1.0.md
│   │   └── standard-agent/
│   │       └── 2026-02-01-v1.0.md
│   └── 2026-03/
├── feedback/                    # Фидбэки (из другого драфта)
├── analytics/                   # Статистика
│   ├── common-issues.json
│   └── improvements.json        # История улучшений документов
└── README.md
```

### 2. Формат архивного файла

Имя: `{YYYY-MM-DD}-v{version}.md`

Содержимое — полный анализ + метаданные:

```markdown
---
analyzed_file: /.instructions/standard-instruction.md
analysis_version: v1.0
analyzed_at: 2026-02-01T14:30:00Z
previous_analysis: 2026-01-15-v1.0.md
issues_found: 8
issues_p1: 2
issues_p2: 4
issues_p3: 2
---

# Анализ: standard-instruction.md
...
```

### 3. Процесс архивации

**Когда архивировать:**
- После сохранения анализа в `drafts/`
- Автоматически (хук или скрипт)

**Алгоритм:**
1. Анализ сохраняется в `/.claude/drafts/2026-02-01-holt-analysis-{name}.md`
2. Скрипт копирует в архив: `/.claude/state/holt/archive/2026-02/{name}/2026-02-01-v1.0.md`
3. Обновляется `improvements.json`

### 4. Отслеживание улучшений

**Файл:** `/.claude/state/holt/analytics/improvements.json`

```json
{
  "documents": {
    "standard-instruction.md": {
      "analyses": [
        {
          "date": "2026-01-15",
          "version": "v1.0",
          "issues": {"p1": 5, "p2": 8, "p3": 3},
          "rating": "Критично"
        },
        {
          "date": "2026-02-01",
          "version": "v1.0",
          "issues": {"p1": 2, "p2": 4, "p3": 2},
          "rating": "Требует правок"
        }
      ],
      "trend": "improving",
      "p1_reduction": "60%"
    }
  }
}
```

### 5. Использование архива

**Сценарии:**
1. **Сравнение версий** — как изменился документ после правок?
2. **Отчёт по качеству** — какие документы улучшились, какие ухудшились?
3. **Переанализ** — после обновления критериев переанализировать по новой версии
4. **Аудит** — история всех анализов для ревью

### 6. Связь с CHANGELOG

При обновлении критериев (новая версия в AGENT.md):
1. Записать изменения в `CHANGELOG.md`
2. Опционально: переанализировать критичные документы
3. В архиве — новые анализы с новой версией (`v1.1`)

**Сравнение версий:**
```
v1.0: 5 категорий
v1.1: 6 категорий (добавлена "Доступность")
```

Старые анализы остаются как есть (v1.0), новые — с v1.1.

### 7. Ротация архива

**Политика хранения:**
- Последние 6 месяцев — полные анализы
- Старше 6 месяцев — только метаданные (issues count, rating)
- Старше 1 года — удаление (опционально)

**Скрипт ротации:**
```python
# .claude/scripts/rotate-holt-archive.py
# Запускать ежемесячно
```
