## Рекомендации по улучшению

### 1. Размер документа — барьер для входа

**Проблема:** 18 секций, ~1300 строк — сложно найти нужное.

**Рекомендация:**
- Добавить "Quick Reference" в начало с основными паттернами
- Или разбить на несколько документов:
  - `standard-workflow-file.md` — базовая структура и триггеры
  - `standard-workflow-advanced.md` — Matrix, Reusable, Composite
  - `standard-workflow-best-practices.md` — Best practices, debugging

### 2. Не описано: локальное тестирование workflow

**Проблема:** Нет способа протестировать workflow без push в репозиторий.

**Рекомендация:** Добавить секцию "Локальное тестирование":
- Инструмент `act` для локального запуска GitHub Actions
- Ограничения: не все Actions работают локально
- Альтернатива: feature-ветка → push → проверка → удаление

### 3. Отсутствует: шаблон минимального CI workflow

**Проблема:** Много примеров отдельных элементов, но нет готового копируемого шаблона для старта.

**Рекомендация:** Добавить в начало:
```yaml
# Минимальный CI workflow (копируйте и адаптируйте)
name: CI
on: [push, pull_request]
permissions:
  contents: read
jobs:
  test:
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/checkout@v4
      - run: make test
```

### 4. Не описано: взаимодействие нескольких workflows

**Проблема:** Описаны отдельные workflows, но не описано:
- Как они взаимодействуют (ci.yml vs deploy.yml)
- Что если один зависит от другого

**Рекомендация:** Добавить секцию "Координация workflows":
- `workflow_run` триггер для зависимостей между workflows
- Пример: deploy.yml запускается после успешного ci.yml

### 5. Неясность: когда использовать Reusable vs Composite

**Проблема:** Оба механизма описаны, но таблица сравнения в §15 недостаточно детальна.

**Рекомендация:** Расширить сравнение:
| Аспект | Reusable Workflow | Composite Action |
|--------|-------------------|------------------|
| Уровень | Jobs | Steps |
| Секреты | Явная передача через `secrets:` | Наследует от вызывающего |
| Outputs | Поддерживает | Поддерживает |
| Кэширование | В каждом job отдельно | Общее с вызывающим |
| Использование | Сложные flow | Повторяющиеся steps |

### 6. Не описано: миграция с других CI систем

**Проблема:** Нет руководства для миграции с GitLab CI, Jenkins, CircleCI.

**Рекомендация:** Это выходит за рамки стандарта, но можно добавить ссылку:
- [GitHub Docs: Migrating to GitHub Actions](https://docs.github.com/en/actions/migrating-to-github-actions)

### 7. Мелкое: примеры на разных языках

**Проблема:** Примеры преимущественно для Node.js (`npm`).

**Рекомендация:** Добавить примеры для:
- Python: `pip install`, `pytest`
- Go: `go build`, `go test`
- Docker: `docker build`, `docker push`

Или ссылку на GitHub Starter Workflows.

### 8. Не описано: self-hosted runners

**Проблема:** Упоминается `runs-on: ubuntu-latest`, но нет информации о self-hosted.

**Рекомендация:** Добавить подсекцию или ссылку:
- Когда использовать: приватные окружения, специфичное железо
- Как настроить: Settings → Actions → Runners
- Безопасность: изоляция, ограничения доступа
