## Контекст

Пользователь хочет создать агента "Капитан Холт", который проверяет работу агента "Эми". Текущий стандарт не описывает паттерны координации агентов.

**Текущее ограничение:** Агенты не могут вызывать других агентов (инструмент `Task` не в списке доступных).

**Причины ограничения:**
- Риск бесконечной рекурсии
- Сложность отладки
- Неконтролируемый рост стоимости API

---

## Проблема

Нужен способ организовать цепочку агентов:
1. Агент A выполняет работу
2. Агент B проверяет результат агента A
3. При ошибках — агент A исправляет

Без возможности агентов вызывать друг друга напрямую.

---

## Решение

**Паттерн "Оркестратор"** — основной LLM координирует агентов через файлы состояния.

### Схема

```
Пользователь: "Создай инструкцию X"
         │
         ▼
    Основной LLM (оркестратор)
         │
         ├─1─► Агент "Эми" (создаёт)
         │         │
         │         └──► /.claude/drafts/result.md
         │
         ├─2─► Агент "Холт" (валидирует)
         │         │
         │         ├──► Читает /.claude/drafts/result.md
         │         └──► Возвращает: ✅ или {ошибки}
         │
         └─3─► Если ошибки:
                   │
                   └──► Агент "Эми" (исправляет)
                            │
                            └──► Обновляет result.md
```

### Принципы

| Принцип | Описание |
|---------|----------|
| Единый координатор | Только основной LLM запускает агентов |
| Коммуникация через файлы | Агенты передают результаты через `drafts/` или `state/` |
| Явные точки контроля | Основной LLM проверяет результат между этапами |
| Ограниченные итерации | Максимум N циклов исправлений |

## Содержание

### Файл: `/.claude/.instructions/agents/standard-agent.md`

**Добавить секцию после "5. Контекст для агента":**

```markdown
## 6. Паттерны координации

### Паттерн "Оркестратор"

> **Основной LLM координирует агентов.** Агенты НЕ вызывают друг друга напрямую.

**Когда использовать:**
- Цепочка "создание → проверка → исправление"
- Параллельная работа нескольких агентов
- Зависимость результата одного агента от другого

**Роли:**

| Роль | Кто | Что делает |
|------|-----|------------|
| Оркестратор | Основной LLM | Запускает агентов, передаёт контекст, принимает решения |
| Исполнитель | Агент (например, Эми) | Выполняет задачу, сохраняет результат |
| Валидатор | Агент (например, Холт) | Проверяет результат, возвращает отчёт |

**Коммуникация:**

| От | К | Через |
|----|---|-------|
| Оркестратор → Агент | prompt | Параметр `prompt` в Task |
| Агент → Оркестратор | результат | Возврат из Task |
| Агент → Агент | файл | `/.claude/drafts/` или `/.claude/state/` |

**Пример цепочки:**

1. Оркестратор запускает Эми:
   ```
   Task: amy-santiago
   prompt: "Создай инструкцию X. Сохрани в /.claude/drafts/instruction-x.md"
   ```

2. Эми сохраняет результат и завершается

3. Оркестратор запускает Холта:
   ```
   Task: captain-holt
   prompt: "Проверь /.claude/drafts/instruction-x.md по стандарту Y"
   ```

4. Холт возвращает отчёт:
   - ✅ Валидация пройдена
   - ❌ Ошибки: [список]

5. Если ошибки — оркестратор запускает Эми снова:
   ```
   Task: amy-santiago
   prompt: "Исправь /.claude/drafts/instruction-x.md. Ошибки: [список]"
   ```

**Ограничение циклов:**

> **Максимум 3 итерации исправлений.** После 3-й — эскалация пользователю.

**Файлы для координации:**

| Файл | Назначение |
|------|------------|
| `/.claude/drafts/{task}.md` | Результат работы агента |
| `/.claude/state/orchestration-{id}.json` | Состояние цепочки |

**Формат orchestration-{id}.json:**

```json
{
  "id": "instruction-x-creation",
  "started_at": "2026-02-01T10:00:00Z",
  "status": "in_progress",
  "iterations": [
    {
      "step": 1,
      "agent": "amy-santiago",
      "action": "create",
      "result": "/.claude/drafts/instruction-x.md",
      "completed_at": "2026-02-01T10:05:00Z"
    },
    {
      "step": 2,
      "agent": "captain-holt",
      "action": "validate",
      "result": {"valid": false, "errors": ["..."]},
      "completed_at": "2026-02-01T10:06:00Z"
    }
  ],
  "current_iteration": 2,
  "max_iterations": 3
}
```
```
