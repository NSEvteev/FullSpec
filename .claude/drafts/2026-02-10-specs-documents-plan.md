### Этап 2: Создание документов (последовательно по объектам)

> **Статус:** В работе (Волна 1 + Discussion завершены)

**Источник контента:** [2026-02-08-specs-architecture.md](./examples/2026-02-08-specs-architecture.md) — разделы указаны в описании каждого документа.

**Подход:** Последовательно по объектам. Для каждого объекта — **полный комплект** (решение #2).

#### Целевой воркфлоу для каждого объекта

Отработан на Discussion. Повторять для каждого последующего объекта (Impact, Design, ADR, Plan Test, Plan Dev, living docs).

**Фаза 1 — Комплект инструкций** (через `/instruction-create`):

| # | Документ | Описание |
|---|----------|----------|
| 1 | `standard-{object}.md` | Стандарт: назначение, именование, frontmatter, статусы, разделы, шаблон, чек-лист, примеры |
| 2 | `validation-{object}.md` | Валидация: шаги проверки, коды ошибок, типичные ошибки |
| 3 | `create-{object}.md` | Воркфлоу создания: Clarify → генерация → валидация → ревью → DRAFT → WAITING |
| 4 | `modify-{object}.md` | Воркфлоу изменения: операции по статусам, переходы жизненного цикла |

**Фаза 2 — Автоматизация** (через `/script-create` и `/skill-create`):

| # | Артефакт | Описание |
|---|----------|----------|
| 1 | `validate-{object}.py` | Скрипт валидации: все проверки из validation, коды ошибок |
| 2 | `create-{object}-file.py` | Скрипт создания файла из шаблона (определение номера, запись) |
| 3 | `/discussion-create` | Скилл создания — SSOT → create-{object}.md |
| 4 | `/discussion-modify` | Скилл изменения — SSOT → modify-{object}.md |
| 5 | `/discussion-validate` | Скилл валидации — SSOT → validation-{object}.md |

**Фаза 3 — Сквозная проверка** (E2E):

| # | Шаг | Описание |
|---|-----|----------|
| 1 | Создать экземпляр через скилл | `/discussion-create` — пройти Clarify, генерацию, предложения |
| 2 | Валидация скриптом | `validate-{object}.py` — 0 ошибок |
| 3 | Ревью и перевод в WAITING | Полный цикл DRAFT → WAITING |
| 4 | Итог | Экземпляр создан, валидация работает, скилл проверен |

**Важно:** Фаза 3 не опциональна. Без сквозной проверки невозможно убедиться, что стандарт, скрипты и скиллы согласованы между собой.

## Решения

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | Порядок: создавать или чистить? | **Сначала удалить** все 11 старых файлов (все конфликтуют или мёртвые), потом создавать новые |
| 2 | validation/create/modify для объектов specs/? | **Полный комплект на каждый объект.** 3 фазы: (1) инструкции (standard + validation + create + modify), (2) автоматизация (скрипты + скиллы), (3) сквозная проверка (создать экземпляр через скилл, пройти до WAITING). Отработано на Discussion |
| 3 | Судьба подпапок discussions/ и impact/ | **Переписать README.md** под новый формат. Стандарты добавить позже (волна 2) |
| 4 | Контентные папки: пустыми или по необходимости? | **Создать сейчас** пустыми с README-заглушками |
