# Драфт: Структура документации объекта

**Дата:** 2026-01-27
**Статус:** DRAFT

---

## Концепция

Для документирования любого **объекта** в проекте используется единая структура из 4 типов документов.

---

## 1. Объект и его свойства

**Объект** — сущность, которую нужно описать, создать и поддерживать.

**Свойства объекта** — составные части или аспекты объекта, которые также требуют документирования.

### Пример: Объект "Структура"

```
Объект: Структура (structure)
├── Свойство: Папки (folders)
├── Свойство: README
├── Свойство: Ссылки (links)
└── Свойство: Frontmatter
```

---

## 2. Типы документов

Для объекта и каждого его свойства могут быть созданы документы 4 типов:

| Тип | Вопрос | Что описывает | Создаёт инструменты? |
|-----|--------|---------------|---------------------|
| **Стандарт** | Что это? Как должно выглядеть? | Формат, структура, правила | Нет |
| **Валидация** | Как проверить, что правильно? | Процедура проверки | Да (скиллы, скрипты) |
| **Создание** | Как создать? | Шаги создания | Да (скиллы, скрипты) |
| **Модификация** | Как изменить/удалить? | Шаги изменения | Да (скиллы, скрипты) |

---

## 3. Формат именования

```
{тип}-{объект}.md
```

| Тип | Префикс | Пример |
|-----|---------|--------|
| Стандарт | `standard-` | `standard-links.md` |
| Валидация | `validation-` | `validation-links.md` |
| Создание | `create-` | `create-structure.md` |
| Модификация | `modify-` | `modify-structure.md` |

---

## 4. Матрица документов

Не каждому свойству нужны все 4 типа документов.

### Правила:

1. **Стандарт** — нужен, если есть формат/правила для описания
2. **Валидация** — нужна, если можно автоматизировать проверку
3. **Создание** — нужно, если объект создаётся как отдельная сущность
4. **Модификация** — нужна, если есть специфичные правила изменения/удаления

### Пример: Объект "Структура"

| Свойство | standard- | validation- | create- | modify- |
|----------|:---------:|:-----------:|:-------:|:-------:|
| **Папки** | — | ✅ `validation-structure` | ✅ `create-structure` | ✅ `modify-structure` |
| **README** | ✅ `standard-readme` | (часть validation-structure) | (часть create-structure) | — |
| **Ссылки** | ✅ `standard-links` | ✅ `validation-links` | — | — |
| **Frontmatter** | ✅ `standard-frontmatter` | (часть validation-links) | — | — |

**Обоснование:**
- Ссылки не "создаются" отдельно — они часть текста
- Frontmatter не "модифицируется" отдельно — это часть файла
- README валидируется в рамках validation-structure

---

## 5. Связи между документами

### Стандарт

```yaml
# standard-{object}.md
Содержит:
  - Правила и форматы
  - Примеры

Ссылается на:
  - Связанную валидацию: validation-{object}.md
  - Связанные воркфлоу: create-{object}.md, modify-{object}.md

НЕ содержит:
  - Секцию "Скиллы и скрипты" (не создаёт инструменты)
```

### Валидация

```yaml
# validation-{object}.md
Содержит:
  - Процедуру проверки
  - Коды ошибок
  - Чек-лист

Ссылается на:
  - Связанный стандарт: standard-{object}.md

Содержит:
  - Секцию "Скиллы и скрипты" (создаёт инструменты)
```

### Создание / Модификация

```yaml
# create-{object}.md / modify-{object}.md
Содержит:
  - Шаги выполнения
  - Чек-лист

Ссылается на:
  - Связанный стандарт: standard-{object}.md
  - Связанную валидацию: validation-{object}.md

Может содержать:
  - Секцию "Скиллы и скрипты" (если автоматизировано)
```

## 6. Секция "Связанные документы"

Каждый документ должен иметь секцию **в начале** (после "Полезные ссылки"), показывающую его место в системе.

**Расположение:**
```markdown
**Полезные ссылки:**
- [Инструкции для .structure](./README.md)
- [SSOT структуры проекта](../README.md)

**Связанные документы:**

| Тип | Документ |
|-----|----------|
| Стандарт | ... |
| Валидация | ... |

## Оглавление
```

**Пример для валидации:**

```markdown
**Связанные документы:**

| Тип | Документ |
|-----|----------|
| Стандарт | [standard-links.md](./standard-links.md) |
| Валидация | Этот документ |
| Создание | — |
| Модификация | — |
```

**Пример для стандарта:**

```markdown
**Связанные документы:**

| Тип | Документ |
|-----|----------|
| Стандарт | Этот документ |
| Валидация | [validation-links.md](./validation-links.md) |
```

> **Примечание:** Заменяет устаревший блок "Связанные инструкции".

## 7. Итоговая структура файлов

```
.structure/.instructions/
├── README.md                    # Индекс всех инструкций
│
├── # Свойство: README
├── standard-readme.md           # Стандарт README для папок
│
├── # Свойство: Ссылки
├── standard-links.md            # Стандарт ссылок
├── validation-links.md          # Валидация ссылок
│
├── # Свойство: Frontmatter
├── standard-frontmatter.md      # Стандарт frontmatter
│
├── # Свойство: Папки (структура)
├── validation-structure.md      # Валидация структуры
├── create-structure.md          # Создание папки
├── modify-structure.md          # Изменение папки
│
└── .scripts/
    ├── validate-structure.py    # Скрипт из validation-structure.md
    ├── validate-links.py        # Скрипт из validation-links.md
    └── generate-readme.py       # Скрипт из create-structure.md
```

## 8. Чек-лист для нового объекта

При документировании нового объекта:

- [ ] Определить объект и его свойства
- [ ] Для каждого свойства определить, какие типы документов нужны
- [ ] Создать документы с правильным именованием
- [ ] Добавить секцию "Связанные документы" в каждый файл
- [ ] Обновить README.md (индекс)
- [ ] Создать скиллы/скрипты для валидации и воркфлоу
