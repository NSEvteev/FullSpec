## Разделение ответственности: СТРУКТУРА vs ИНСТРУКЦИИ

**КРИТИЧНО:** Две области инструкций имеют чёткое разделение ответственности.

### Матрица ответственности

| Область | Отвечает за | Файлы |
|---------|-------------|-------|
| **`.structure/`** | ГДЕ создавать (папки, размещение) | principles.md, readme-index.md |
| **`.claude/.instructions/`** | ЧТО создавать (контент, формат) | types.md, validation.md, workflow*.md |

### Правило создания инструкции

```
При создании инструкции:

1. Нужна НОВАЯ ПАПКА?
   │
   ├─ ДА → Использовать правила СТРУКТУРЫ
   │       ├─ Создать папку по /.instructions/.structure/principles.md
   │       ├─ Создать README.md по /.instructions/.structure/readme-index.md
   │       └─ Затем создать инструкцию по /.instructions/.claude/.instructions/
   │
   └─ НЕТ → Папка уже существует
           ├─ Создать инструкцию по /.instructions/.claude/.instructions/
           └─ Обновить README.md папки (добавить новую секцию)
```

### Что куда переносится

| Правило | Было в | Станет в |
|---------|--------|----------|
| Правило "1 уровень" | template-readme.md | `.structure/readme-index.md` |
| Формат README-индекса | template-readme.md | `.structure/readme-index.md` |
| Шаблон README | template-readme.md | Остаётся (ссылка из readme-index.md) |
| Формат инструкции | template-instruction.md | `.instructions/` (без изменений) |
| Workflow создания | workflow-create.md | `.instructions/` (без изменений) |

### Связь между областями

```
/.claude/.instructions/.claude/.instructions/workflow-create.md
  │
  └─ Шаг "Создание папки (если нужна)"
       └─ ССЫЛКА → /.claude/.instructions/.structure/readme-index.md
```

## Проблема

Папки `/.structure/` и `/.claude/.instructions/.structure/` смешивают факты и правила:

| Файл | Проблема |
|------|----------|
| `/.structure/project.md` | Содержит "Общие правила" — это не факт |
| `/.structure/instructions.md` | Содержит "Принцип" — это не факт; дублирует SSOT инструкций |
| `/.structure/mapping.md` | Содержит "Принцип" в конце — это не факт |
| `/.instructions/.structure/README.md` | Дублирует таблицы; не соответствует формату шаблона |
| `template-readme.md` | Содержит правила структуры, которые должны быть в `.structure/` |

## Принцип разделения ФАКТ / ПРАВИЛО

| Тип | Назначение | Папка |
|-----|------------|-------|
| **project** | ЧТО есть (факты, специфика) | `/.structure/` |
| **standard** | КАК делать (правила, универсальные) | `/.claude/.instructions/.structure/` |

### Определение

- **ФАКТ** = текущее состояние, которое можно проверить `ls`, `tree`
- **ПРАВИЛО** = требование, которое должно соблюдаться

## Правило "1 уровень" для README.md

**КРИТИЧНО:** README.md папки показывает **только прямых потомков**:

| Элемент | Что показывать |
|---------|---------------|
| Файлы текущей папки | Полное описание, оглавление секций |
| Подпапки | **Только ссылка на README.md подпапки** |

### Пример для `.structure/`

```
/.claude/.instructions/.structure/
├── README.md               # Индекс (этот файл)
├── principles.md           # ← описание + оглавление
├── lifecycle.md            # ← описание + оглавление
├── responsibilities.md     # ← описание + оглавление
├── readme-index.md         # ← описание + оглавление
├── examples.md             # ← описание + оглавление
└── links/                  # ← ТОЛЬКО ссылка: [links/README.md](./links/README.md)
    ├── README.md           #    (содержимое НЕ раскрывается здесь)
    ├── format.md
    └── ...
```

**Неправильно:** Показывать в README.md родителя содержимое `links/format.md`
**Правильно:** Ссылка на `links/README.md`, который сам описывает свои файлы

## Правило "1 уровень"

**КРИТИЧНО:** README.md папки показывает **только прямых потомков**.

| Элемент | Что показывать |
|---------|---------------|
| Файлы текущей папки | Полное описание, оглавление секций |
| Подпапки | **Только ссылка на README.md подпапки** |

### Почему

- Избегаем дублирования информации
- Каждая папка отвечает за свой README
- Изменения в подпапке не требуют обновления родителя

### Пример

```
parent/
├── README.md       # Описывает file.md + ссылка на child/README.md
├── file.md         # ← полное описание здесь
└── child/
    ├── README.md   # ← содержимое описано ЗДЕСЬ, не в parent/README.md
    └── nested.md
```

**В parent/README.md:**
```markdown
# 3. Child

Подпапка с {описание}.

**Индекс:** [child/README.md](./child/README.md)
```

**НЕ показывать:** содержимое `nested.md` в `parent/README.md`

## Зеркалирование

Папка проекта `/X/` → инструкции `/.claude/.instructions/X/`

```
Папка /X/  →  Инструкции /.claude/.instructions/X/
Правила    →  Инструкции /.claude/.instructions/.claude/
```

---

## Общие правила

1. **README.md обязателен** — каждая папка ДОЛЖНА иметь README.md как индекс
2. **SSOT в drafts/** — документы-первоисточники в `/.claude/drafts/`
3. **settings.local.json не в git** — локальные настройки игнорируются
4. **state/ не в git** — состояния агентов игнорируются

---

## Именование

- Папки с точкой в проекте → папки с точкой в инструкциях
- Пример: `/.github/` → `/.claude/.instructions/.github/`
```

## Валидация после выполнения

### `/.structure/` должен содержать только:
- [ ] Деревья папок (можно сгенерировать `tree`)
- [ ] Таблицы маппинга
- [ ] Ссылки на SSOT (не копии)
- [ ] Никаких слов: "должен", "обязателен", "правило", "требуется", "принцип"

### `/.instructions/.structure/` должен содержать только:
- [ ] README.md по формату шаблона (правило "1 уровень")
- [ ] Правила и требования
- [ ] Decision trees
- [ ] IN/OUT матрицы
- [ ] Ссылки на SSOT для актуальных данных

### Проверка правила "1 уровень":
- [ ] README.md `.structure/` НЕ раскрывает содержимое `links/`
- [ ] `links/` упоминается только как ссылка на `links/README.md`

### Проверка разделения ответственности:
- [ ] `.structure/readme-index.md` содержит правила README-индексов
- [ ] `.instructions/template-readme.md` содержит только шаблон
