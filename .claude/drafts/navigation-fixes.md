# Драфт: Исправления навигации

## Проблемы

### 1. Текст ссылки слишком краткий

**Сейчас:**
```markdown
> [Инструкции .structure](./README.md)
```

**Должно быть:**
```markdown
> [Инструкции по работе со структурой проекта](./README.md)
```

Более понятный контекст. Текст должен быть понятен без контекста файла.

---

### 2. Неправильная иерархия для `links/`

**Проблема:** `links/` — это инструкции по работе со ссылками **во всём проекте**, не только в `.structure`.

**Сейчас:**
```
.structure/.instructions/links/format.md
> [Работа со ссылками в .structure](./README.md)  ← 1 уровень, привязка к .structure
```

**Должно быть:** 2 уровня навигации:
1. Ссылка на `links/README.md` — контекст работы со ссылками
2. В `links/README.md` — ссылка на `.structure/.instructions/README.md` — контекст работы со структурой

**Решение:** ✅ Оставить в `.structure/.instructions/`, текст = `Инструкции по работе со ссылками`

---

### 3. Дублирование в `/.instructions/validation.md`

**Проблема:** Я добавил блок "Ссылка на контекст" с правилами, но SSOT уже есть:
- **SSOT:** `/.structure/.instructions/format.md#ссылка-на-контекст`
- **Дубликат:** `/.instructions/validation.md#ссылка-на-контекст`

**Решение:** Заменить дублированный блок на ссылку на SSOT.

---

### 4. КРИТИЧНО: `/.instructions/validation.md` не имеет blockquote

**Проблема:** Файл не следует собственным правилам! После заголовка нет ссылки на контекст.

**Сейчас:**
```markdown
# Валидация инструкций

Правила валидации путей...
```

**Должно быть:**
```markdown
# Валидация инструкций

> [Инструкции по работе с инструкциями](./README.md)

Правила валидации путей...
```

---

### 5. Подпапки specs — специфичные тексты

**Проблема:** Подпапки `specs/.instructions/` имеют особый формат:
- `impact/README.md` → `Инструкции specs`
- `discussions/README.md` → `Инструкции specs`
- `services/plans/README.md` → `Планы в specs`
- `services/adr/README.md` → `ADR в specs`

**Решение:** Оставить специфичные тексты для подпапок services/ (они описывают контент, не инструкции). Для impact/ и discussions/ — унифицировать с общим форматом.
