## Краткая оценка

Документ **хорошо структурирован** с полным описанием жизненного цикла Issue. Визуальная диаграмма состояний помогает пониманию.

## Рекомендации по улучшению

### 2. Неясность: когда добавлять метку status:wip

**Проблема:** Написано "label: status:wip (опционально)", но не ясно критерий добавления.

**Рекомендация:** Уточнить:
- Добавлять `status:wip` при создании ветки (если команда >3 человек)
- Для команды из 2 человек — опционально (прозрачность ниже)
- Убирать `status:wip` при создании PR (заменять на `status:in-review`)

### 3. Не описано: дублирование Issues

**Проблема:** Нет процедуры обнаружения и обработки дублирующих Issues.

**Рекомендация:** Добавить подсекцию "Дубликаты":
- Перед созданием: поиск `gh issue list --search "keyword"`
- При обнаружении дубликата: закрыть с комментарием "Дубликат #XXX"
- Метка `status:duplicate` (если используется)

### 5. Неясность: формат зависимостей между Issues

**Проблема:** Показан формат `**Зависит от:** #123, #124`, но не описано:
- В какой секции body указывать
- Автоматизация проверки (или только ручная)

**Рекомендация:** Уточнить в секции "§9. Зависимости":
- Указывать в начале body Issue, отдельной строкой
- Проверка ручная перед закрытием
- GitHub не поддерживает автоматическую блокировку по зависимостям

### 6. Не описано: Sub-issues / Checklist

**Проблема:** Не описано использование чек-листов внутри Issue как способ разбивки на подзадачи.

**Рекомендация:** Добавить подсекцию "Подзадачи внутри Issue":
- Использовать `- [ ] Подзадача` в body
- GitHub показывает прогресс (3/5 tasks)
- Когда выносить в отдельные Issues: если подзадача требует отдельного PR

### 7. Мелкое: нет примера полного Issue body

**Проблема:** Есть отдельные примеры структур для багов и фич, но нет примера заполненного Issue body.

**Рекомендация:** Добавить полный пример:
```markdown
## Описание

Добавить возможность экспорта данных в CSV формат.

## Критерии готовности

- [ ] Эндпоинт GET /api/export/csv
- [ ] Тесты покрывают happy path
- [ ] Документация обновлена

**Зависит от:** #120
```
