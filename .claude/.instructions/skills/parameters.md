## Общие флаги

| Флаг | Описание | Тип | Пример |
|------|----------|-----|--------|
| `--dry-run` | Показать план без выполнения | boolean | `--dry-run` |
| `--auto` | Автоматический режим без подтверждений | boolean | `--auto` |
| `--json` | Вывод в JSON формате (для CI/скриптов) | boolean | `--json` |
| `--verbose` | Подробный вывод с деталями | boolean | `--verbose` |
| `--depth N` | Глубина обхода связей (по умолчанию 1) | number | `--depth 2` |
| `--diff` | Показать diff изменений | boolean | `--diff` |
| `--no-issue` | Не создавать/не связывать Issue | boolean | `--no-issue` |

## Правила использования

### Обязательная поддержка

**Правило:** Все скиллы должны поддерживать `--dry-run`.

```
/skill-create my-skill --dry-run

📋 Предварительный просмотр (--dry-run)

Будет создано:
- /.claude/skills/my-skill/SKILL.md

Будет обновлено:
- /.claude/skills/README.md

ℹ️ Изменения НЕ применены (--dry-run)
```

### Именование флагов

| Правило | Пример |
|---------|--------|
| Флаги с отрицанием: `--no-{action}` | `--no-issue`, `--no-verify` |
| Числовые параметры через пробел | `--depth 2`, `--limit 10` |
| Булевы флаги без значения | `--verbose`, `--auto` |

### Приоритет флагов

1. `--dry-run` — всегда приоритетнее (auto игнорируется)
2. `--json` — меняет формат вывода, совместим с другими
3. `--verbose` — расширяет вывод, совместим с --json

---

## Совместимость флагов

| Комбинация | Результат |
|------------|-----------|
| `--dry-run --auto` | Показать план (auto игнорируется) |
| `--depth 0` | Без рекурсивного обхода |
| `--diff --dry-run` | Показать предполагаемые изменения |
| `--json --verbose` | JSON с расширенными полями |
| `--json --dry-run` | JSON с планом изменений |

## Примеры

### --dry-run

```bash
/issue-create --dry-run
# Показывает, какой Issue будет создан, без создания

/links-update file.md --dry-run
# Показывает, какие ссылки будут обновлены
```

### --auto

```bash
/skill-create my-skill --auto
# Создаёт скилл без подтверждений

/instruction-create src/api/design.md --auto
# Применяет все рекомендованные улучшения
```

### --json

```bash
/health-check --json
# Вывод для CI/CD:
# {"status": "success", "checks": {"links": 342, "broken": 0}}

/test-execute --json
# {"status": "success", "passed": 15, "failed": 0}
```

### --verbose

```bash
/links-validate --verbose
# Показывает каждую проверенную ссылку

/doc-reindex --verbose
# Показывает все переиндексированные файлы
```

### --depth

```bash
/context-update file.md --depth 2
# Обновляет контекст на 2 уровня связей (A→B→C)

/links-update file.md --depth 0
# Только прямые ссылки, без рекурсии
```
