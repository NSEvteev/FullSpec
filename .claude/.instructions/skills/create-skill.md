## 1. Когда создавать

**Создавать скилл когда:**

| Критерий | Условие |
|----------|---------|
| SSOT-инструкция | ✅ Существует |
| Воркфлоу | ✅ Есть в инструкции |
| Повторяемость | > 2 раз |

**НЕ создавать скилл когда:**

- ❌ Нет SSOT-инструкции → сначала создать инструкцию
- ❌ Разовое действие
- ❌ Простое действие без воркфлоу

**Порядок создания:**

```
1. Инструкция (SSOT)     ← Сначала
2. Скилл (триггер)       ← Потом
```

## 2. Подготовка

### 2.1. Проверить SSOT-инструкцию

```
Вопросы:
1. Инструкция существует?
2. Есть секция "Воркфлоу" или "Шаги"?
3. Есть секция "Чек-лист"?
```

Если нет → сначала создать инструкцию.

### 2.2. Определить имя скилла

Формат: `{object}-{action}`

| Объект | Действие | Имя скилла |
|--------|----------|------------|
| structure | create | `structure-create` |
| links | validate | `links-validate` |
| spec | status | `spec-status` |

### 2.3. Определить категорию

| Категория | Когда использовать |
|-----------|-------------------|
| `documentation` | Работа с документацией |
| `code` | Работа с кодом |
| `testing` | Тестирование |
| `deployment` | Деплой |

### 2.4. Определить триггеры

- Команда: `/{skill-name}`
- Фразы ru: 2-3 варианта
- Фразы en: 1-2 варианта

## 3. Шаги создания

```
1. Проверить SSOT-инструкцию
   └── Существует, есть воркфлоу и чек-лист

2. Проверить уникальность
   └── Скилл с таким именем не существует

3. Создать папку
   └── /.claude/skills/{skill-name}/

4. Создать SKILL.md
   └── По шаблону (секция 4)

5. Заполнить frontmatter
   ├── name, description
   ├── allowed-tools, category
   └── triggers (commands, phrases)

6. Добавить SSOT-ссылку
   └── **SSOT:** [инструкция](путь)

7. Заполнить "Формат вызова"
   └── Синтаксис + таблица параметров

8. Заполнить "Воркфлоу"
   └── Предупреждение + "→ Выполнить из SSOT"

9. Заполнить "Чек-лист"
   └── Ссылка на SSOT#чек-лист

10. Добавить примеры
    └── 2-3 примера вызова

11. Валидировать
    └── Проверить по чек-листу

12. Обновить /.claude/skills/README.md
    └── Добавить скилл в таблицу категории
```

## 4. Шаблон

```markdown
---
name: {skill-name}
description: {Краткое описание}
allowed-tools: Read, Bash, Glob, Grep, Write, Edit
category: documentation
triggers:
  commands:
    - /{skill-name}
  phrases:
    ru:
      - {фраза 1}
      - {фраза 2}
    en:
      - {phrase}
---

# {Название}

**SSOT:** [{инструкция}.md]({путь к SSOT-инструкции})

## Формат вызова

```
/{skill-name} <параметр> [--опция]
```

| Параметр | Описание | Обязательный |
|----------|----------|--------------|
| `<параметр>` | {описание} | Да |
| `--опция` | {описание} | Нет |

## Воркфлоу

> ⚠️ **Перед выполнением** прочитать [{инструкция}.md]({путь})

→ Выполнить шаги из SSOT-инструкции.

## Чек-лист

→ См. [{инструкция}.md#чек-лист]({путь}#чек-лист)

## Примеры

```
/{skill-name} example1
/{skill-name} example2 --option
```
```
