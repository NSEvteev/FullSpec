## 1. Когда валидировать

**Обязательно:**
- Перед коммитом нового скилла
- После изменения скилла
- При code review

**Опционально:**
- При рефакторинге
- В CI/CD pipeline

## 2. Что проверяется

### Frontmatter

| Проверка | Обязательно | Описание |
|----------|:-----------:|----------|
| `name` | ✅ | Присутствует, kebab-case |
| `description` | ✅ | Присутствует, не пустое |
| `allowed-tools` | ✅ | Присутствует |
| `category` | ✅ | Одна из: documentation, code, testing, deployment |
| `triggers.commands` | ✅ | Минимум 1 команда |
| `triggers.phrases` | ✅ | ru и en присутствуют |

### Структура

| Проверка | Обязательно | Описание |
|----------|:-----------:|----------|
| Заголовок H1 | ✅ | Единственный `#` |
| SSOT-ссылка | ✅ | `**SSOT:**` после заголовка |
| Формат вызова | ✅ | Секция с таблицей параметров |
| Воркфлоу | ✅ | Секция с "→ Выполнить из SSOT" |
| Чек-лист | ✅ | Ссылка на SSOT |
| Примеры | ✅ | Минимум 1 пример |

### Ссылки

| Проверка | Обязательно | Описание |
|----------|:-----------:|----------|
| SSOT существует | ✅ | Файл инструкции существует |
| Якорь чек-листа | ⚠️ | Секция #чек-лист в SSOT |

### Антипаттерны

| Проверка | Описание |
|----------|----------|
| Нет дублирования | Шаги воркфлоу НЕ скопированы из инструкции |
| Нет логики | Скилл НЕ содержит алгоритмов и условий |
| Размер | < 80 строк |

## 3. Коды ошибок

### Frontmatter (K001-K008)

| Код | Описание |
|-----|----------|
| **K001** | Отсутствует `name` |
| **K002** | Неверный формат `name` (не kebab-case) |
| **K003** | Отсутствует `description` |
| **K004** | Отсутствует `allowed-tools` |
| **K005** | Отсутствует `category` |
| **K006** | Неверная `category` |
| **K007** | Отсутствует `triggers.commands` |
| **K008** | Отсутствует `triggers.phrases` |

### Структура (K010-K016)

| Код | Описание |
|-----|----------|
| **K010** | Отсутствует заголовок H1 |
| **K011** | Несколько заголовков H1 |
| **K012** | Отсутствует SSOT-ссылка |
| **K013** | Отсутствует "Формат вызова" |
| **K014** | Отсутствует "Воркфлоу" |
| **K015** | Отсутствует "Чек-лист" |
| **K016** | Отсутствует "Примеры" |

### Ссылки (K020-K021)

| Код | Описание |
|-----|----------|
| **K020** | SSOT-файл не существует |
| **K021** | Битая ссылка в чек-листе |

### Антипаттерны (K030-K031)

| Код | Описание |
|-----|----------|
| **K030** | Дублирование шагов из инструкции |
| **K031** | Скилл > 80 строк |

## 4. Запуск валидации

### Один скилл

```bash
python .claude/.instructions/skills/.scripts/validate-skill.py structure-create
```

### Все скиллы

```bash
python .claude/.instructions/skills/.scripts/validate-skill.py --all
```

### JSON вывод

```bash
python .claude/.instructions/skills/.scripts/validate-skill.py --json structure-create
```

### Вывод

```
✅ structure-create — валидация пройдена (48 строк)

❌ old-skill — 3 ошибки:
   K012: Отсутствует SSOT-ссылка
   K030: Дублирование шагов из инструкции (строки 25-89)
   K031: Скилл 156 строк (макс 80)
```

## 5. Чек-лист

```markdown
## Чек-лист валидации скилла

### Frontmatter
- [ ] `name` присутствует, kebab-case
- [ ] `description` заполнено
- [ ] `allowed-tools` указаны
- [ ] `category` корректная
- [ ] `triggers.commands` минимум 1
- [ ] `triggers.phrases` ru и en

### Структура
- [ ] Один заголовок H1
- [ ] SSOT-ссылка сразу после заголовка
- [ ] Секция "Формат вызова" с таблицей
- [ ] Секция "Воркфлоу" с "→ Выполнить из SSOT"
- [ ] Секция "Чек-лист" со ссылкой
- [ ] Секция "Примеры" минимум 1

### Ссылки
- [ ] SSOT-файл существует
- [ ] Якорь #чек-лист валиден

### Антипаттерны
- [ ] Нет дублирования шагов
- [ ] Размер < 80 строк
```
