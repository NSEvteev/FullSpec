## Правила

### Принцип интеграции

При добавлении нового скилла — проверить, может ли он быть полезен существующим скиллам.

### Правило применимости

Скилл применим, если:
- Дополняет воркфлоу существующего скилла
- Работает с тем же типом объектов
- Добавляет проверки/валидацию/автоматизацию

### Правило неприменимости

Скилл НЕ применим, если:
- Нет логической связи
- Дублирует функциональность
- Усложняет без пользы

## Матрица применимости

Какой новый скилл куда интегрировать:

| Новый скилл | Применим к | Место интеграции |
|-------------|-----------|------------------|
| `*-validate` | Скиллы с созданием/изменением | После создания (валидация) |
| `*-update` | Скиллы с созданием | После создания (синхронизация) |
| `*-review` | Скиллы с шагом ревью | Заменить/дополнить шаг ревью |
| `*-execute` | Скиллы с шагом выполнения | Интеграция в воркфлоу выполнения |
| `context-*` | Все скиллы с .md файлами | После изменения документа |
| `links-*` | Скиллы, создающие .md | После создания/изменения |
| `issue-*` | Скиллы категории git | Связанные скиллы, воркфлоу |
| `doc-*` | Скиллы работы с документами | После создания документа |

## Критерии применимости

### Что считается "применимым"

Новый скилл применим к существующему, если:

1. **Дополняет воркфлоу** — может быть вызван на одном из шагов
2. **Улучшает качество** — добавляет проверки, валидацию, автоматизацию
3. **Связан по объекту** — работает с тем же типом объектов (скиллы, инструкции, документы)

### Что НЕ считается применимым

- Скилл из другой категории без логической связи
- Скилл, который дублирует существующую функциональность
- Скилл, который усложняет воркфлоу без явной пользы

### По категориям

| Приоритет | Категории |
|-----------|-----------|
| 1 (первый) | Та же категория |
| 2 | Связанные категории |
| 3 | Другие категории (по объекту и воркфлоу) |

### По действиям

| Действие нового | Применим к действиям |
|-----------------|---------------------|
| `*-create` | — (создание не интегрируется) |
| `*-update` | `*-create` (после создания синхронизировать) |
| `*-validate` | `*-create`, `*-update` (после изменения проверить) |
| `*-delete` | — (удаление не интегрируется) |

### По объектам

| Объект нового | Применим к объектам |
|---------------|---------------------|
| `skill-*` | skill-*, context-* |
| `links-*` | все, работающие с .md |
| `context-*` | все, работающие с .md |
| `instruction-*` | instruction-*, context-*, links-* |
| `issue-*` | issue-*, другие скиллы категории git |
| `doc-*` | doc-*, instruction-*, links-* |
| `prompt-*` | skill-*, instruction-* (мета-скиллы) |

## Куда добавлять интеграцию

| Место | Когда |
|-------|-------|
| Воркфлоу (шаг) | Новый скилл выполняет действие, нужное на этом шаге |
| Связанные скиллы | Новый скилл семантически связан |
| Чек-лист | Новый скилл добавляет проверку |

## Типы ссылок

При удалении скилла нужно найти и удалить ссылки следующих типов:

### Связанные скиллы

**Паттерн:**
```regex
- \[{имя}\]\(.*\).*
```

**Пример:**
```markdown
- [links-validate](/.claude/skills/links-validate/SKILL.md) — проверка ссылок
```

**Действие:** Удалить строку целиком

### Блок интеграции

**Паттерн:**
```regex
\*\*Интеграция с {имя}:\*\*[\s\S]*?(?=\n\n|\n###|\n\*\*|$)
```

**Пример:**
```markdown
**Интеграция с links-validate:**
Вызвать [/links-validate](/.claude/skills/links-validate/SKILL.md) для проверки ссылок.
```

**Действие:** Удалить весь блок (от заголовка до пустой строки)

### Вызов в тексте

**Паттерн:**
```regex
Вызвать.*\[?/?{имя}\]?.*
```

**Пример:**
```markdown
Вызвать /links-validate для проверки.
```

**Действие:** Удалить строку или предложение

### Чек-лист

**Паттерн:**
```regex
- \[ \].*{имя}.*
```

**Пример:**
```markdown
- [ ] **Шаг 7:** Вызвал /links-validate
```

**Действие:** Удалить строку

### Frontmatter (агенты)

**Паттерн:**
```regex
skills:.*{имя}
```

**Пример до:**
```yaml
skills: [skill-create, skill-update, links-validate]
```

**Пример после:**
```yaml
skills: [skill-create, skill-update]
```

**Алгоритм удаления из массива YAML:**
1. Найти строку `skills: [...]`
2. Распарсить массив
3. Удалить элемент `{имя}`
4. Если массив пустой — удалить всю строку `skills:`

## Формат интеграции в воркфлоу

### Добавление в шаг

```markdown
**Интеграция с {новый-скилл}:**
Вызвать [/{новый-скилл}](/.claude/skills/{новый-скилл}/SKILL.md) для {описание}.
```

### Добавление в чек-лист

```markdown
- [ ] **Шаг {N}:** Вызвал [/{новый-скилл}](/.claude/skills/{новый-скилл}/SKILL.md)
```

### Добавление в связанные скиллы

```markdown
**Связанные скиллы:**
- [{существующий}](/.claude/skills/{существующий}/SKILL.md) — описание
- [{новый-скилл}](/.claude/skills/{новый-скилл}/SKILL.md) — описание
```

## Примеры

### Пример 1: Интеграция issue-review

**Контекст:** Создан скилл `issue-review` для ревью решения перед закрытием Issue.

```
/skill-update issue-review

📋 Предложения по интеграции скилла issue-review

📄 issue-execute — Шаг 5 (Завершение работы)
   Польза: Вызвать ревью перед закрытием Issue

📄 issue-complete — Шаг 1 (Проверка готовности)
   Польза: Убедиться, что ревью пройдено

Применить? [Y/n]
```

### Пример 2: Удаление ссылок на скилл

**Контекст:** Удалён скилл `links-validate`.

```
/skill-delete links-validate

📄 skill-create — найдено 2 ссылки
   - Связанные скиллы (строка 28)
   - Шаг 7 — Валидация (строки 156-158)

Удалить? [Y/n]
```
