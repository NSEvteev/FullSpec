## Обновление

**Когда обновлять:**
- Изменился путь к SSOT-инструкции
- Изменились параметры вызова
- Добавляются новые триггеры

### Шаг 1: Найти скилл

```
/.claude/skills/{name}/SKILL.md
```

### Шаг 2: Внести изменения

- Frontmatter (если нужно)
- SSOT-ссылка (если изменился путь)
- Параметры (если изменились)
- Примеры (если изменились)

**Что НЕ менять в скилле:**
- Логику (она в SSOT-инструкции)
- Шаги воркфлоу (они в SSOT)
- Чек-лист (он в SSOT)

### Шаг 3: Валидация

```bash
python .claude/.instructions/skills/.scripts/validate-skill.py {name}
```

---

## Деактивация

**Когда деактивировать:**
- SSOT-инструкция удалена
- Скилл заменён другим
- Функционал больше не нужен

### Шаг 1: Найти все ссылки

```bash
python .claude/.instructions/skills/.scripts/find-skill-references.py {skill-name}
```

**Если скрипт недоступен:** использовать паттерны из секции [Обновление ссылок](#обновление-ссылок).

### Шаг 2: Обновить ссылающиеся документы

- Удалить упоминания
- Заменить на альтернативу
- Удалить из `/.claude/skills/README.md`

### Шаг 3: Пометить папку _old

```bash
mv /.claude/skills/{name}/ /.claude/skills/{name}_old/
git commit -m "chore: деактивирован скилл {name} — {причина}"
```

## Миграция

**Когда мигрировать:**
- Переименование скилла
- Изменение структуры (object-action)

### Шаг 1: Найти все ссылки на старое имя

```bash
python .claude/.instructions/skills/.scripts/find-skill-references.py {old-name}
```

**Если скрипт недоступен:** использовать паттерны из секции [Обновление ссылок](#обновление-ссылок).

### Шаг 2: Создать новый скилл

```bash
# Использовать /skill-create {new-name}
```

### Шаг 3: Обновить все ссылки

Заменить старое имя на новое во всех документах.

### Шаг 4: Пометить старый скилл _old

```bash
mv /.claude/skills/{old-name}/ /.claude/skills/{old-name}_old/
```

## Обновление ссылок

### Где искать

| Место | Glob-паттерн |
|-------|--------------|
| Скиллы | `/.claude/skills/*/SKILL.md` |
| Инструкции | `**/.instructions/**/*.md` |
| README | `**/README.md` |
| CLAUDE.md | `/CLAUDE.md` |
| Rules | `/.claude/rules/*.md` |

### Типы ссылок

```markdown
# Markdown-ссылка
[/skill-name](/.claude/skills/skill-name/SKILL.md)

# Упоминание команды
`/skill-name`

# В таблице
| [/skill-name](path) | описание |
```

### Команды поиска

```bash
# Все упоминания
grep -r "skill-name" --include="*.md"

# Ссылки на SKILL.md
grep -r "skills/skill-name" --include="*.md"

# Команды
grep -r "/skill-name" --include="*.md"
```

## Примеры

### Обновление SSOT-ссылки

```bash
# 1. Найти скилл
cat /.claude/skills/structure-create/SKILL.md

# 2. Изменить путь к SSOT-инструкции
# **SSOT:** [create-structure.md](новый/путь)

# 3. Валидация
python .claude/.instructions/skills/.scripts/validate-skill.py structure-create
```

### Деактивация устаревшего скилла

```bash
# 1. Поиск ссылок
grep -r "skill-old" --include="*.md"

# 2. Обновить ссылающиеся документы

# 3. Пометить папку _old
mv /.claude/skills/skill-old/ /.claude/skills/skill-old_old/
```
