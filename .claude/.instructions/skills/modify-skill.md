## Обновление

**Когда обновлять:**
- Изменился путь к SSOT-инструкции
- Изменились параметры вызова
- Добавляются новые триггеры

### Шаг 1: Найти скилл

```
/.claude/skills/{name}/SKILL.md
```

### Шаг 2: Внести изменения

- Frontmatter (если нужно)
- SSOT-ссылка (если изменился путь)
- Параметры (если изменились)
- Примеры (если изменились)

**Что НЕ менять в скилле:**
- Логику (она в SSOT-инструкции)
- Шаги воркфлоу (они в SSOT)
- Чек-лист (он в SSOT)

### Шаг 3: Подтверждение изменений

> **ОБЯЗАТЕЛЬНО** использовать `AskUserQuestion` tool с кликабельными опциями.

**Вопрос:** "Изменения в скилле внесены. Проверьте."

**Опции:**
1. Подтверждаю — продолжить валидацию
2. Дорабатываем — нужны изменения в связанной инструкции
3. Другое — свободный ввод

**При выборе "Дорабатываем":**

> **ВАЖНО:** Дорабатывается НЕ скилл, а связанная SSOT-инструкция.

1. **Определить тип SSOT-инструкции:**
   - Если это `create-`, `modify-` или `validation-` инструкция

2. **Изучить ВСЕ 4 связанных документа:**
   - `standard-{object}.md` — стандарт
   - `create-{object}.md` — создание
   - `modify-{object}.md` — модификация
   - `validation-{object}.md` — валидация

3. **Анализ контекста:**
   - Как изменения в одном документе влияют на остальные?
   - Какие секции нужно синхронизировать?
   - Какие ссылки нужно обновить?

4. **Сформулировать план изменений:**
   ```
   Я буду теперь делать так:

   **Документы для обновления:**
   - `{путь 1}`: {что изменить}
   - `{путь 2}`: {что изменить}
   - ...

   **Изменения обеспечивают:**
   - {согласованность 1}
   - {согласованность 2}
   ```

5. **Внести изменения во все 4 документа** (если нужно)

6. **Повторить Шаг 3** (цикл до "Подтверждаю")

### Шаг 4: Валидация

```bash
python .claude/.instructions/skills/.scripts/validate-skill.py {name}
```

### Шаг 5: Отчёт о проделанной работе

```
## Отчёт об изменении скилла

**Изменён скилл:** `/{skill-name}`

**Тип изменения:** Обновление

**Что изменено в скилле:**
- {список изменений}

**Связанные документы обновлены:**
- `{путь 1}`: {что изменено}
- или "не требовалось"

**README обновлён:** {да/нет}

**Валидация:** пройдена ✅
```

## Деактивация

**Когда деактивировать:**
- SSOT-инструкция удалена
- Скилл заменён другим
- Функционал больше не нужен

### Шаг 1: Найти все ссылки

```bash
python .claude/.instructions/skills/.scripts/find-skill-references.py {skill-name}
```

**Если скрипт недоступен:** использовать паттерны из секции [Обновление ссылок](#обновление-ссылок).

### Шаг 2: Обновить ссылающиеся документы

- Удалить упоминания
- Заменить на альтернативу
- Удалить из `/.claude/skills/README.md`

### Шаг 3: Пометить папку _old

```bash
mv /.claude/skills/{name}/ /.claude/skills/{name}_old/
git commit -m "chore: деактивирован скилл {name} — {причина}"
```

### Шаг 4: Отчёт о проделанной работе

```
## Отчёт о деактивации скилла

**Деактивирован скилл:** `/{skill-name}`

**Причина:** {причина деактивации}

**Обновлены документы:**
- {список документов, из которых удалены ссылки}

**README обновлён:** ✅

**Статус:** деактивирован ✅
```

## Миграция

**Когда мигрировать:**
- Переименование скилла
- Изменение структуры (object-action)

### Шаг 1: Найти все ссылки на старое имя

```bash
python .claude/.instructions/skills/.scripts/find-skill-references.py {old-name}
```

**Если скрипт недоступен:** использовать паттерны из секции [Обновление ссылок](#обновление-ссылок).

### Шаг 2: Создать новый скилл

```bash
# Использовать /skill-create {new-name}
```

### Шаг 3: Обновить все ссылки

Заменить старое имя на новое во всех документах.

### Шаг 4: Пометить старый скилл _old

```bash
mv /.claude/skills/{old-name}/ /.claude/skills/{old-name}_old/
```

### Шаг 5: Отчёт (миграция)

```
## Отчёт о миграции скилла

**Мигрирован скилл:**
- Старое имя: `/{old-name}`
- Новое имя: `/{new-name}`

**Обновлены ссылки в:**
- {список документов с обновлёнными ссылками}

**README обновлён:** ✅

**Валидация:** пройдена ✅
```

## Обновление ссылок

### Где искать

| Место | Glob-паттерн |
|-------|--------------|
| Скиллы | `/.claude/skills/*/SKILL.md` |
| Инструкции | `**/.instructions/**/*.md` |
| README | `**/README.md` |
| CLAUDE.md | `/CLAUDE.md` |
| Rules | `/.claude/rules/*.md` |

### Типы ссылок

```markdown
# Markdown-ссылка
[/skill-name](/.claude/skills/skill-name/SKILL.md)

# Упоминание команды
`/skill-name`

# В таблице
| [/skill-name](path) | описание |
```

### Команды поиска

```bash
# Все упоминания
grep -r "skill-name" --include="*.md"

# Ссылки на SKILL.md
grep -r "skills/skill-name" --include="*.md"

# Команды
grep -r "/skill-name" --include="*.md"
```

## Примеры

### Обновление SSOT-ссылки

```bash
# 1. Найти скилл
cat /.claude/skills/structure-create/SKILL.md

# 2. Изменить путь к SSOT-инструкции
# **SSOT:** [create-structure.md](новый/путь)

# 3. Валидация
python .claude/.instructions/skills/.scripts/validate-skill.py structure-create
```

```bash
# 1. Поиск ссылок
grep -r "skill-old" --include="*.md"

# 3. Пометить папку _old
mv /.claude/skills/skill-old/ /.claude/skills/skill-old_old/
```
