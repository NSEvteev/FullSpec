## Принцип

Вывод должен быть:
- **Узнаваемым** — пользователь сразу понимает тип сообщения по иконке
- **Консистентным** — одинаковый формат во всех скиллах
- **Информативным** — содержит всё необходимое для понимания

---

## Иконки и их значение

| Иконка | Тип | Когда использовать |
|--------|-----|-------------------|
| ✅ | Успех | Операция завершена успешно |
| ❌ | Ошибка | Операция не выполнена |
| ⚠️ | Предупреждение | Требует внимания, но не блокирует |
| 📋 | Ревью/Список | Обзор, чек-лист, результаты проверки |
| ⏳ | Прогресс | Операция выполняется |
| ℹ️ | Информация | Нейтральное сообщение |
| 💡 | Предложение | Рекомендация, улучшение |
| ❓ | Вопрос | Требуется ввод пользователя |

## Форматы по типу сообщения

### Успех (✅)

```
✅ {Объект} {действие}

{Детали результата}

Следующие шаги:
- {шаг 1}
- {шаг 2}
```

**Пример:**
```
✅ Скилл создан успешно

Файл: /.claude/skills/my-skill/SKILL.md
Категория: utility

Следующие шаги:
- Проверить и доработать инструкции
- Протестировать скилл
```

### Ошибка (❌)

> **Детальный формат:** см. [errors.md](./errors.md)

```
❌ Ошибка: {краткое описание}

Причина: {детали}
Решение: {что сделать}
```

### Предупреждение (⚠️)

```
⚠️ {Краткое описание ситуации}

{Детали}

Варианты:
[1] {Вариант 1}
[2] {Вариант 2}
[3] Отменить
```

### Ревью / Список (📋)

**Формат ревью:**
```
📋 Ревью {объекта}

✅ Сильные стороны:
- {пункт 1}

💡 Предложения по улучшению:
1. {предложение 1}
2. {предложение 2}

Применить предложения? (все / выборочно / пропустить)
```

**Формат чек-листа:**
```
✅ Шаг 1: Получил номер Issue
✅ Шаг 2: Проверил статус
⏳ Шаг 3: Закрываю Issue...
⬜ Шаг 4: Обновить документацию
```

### Прогресс (⏳)

```
⏳ {Действие}...

{Детали, если есть}
```

### Информация (ℹ️)

```
ℹ️ {Сообщение}

{Детали, если нужны}
```

### Предложение (💡)

```
💡 {Предложение}

{Обоснование}

Применить? [Y/n]
```

### Вопрос (❓)

```
❓ {Вопрос}

[1] {Вариант 1} — {описание}
[2] {Вариант 2} — {описание}

Что выбрать? [1/2]
```

## Составные форматы

### Предварительный просмотр (--dry-run)

```
📋 Предварительный просмотр (--dry-run)

Будет создано:
- {путь 1}

Будет обновлено:
- {путь 2}

Будет вызвано:
- /{скилл-1}

ℹ️ Изменения НЕ применены (--dry-run)
```

### Результат с метриками

```
✅ {Операция} завершена

Статистика:
| Метрика | Значение |
|---------|----------|
| Обработано | {N} |
| Успешно | {M} |
| Пропущено | {K} |

Время: {X} сек
```

## Таблицы

### Формат таблицы статусов

```
| Объект | Статус | Примечание |
|--------|--------|------------|
| file1.md | ✅ Обновлён | — |
| file2.md | ⚠️ Пропущен | Нет изменений |
| file3.md | ❌ Ошибка | Нет доступа |
```

## JSON формат

Для интеграции с CI/CD скиллы поддерживают `--json`.

### Структура

```json
{
  "status": "success|error|warning",
  "message": "Краткое описание",
  "data": { },
  "meta": {
    "skill": "skill-name",
    "timestamp": "2026-01-20T14:30:00Z"
  }
}
```

### Пример успеха

```json
{
  "status": "success",
  "message": "Скилл создан успешно",
  "data": {
    "path": "/.claude/skills/my-skill/SKILL.md",
    "category": "utility"
  }
}
```

### Пример ошибки

```json
{
  "status": "error",
  "message": "Файл уже существует",
  "error": {
    "code": "FILE_EXISTS",
    "details": "/.claude/skills/my-skill/SKILL.md"
  }
}
```
