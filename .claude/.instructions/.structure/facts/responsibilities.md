# Формат responsibilities.md

Правила формата и обновления файла `/.structure/responsibilities.md`.

---

## Назначение

Файл `/.structure/responsibilities.md` — **SSOT всех зон ответственности** папок проекта.

**SSOT:** [/.structure/responsibilities.md](/.structure/responsibilities.md)

---

## Формат секции

Каждая папка описывается по шаблону:

```markdown
### /path/to/folder/

**IN:** Что хранится — типы файлов, назначение

**Границы:**
- что относится сюда → здесь
- что НЕ относится → куда класть
```

---

## Элементы

| Элемент | Обязателен | Описание |
|---------|------------|----------|
| `### /path/` | ✅ | Заголовок = путь к папке |
| `**IN:**` | ✅ | Что хранится |
| `**Границы:**` | ✅ | Разделение ответственности |

---

## Правила описания

1. **IN — максимально конкретно**
   - ❌ "код"
   - ✅ "handlers, services, models, utils"

2. **Границы — с редиректами**
   - ❌ "общий код не здесь"
   - ✅ "общий код → /shared/"

3. **Иерархия** — вложенная папка уточняет родительскую

---

## Структура файла

```markdown
# Зоны ответственности папок проекта (SSOT)

> **Правила:** /.claude/.instructions/.structure/facts/responsibilities.md

---

## Корневые файлы

### README.md
**IN:** ...
**Границы:** ...

---

## /src/ — Исходный код сервисов

> **Инструкция:** /.claude/.instructions/src/

### /src/
**IN:** ...
**Границы:** ...

### /src/{service}/
**IN:** ...
**Границы:** ...

---

## /platform/ — ...

...
```

---

## Когда обновлять

| Событие | Действие |
|---------|----------|
| Создана папка | Добавить секцию с зоной |
| Удалена папка | Убрать секцию |
| Изменилась зона | Обновить IN и Границы |

---

## Связь с README папок

После обновления `/.structure/responsibilities.md`:

1. **Скопировать** зону в README папки
2. **Добавить ссылку** на полный файл

```markdown
## Зона ответственности

{Копия из responsibilities.md}

> **Все зоны:** [/.structure/responsibilities.md](/.structure/responsibilities.md)
```
