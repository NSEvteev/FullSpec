## Правило "1 уровень"

**КРИТИЧНО:** README.md папки показывает **только прямых потомков**.

| Элемент | Что показывать |
|---------|---------------|
| Файлы текущей папки | Полное описание, оглавление секций |
| Подпапки | **Только ссылка на README.md подпапки** |

### Почему

- Избегаем дублирования информации
- Каждая папка отвечает за свой README
- Изменения в подпапке не требуют обновления родителя

### Пример

```
parent/
├── README.md       # Описывает file.md + ссылка на child/README.md
├── file.md         # ← полное описание здесь
└── child/
    ├── README.md   # ← содержимое описано ЗДЕСЬ, не в parent/README.md
    └── nested.md
```

**В parent/README.md:**
```markdown
# 3. Child

Подпапка с {описание}.

**Индекс:** [child/README.md](./child/README.md)
```

**НЕ показывать:** содержимое `nested.md` в `parent/README.md`

## Обязательные секции

| Секция | Обязательна | Описание |
|--------|-------------|----------|
| Заголовок | ✅ | `# Инструкции /{папка}/` |
| Описание | ✅ | Одно предложение |
| Содержание | ✅ | Перечисление тем |
| Оглавление | ✅ | Таблица секций |
| Дерево файлов | ✅ | Структура папки |
| Секции файлов | ✅ | По одной на файл |
| Секции подпапок | Если есть | Только ссылка на README |
| Шаблоны | Если есть | Таблица шаблонов |
| Скиллы | Если есть | Таблица скиллов |
| Скрипты | Если есть | Таблица скриптов |

## Формат подпапок

### В таблице оглавления

```markdown
| [N. {Название}](#n-название) | [{subfolder}/](./{subfolder}/) | → [README.md](./{subfolder}/README.md) |
```

### В секции

```markdown
# N. {Название}

{Краткое описание подпапки — одно предложение.}

**Индекс:** [{subfolder}/README.md](./{subfolder}/README.md)
```

**Никакого дополнительного содержимого!**

---

## Когда создавать

README.md создаётся:
1. При создании новой папки инструкций
2. Автоматически скиллом `/instruction-create` (если папка новая)

README.md обновляется:
1. При добавлении файла в существующую папку
2. При добавлении подпапки
3. При удалении/переименовании файлов
