# README для папок проекта

Формат README.md для папок проекта (`/src/`, `/platform/`, etc.).

---

## Назначение

README папки проекта описывает **зону ответственности** — что здесь хранится и что нет.

---

## Обязательная структура

```markdown
# {Название папки}

## Зона ответственности

{Копия из /.structure/responsibilities.md}

**IN:** {что хранится}

**Границы:**
- {что здесь} → здесь
- {что не здесь} → {куда}

> **Все зоны:** [/.structure/responsibilities.md](/.structure/responsibilities.md)

---

## Содержимое

{Описание файлов и подпапок}

---

## См. также

- {ссылки на связанные ресурсы}
```

---

## Секции

| Секция | Обязательна | Описание |
|--------|-------------|----------|
| Заголовок | ✅ | `# {Название}` |
| Зона ответственности | ✅ | Копия из SSOT + ссылка |
| Содержимое | ✅ | Описание содержимого |
| См. также | ⚠️ | Если есть связи |

---

## Правила

1. **Зона = копия из SSOT**
   - Копировать из `/.structure/responsibilities.md`
   - Добавить ссылку на полный файл

2. **Содержимое — только прямые потомки**
   - Файлы текущей папки — описать
   - Подпапки — только ссылка на их README

3. **Не дублировать**
   - Если есть зеркало в инструкциях — сослаться

---

## Пример

```markdown
# /src/auth/

## Зона ответственности

Сервис аутентификации и авторизации.

**IN:** backend/, database/, tests/, docs/, README.md, Makefile

**Границы:**
- код сервиса auth → здесь
- общие утилиты авторизации → /shared/libs/auth/
- E2E тесты → /tests/e2e/
- ADR → /specs/services/auth/adr/

> **Все зоны:** [/.structure/responsibilities.md](/.structure/responsibilities.md)

---

## Содержимое

| Папка/Файл | Описание |
|------------|----------|
| `backend/` | Серверный код |
| `database/` | Схемы и миграции |
| `tests/` | Unit и integration тесты |
| `docs/` | Документация API |

---

## См. также

- **Инструкции:** [/.claude/.instructions/src/](/.claude/.instructions/src/)
- **ADR:** [/specs/services/auth/adr/](/specs/services/auth/adr/)
```
