# README для папок инструкций

Формат README.md для папок инструкций (`/.claude/.instructions/*`).

---

## Назначение

README папки инструкций — **индекс правил** для LLM.

---

## Обязательная структура

```markdown
# Инструкции /{область}/

{Одно предложение — что покрывают инструкции}

---

## Оглавление

| Секция | Инструкция | Описание |
|--------|------------|----------|
| [1. {Тема}](#1-тема) | [{file}.md](./{file}.md) | {Описание} |
| [2. {Подпапка}](#2-подпапка) | [{folder}/](./{folder}/) | → [README.md](./{folder}/README.md) |

\`\`\`
/.claude/.instructions/{область}/
├── README.md           # Этот файл
├── {file}.md           # Описание
└── {folder}/           # → README.md
\`\`\`

---

# 1. {Тема}

{Краткое описание}

**Оглавление:**
- [{Секция 1}](./{file}.md#секция-1)
- [{Секция 2}](./{file}.md#секция-2)

**Инструкция:** [{file}.md](./{file}.md)

---

# 2. {Подпапка}

{Краткое описание подпапки}

**Индекс:** [{folder}/README.md](./{folder}/README.md)
```

---

## Секции

| Секция | Обязательна | Описание |
|--------|-------------|----------|
| Заголовок | ✅ | `# Инструкции /{область}/` |
| Описание | ✅ | Одно предложение |
| Оглавление | ✅ | Таблица + дерево файлов |
| Секции файлов | ✅ | По одной на файл |
| Секции подпапок | Если есть | Только ссылка на README |

---

## Правила

### Правило "1 уровень"

README показывает **только прямых потомков**:
- Файлы текущей папки → полное описание
- Подпапки → **только ссылка на их README**

### Формат подпапок

В таблице:
```markdown
| [N. {Название}](#n-название) | [{folder}/](./{folder}/) | → [README.md](./{folder}/README.md) |
```

В секции:
```markdown
# N. {Название}

{Краткое описание — одно предложение}

**Индекс:** [{folder}/README.md](./{folder}/README.md)
```

**Никакого дополнительного содержимого подпапки!**

---

## Почему так

- Избегаем дублирования информации
- Каждая папка отвечает за свой README
- Изменения в подпапке не требуют обновления родителя
