## Обновление

Изменение содержимого rule или frontmatter.

### Шаг 1: Прочитать rule

```bash
# Прочитать существующий rule
cat .claude/rules/{name}.md
```

**Цель:** Понять текущее содержимое и структуру.

### Шаг 2: Внести изменения

**Что можно изменять:**

| Элемент | Описание |
|---------|----------|
| `description` | Описание в frontmatter |
| `paths` | Glob-паттерны для условного rule |
| Содержимое | Текст rule (H1, описание, ссылки на скиллы) |

**Ограничения:**
- Не изменять `standard` и `index` в frontmatter
- Сохранять формат ссылок на скиллы: `→ /skill-name`
- Не превышать 50 строк

**Изменение типа (глобальный ↔ условный):**

```yaml
# Из глобального в условный
# Было:
---
description: ...
standard: ...
index: ...
---

# Стало:
---
description: ...
standard: ...
index: ...
paths:
  - "паттерн/**"
---

# Из условного в глобальный — удалить поле paths
```

### Шаг 3: Валидация

```bash
python .claude/.instructions/rules/.scripts/validate-rule.py {name}
```

### Шаг 4: Отчёт

```
## 📋 Отчёт об обновлении rule

✏️ **Изменён rule:** `{name}.md`

📝 **Что изменено:**
- {список изменений}

✅ **Валидация:** пройдена

---

⚠️ **ТРЕБУЕТСЯ НОВАЯ СЕССИЯ**

Изменения в rule вступят в силу только после:
1. Завершения текущей сессии
2. Начала НОВОЙ сессии Claude Code
```

## Деактивация

Rule больше не используется, но не удаляется (для истории).

**Принцип:** Не удалять файлы — переименовать в `_old-{name}.md`.

### Шаг 1: Найти ссылки на rule

> **Проверить:** Есть ли документация, которая ссылается на этот rule?

**Где искать:**
- README файлы
- Документация проекта
- Комментарии в коде

**Если найдены ссылки:**
- Удалить или обновить их

### Шаг 2: Переименовать файл

```bash
# Переименовать в _old-{name}.md
mv .claude/rules/{name}.md .claude/rules/_old-{name}.md
```

**Соглашение:**
- Префикс `_old-` — файл деактивирован
- Claude Code игнорирует файлы с `_` в начале
- Файл остаётся в git для истории

### Шаг 3: Отчёт

```
## 📋 Отчёт о деактивации rule

🚫 **Деактивирован rule:** `{name}.md`

**Новое имя:** `_old-{name}.md`

**Причина деактивации:** {причина}

---

⚠️ **ТРЕБУЕТСЯ НОВАЯ СЕССИЯ**

Для применения изменений необходимо:
1. Завершить текущую сессию
2. Начать НОВУЮ сессию Claude Code

Деактивированный rule больше не будет загружаться.
```

## Миграция

Переименование rule (например, для улучшения именования).

### Шаг 1: Переименовать rule

```bash
# Переименовать файл
mv .claude/rules/{old-name}.md .claude/rules/{new-name}.md
```

### Шаг 2: Обновить ссылки

**Где обновлять:**
- Документация, ссылающаяся на rule
- README файлы
- Комментарии в коде

**Формат поиска:**
```bash
# Найти упоминания старого имени
grep -r "{old-name}" .
```

### Шаг 3: Валидация после миграции

```bash
python .claude/.instructions/rules/.scripts/validate-rule.py {new-name}
```

### Шаг 4: Отчёт о миграции

```
## 📋 Отчёт о миграции rule

**Старое имя:** `{old-name}.md`

**Новое имя:** `{new-name}.md`

**Обновлено ссылок:** {количество}

✅ **Валидация:** пройдена

---

⚠️ **ТРЕБУЕТСЯ НОВАЯ СЕССИЯ**

Для применения изменений необходимо:
1. Завершить текущую сессию
2. Начать НОВУЮ сессию Claude Code

Rule будет загружаться под новым именем.
```

## Добавление скиллов (из /instruction-create)

> **Контекст:** Вызывается из `/instruction-create` после создания инструкций и скиллов.

**Когда использовать:**
- При создании инструкций для области, которая уже покрыта существующим rule
- Нужно добавить новые скиллы в rule

**Порядок скиллов:** validate → create → modify

### Шаг 1: Прочитать rule

```bash
cat .claude/rules/{name}.md
```

### Шаг 2: Добавить секцию скиллов

**Формат:**
```markdown
При работе с [область] ОБЯЗАТЕЛЬНО использовать скиллы:
  - /[область]-validate - валидация [сущность]
  - /[область]-create - создание [сущность]
  - /[область]-modify - изменение [сущность]
```

**Примечание:** Добавлять только те скиллы, которые были созданы.

### Шаг 3: Валидация

```bash
python .claude/.instructions/rules/.scripts/validate-rule.py {name}
```

### Шаг 4: Отчёт

```
## 📋 Отчёт о добавлении скиллов в rule

✏️ **Изменён rule:** `{name}.md`

**Добавлены скиллы:**
- /[область]-validate
- /[область]-create
- /[область]-modify

✅ **Валидация:** пройдена

---

⚠️ **ТРЕБУЕТСЯ НОВАЯ СЕССИЯ**
```

## Обновление ссылок

При миграции или деактивации rule нужно обновить ссылки.

**Типы ссылок:**

| Тип | Формат | Где встречается |
|-----|--------|-----------------|
| Markdown-ссылки | `[текст](путь)` | README, документация |
| Упоминания | `{name}.md` | Комментарии, тексты |

**Процесс:**

1. **Найти все ссылки:**
   ```bash
   grep -r "{old-name}" .
   ```

2. **Обновить каждую ссылку:**
   - Для деактивации — удалить ссылку
   - Для миграции — заменить на новое имя

3. **Проверить результат:**
   ```bash
   /links-validate
   ```

### Обновление: Добавить скилл в триггеры

```bash
# Шаг 1: Прочитать rule
cat .claude/rules/instructions.md

# Текущее содержимое:
# При создании инструкции → `/instruction-create`

# Шаг 2: Внести изменения
# Добавить строку:
# При валидации инструкции → `/instruction-validate`

# Шаг 3: Валидация

# Шаг 4: Отчёт
# ⚠️ Требуется новая сессия
```

### Обновление: Изменить paths

```bash
# Шаг 1: Прочитать rule
cat .claude/rules/scripts.md

# Текущий frontmatter:
# paths:
#   - ".scripts/**"

# Шаг 2: Изменить paths
# Новый frontmatter:
# paths:
#   - "**/.scripts/**"

# Шаг 3: Валидация
python .claude/.instructions/rules/.scripts/validate-rule.py scripts

# Шаг 4: Отчёт
# ⚠️ Требуется новая сессия
```

### Деактивация: Устаревший rule

```bash
# Шаг 1: Найти ссылки
grep -r "old-workflow" .
# Найдено: README.md, docs/guide.md

# Удалить упоминания из документации

# Шаг 2: Переименовать
mv .claude/rules/old-workflow.md .claude/rules/_old-old-workflow.md

# Шаг 3: Отчёт
# ⚠️ Требуется новая сессия
```

### Миграция: Улучшить именование

```bash
# Шаг 1: Переименовать
mv .claude/rules/core.md .claude/rules/critical-rules.md

# Шаг 2: Найти ссылки
grep -r "core\.md" .
# Найдено: README.md

# Обновить ссылки:
# core.md → critical-rules.md

# Шаг 3: Валидация
python .claude/.instructions/rules/.scripts/validate-rule.py critical-rules

# Шаг 4: Отчёт
# ⚠️ Требуется новая сессия
```

```bash

#   - /hooks-create - создание хука
#   - /hooks-modify - изменение хука
```
