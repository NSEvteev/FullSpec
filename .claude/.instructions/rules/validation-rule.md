## Шаги

### Шаг 1: Автоматическая валидация

**Автоматически:**
```bash
python .claude/.instructions/rules/.scripts/validate-rule.py {name}
```

Где `{name}` — имя rule без расширения (например, `core` для `core.md`).

**Что проверяет:**
- Расположение файла в `/.claude/rules/`
- Расширение `.md`
- Валидность frontmatter
- Обязательные поля: `description`, `standard`, `index`
- Опциональное поле `paths` (если есть — проверка валидности glob-паттернов)

### Шаг 2: Ручная проверка чек-листа

**Когда использовать:** Если скрипт недоступен или требуется детальная проверка.

**Чек-лист:**

#### Файл
- [ ] Файл находится в `/.claude/rules/`
- [ ] Расширение файла `.md`
- [ ] Имя файла в kebab-case (например, `core.md`)

#### Frontmatter
- [ ] Frontmatter валиден (YAML)
- [ ] Поле `description` присутствует и не пустое
- [ ] Поле `standard` указывает на `.claude/.instructions/rules/standard-rule.md`
- [ ] Поле `index` указывает на `.claude/.instructions/rules/README.md`
- [ ] Если есть `paths` — это массив строк с валидными glob-паттернами

#### Структура
- [ ] Содержимое не превышает 50 строк (рекомендация)

#### Содержимое
- [ ] Rule содержит краткое описание или триггеры
- [ ] Ссылки на скиллы в формате `→ /skill-name`
- [ ] Нет дублирования содержимого инструкций (только ссылки на них)

### Шаг 3: Тестирование после рестарта

> **⚠️ КРИТИЧЕСКОЕ ПРАВИЛО:** Rules загружаются ТОЛЬКО при старте сессии Claude Code.

**Для применения rule необходимо:**
1. Завершить текущую сессию
2. Начать НОВУЮ сессию Claude Code

**Проверка работоспособности:**

#### Метод 1: Команда /memory

```
/memory
```

**Что проверить:**
- Rule должен быть в списке загруженных rules
- Для условных rules — проверить указанные `paths`

#### Метод 2: Триггер по paths (для условных rules)

1. Открыть файл, соответствующий `paths` из rule
2. Попросить Claude выполнить действие из rule
3. Убедиться, что Claude следует правилам rule

**Пример для `instructions.md` с `paths: [".claude/rules/**"]`:**

```
1. Открыть .claude/rules/core.md
2. Попросить: "Создай новый rule"
3. Claude должен использовать /rule-create (как указано в instructions.md)
```

## Чек-лист

### Автоматическая валидация
- [ ] Запущен `validate-rule.py`
- [ ] Все проверки пройдены (exit code 0)

### Ручная валидация (если скрипт недоступен)
- [ ] Файл в правильном расположении
- [ ] Frontmatter валиден
- [ ] Обязательные поля присутствуют
- [ ] Структура соответствует стандарту

### Тестирование
- [ ] Текущая сессия завершена
- [ ] Начата НОВАЯ сессия
- [ ] Rule загружен (проверено через `/memory`)
- [ ] Триггеры работают (для условных rules)

## Типичные ошибки

| Код | Ошибка | Причина | Решение |
|-----|--------|---------|---------|
| R001 | Неверное расположение | Файл не в `/.claude/rules/` | Переместить файл в `/.claude/rules/` |
| R002 | Неверное расширение | Файл не `.md` | Переименовать файл с расширением `.md` |
| R010 | Невалидный frontmatter | YAML-синтаксис нарушен | Проверить синтаксис YAML, закрыть кавычки |
| R011 | Отсутствует `description` | Поле не указано или пустое | Добавить `description: {описание}` |
| R012 | Отсутствует `standard` | Поле не указано | Добавить `standard: .claude/.instructions/rules/standard-rule.md` |
| R013 | Отсутствует `index` | Поле не указано | Добавить `index: .claude/.instructions/rules/README.md` |
| R014 | `paths` не массив строк | `paths` указан, но не массив | Исправить формат: `paths: ["pattern/**"]` |
| R015 | Невалидный glob-паттерн | Паттерн содержит ошибки | Проверить синтаксис glob (например, `**/*.md`) |
