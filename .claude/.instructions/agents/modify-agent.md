## Принципы

> **Шаблоны — из примеров SSOT.** При изменении файлов использовать шаблоны из [standard-agent.md](./standard-agent.md). Запрещено придумывать свой формат.

> **Агенты НЕ удаляются, а деактивируются.** Конфигурации содержат знания о задачах, которые могут понадобиться позже.

> **Изменение промпта требует подтверждения.** Промпт — критическая часть агента, изменения нужно согласовать.

> **При добавлении Edit/Write — добавить секцию "Удаление файлов"** в промпте (правило `_old_` префикса).

## Обновление

Изменение конфигурации агента без изменения имени.

### Шаг 1: Найти агента

```
/.claude/agents/{agent-name}/AGENT.md
```

### Шаг 2: Проанализировать конфигурацию

**Проверить на соответствие стандарту:**
- Обязательные поля заполнены (name, description, type, prompt)
- Промпт содержит обязательные секции (Роль, Задача, Ограничения, Формат вывода)
- Тип агента корректен

**Сформировать предложения по доработке:**

| Что проверить | Возможные проблемы |
|---------------|-------------------|
| Актуальность | Устаревшие ссылки на инструкции/скиллы |
| Полнота | Отсутствующие секции промпта |
| Ясность | Неконкретные ограничения |
| Безопасность | Отсутствие ограничений на деструктивные операции |

**Предложить пользователю:** список найденных проблем и рекомендации.

### Шаг 3: Внести изменения

**Типичные изменения:**

| Изменение | Что обновить |
|-----------|--------------|
| Добавить ограничение | Секция `## Ограничения` в промпте |
| Изменить модель | Поле `model` во frontmatter |
| Добавить инструмент | Поле `tools` во frontmatter |
| Запретить инструмент | Поле `disallowedTools` во frontmatter |
| Изменить режим разрешений | Поле `permissionMode` во frontmatter (acceptEdits не затрагивает критичные файлы) |
| Добавить скилл | Поле `skills` во frontmatter + секция в промпте |
| Изменить max_turns | Поле `max_turns` во frontmatter |
| Обновить SSOT-ссылки | Секция `## Инструкции и SSOT` в промпте |
| Добавить hook | Поле `hooks` во frontmatter |
| Добавить Edit/Write | + секция `## Удаление файлов` (правило `_old_` префикса) |
| Обновить standard-version | Если стандарт обновился — обновить `standard-version` во frontmatter |

**ОБЯЗАТЕЛЬНО при любом изменении:**

1. Обновить `version` во frontmatter: `v1.0` → `v1.1`
2. Добавить запись в `CHANGELOG.md`:

```markdown
## v1.1 (YYYY-MM-DD)

### Изменено
- {что изменено}
```

### Шаг 4: Подтверждение изменений

> **ОБЯЗАТЕЛЬНО** использовать `AskUserQuestion` tool.

**Вопрос:** "Изменения внесены. Проверьте и подтвердите."

**Опции:**
1. Подтверждаю — изменения корректны
2. Нужны правки — укажу что исправить

**При выборе "Нужны правки":**

1. Спросить: "Что нужно исправить?"
2. Внести изменения
3. Повторить Шаг 4

### Шаг 5: Валидация

```bash
python .claude/.instructions/agents/.scripts/validate-agent.py .claude/agents/{agent-name}.yaml
```

### Шаг 6: Обновить README (если нужно)

Если изменились `description` или `type` — обновить `/.claude/agents/README.md`.

### Шаг 7: Отчёт (обновление)

```
## 📋 Отчёт об изменении агента

✏️ **Изменён агент:** `/.claude/agents/{name}/AGENT.md`

🏷️ **Тип изменения:** Обновление

**Версия:** {old-version} → {new-version}

📝 **Что изменено:**
- {список изменений}

📊 **Текущее состояние:**
- Описание: {description}
- Тип: {type}
- Модель: {model}

**Обновлённые файлы:**
- AGENT.md (frontmatter: version)
- CHANGELOG.md (новая запись)

✅ **Валидация:** пройдена
```

## Деактивация

> **Агенты НЕ удаляются. Они деактивируются с префиксом `DELETE_`.**

**Когда деактивировать:**
- Задача агента больше не актуальна
- Агент заменён другим
- Агент не используется длительное время

### Шаг 1: Найти все ссылки

```bash
grep -r "{agent-name}" --include="*.md" --include="*.yaml" .
```

**Где искать:**
- `/.claude/agents/README.md` — индекс агентов
- `**/.instructions/**/*.md` — инструкции
- `/.claude/drafts/*.md` — драфты
- Промпты других агентов

### Шаг 2: Пометить агента как удалённого

> **ПРАВИЛО:** Папки агентов НЕ удаляются, а помечаются `DELETE_`.

Переименовать папку:
```bash
mv .claude/agents/{agent-name} .claude/agents/DELETE_{agent-name}
```

### Шаг 3: Обновить README

В `/.claude/agents/README.md`:
- Удалить строку агента из таблицы
- Или пометить: `~~[agent-name]~~ (деактивирован)`

### Шаг 4: Обновить ссылающиеся документы

- Инструкции — удалить/пометить ссылки
- Драфты — пометить `<!-- BROKEN: агент деактивирован -->`
- Промпты других агентов — удалить из списка доступных

### Шаг 5: Отчёт (деактивация)

```
## 📋 Отчёт о деактивации агента

🚫 **Деактивирован агент:** `/.claude/agents/{name}.yaml`

🏷️ **Тип изменения:** Деактивация

**Причина:** {причина деактивации}

**Обновлены документы:**
- {список обновлённых ссылающихся документов}

**Статус:** деактивирован ✅
```

## Миграция

Переименование агента с сохранением функциональности.

**Когда мигрировать:**
- Имя не отражает назначение
- Унификация именования
- Слияние с другим агентом

### Шаг 1: Найти все ссылки на старое имя

```bash
python .claude/.instructions/agents/.scripts/find-agent-refs.py {old-agent-name}
```

**Если скрипт недоступен:**
```bash
grep -r "{old-agent-name}" --include="*.md" --include="*.yaml" .
```

### Шаг 2: Переименовать папку

```bash
mv .claude/agents/{old-name} .claude/agents/{new-name}
```

### Шаг 3: Обновить поле name

В файле `{new-name}/AGENT.md` обновить frontmatter:
```yaml
name: {new-name}  # было: {old-name}
```

### Шаг 4: Обновить все ссылки

Заменить все вхождения `{old-name}` на `{new-name}` в:
- `/.claude/agents/README.md`
- Инструкциях
- Драфтах
- Промптах других агентов

### Шаг 5: Обновить README

В `/.claude/agents/README.md` обновить имя агента в таблице.

### Шаг 6: Валидация (миграция)

```bash
python .claude/.instructions/agents/.scripts/validate-agent.py .claude/agents/{new-name}.yaml
```

### Шаг 7: Отчёт (миграция)

```
## 📋 Отчёт о миграции агента

**Мигрирован агент:**
- Старое имя: `{old-name}`
- Новое имя: `{new-name}`

🏷️ **Тип изменения:** Миграция

**Обновлены ссылки в:**
- {список документов с обновлёнными ссылками}

✅ **Валидация:** пройдена
```

## Обновление ссылок

### Где искать ссылки на агента

| Место | Glob-паттерн |
|-------|--------------|
| Индекс агентов | `/.claude/agents/README.md` |
| Инструкции | `**/.instructions/**/*.md` |
| Драфты | `/.claude/drafts/*.md` |
| Промпты агентов | `/.claude/agents/*/AGENT.md` |

### Типы ссылок

**Markdown-ссылка:**
```
[agent-name](./{agent-name}.yaml)
```

**Упоминание в промпте:**
```yaml
prompt: |
  Другие доступные агенты:
  - {agent-name}: {описание}
```

### Действия при обнаружении

| Тип ссылки | При деактивации | При миграции |
|------------|-----------------|--------------|
| Markdown | Удалить/пометить | Обновить путь |
| Промпт | Удалить из списка | Обновить имя |

## Примеры

### Обновление: добавление ограничения

```markdown
# Было (в теле AGENT.md):
## Ограничения
- НЕ модифицировать файлы

# Стало:
## Ограничения
- НЕ модифицировать файлы
- НЕ читать файлы с секретами (.env, credentials)
```

### Обновление: изменение модели и добавление disallowedTools

```yaml
# Было (frontmatter):
---
model: haiku
tools: Read, Grep, Glob
max_turns: 10
---

# Стало:
---
model: sonnet
tools: Read, Grep, Glob, Bash
disallowedTools: Write, Edit
max_turns: 20
---
```

### Деактивация: устаревший агент

```bash
# 1. Найти ссылки
grep -r "old-api-checker" --include="*.md" .

# 2. Переименовать папку
mv .claude/agents/old-api-checker .claude/agents/DELETE_old-api-checker

# 3. Обновить README — удалить строку из таблицы

# 4. Обновить ссылающиеся документы
```

### Миграция: переименование

```bash
# 1. Найти ссылки
grep -r "todo-finder" --include="*.md" .

# 2. Переименовать папку
mv .claude/agents/todo-finder .claude/agents/tech-debt-finder

# 3. Обновить name в AGENT.md
# name: tech-debt-finder

# 4. Заменить ссылки
# todo-finder → tech-debt-finder

# 5. Обновить README

# 6. Валидация
python .claude/.instructions/agents/.scripts/validate-agent.py .claude/agents/tech-debt-finder
```
