## Принципы

> **Шаблоны — из примеров SSOT.** При создании файлов использовать шаблоны из [standard-agent.md](./standard-agent.md). Запрещено придумывать свой формат.

> **Промпт формируется итеративно.** Сначала черновик на основе контекста, затем подтверждение пользователем.

> **Ограничения важнее инструкций.** Что агент НЕ должен делать — критичнее того, что должен.

> **Координация автоматизирована.** Хуки `SubagentStart`/`SubagentStop` автоматически регистрируют агента и выполняют cleanup. Агент должен только работать с блокировками файлов. См. [standard-state.md](../state/standard-state.md).

## Шаги

### Шаг 0: Проверить существующих агентов

> **ОБЯЗАТЕЛЬНО перед созданием** — убедиться, что нет агента с похожим функционалом.

```bash
python .claude/.instructions/agents/.scripts/list-agents.py
```

**LLM анализирует:**
1. Есть ли агент с похожим функционалом?
2. Можно ли расширить существующего?

**Если найден похожий:** использовать `AskUserQuestion` с вариантами:
1. Расширить существующий `{name}` → `/agent-modify`
2. Создать нового агента

### Шаг 1: Определить необходимость

**Вопросы перед созданием:**

| Вопрос | Если "нет" — не создавать агента |
|--------|----------------------------------|
| Задача повторяется > 3 раз? | Разовая задача → выполнить напрямую |
| Задача автономна? | Требует диалога → основной контекст |
| Задача специализирована? | Простая команда → Bash |

**Если все ответы "да" — продолжить.**

### Шаг 2: Выбрать тип агента

> **SSOT:** [standard-agent.md#2-типы-агентов](./standard-agent.md#2-типы-агентов)

| Тип | Когда использовать |
|-----|--------------------|
| `explore` | Поиск по кодовой базе, анализ структуры |
| `bash` | Git операции, сборка, тесты |
| `plan` | Планирование реализации, архитектура |
| `general-purpose` | Сложные многошаговые задачи |

### Шаг 3: Собрать контекст

**Автоматически выполнить:**

```bash
python .instructions/.scripts/list-instructions.py
```

**LLM анализирует:**

1. Какие инструкции релевантны задаче агента?
2. Какие скиллы могут понадобиться (для `general-purpose`)?
3. Какие правила из `/.claude/rules/` применимы?

**Результат:** список SSOT-ссылок и скиллов для включения в промпт.

### Шаг 4: Создать файл конфигурации

```bash
python .claude/.instructions/agents/.scripts/create-agent-file.py {name} {type} [--description "{text}"]
```

**Примеры:**
```bash
python .claude/.instructions/agents/.scripts/create-agent-file.py todo-finder explore
python .claude/.instructions/agents/.scripts/create-agent-file.py code-reviewer general-purpose --description "Код-ревью"
```

**Вручную (если нужно):**

**Структура:**
```
/.claude/agents/{agent-name}/
└── AGENT.md          # Конфигурация агента
```

**Шаблон AGENT.md:**

```markdown
---
name: {agent-name}
description: {Когда Claude должен использовать агента}
type: {explore | bash | plan | general-purpose}
model: {haiku | sonnet | opus | inherit}
tools: {Read, Grep, Glob, ...}
disallowedTools: {Write, Edit, ...}
permissionMode: {default | acceptEdits | plan}
max_turns: {5-50}
skills:
  - {skill-1}
  - {skill-2}
---

{Системный промпт — см. Шаг 5}
```

**Для агентов с Edit/Write — создать файл лога операций:**

```bash
# Создать файл лога операций
echo '{"agent": "{agent-name}", "description": "", "started_at": null, "finished_at": null, "context": "", "operations": []}' > .claude/state/agent-{agent-name}-operation.json
```

### Шаг 5: Написать промпт

> **SSOT:** [standard-agent.md#4-правила-написания-промптов](./standard-agent.md#4-правила-написания-промптов)

**Обязательные секции промпта:**

| Секция | Содержание |
|--------|------------|
| `## Роль` | Кто агент, специализация |
| `## Задача` | Что конкретно делать |
| `## Ограничения` | Чего НЕ делать (критично!) |
| `## Формат вывода` | Как оформить результат |

**Рекомендуемые секции:**

| Секция | Когда добавлять |
|--------|-----------------|
| `## Инструкции и SSOT` | Если есть релевантные SSOT |
| `## Скиллы` | Для `general-purpose` (кратко, без дублирования frontmatter) |
| `## Область работы` | Если ограничена папка/файлы |
| `## Критерии успеха` | Для сложных задач |
| `## Работа со state` | **Обязательно** для агентов с Edit/Write |

**Шаблон полного промпта:**

```markdown
## Роль
{Кто агент: специализация, область экспертизы}

## Задача
{Что конкретно должен сделать агент}

## Область работы
- Путь: {папка или паттерн}
- Типы файлов: {*.py, *.ts, etc}
- Исключения: {что игнорировать}

## Инструкции и SSOT
Релевантные инструкции:
- {путь к инструкции 1}
- {путь к инструкции 2}

## Скиллы
Используй скиллы из frontmatter вместо ручных операций.

## Работа со state (для агентов с Edit/Write)

> **SSOT:** [standard-state.md](../state/standard-state.md)

Перед редактированием файла:
1. Проверить `/.claude/state/locks.json`
2. Добавить блокировку
3. Выполнить операцию
4. Снять блокировку сразу после

Вести лог в `/.claude/state/agent-{name}-operation.json`.

## Ограничения
- НЕ {ограничение 1}
- НЕ {ограничение 2}
- ВСЕГДА {обязательное поведение}

## Формат вывода
{Точный формат результата}
```

### Шаг 6: Настроить параметры

**Выбор модели (`model`):**

| Модель | Когда использовать |
|--------|--------------------|
| `haiku` | Простой поиск, быстрые задачи |
| `sonnet` | Анализ, код-ревью, планирование |
| `opus` | Сложные многошаговые задачи |
| `inherit` | Использовать модель основной сессии |

**Выбор инструментов (`tools`):**

| Тип агента | Рекомендуемые tools |
|------------|---------------------|
| `explore` | `Read, Grep, Glob` |
| `plan` | `Read, Grep, Glob` |
| `bash` | `Bash, Read` |
| `general-purpose` | Все нужные инструменты |

**Выбор запрещённых инструментов (`disallowedTools`):**

| Тип агента | Рекомендуемые disallowedTools |
|------------|-------------------------------|
| `explore`, `plan` | `Write, Edit, Bash` |
| Код-ревью | `Write, Edit` |

**Выбор режима разрешений (`permissionMode`):**

| Режим | Когда использовать |
|-------|-------------------|
| `plan` | Агенты только для анализа |
| `default` | Агенты с записью — требуют подтверждения |
| `acceptEdits` | Доверенные агенты для автоправок |

**Выбор max_turns:**

| Тип задачи | Рекомендуемое значение |
|------------|------------------------|
| Простой поиск | 5-10 |
| Анализ папки | 10-20 |
| Код-ревью | 15-30 |
| Сложная задача | 30-50 |

**Для `general-purpose` — выбор скиллов (`skills`):**

> **ОБЯЗАТЕЛЬНО** использовать `AskUserQuestion` tool с multiSelect.

**Вопрос:** "Какие скиллы разрешить агенту?"

**Опции:** список релевантных скиллов из Шага 3.

**Настройка hooks (опционально):**

Для автоматической валидации или логирования (редко используется):

```yaml
hooks:
  PostToolUse:
    - matcher: "Edit|Write"
      hooks:
        - type: command
          command: "python .instructions/.scripts/validate-instruction.py"
```

### Шаг 7: Подтверждение промпта

> **ОБЯЗАТЕЛЬНО** использовать `AskUserQuestion` tool.

**Вопрос:** "Промпт агента корректен?"

**Опции:**
1. Да, всё корректно
2. Нужны правки

**При выборе "Нужны правки":**

1. Спросить: "Что нужно исправить?"
2. Внести изменения
3. Повторить Шаг 7

### Шаг 8: Валидация

```bash
python .claude/.instructions/agents/.scripts/validate-agent.py .claude/agents/{agent-name}.yaml
```

### Шаг 9: Обновить README

Добавить агента в `/.claude/agents/README.md`:

**Секция "Агенты":**
```markdown
| [{agent-name}](./{agent-name}.yaml) | {description} | {type} |
```

### Шаг 10: Отчёт о проделанной работе

```
## Отчёт о создании агента

**Создан агент:** `/.claude/agents/{name}.yaml`

**Тип:** {explore | bash | plan | general-purpose}

**Описание:** {description}

**Настройки:**
- Модель: {model}
- Max turns: {max_turns}

**Скиллы:** {список или "не применимо"}

**Валидация:** пройдена ✅
```

## Примеры

### Создание explore-агента

**Структура:**
```
/.claude/agents/todo-finder/
└── AGENT.md
```

**Файл AGENT.md:**

```markdown
---
name: todo-finder
description: Поиск TODO/FIXME комментариев. Используй для анализа технического долга.
type: explore
model: haiku
tools: Read, Grep, Glob
permissionMode: plan
max_turns: 10
---

## Роль
Агент для поиска технического долга в коде.

## Задача
Найти все комментарии с маркерами: TODO, FIXME, HACK, XXX, BUG.

## Область работы
- Путь: указывается в запросе (по умолчанию — весь src/)
- Типы файлов: Python, TypeScript, JavaScript, Markdown
- Исключения: node_modules/, .git/, __pycache__/

## Ограничения
- НЕ модифицировать файлы
- НЕ предлагать исправления
- Только сбор и группировка

## Формат вывода
Markdown таблица:
| Файл | Строка | Тип | Комментарий |
|------|--------|-----|-------------|
```

### Создание general-purpose агента

**Структура:**
```
/.claude/agents/code-reviewer/
└── AGENT.md
```

**Файл AGENT.md:**

```markdown
---
name: code-reviewer
description: Код-ревью с проверкой принципов. Используй после написания кода.
type: general-purpose
model: sonnet
tools: Read, Grep, Glob, Bash, AskUserQuestion
disallowedTools: Write, Edit
permissionMode: default
max_turns: 30
skills:
  - principles-validate
  - links-validate
---

## Роль
Код-ревьюер для проверки качества кода.

## Задача
Проверить указанные файлы на соответствие стандартам проекта.

## Инструкции и SSOT
SSOT: /.instructions/standard-principles.md

## Скиллы
Используй скиллы из frontmatter вместо ручных операций.

## Ограничения
- НЕ автоисправлять код (нет доступа к Edit/Write)
- Только анализ и отчёт
- Группировать замечания по серьёзности

## Формат вывода
Таблица markdown:
| Серьёзность | Файл | Строка | Проблема | Рекомендация |
|-------------|------|--------|----------|--------------|
```
