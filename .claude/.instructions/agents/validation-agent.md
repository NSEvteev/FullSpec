После любого изменения агента:

- Создание → [create-agent.md](./create-agent.md)

**Обязательно:**

**Опционально:**
- В CI/CD pipeline
- При обновлении стандарта агентов

## Шаги

### Шаг 0: Автоматическая валидация

```bash
python .claude/.instructions/agents/.scripts/validate-agent.py {path-to-agent}
```

Скрипт проверяет структуру, frontmatter, именование, тип, промпт, безопасность и версионирование. Если валидация пройдена — **готово**, шаги 1-8 не нужны.

**Все агенты:**
```bash
python .claude/.instructions/agents/.scripts/validate-agent.py --all
```

**Если скрипт недоступен** — выполнить шаги 1-8 вручную.

### Шаг 1: Проверить структуру файла

**Расположение:**
```
/.claude/agents/{agent-name}/
└── AGENT.md          # Конфигурация агента
```

**Проверки:**
- [ ] Папка в `/.claude/agents/`
- [ ] Файл `AGENT.md` существует
- [ ] YAML frontmatter валидный
- [ ] Имя папки = поле `name` во frontmatter

**Вручную:**
```bash
# Проверить YAML frontmatter
python -c "
import re
content = open('.claude/agents/{agent-name}/AGENT.md').read()
match = re.match(r'^---\n(.*?)\n---', content, re.DOTALL)
if match:
    import yaml
    yaml.safe_load(match.group(1))
    print('✅ YAML валиден')
else:
    print('❌ Frontmatter не найден')
"
```

### Шаг 2: Проверить обязательные поля

**SSOT:** [standard-agent.md § 3](./standard-agent.md#3-формат-конфигурации)

**Обязательные поля (frontmatter):**

| Поле | Тип | Проверка |
|------|-----|----------|
| `name` | string | Непустая строка, kebab-case, латиница |
| `description` | string | Непустая строка — когда использовать агента |
| `standard` | string | Путь к стандарту (`.claude/.instructions/agents/standard-agent.md`) |
| `standard-version` | string | Версия стандарта (`v1.1`) |
| `index` | string | Путь к индексу (`.claude/.instructions/agents/README.md`) |

**Рекомендуемые поля (frontmatter):**

| Поле | Тип | Проверка |
|------|-----|----------|
| `model` | enum | `haiku`, `sonnet`, `opus`, `inherit` |
| `tools` | string/list | Валидные инструменты через запятую или список |
| `permissionMode` | enum | `default`, `acceptEdits`, `dontAsk`, `bypassPermissions`, `plan` |

**Кастомные поля проекта:**

| Поле | Тип | Проверка |
|------|-----|----------|
| `type` | enum | `explore`, `bash`, `plan`, `general-purpose` |
| `max_turns` | int | Положительное число |
| `version` | string | Формат `vX.Y` (например: `v1.0`, `v2.1`) |

**Пример корректной конфигурации:**
```markdown
---
name: todo-finder
description: Поиск TODO/FIXME комментариев. Используй для анализа технического долга.
standard: .claude/.instructions/agents/standard-agent.md
standard-version: v1.3
index: .claude/.instructions/agents/README.md
type: explore
model: haiku
tools: Read, Grep, Glob
permissionMode: plan
max_turns: 10
version: v1.0
---

## Роль
Агент для поиска технического долга в коде.
...
```

### Шаг 3: Проверить именование

**SSOT:** [standard-agent.md § 7](./standard-agent.md#7-расположение-и-именование)

**Правила:**

| Правило | Пример ✅ | Пример ❌ |
|---------|----------|----------|
| Kebab-case | `todo-finder/` | `todoFinder/`, `todo_finder/` |
| Описательное | `unused-imports/` | `cleaner/` |
| Без префиксов | `test-runner/` | `agent-test-runner/` |
| Латиница | `code-review/` | `проверка-кода/` |

**Проверки:**
- [ ] Имя папки соответствует полю `name` во frontmatter
- [ ] Формат kebab-case
- [ ] Без префикса `agent-`
- [ ] Только латиница
- [ ] Файл называется `AGENT.md` (uppercase)

### Шаг 4: Проверить тип агента

**SSOT:** [standard-agent.md § 2](./standard-agent.md#2-типы-агентов)

**Допустимые типы:**

| Тип | Инструменты | Назначение |
|-----|-------------|------------|
| `explore` | Glob, Grep, Read | Исследование кодовой базы |
| `bash` | Bash | Выполнение команд |
| `plan` | Read, Glob, Grep | Планирование реализации |
| `general-purpose` | Все инструменты | Универсальные задачи |

**Проверки:**
- [ ] Тип из допустимого списка (если указан)
- [ ] Инструменты соответствуют типу (если указаны)
- [ ] Скиллы только для `general-purpose` (если указаны)

### Шаг 5: Проверить официальные поля

**SSOT:** [standard-agent.md § 3](./standard-agent.md#3-формат-конфигурации)

**Поле `model`:**

| Значение | Допустимо |
|----------|-----------|
| `haiku` | ✅ |
| `sonnet` | ✅ |
| `opus` | ✅ |
| `inherit` | ✅ |
| Другое | ❌ |

**Поле `tools`:**

Допустимые инструменты:
```
Read, Write, Edit, Bash, Glob, Grep, WebSearch, WebFetch, AskUserQuestion
```

- [ ] Все указанные инструменты из допустимого списка
- [ ] Формат: строка через запятую или YAML-список

**Поле `disallowedTools`:**

- [ ] Все указанные инструменты из допустимого списка
- [ ] Нет пересечения с `tools`

**Поле `permissionMode`:**

| Значение | Допустимо | Примечание |
|----------|-----------|------------|
| `default` | ✅ | Стандартный режим |
| `acceptEdits` | ✅ | Автопринятие правок (не затрагивает критичные файлы: .env, package.json, конфиги) |
| `dontAsk` | ✅ | Автоотклонение |
| `bypassPermissions` | ⚠️ | Опасно для кастомных агентов |
| `plan` | ✅ | Только чтение |
| Другое | ❌ | |

**Поле `skills`:**

- [ ] Формат: YAML-список строк
- [ ] Скиллы существуют в проекте

**Поле `hooks`:**

- [ ] Формат: YAML-объект
- [ ] Допустимые события: `PreToolUse`, `PostToolUse`, `Stop`
- [ ] Скрипты в hooks существуют (если указаны пути)

**Поле `version` (кастомное):**

- [ ] Формат: строка `vX.Y` (например: `v1.0`, `v2.1`)
- [ ] Соответствует последней записи в CHANGELOG.md

### Шаг 6: Проверить промпт

**SSOT:** [standard-agent.md § 4](./standard-agent.md#4-правила-написания-промптов)

**Рекомендуемая структура промпта:**

```markdown
## Роль
{Кто агент и какова его специализация}

## Задача
{Что агент должен делать}

## Ограничения
{Чего агент НЕ должен делать}

## Формат вывода
{Как агент должен оформлять результат}
```

**Обязательные проверки:**
- [ ] Промпт не пустой
- [ ] Есть секция "Роль" или "Задача"
- [ ] Есть секция "Ограничения" (явные запреты)
- [ ] Есть секция "Формат вывода"

**Антипаттерны (WARNING):**

| Антипаттерн | Пример ❌ | Исправление ✅ |
|-------------|----------|----------------|
| Слишком общий | "Помоги с кодом" | "Найди неиспользуемые импорты в src/" |
| Без ограничений | "Исправь баги" | "Найди баги, НЕ исправляй автоматически" |
| Без формата вывода | "Проанализируй" | "Выведи список в формате markdown" |

**Дополнительные проверки для review-агентов** (discussion-reviewer, captain-holt и др.):

| Проверка | Ожидание |
|----------|----------|
| Антигаллюцинации | Промпт содержит секцию с инструкциями «не выдумывай» или аналогичные |
| Контекстная изоляция | Промпт указывает, какой контекст агент НЕ должен запрашивать |
| Градация серьёзности | Выход агента имеет приоритеты (P1/P2/P3 или аналог) |

### Шаг 7: Проверить безопасность

**SSOT:** [standard-agent.md § 10](./standard-agent.md#10-безопасность)

**Уровни доступа:**

| Уровень | Описание | Допустим для агентов |
|---------|----------|---------------------|
| `read-only` | Только чтение | ✅ Да |
| `safe-write` | Безопасная запись | ✅ Да |
| `full` | Полный доступ | ❌ Только основной LLM |

**Проверки:**
- [ ] `access_level` не `full` (или отсутствует)
- [ ] Нет разрешения на `delete`, `rm`
- [ ] Нет разрешения на `git push`, `git reset`
- [ ] В промпте есть секция "Ограничения безопасности" (рекомендуется)

**Запрещённые паттерны в settings:**
```yaml
# ❌ НЕДОПУСТИМО для агентов
settings:
  access_level: full
  allowed_operations:
    - delete
    - rm -rf
```

### Шаг 8: Проверить версионирование

**SSOT:** [standard-agent.md § 11](./standard-agent.md#11-версионирование-агентов)

**Проверки:**

- [ ] Поле `version` указано в frontmatter
- [ ] Формат версии: `vX.Y` (например: `v1.0`, `v2.1`)
- [ ] Файл `CHANGELOG.md` существует в папке агента
- [ ] Версия в CHANGELOG соответствует версии в frontmatter
- [ ] CHANGELOG содержит описание изменений для текущей версии

**Пример корректной структуры:**

```
/.claude/agents/todo-finder/
├── AGENT.md          # version: v1.1
└── CHANGELOG.md      # ## v1.1 (2026-02-01)
```

- [ ] `index` — путь к индексу агентов

### Именование
- [ ] Имя папки = поле `name`
- [ ] Kebab-case
- [ ] Без префикса `agent-`
- [ ] Латиница

### Официальные поля
- [ ] `model` — допустимое значение (если указан)
- [ ] `tools` — валидные инструменты (если указан)
- [ ] `disallowedTools` — валидные инструменты (если указан)
- [ ] `permissionMode` — допустимое значение (если указан)
- [ ] `skills` — существующие скиллы (если указан)
- [ ] `hooks` — корректный формат (если указан)

### Кастомные поля
- [ ] `type` — допустимое значение (если указан)
- [ ] `max_turns` — положительное число (если указан)
- [ ] `version` — формат `vX.Y` (если указан)

### Версионирование
- [ ] Поле `version` указано в frontmatter
- [ ] Файл `CHANGELOG.md` существует в папке агента
- [ ] Версия в CHANGELOG соответствует версии в frontmatter

### Промпт (тело AGENT.md)
- [ ] Есть "Роль" или "Задача"
- [ ] Есть "Ограничения"
- [ ] Есть "Формат вывода"
- [ ] Нет антипаттернов
- [ ] Для review-агентов: есть секция "Антигаллюцинации"
- [ ] Для review-агентов: указана контекстная изоляция
- [ ] Для review-агентов: выход имеет градацию серьёзности (P1/P2/P3)

- [ ] `permissionMode` не `bypassPermissions` (для кастомных агентов)

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Нет AGENT.md | `MISSING_AGENT_MD` | Файл AGENT.md отсутствует | Создать файл AGENT.md в папке агента |
| Невалидный frontmatter | `YAML_SYNTAX` | Неправильный YAML во frontmatter | Проверить синтаксис между `---` |
| Отсутствует name | `MISSING_NAME` | Поле name не указано | Добавить поле `name: agent-name` |
| Имя не совпадает | `NAME_MISMATCH` | Имя папки ≠ поле name | Синхронизировать имя папки и поле |
| Неверный тип | `INVALID_TYPE` | Тип не из допустимого списка | Использовать: explore, bash, plan, general-purpose |
| Неверная модель | `INVALID_MODEL` | Модель не из допустимого списка | Использовать: haiku, sonnet, opus, inherit |
| Неверный tools | `INVALID_TOOLS` | Инструмент не существует | Проверить список допустимых инструментов |
| Неверный permissionMode | `INVALID_PERMISSION` | Режим не из списка | Использовать: default, acceptEdits, dontAsk, plan |
| Пустой промпт | `EMPTY_PROMPT` | Тело AGENT.md пустое | Написать промпт после frontmatter |
| Нет ограничений | `NO_CONSTRAINTS` | В промпте нет секции "Ограничения" | Добавить явные запреты |
| Нет формата вывода | `NO_OUTPUT_FORMAT` | В промпте нет "Формат вывода" | Указать ожидаемый формат результата |
| Неверное именование | `INVALID_NAME` | camelCase или кириллица | Использовать kebab-case, латиницу |
| Опасный режим | `UNSAFE_PERMISSION` | bypassPermissions для кастомного агента | Использовать default или plan |
| Скилл не существует | `MISSING_SKILL` | Указанный скилл не найден | Проверить имя скилла |
| Нет версии | `MISSING_VERSION` | Поле version не указано | Добавить `version: v1.0` |
| Неверный формат версии | `INVALID_VERSION` | Формат не `vX.Y` | Исправить на формат `v1.0` |
| Нет CHANGELOG | `MISSING_CHANGELOG` | Файл CHANGELOG.md отсутствует | Создать CHANGELOG.md в папке агента |
| Нет standard-version | `MISSING_STANDARD_VERSION` | Поле standard-version не указано | Добавить `standard-version: v1.1` |
| Нет правила удаления | `MISSING_DELETE_RULE` | Агент с Edit/Write без правила `_old_` | Добавить секцию "Удаление файлов" в промпт |
