## Шаги

### Шаг 1: Проверить структуру файла

**Расположение:**
```
/.claude/agents/{agent-name}.yaml
```

**Проверки:**
- [ ] Файл в папке `/.claude/agents/`
- [ ] Расширение `.yaml`
- [ ] Валидный YAML-синтаксис

**Вручную:**
```bash
# Проверить YAML-синтаксис
python -c "import yaml; yaml.safe_load(open('.claude/agents/{agent-name}.yaml'))"
```

### Шаг 2: Проверить обязательные поля

**SSOT:** [standard-agent.md § 3](./standard-agent.md#3-формат-конфигурации)

**Обязательные поля:**

| Поле | Тип | Проверка |
|------|-----|----------|
| `name` | string | Непустая строка, kebab-case |
| `description` | string | Непустая строка, 1 предложение |
| `type` | enum | Одно из: `explore`, `bash`, `plan`, `general-purpose` |
| `prompt` | string | Непустая строка, многострочная |

**Пример корректной конфигурации:**
```yaml
name: todo-finder
description: Поиск TODO/FIXME комментариев в кодовой базе
type: explore

prompt: |
  ## Роль
  Агент для поиска технического долга в коде.
  ...
```

### Шаг 3: Проверить именование

**SSOT:** [standard-agent.md § 7](./standard-agent.md#7-расположение-и-именование)

**Правила:**

| Правило | Пример ✅ | Пример ❌ |
|---------|----------|----------|
| Kebab-case | `todo-finder` | `todoFinder`, `todo_finder` |
| Описательное | `unused-imports` | `cleaner` |
| Без префиксов | `test-runner` | `agent-test-runner` |
| Латиница | `code-review` | `проверка-кода` |

**Проверки:**
- [ ] Имя файла соответствует полю `name`
- [ ] Формат kebab-case
- [ ] Без префикса `agent-`
- [ ] Только латиница

### Шаг 4: Проверить тип агента

**SSOT:** [standard-agent.md § 2](./standard-agent.md#2-типы-агентов)

**Допустимые типы:**

| Тип | Инструменты | Назначение |
|-----|-------------|------------|
| `explore` | Glob, Grep, Read | Исследование кодовой базы |
| `bash` | Bash | Выполнение команд |
| `plan` | Read, Glob, Grep | Планирование реализации |
| `general-purpose` | Все инструменты | Универсальные задачи |

**Проверки:**
- [ ] Тип из допустимого списка
- [ ] Инструменты соответствуют типу (если указаны)
- [ ] Скиллы только для `general-purpose` (если указаны)

### Шаг 5: Проверить промпт

**SSOT:** [standard-agent.md § 4](./standard-agent.md#4-правила-написания-промптов)

**Рекомендуемая структура промпта:**

```markdown
## Роль
{Кто агент и какова его специализация}

## Задача
{Что агент должен делать}

## Ограничения
{Чего агент НЕ должен делать}

## Формат вывода
{Как агент должен оформлять результат}
```

**Обязательные проверки:**
- [ ] Промпт не пустой
- [ ] Есть секция "Роль" или "Задача"
- [ ] Есть секция "Ограничения" (явные запреты)
- [ ] Есть секция "Формат вывода"

**Антипаттерны (WARNING):**

| Антипаттерн | Пример ❌ | Исправление ✅ |
|-------------|----------|----------------|
| Слишком общий | "Помоги с кодом" | "Найди неиспользуемые импорты в src/" |
| Без ограничений | "Исправь баги" | "Найди баги, НЕ исправляй автоматически" |
| Без формата вывода | "Проанализируй" | "Выведи список в формате markdown" |

### Шаг 6: Проверить безопасность

**SSOT:** [standard-agent.md § 10](./standard-agent.md#10-безопасность)

**Уровни доступа:**

| Уровень | Описание | Допустим для агентов |
|---------|----------|---------------------|
| `read-only` | Только чтение | ✅ Да |
| `safe-write` | Безопасная запись | ✅ Да |
| `full` | Полный доступ | ❌ Только основной LLM |

**Проверки:**
- [ ] `access_level` не `full` (или отсутствует)
- [ ] Нет разрешения на `delete`, `rm`
- [ ] Нет разрешения на `git push`, `git reset`
- [ ] В промпте есть секция "Ограничения безопасности" (рекомендуется)

**Запрещённые паттерны в settings:**
```yaml
# ❌ НЕДОПУСТИМО для агентов
settings:
  access_level: full
  allowed_operations:
    - delete
    - rm -rf
```

## Чек-лист

### Структура
- [ ] Файл в `/.claude/agents/`
- [ ] Расширение `.yaml`
- [ ] Валидный YAML

### Обязательные поля
- [ ] `name` — непустая строка
- [ ] `description` — одно предложение
- [ ] `type` — допустимое значение
- [ ] `prompt` — непустая строка

### Именование
- [ ] Имя файла = поле `name`
- [ ] Kebab-case
- [ ] Без префикса `agent-`
- [ ] Латиница

### Промпт
- [ ] Есть "Роль" или "Задача"
- [ ] Есть "Ограничения"
- [ ] Есть "Формат вывода"
- [ ] Нет антипаттернов

### Безопасность
- [ ] Уровень доступа не `full`
- [ ] Нет разрешений на удаление
- [ ] Нет разрешений на git push/reset

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Невалидный YAML | `YAML_SYNTAX` | Неправильный отступ или спецсимволы | Проверить синтаксис YAML |
| Отсутствует name | `MISSING_NAME` | Поле name не указано | Добавить поле `name: agent-name` |
| Неверный тип | `INVALID_TYPE` | Тип не из допустимого списка | Использовать: explore, bash, plan, general-purpose |
| Пустой промпт | `EMPTY_PROMPT` | Промпт пустой или отсутствует | Написать промпт согласно стандарту |
| Нет ограничений | `NO_CONSTRAINTS` | В промпте нет секции "Ограничения" | Добавить явные запреты |
| Нет формата вывода | `NO_OUTPUT_FORMAT` | В промпте нет "Формат вывода" | Указать ожидаемый формат результата |
| Неверное именование | `INVALID_NAME` | camelCase или кириллица | Использовать kebab-case, латиницу |
| Full-доступ | `UNSAFE_ACCESS` | access_level: full | Использовать read-only или safe-write |
| Разрешено удаление | `UNSAFE_DELETE` | delete в allowed_operations | Убрать delete, использовать _old_ префикс |
