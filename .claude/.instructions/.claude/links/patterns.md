## Поиск ссылок

### Markdown-ссылка

**Паттерн:**
```regex
\[([^\]]+)\]\(([^)]+)\)
```

**Группы захвата:**
- `$1` — текст ссылки
- `$2` — путь

**Пример:**
```
[skills.md](/.claude/skills/README.md)
$1 = "skills.md"
$2 = "/.claude/skills/README.md"
```

### Помеченная ссылка

**Паттерн:**
```regex
(\S+)\s*\*\(ссылка удалена:\s*([^)]+)\)\*
```

**Группы захвата:**
- `$1` — текст (бывшее имя)
- `$2` — путь (бывший)

## Поиск упоминаний

Паттерны для поиска упоминаний файлов/папок, которые ещё не оформлены как ссылки.

| Тип | Паттерн | Примеры |
|-----|---------|---------|
| Абсолютный путь к файлу | `/[\w\-./]+\.\w+` | `/config/settings.json` |
| Относительный путь | `\.?/[\w\-./]+\.\w+` | `./src/index.ts` |
| Имя файла | `[\w\-]+\.\w+` | `package.json`, `README.md` |
| Путь к папке | `/[\w\-./]+/` | `/.claude/scripts/` |

### Алгоритм обнаружения

```
1. Разбить документ на сегменты: текст / блок кода
2. В текстовых сегментах применить паттерны
3. Для каждого совпадения проверить:
   - Не внутри ссылки?
   - Не внутри команды?
   - Файл существует?
4. Собрать список: {текст, строка, тип: файл|папка}
```

## Исключения

**Не обрабатывать ссылки внутри:**

| Контекст | Почему |
|----------|--------|
| Блоки кода (``` ... ```) | Примеры, не реальные ссылки |
| Уже оформленные ссылки `[](...)` | Дублирование |
| Bash-команды (mkdir, rm, cd) | Команды, не документация |
| URL (http://, https://) | Внешние ссылки |
| Примеры вывода команд | Не редактируемый контент |

### Bash-команды для исключения

```
mkdir, rm, cd, ls, cp, mv, touch, cat, grep
```

## Примеры

### Поиск всех ссылок в файле

```bash
grep -oP '\[([^\]]+)\]\(([^)]+)\)' файл.md
```

### Поиск помеченных ссылок

```bash
grep -r "ссылка удалена:" --include="*.md" .
```

### Поиск ссылок на конкретный файл

```bash
grep -r "design.md" --include="*.md" .claude/.instructions/
```

### Замена пути в ссылках

```regex
# Найти
\]\(/old/path/([^)]+)\)
# Заменить
](/new/path/$1)
```
