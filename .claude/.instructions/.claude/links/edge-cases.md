## Циклические ссылки

**Ситуация:** A ссылается на B, B ссылается на A.

**Поведение:** Каждый файл обрабатывается один раз. При повторном обнаружении — пропуск.

**Пример:**
```
skill-create/SKILL.md ← → skill-update/SKILL.md
```

**Результат:**
```
Обновлено: skill-create/SKILL.md
Обновлено: skill-update/SKILL.md
Пропущено (уже обработано): skill-create/SKILL.md
```

## Конфликты путей

**Ситуация:** `--old-name` совпадает с частью другого пути.

**Пример:**
```
/links-update /src/auth.ts --old-name /src/auth
```

**Риск:** Может затронуть `/src/auth/`, `/src/auth-utils.ts`.

**Поведение:** Показать все затронутые пути для подтверждения:
```
⚠️ Найдены пути с частичным совпадением:

- /src/auth.ts (точное совпадение)
- /src/auth/ (частичное)
- /src/auth-utils.ts (частичное)

Обновить только точные совпадения? [Y/n/все]
```

## Блоки кода

**Ситуация:** Ссылка находится внутри ``` блока.

**Поведение по умолчанию:** НЕ обновлять ссылки внутри блоков кода.

**Причина:** Это примеры, не реальные ссылки.

**Переопределение:**
```
/links-update /new/path.md --old-name /old/path.md --include-code-blocks
```

## Массовые изменения

**Ситуация:** Переименование затрагивает 50+ файлов.

**Поведение:** Предупреждение + рекомендация:
```
⚠️ Обнаружено 57 файлов для обновления

Рекомендация: используйте скрипт для массовых изменений:
python .claude/scripts/update_links.py "old/path" "new/path" --dry-run

Продолжить через скилл? [Y/n/скрипт]
```

**Преимущества скрипта:**
- ~1 секунда на любое количество замен
- Автоматический поиск всех .md файлов
- Пропуск блоков кода
- Предпросмотр diff

## Параллельное выполнение

**Ситуация:** Несколько скиллов вызывают links-update одновременно.

**Риск:** Конфликты при записи в один файл.

**Рекомендация:** Не вызывать links-update параллельно для пересекающихся путей.

**Поведение при конфликте:**
```
⚠️ Файл skills.md заблокирован другим процессом

Повторить через 5 секунд? [Y/n]
```

## Оба файла существуют

**Ситуация:** Файл скопирован, а не перемещён (old и new оба существуют).

**Поведение:** Предупреждение:
```
⚠️ Оба файла существуют:
- /old/path.md (старый)
- /new/path.md (новый)

Варианты:
[1] Обновить ссылки на новый путь (рекомендуется)
[2] Оставить ссылки на старый путь
[3] Отменить
```

## Помеченные без --old-name

**Ситуация:** В проекте есть пометки `*(ссылка удалена: path)*`, но `--old-name` не указан.

**Поведение:** Информирование:
```
ℹ️ Найдены помеченные ссылки (3 шт.)

Эти ссылки могут быть восстановлены, если указать --old-name:
- /doc/api.md:45 — *(ссылка удалена: /src/old.ts)*
- /doc/guide.md:12 — *(ссылка удалена: /src/old.ts)*

Показать подробности? [Y/n]
```
