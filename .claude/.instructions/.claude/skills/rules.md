## Правило одного действия

**Скилл выполняет 1 действие над объектом.**

| Действие | Описание | Пример |
|----------|----------|--------|
| `create` | Создание нового объекта | `skill-create`, `doc-create` |
| `update` | Обновление существующего | `skill-update`, `doc-update` |
| `delete` | Удаление объекта | `skill-delete`, `doc-delete` |

**Исключения:**
- `execute` — выполнение (взятие в работу): `issue-execute`
- `complete` — завершение: `issue-complete`, `test-complete`
- `review` — проверка: `issue-review`, `test-review`
- `migrate` — миграция (переименование + обновление ссылок): `skill-migrate`

**Правило:** Если пользователь просит "управление X" (create + update + delete) — разделить на 3 скилла.

**Важно:** Скилл может вызывать другие скиллы в рамках своего воркфлоу — это НЕ нарушает правило одного действия. Пример: `/instruction-create` вызывает `/links-update` после создания инструкции.

## Формат названия

```
{объект}-{действие}
```

**Требования:**
- Только латиница, цифры, дефис
- Объект в единственном числе
- Действие из списка выше

**Примеры:**
- ✅ `skill-create`, `agent-update`, `doc-delete`
- ✅ `issue-execute`, `test-review`
- ❌ `skills-create` (множественное число)
- ❌ `create-skill` (неверный порядок)
- ❌ `skillCreate` (camelCase)

## Расположение файлов

```
/.claude/skills/{название}/
    SKILL.md          # Обязательно
    tests.md          # Опционально (если много тестов)
```

**Правило:** Один скилл = одна папка.

## Справочник allowed-tools

Инструменты, доступные скиллам в frontmatter `allowed-tools`:

| Инструмент | Описание | Когда использовать |
|------------|----------|-------------------|
| `Bash` | Выполнение команд | git, gh, npm, make |
| `Read` | Чтение файлов | Анализ содержимого |
| `Write` | Создание файлов | Новые файлы |
| `Edit` | Редактирование файлов | Изменение существующих |
| `Glob` | Поиск файлов по паттерну | `**/*.md`, `src/**/*.ts` |
| `Grep` | Поиск в содержимом | Поиск по тексту |
| `WebFetch` | HTTP запросы | API, загрузка |
| `WebSearch` | Поиск в интернете | Документация, примеры |
| `Task` | Запуск подзадач | Агенты, параллелизм |

**Правило:** Указывать только необходимые инструменты.

## FAQ

### Как добавить новую категорию скиллов?

**Когда нужна новая категория:**
- 2+ скиллов с общей темой
- Не подходят в существующие категории

**Шаги:**
1. Добавить секцию в [/.claude/skills/README.md](/.claude/skills/README.md)
2. Обновить `/skill-create` — добавить категорию в список
3. Создать скиллы категории

**Формат категории в индексе:**

```markdown
### {category-name}

{Описание — 1 строка.}

| Скилл | Описание | Триггеры |
|-------|----------|----------|
| [skill-name](./skill-name/SKILL.md) | Описание | `/skill-name`, "фраза" |
```

### Что делать при конфликте между скиллами?

**Причины конфликтов:**

| Причина | Решение |
|---------|---------|
| Перекрытие зон ответственности | Вызывать последовательно |
| Конкурентные изменения одного файла | Не запускать параллельно |
| Циклическая зависимость | Переструктурировать воркфлоу |

**Предотвращение:**
- Вызывать связанные скиллы последовательно
- Использовать `--dry-run` для проверки
- Не запускать один скилл несколько раз параллельно
