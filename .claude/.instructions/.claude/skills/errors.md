## Принцип

Обработка ошибок должна быть:
- **Предсказуемой** — пользователь знает, чего ожидать
- **Информативной** — понятно, что пошло не так
- **Восстановимой** — есть способ исправить ситуацию

---

## Паттерн обработки

```
1. Обнаружить ошибку
2. Сообщить пользователю (формат ниже)
3. Откатить изменения (если были)
4. Предложить решение
```

---

## Формат сообщения об ошибке

### Базовый формат

```
❌ Ошибка: {краткое описание}

Шаг: {номер и название шага}
Причина: {детальное описание}

Откат:
- {что откачено}

Решение:
1. {вариант 1}
2. {вариант 2}
```

### Пример

```
❌ Ошибка: Файл уже существует

Шаг: 4. Создать файлы
Причина: Файл /.claude/skills/my-skill/SKILL.md уже существует

Откат:
- Изменения не вносились

Решение:
1. Перезаписать существующий файл
2. Открыть для редактирования (/skill-update)
3. Отменить операцию
```

## Типовые ошибки

### Категория: Файлы

| Ошибка | Код | Сообщение | Решение |
|--------|-----|-----------|---------|
| Файл существует | `FILE_EXISTS` | "Файл {путь} уже существует" | Перезаписать / Редактировать / Отменить |
| Файл не найден | `FILE_NOT_FOUND` | "Файл {путь} не найден" | Проверить путь / Создать |
| Папка не найдена | `DIR_NOT_FOUND` | "Папка {путь} не найдена" | Создать папку |
| Нет прав | `PERMISSION_DENIED` | "Нет прав на запись в {путь}" | Проверить права |

### Категория: Git

| Ошибка | Код | Сообщение | Решение |
|--------|-----|-----------|---------|
| Незакоммиченные изменения | `GIT_DIRTY` | "Есть незакоммиченные изменения" | Закоммитить / Stash / Отменить |
| Ветка существует | `BRANCH_EXISTS` | "Ветка {имя} уже существует" | Использовать / Удалить / Переименовать |
| Конфликт merge | `MERGE_CONFLICT` | "Конфликт при слиянии" | Разрешить вручную |
| gh не авторизован | `GH_NOT_AUTH` | "GitHub CLI не авторизован" | `gh auth login` |

### Категория: Валидация

| Ошибка | Код | Сообщение | Решение |
|--------|-----|-----------|---------|
| Неверный формат | `INVALID_FORMAT` | "{поле} имеет неверный формат" | Исправить формат |
| Обязательное поле | `REQUIRED_FIELD` | "Поле {имя} обязательно" | Указать значение |
| Неверный путь | `INVALID_PATH` | "Путь {путь} не соответствует паттерну" | Проверить путь |

### Категория: Зависимости

| Ошибка | Код | Сообщение | Решение |
|--------|-----|-----------|---------|
| Скилл не найден | `SKILL_NOT_FOUND` | "Скилл {имя} не существует" | Создать / Проверить имя |
| Циклическая зависимость | `CIRCULAR_DEP` | "Обнаружена циклическая зависимость" | Переструктурировать |
| Внешняя зависимость | `EXTERNAL_DEP` | "Требуется {зависимость}" | Установить зависимость |

## Откат изменений

### Принцип

**Правило:** При ошибке после внесения изменений — откатить ВСЕ изменения этого вызова скилла.

### Типовые команды отката

```bash
# Откат файла
git checkout -- {путь}

# Удаление созданного файла
rm -f {путь}

# Удаление созданной папки
rm -rf {путь}/

# Откат всех изменений
git checkout -- .

# Откат коммита (если был)
git revert HEAD --no-commit
```

### Порядок отката

1. Откатить изменения в обратном порядке (последнее → первое)
2. Сообщить пользователю о каждом откате
3. Подтвердить успешный откат

```
Откат выполнен:
- Удалён: /.claude/skills/my-skill/SKILL.md
- Восстановлен: /.claude/skills/README.md
- Восстановлен: /.claude/agents/README.md

Можно попробовать снова.
```

## Предложение решений

### Формат

```
Решение:
1. {Вариант 1} — {когда использовать}
2. {Вариант 2} — {когда использовать}
3. {Вариант 3} — {когда использовать}

Что выбрать? [1/2/3]
```

### Типовые решения

| Ситуация | Варианты |
|----------|----------|
| Файл существует | Перезаписать / Редактировать / Отменить |
| Объект не найден | Создать / Проверить имя / Отменить |
| Нет прав | Запросить права / Другой путь / Отменить |
| Конфликт | Разрешить / Отменить / Пропустить |

## Частичный успех

При частичном успехе (часть операций выполнена):

```
⚠️ Частичный успех

Выполнено:
- ✅ Создан файл SKILL.md
- ✅ Обновлён skills.md

Не выполнено:
- ❌ Обновление agents.md — ошибка доступа

Варианты:
1. Откатить всё и повторить
2. Оставить выполненное, исправить ошибку вручную
3. Продолжить без agents.md

Что выбрать?
```
