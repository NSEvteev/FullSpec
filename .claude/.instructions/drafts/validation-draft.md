## Когда валидировать

После создания или изменения черновика в `/.claude/drafts/`:

- Создание нового черновика
- Изменение структуры черновика
- Перед добавлением в git

**Важно:** Валидация — опциональная. Черновики создаются спонтанно и не требуют формального процесса.

## Шаги

### Шаг 1: Проверка именования

**Автоматически:**
```bash
python .claude/.instructions/drafts/.scripts/validate-draft.py <путь к черновику>
```

**Вручную (если нужно):**
- Проверить формат: `YYYY-MM-DD-<topic>.md`
- Проверить kebab-case в `<topic>`
- Проверить латиницу (не кириллица)

### Шаг 2: Проверка структуры

**Автоматически:**
```bash
python .claude/.instructions/drafts/.scripts/validate-draft.py <путь к черновику>
```

**Вручную (если нужно):**
- Наличие заголовка `# {Тема}`
- Наличие оглавления
- Наличие секции "Контекст"
- Наличие секции "Содержание"

### Шаг 3: Проверка расположения

**Автоматически:**
```bash
python .claude/.instructions/drafts/.scripts/validate-draft.py <путь к черновику>
```

**Вручную (если нужно):**
- Файл находится в `/.claude/drafts/`
- Не в подпапках (плоская структура)

### Шаг 4: Проверка всех черновиков

**Автоматически:**
```bash
python .claude/.instructions/drafts/.scripts/validate-draft.py --all
```

## Чек-лист

- [ ] Именование соответствует формату `YYYY-MM-DD-<topic>.md`
- [ ] Файл находится в `/.claude/drafts/`
- [ ] Есть заголовок `# {Тема}`
- [ ] Есть оглавление
- [ ] Есть секция "Контекст"
- [ ] Есть секция "Содержание"
- [ ] README.md обновлён (если черновик добавлен/удалён)

## Типичные ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Неверный формат даты | Использован формат `DD-MM-YYYY` | Использовать `YYYY-MM-DD` |
| Кириллица в имени | Имя файла на русском | Транслитерировать в латиницу |
| Snake_case в topic | Использован `_` | Заменить на `-` (kebab-case) |
| Отсутствует контекст | Не указано зачем создан черновик | Добавить секцию "Контекст" |
| Черновик в подпапке | Создана подпапка в drafts/ | Переместить в корень drafts/ |
