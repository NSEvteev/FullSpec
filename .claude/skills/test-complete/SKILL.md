# Отметка теста

Команда для отметки теста как пройденного, провального или пропущенного. Ведёт историю запусков и статистику.

**Связанные скиллы:**

- [test-execute](/.claude/skills/test-execute/SKILL.md) — выполнение тестов
- [test-review](/.claude/skills/test-review/SKILL.md) — проверка полноты теста
- [test-delete](/.claude/skills/test-delete/SKILL.md) — удаление теста

## Формат вызова

```
/test-complete [путь-к-тесту] [--status passed|failed|skipped] [--note "комментарий"]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь-к-тесту` | Путь к тесту или скиллу | Последний выполненный |
| `--status` | Статус результата | `passed` |
| `--note` | Комментарий к результату | — |
| `--batch` | Отметить несколько тестов | — |
| `--revert` | Откатить последнюю отметку | — |

---

## Статусы тестов

| Статус | Символ | Описание | Когда использовать |
|--------|--------|----------|-------------------|
| `passed` | ✅ | Тест пройден | Все проверки успешны |
| `failed` | ❌ | Тест провален | Хотя бы одна проверка не прошла |
| `skipped` | ⏭️ | Тест пропущен | Тест не применим или заблокирован |
| `not_run` | ⬜ | Не выполнен | Начальное состояние |
| `flaky` | ⚠️ | Нестабильный | Проходит не всегда |

## Правила

### Правило истории

**Правило:** Каждое изменение статуса записывается в историю с датой и комментарием.

**Формат истории:**
```
### История запусков

| Дата | Статус | Комментарий |
|------|--------|-------------|
| 2026-01-20 | ✅ Passed | — |
| 2026-01-19 | ❌ Failed | Ошибка в шаге 3 |
| 2026-01-18 | ✅ Passed | — |
```

### Правило flaky-тестов

**Правило:** Если тест переключается между passed/failed более 3 раз за 7 дней — пометить как `flaky`.

**Действия для flaky:**
1. Добавить метку ⚠️
2. Уведомить о нестабильности
3. Предложить исправить

### Правило автоматической отметки

**Правило:** После `/test-execute` автоматически предлагается `/test-complete`.

**Исключение:** Можно отключить через `--no-complete` в test-execute.

### Правило batch-отметки

**Правило:** При `--batch` можно отметить несколько тестов одним статусом.

```
/test-complete --batch .claude/skills/issue-*/SKILL.md --status passed
```

## Воркфлоу

### Шаг 1: Определить тест

1. Из аргумента: `/test-complete .claude/skills/issue-create/SKILL.md`
2. Из контекста: последний выполненный тест
3. Или спросить: "Какой тест отметить?"

**Если из контекста:**
```
📋 Последний выполненный тест

Тест: issue-create
Время: 2026-01-20 15:30
Результат: ✅ 3/3 passed

Отметить этот тест? [Y/n]
```

**Batch режим:**
```
📋 Batch отметка

Паттерн: .claude/skills/issue-*/SKILL.md

Найдено тестов: 5
- issue-create: ⬜ не отмечен
- issue-update: ⬜ не отмечен
- issue-execute: ⬜ не отмечен
- issue-review: ✅ passed (2026-01-19)
- issue-complete: ⬜ не отмечен

Отметить все как {status}? [Y/n/выборочно]
```

### Шаг 2: Проверить текущий статус

1. Прочитать файл теста
2. Найти текущий статус
3. Проверить историю

**Scope claude:**
```
📋 Текущий статус: issue-create

Статус: ⬜ Не выполнен
Последний запуск: —
История: 0 записей

Новый статус: passed
Продолжить? [Y/n]
```

**Если уже отмечен:**
```
📋 Текущий статус: issue-create

Статус: ✅ Passed
Дата: 2026-01-19
Комментарий: —

Перезаписать статус? [Y/n]
```

### Шаг 3: Обновить статус

**Scope claude (SKILL.md):**

Найти строку:
```
Результат: ⬜ Не выполнен
```

Заменить на:
```
Результат: ✅ Passed (2026-01-20)
```

**Scope project:**

Обновить файл результатов тестов или CI badge.

### Шаг 4: Обновить историю

**Добавить запись в историю:**

```markdown
### История запусков

| Дата | Статус | Комментарий |
|------|--------|-------------|
| 2026-01-20 | ✅ Passed | — |
```

**Если история не существует — создать раздел.**

**Проверка на flaky:**
```
⚠️ Обнаружен нестабильный тест

Тест: issue-review
История за 7 дней:
- 2026-01-20: ✅ Passed
- 2026-01-19: ❌ Failed
- 2026-01-18: ✅ Passed
- 2026-01-17: ❌ Failed

Переключений: 4
Порог: 3

Пометить как flaky? [Y/n]
```

### Шаг 5: Уведомления

**При failed:**
```
❌ Тест провален: issue-create

Рекомендации:
1. Проверить код: /test-review {путь}
2. Исправить: /test-update {путь} --reason fix
3. Создать Issue: /issue-create --title "Fix test: issue-create"

Создать Issue для исправления? [Y/n]
```

**При failed на критичном скилле:**

> **Критичные скиллы:** `skill-*`, `instruction-*`, `issue-*` (см. [CLAUDE.md](/CLAUDE.md#критичные-скиллы))

```
🚨 Тест провален: skill-create (КРИТИЧНЫЙ)

Скилл skill-create относится к критичным (skill-management).
Failed тест критичного скилла требует немедленного внимания.

Рекомендации:
1. Проверить: /test-review .claude/skills/skill-create/SKILL.md
2. Исправить: /test-update .claude/skills/skill-create/SKILL.md --reason fix

⚠️ Рекомендуется создать Issue с высоким приоритетом.

Создать Issue? [Y/n]
> Y

/issue-create --title "[CRITICAL] Fix test: skill-create" --label "priority:high"
```

**При первом passed после failed:**
```
🎉 Тест восстановлен: issue-create

Предыдущий статус: ❌ Failed (2026-01-19)
Новый статус: ✅ Passed

Тест снова работает!
```

**При flaky:**
```
⚠️ Нестабильный тест: issue-review

Статистика:
- Passed: 60%
- Failed: 40%

Рекомендации:
1. Проверить зависимости
2. Добавить retries
3. Изолировать тест

Исправить сейчас? [Y/n]
```

### Шаг 6: Результат

```
✅ Тест отмечен

Тест: {путь}
Статус: ✅ Passed
Дата: 2026-01-20 15:45
Комментарий: {note или "—"}

История:
- Всего запусков: {N}
- Passed: {X}
- Failed: {Y}
- Успешность: {X/N * 100}%

Следующие шаги:
- Проверить покрытие: /test-review {путь}
- Запустить все тесты: /test-execute --scope claude
```

## FAQ / Troubleshooting

### Когда создавать Issue при failed?

**Автоматическое предложение Issue если:**
- Тест упал впервые за последние 5 запусков
- Тест связан с критичным скиллом (см. CLAUDE.md)
- Тест падает 3+ раза подряд

**Создать Issue вручную:**
```bash
/issue-create --type bug --title "Test failed: название-теста"
```

### Как посмотреть историю запусков?

```bash
/test-complete путь-к-тесту --history
```

Или прочитать раздел "История запусков" в файле теста.

### Можно ли откатить отметку?

Да, используйте флаг `--rollback`:
```bash
/test-complete путь-к-тесту --rollback
```

### Какие статусы доступны?

| Статус | Emoji | Описание |
|--------|-------|----------|
| `passed` | ✅ | Тест успешно пройден |
| `failed` | ❌ | Тест провален |
| `skipped` | ⏭️ | Тест пропущен |
| `pending` | ⬜ | Тест не выполнялся |

### Как добавить примечание к статусу?

```bash
/test-complete путь --status passed --note "Fixed after refactoring"
```

### Scope определён неверно

> **SSOT:** См. [scope-detection.md](/.claude/templates/scope-detection.md)

Используйте явное указание:
```bash
/test-complete путь --scope project
```

## Следующие шаги

После отметки теста:

**При passed:**
Работа завершена.

**При failed:**
```bash
# 1. Найти проблему
/test-review {путь}

# 2. Исправить
/test-update {путь} --reason fix

# 3. Или создать Issue
/issue-create --type bug
```

**При skipped:**
```bash
# Если тест больше не нужен
/test-delete {путь}
```

**Типичные цепочки:**
```
failed:  /test-complete → /test-review → /test-update → /test-execute
skipped: /test-complete → /test-delete (если устарел)
```
