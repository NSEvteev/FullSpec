# Удаление теста

Команда для удаления теста: удаление раздела тестирования из SKILL.md или удаление файла теста.

**Связанные скиллы:**
- [test-create](/.claude/skills/test-create/SKILL.md) — создание теста
- [test-update](/.claude/skills/test-update/SKILL.md) — изменение теста
- [test-execute](/.claude/skills/test-execute/SKILL.md) — выполнение тестов
- [test-review](/.claude/skills/test-review/SKILL.md) — проверка полноты теста
- [test-complete](/.claude/skills/test-complete/SKILL.md) — отметка о прохождении

**Связанные инструкции:**
- [tests/claude-testing.md](/.claude/instructions/tests/claude-testing.md) — тестирование Claude Code

**Шаблоны:**

- [scope-detection.md](/.claude/templates/scope-detection.md) — определение scope (SSOT)
- [output-formats.md](/.claude/templates/output-formats.md) — форматы вывода (SSOT)

## Формат вызова

```
/test-delete [путь-к-тесту] [--force] [--reason "причина"]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь-к-тесту` | Путь к тесту или скиллу | Спросить |
| `--force` | Удалить без подтверждения | — |
| `--reason` | Причина удаления | Спросить |
| `--keep-history` | Сохранить историю в архив | — |
| `--batch` | Удалить несколько тестов | — |

## Типы удаления

| Scope | Что удаляется | Обратимость |
|-------|---------------|-------------|
| claude (раздел) | Раздел "## Тестирование" в SKILL.md | `git checkout` |
| claude (файл) | Файл `tests.md` в папке скилла | `git checkout` |
| project | Файл `*.test.ts` или `*.spec.ts` | `git checkout` |

## Правила

### Правило подтверждения

**Правило:** Удаление теста требует явного подтверждения (кроме `--force`).

**Обязательная информация перед удалением:**
- Путь к тесту
- Тип теста (smoke/functional/integration)
- Последний результат
- Количество сценариев

### Правило причины

**Правило:** Рекомендуется указывать причину удаления для аудита.

**Типичные причины:**
- `deprecated` — функционал устарел
- `replaced` — тест заменён другим
- `invalid` — тест некорректен и не подлежит исправлению
- `merged` — объединён с другим тестом

### Правило архивации

**Правило:** При `--keep-history` история запусков сохраняется в архив.

**Формат архива:**
```
/.claude/archive/tests/{skill-name}-{date}.md
```

### Правило защиты

**Правило:** Нельзя удалить тест, если он:
- Единственный тест для критичного скилла
- Последний запуск был failed (нужно сначала исправить)

**Обход:** Использовать `--force` с явным подтверждением.

## Воркфлоу

### Шаг 1: Определить тест

1. Из аргумента: `/test-delete .claude/skills/old-skill/SKILL.md`
2. Или спросить: "Какой тест удалить?"

**Валидация:**
- Путь существует
- Объект имеет тесты

**Batch режим:**
```
📋 Batch удаление

Паттерн: .claude/skills/deprecated-*/SKILL.md

Найдено тестов: 3
- deprecated-feature-1: smoke test (✅ passed)
- deprecated-feature-2: smoke test (⬜ not run)
- deprecated-feature-3: functional test (❌ failed)

⚠️ deprecated-feature-3 имеет статус failed. Удалить всё равно? [y/N]
```

### Шаг 2: Анализ теста

1. Прочитать тест
2. Извлечь метаданные:
   - Тип теста
   - Количество сценариев
   - Последний результат
   - История запусков

```
📋 Информация о тесте

Путь: .claude/skills/old-feature/SKILL.md
Scope: claude
Тип: smoke

Содержимое:
- Сценариев: 3
- Последний результат: ✅ Passed (2026-01-15)

История:
- Запусков: 5
- Passed: 4 (80%)
- Failed: 1 (20%)

Связанные тесты: нет
```

**Проверка защиты:**
```
⚠️ Защита от удаления

Тест: critical-feature
Причина: Единственный тест для критичного скилла

Варианты:
[1] Создать замену перед удалением
[2] Продолжить с --force
[3] Отмена

> _
```

### Шаг 3: Подтверждение

```
⚠️ Удаление теста

Тест: old-feature
Путь: .claude/skills/old-feature/SKILL.md
Тип: smoke (3 сценария)
Последний результат: ✅ Passed

Причина удаления: {reason или "не указана"}

Действие: Удалить раздел "## Тестирование" из SKILL.md

Продолжить? [y/N]
```

**С --force:**
```
⚠️ Принудительное удаление (--force)

Тест: old-feature
Статус: ❌ Failed

ВНИМАНИЕ: Тест имеет статус failed!
Рекомендуется сначала исправить.

Точно удалить? Введите "DELETE" для подтверждения:
> _
```

### Шаг 4: Удаление

**Scope claude (раздел в SKILL.md):**

1. Найти раздел `## Тестирование`
2. Удалить всё до следующего `##` или конца файла
3. Если `--keep-history` — сохранить историю в архив

```bash
# Создать архив
mkdir -p .claude/archive/tests
# Сохранить историю
echo "..." > .claude/archive/tests/old-feature-2026-01-20.md
```

**Scope claude (файл tests.md):**

```bash
rm .claude/skills/old-feature/tests.md
```

**Scope project:**

```bash
rm src/auth/services/tests/old-feature.test.ts
```

### Шаг 5: Очистка связей

1. Проверить, есть ли ссылки на тест в других файлах
2. Если есть — предложить удалить или пометить

```
📋 Найдены ссылки на удалённый тест

Файлы со ссылками:
- .claude/instructions/tests/claude-testing.md:45 — пример теста
- .claude/skills/other-skill/SKILL.md:78 — связанный тест

Действие:
[1] Удалить ссылки
[2] Пометить как устаревшие
[3] Оставить как есть

> _
```

### Шаг 6: Результат

```
✅ Тест удалён

Тест: old-feature
Путь: .claude/skills/old-feature/SKILL.md
Тип удаления: раздел

Причина: {reason}

Архив: {путь к архиву или "не создан"}
Ссылки: {N} удалено / {M} помечено

Откат:
git checkout -- .claude/skills/old-feature/SKILL.md

Следующие шаги:
- Убедиться, что функционал покрыт другими тестами
- Проверить связанные скиллы
```

## FAQ / Troubleshooting

### Как восстановить удалённый тест?

**Из git (если не был закоммичен):**
```bash
git checkout -- {путь-к-тесту}
```

**Из архива (если использовался `--keep-history`):**
```bash
# Посмотреть архив
cat .claude/archive/tests/{skill-name}-{date}.md

# Восстановить вручную через /test-create
/test-create {путь-к-объекту}
# Скопировать содержимое из архива
```

**Из git history (если был закоммичен):**
```bash
# Найти коммит с удалением
git log --oneline -- {путь-к-тесту}

# Восстановить из конкретного коммита
git checkout {commit-hash}^ -- {путь-к-тесту}
```

### Когда использовать `--keep-history`?

| Ситуация | `--keep-history` | Почему |
|----------|:----------------:|--------|
| Тест устарел, функционал удалён | ❌ Нет | История не нужна |
| Тест заменяется новым | ✅ Да | Сохранить метрики для сравнения |
| Тест объединяется с другим | ✅ Да | Документировать объединение |
| Временное удаление | ✅ Да | Облегчит восстановление |
| Flaky тест для анализа | ✅ Да | Сохранить паттерн нестабильности |

### Что происходит со связанными тестами?

**При удалении теста:**
1. Скилл ищет ссылки на тест в других файлах
2. Предлагает действия: удалить / пометить / оставить

**Типы связей:**
| Связь | Обработка |
|-------|-----------|
| Ссылка в документации | Пометить `*(удалён)*` |
| Связанный тест (depends on) | Предупреждение + выбор |
| Импорт в другом тесте | Ошибка компиляции (project scope) |

**Если связанный тест зависит от удаляемого:**
```
⚠️ Найден зависимый тест

Тест: integration-flow
Зависит от: {удаляемый тест}

Удаление может сломать integration-flow.

Варианты:
[1] Удалить оба теста
[2] Обновить зависимый тест
[3] Отмена
```

### Как удалить все тесты объекта?

**Batch режим:**
```bash
/test-delete --batch .claude/skills/{skill-name}/**/*.test.md
```

**Для scope claude (все тесты скилла):**
```bash
/test-delete .claude/skills/{skill-name}/SKILL.md --batch
# Удалит и раздел "Тестирование", и файл tests.md (если есть)
```

**Для scope project (все тесты модуля):**
```bash
/test-delete --batch src/{module}/tests/*.test.ts
```

### Что такое "критичный скилл" в контексте защиты?

**Критичные скиллы** определены в [CLAUDE.md](/CLAUDE.md#критичные-скиллы):
- `skill-*` (skill-create, skill-update, skill-delete)
- `instruction-*` (instruction-create, instruction-update, instruction-delete)
- `issue-*` (issue-create, issue-update, issue-execute, issue-review, issue-complete, issue-delete)

**Защита:** Нельзя удалить **единственный тест** критичного скилла.

**Обход:**
1. Создать новый тест перед удалением
2. Или использовать `--force` с явным подтверждением

### Можно ли отменить batch удаление?

**До подтверждения:** Просто выберите `N` или Ctrl+C.

**После подтверждения:** Используйте git:
```bash
# Отменить все изменения в рабочей директории
git checkout -- .claude/skills/{pattern}/*

# Или через stash (если были другие изменения)
git stash
git checkout -- {файлы}
git stash pop
```

### Почему тест не удаляется?

| Симптом | Причина | Решение |
|---------|---------|---------|
| "Защита от удаления" | Критичный скилл, единственный тест | Создать замену или `--force` |
| "Тест имеет статус failed" | Защита от скрытия проблем | Исправить тест или `--force` |
| "Файл не найден" | Неверный путь | Проверить путь через `ls` |
| "Нет раздела Тестирование" | Тест уже удалён или не существует | Проверить содержимое SKILL.md |
