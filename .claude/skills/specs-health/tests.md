# Тесты: specs-health

## Smoke-тесты

### smoke-1: Базовый запуск

**Команда:** `/specs-health`

**Триггеры:**
- `проверь спецификации`
- `здоровье specs`
- `найди проблемы в specs`
- `check specs`
- `specs health`
- `find specs issues`

**Ожидание:** Отчёт о целостности /specs/

**Проверка:**
- [ ] Скилл распознаётся по команде `/specs-health`
- [ ] Показано количество проверенных документов
- [ ] Показано количество проверенных сервисов
- [ ] Выведён итоговый статус (проблемы или "Проблем не обнаружено")
- [ ] Нет ошибок выполнения

**Статус:** ⬜ Не выполнен

## Функциональные тесты

### func-1: Обнаружение Orphan Discussion

**Подготовка:**
1. Создать Discussion в статусе APPROVED
2. Убедиться, что нет связанного Impact >7 дней

**Команда:** `/specs-health`

**Ожидание:** Обнаружена проблема "Orphan Discussion"

**Проверка:**
- [ ] В отчёте присутствует предупреждение
- [ ] Указана рекомендация: создать Impact или REJECTED

**Очистка:** Удалить тестовый Discussion

**Статус:** ⬜ Не выполнен

---

### func-2: Обнаружение Inconsistent status

**Подготовка:**
1. Создать Impact в статусе RUNNING
2. Создать дочерние ADR в статусе DONE

**Команда:** `/specs-health`

**Ожидание:** Обнаружена проблема "Inconsistent status"

**Проверка:**
- [ ] Проблема отмечена как критичная
- [ ] Указана рекомендация: /spec-status для обновления родителя

**Очистка:** Удалить тестовые документы

**Статус:** ⬜ Не выполнен

---

### func-3: Обнаружение Stuck Plan

**Подготовка:**
1. Создать Plan в статусе RUNNING (>14 дней)
2. Убедиться, что все связанные GitHub Issues закрыты

**Команда:** `/specs-health`

**Ожидание:** Обнаружена проблема "Stuck Plan"

**Проверка:**
- [ ] Проблема отмечена как критичная
- [ ] Показано время в статусе RUNNING
- [ ] Указана рекомендация: перевести в DONE

**Очистка:** Удалить тестовый Plan

**Статус:** ⬜ Не выполнен

---

### func-4: Обнаружение Broken links

**Подготовка:**
1. Добавить в документ /specs/ ссылку на несуществующий файл

**Команда:** `/specs-health`

**Ожидание:** Обнаружена проблема "Broken links"

**Проверка:**
- [ ] В отчёте указаны битые ссылки
- [ ] Указана рекомендация: исправить ссылки

**Очистка:** Восстановить документ

**Статус:** ⬜ Не выполнен

---

### func-5: Обнаружение Service without specs

**Подготовка:**
1. Создать папку /src/test-service/
2. НЕ создавать /specs/services/test-service/

**Команда:** `/specs-health`

**Ожидание:** Обнаружена проблема "Service without specs"

**Проверка:**
- [ ] В отчёте указан сервис без specs
- [ ] Указана рекомендация: /spec-create adr для сервиса

**Очистка:** Удалить /src/test-service/

**Статус:** ⬜ Не выполнен

---

### func-6: Режим --fix с подтверждением

**Команда:** `/specs-health --fix`

**Ожидание:** Предложение исправить найденные проблемы

**Проверка:**
- [ ] Показан список проблем
- [ ] Выведен запрос: "Исправить проблемы? [все / критичные / выборочно / пропустить]"
- [ ] Claude ожидает ответа пользователя (блокирующее подтверждение)

**Статус:** ⬜ Не выполнен

## Edge cases

### edge-1: Пустая папка /specs/

**Подготовка:** Папка /specs/ пуста (только README.md)

**Команда:** `/specs-health`

**Ожидание:** Корректная обработка

**Проверка:**
- [ ] Нет ошибок выполнения
- [ ] Показано: "Проверено документов: 0"

**Статус:** ⬜ Не выполнен

---

### edge-2: Все проверки пройдены

**Подготовка:** Корректная структура /specs/

**Команда:** `/specs-health`

**Ожидание:** Успех

**Проверка:**
- [ ] Статус: "Проблем не обнаружено"
- [ ] Показано количество документов без проблем

**Статус:** ⬜ Не выполнен

---

**Проверка:**
- [ ] Нет падения с ошибкой

**Статус:** ⬜ Не выполнен

- [ ] Вызывается по английской фразе "check specs"
- [ ] Показывает отчёт о целостности
- [ ] Не падает с ошибкой при базовом вызове
