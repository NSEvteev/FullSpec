# Обновление документации Claude

Скилл для синхронизации CLAUDE.md и llm_instructions/llm_instructions.md при важных изменениях в проекте.

## Когда использовать

**Автоматически запускать после:**
1. Добавления новых инструкций в `llm_instructions/`
2. Создания новых скиллов или агентов
3. Изменения структуры проекта
4. Добавления новых служебных скриптов
5. Изменения workflow документации
6. Добавления новых команд Makefile
7. Изменения системы управления задачами

## Принципы

**Разделение информации:**
- **CLAUDE.md** — краткая справочная информация для быстрого старта
- **llm_instructions.md** — полная детальная информация с контекстом

**Правило против дублирования:**
- CLAUDE.md содержит только ссылки и краткие описания (1-2 предложения)
- llm_instructions.md содержит полное описание с примерами и деталями
- Избегать копирования одинаковых абзацев в оба файла

## Инструкции

### Шаг 1: Понять изменение

Определить тип изменения:
- [ ] Добавлена новая инструкция в `llm_instructions/`
- [ ] Создан новый скилл или агент
- [ ] Изменена структура проекта
- [ ] Добавлен новый скрипт
- [ ] Изменён workflow

### Шаг 2: Прочитать текущие документы

```bash
# Прочитать CLAUDE.md
cat CLAUDE.md

# Прочитать llm_instructions.md
cat llm_instructions/llm_instructions.md
```

Обратить внимание на:
- Структуру разделов
- Существующие ссылки
- Уровень детализации

### Шаг 3: Обновить llm_instructions.md

**Что обновлять:**

1. **Таблица инструкций** (если добавлена новая инструкция):
```markdown
| Файл | Назначение |
|------|------------|
| [instructions_new.md](instructions_new.md) | Краткое описание |
```

2. **Структура проекта** (если изменилась):
```
llm_instructions/
├── llm_instructions.md
├── instructions_*.md
└── templates/
```

3. **Ключевые особенности** (если добавлен новый функционал):
```markdown
**Новый раздел:**
- Описание функционала
- Как использовать
```

### Шаг 4: Обновить CLAUDE.md

**Что обновлять:**

1. **Таблица "Инструкции для LLM"** (всегда актуальная):
```markdown
| Инструкция | Назначение |
|------------|------------|
| [instructions_new.md](llm_instructions/instructions_new.md) | Краткое описание |
```

2. **Раздел "Быстрый старт LLM"** (если изменился workflow):
```markdown
1. **Контекст проекта:** Ознакомиться с llm_instructions.md
2. **Новый шаг:** Краткое описание
```

3. **Ссылки на новые ресурсы** (при добавлении скриптов/скиллов):
```markdown
## Команды

make new-command     # Краткое описание
```

### Шаг 5: Проверить на дублирование

**Чек-лист:**
- [ ] CLAUDE.md содержит только краткие описания (1-2 предложения)
- [ ] Детальные инструкции только в llm_instructions.md
- [ ] Нет копирования целых абзацев между файлами
- [ ] Все ссылки из CLAUDE.md ведут в llm_instructions/ или другие детальные документы
- [ ] Таблицы инструкций синхронизированы (одинаковый список, разная детализация)

**Примеры правильного разделения:**

✅ **CLAUDE.md:**
```markdown
| [instructions_tasks.md](llm_instructions/instructions_tasks.md) | Управление задачами через llm_tasks/ |
```

✅ **llm_instructions.md:**
```markdown
| [instructions_tasks.md](instructions_tasks.md) | **Управление задачами через `llm_tasks/`** (current_tasks.md, future_tasks.md), workflow статусов, правила синхронизации с TodoWrite |
```

❌ **Плохо (дублирование):**
Одинаковый длинный текст в обоих файлах.

### Шаг 6: Проверить ссылки

```bash
# Проверить, что все ссылки работают
python scripts/check_doc_health.py --check links CLAUDE.md llm_instructions/llm_instructions.md
```

### Шаг 7: Резюме изменений

Сообщить пользователю:
1. Какие файлы были обновлены
2. Что было добавлено/изменено в каждом
3. Проверены ли ссылки
4. Проверено ли дублирование

## Правила синхронизации

### Раздел "Инструкции для LLM"

**Требование:** Всегда актуален в обоих файлах

**CLAUDE.md** — краткая таблица:
```markdown
| Инструкция | Назначение |
|------------|------------|
| [instructions_X.md](llm_instructions/instructions_X.md) | Краткое (1-5 слов) |
```

**llm_instructions.md** — детальная таблица:
```markdown
| Файл | Назначение |
|------|------------|
| [instructions_X.md](instructions_X.md) | **Основная тема** — детали, контекст, связанные документы |
```

### Раздел "Быстрый старт LLM"

**Требование:** Синхронизирован, но разный уровень детализации

**CLAUDE.md:**
```markdown
1. **Шаг:** Краткое действие
2. **Шаг:** Краткое действие
```

**llm_instructions.md:**
```markdown
## Быстрый старт LLM

См. раздел «Быстрый старт LLM» в [CLAUDE.md](../CLAUDE.md#быстрый-старт-llm)
```

(Избегать дублирования — в llm_instructions.md просто ссылка)

### Структура проекта

**Требование:** Полная только в llm_instructions.md

**CLAUDE.md:**
```markdown
## Архитектура

См. [llm_instructions.md](llm_instructions/llm_instructions.md#структура-проекта)
```

**llm_instructions.md:**
```markdown
## Структура проекта

Полное ASCII-дерево со всеми папками и комментариями
```

## Примеры использования

### Пример 1: Добавлена новая инструкция instructions_security.md

**Действия:**
1. Прочитать CLAUDE.md и llm_instructions.md
2. Добавить в таблицу инструкций в llm_instructions.md:
   ```markdown
   | [instructions_security.md](instructions_security.md) | **Правила безопасности кода** — валидация, аутентификация, работа с секретами, checklist перед релизом |
   ```
3. Добавить в таблицу инструкций в CLAUDE.md:
   ```markdown
   | [instructions_security.md](llm_instructions/instructions_security.md) | Правила безопасности |
   ```
4. Проверить ссылки
5. Сообщить пользователю об обновлении

### Пример 2: Создан новый скилл /test-runner

**Действия:**
1. Обновить llm_instructions.md в разделе "Скиллы":
   ```markdown
   - `/test-runner` — автоматический запуск тестов после изменений кода
   ```
2. Обновить CLAUDE.md в разделе "Команды":
   ```markdown
   /test-runner     # Запуск тестов
   ```
3. Проверить, что информация не дублируется
4. Сообщить пользователю
