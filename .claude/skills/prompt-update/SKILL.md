# Улучшение промта

Мета-скилл для улучшения и обогащения промтов перед их выполнением. Анализирует запрос пользователя, добавляет контекст и детали, предлагает улучшенную версию.

> ⚠️ **КРИТИЧЕСКОЕ ПРАВИЛО:** Улучшенный промт ВСЕГДА показывается пользователю перед выполнением. Нельзя выполнять задачу без явного подтверждения пользователя. Шаг 5 (показать пользователю) — блокирующий.

**Связанные скиллы:**
- [skill-create](/.claude/skills/skill-create/SKILL.md) — если промт про создание скилла
- [instruction-create](/.claude/skills/instruction-create/SKILL.md) — если промт про создание инструкции
- [context-update](/.claude/skills/context-update/SKILL.md) — распространение контекста

**Шаблоны:**
- [output-formats.md](/.claude/templates/output-formats.md) — форматы вывода (SSOT)

## Формат вызова

```
/prompt-update <промт>
```

Или использовать триггерные фразы:
```
улучши промт: добавь кнопку логаута
промт: сделай авторизацию
```

**Примеры:**
- `/prompt-update добавь тесты` — улучшить и выполнить
- `улучши промт: рефакторинг` — улучшить и выполнить
- `промт: исправь баг` — улучшить и выполнить

## Когда использовать

| Ситуация | Польза |
|----------|--------|
| Короткий запрос | Добавить детали и контекст |
| Неоднозначная задача | Уточнить требования |
| Сложная задача | Разбить на шаги |
| Новый в проекте | Добавить контекст проекта |

## Правила улучшения

### Что добавлять

1. **Контекст проекта** — структура, технологии, паттерны
2. **Уточнения** — что именно, где, как
3. **Ограничения** — что НЕ делать
4. **Критерии успеха** — как понять, что готово
5. **Связанные файлы** — какие файлы затронуты

### Чего избегать

- Не менять суть запроса
- Не добавлять лишнего функционала
- Не усложнять без необходимости
- Не терять исходное намерение

### Принцип минимального улучшения

Добавлять только то, что:
- Устраняет неоднозначность
- Добавляет необходимый контекст
- Помогает выполнить задачу качественнее

## Воркфлоу

### Шаг 1: Получить промт

1. Извлечь промт из сообщения пользователя
2. Определить тип задачи:
   - Создание (добавь, создай, сделай)
   - Изменение (измени, обнови, рефакторинг)
   - Исправление (исправь, почини, баг)
   - Исследование (найди, покажи, объясни)

### Шаг 2: Анализ промта

Определить:

| Аспект | Вопрос | Пример |
|--------|--------|--------|
| Объект | Что именно? | "кнопку" → какую кнопку? где? |
| Действие | Что сделать? | "добавь" → создать новую или изменить? |
| Место | Где? | В каком файле/компоненте? |
| Контекст | Зачем? | Какая бизнес-цель? |
| Ограничения | Что не трогать? | Какие файлы не менять? |

**Матрица неоднозначности:**

| Промт | Неоднозначность | Уточнение |
|-------|-----------------|-----------|
| "добавь тесты" | Какие? Куда? | Юнит-тесты для последних изменений |
| "рефакторинг" | Чего? Как? | Какой код? Какие паттерны? |
| "исправь баг" | Какой? Где? | Описание бага, файл |
| "сделай быстрее" | Что именно? | Какая функция? Метрики? |

### Шаг 3: Собрать контекст

**3.1. Контекст проекта:**
- Прочитать [CLAUDE.md](/CLAUDE.md) для понимания структуры
- Определить релевантные инструкции из `/.claude/instructions/`

**3.2. Контекст задачи:**
- Найти упомянутые файлы/компоненты
- Понять текущее состояние кода
- Определить связанные файлы

**3.3. Контекст истории:**
- Посмотреть недавние изменения (git log)
- Учесть предыдущие сообщения в диалоге

## Техники улучшения

### По типу задачи

**Создание (добавь, создай):**
```
Исходный: "добавь кнопку логаута"

Улучшенный:
- Задача: Добавить кнопку логаута в header
- Файл: src/components/Header.tsx
- Требования:
  - Использовать существующий Button компонент
  - Вызывать auth.logout() при клике
  - Показывать только авторизованным
- Ограничения: Не менять стили header
```

**Изменение (измени, рефакторинг):**
```
Исходный: "рефакторинг авторизации"

Улучшенный:
- Задача: Рефакторинг модуля авторизации
- Файлы: src/auth/*.ts
- Требования:
  - Выделить логику в отдельные функции
  - Добавить типизацию
  - Сохранить обратную совместимость
- Ограничения: Не менять API эндпоинты
```

**Исправление (исправь, баг):**
```
Исходный: "исправь баг с логином"

Улучшенный:
- Задача: Исправить баг с логином
- Вопросы: Какой именно баг? Воспроизведение?
- Файлы: Вероятно src/auth/login.ts
- Требования:
  - Найти причину бага
  - Исправить с минимальными изменениями
  - Добавить тест на регрессию
```

### Добавление контекста

**Из структуры проекта:**
- Технологии: React, TypeScript, etc.
- Паттерны: как организован код
- Конвенции: именование, стиль

**Из истории:**
- Последние изменения в области
- Связанные коммиты/PR

**Из инструкций:**
- Правила для данной области кода
- Ограничения и требования

## FAQ / Troubleshooting

### Когда использовать prompt-update?

| Ситуация | Использовать |
|----------|--------------|
| Запрос неоднозначен | Да |
| Много вариантов реализации | Да |
| Нужно уточнить детали | Да |
| Запрос простой и конкретный | Нет — выполнить сразу |

### Можно ли отклонить улучшенный промт?

Да. На Шаге 5 выберите `[2] Отклонить` и скилл:
- Сохранит исходный промт
- Попросит уточнить требования
- Предложит альтернативу

### Что если улучшенный промт неверный?

1. Выберите `[3] Редактировать`
2. Укажите, что исправить
3. Скилл сгенерирует новую версию

### Почему подтверждение обязательно?

**Блокирующее правило:** Улучшенный промт может изменить смысл задачи. Без подтверждения пользователя скилл не выполняет действия.

### Какие скиллы интегрированы с prompt-update?

| Категория | Скиллы |
|-----------|--------|
| skill-management | skill-create |
| instruction-management | instruction-create |
| Любой скилл с неоднозначным промтом | По рекомендации |
