# Скилл: Документирование выполненной задачи

## Назначение

Автоматически вызывается при завершении задачи (через `scripts/task_complete.py`) для обновления связанной документации.

## Входные параметры

Скилл получает ID выполненной задачи через аргументы командной строки:

```bash
# Вызов из task_complete.py
python scripts/task_complete.py FEAT-00001
# Автоматически вызывает Amy с этим скиллом

# Ручной вызов
# TODO: определить API для вызова
```

## Процесс работы

### 1. Прочитать файл выполненной задачи

- Найти файл в `llm_tasks/completed/YYYY-MM/{assignee}/{task_id}.md`
- Извлечь из frontmatter: `id`, `title`, `category`, `completed`, `assignee`
- Извлечь секцию "Связанные документы"
- Извлечь секцию "Результат"

### 2. Обновить связанные документы

Для каждого документа из секции "Связанные документы":

#### Если это Дискуссия (general_docs/01_discuss/)

- Добавить или обновить раздел "Связанные задачи"
- Формат:
  ```markdown
  ## Связанные задачи

  - [FEAT-00001](../../../llm_tasks/completed/2026/01-january/FEAT-00001.md) — Название задачи (завершено: 2026-01-15)
  ```

#### Если это Архитектура (general_docs/02_architecture/)

- Обновить раздел "История изменений"
- Добавить запись:
  ```markdown
  ## История изменений

  ### 2026-01-15

  - **[FEAT-00001](../../../llm_tasks/completed/2026/01-january/FEAT-00001.md):** Название задачи
    - Краткое описание изменений из секции "Результат"
  ```

#### Если это План реализации (general_docs/06_imp_plans/)

- Найти связанные подзадачи в плане
- Отметить как выполненные `- [x]`
- Добавить ссылку на задачу

### 3. Обновить индексы документации

Проверить и обновить индексы `000_*_index.md` в:
- `general_docs/01_discuss/`
- `general_docs/02_architecture/`
- `general_docs/06_imp_plans/`

### 4. Обновить индекс завершённых задач

Обновить файл `llm_tasks/completed/YYYY-MM/{assignee}/0_task_index.md`:

1. Добавить строку в таблицу с данными задачи:
   - ID задачи с ссылкой (например, `[FEAT-00001](FEAT-00001.md)`)
   - Название
   - Дата завершения
   - Фактическое время
   - Фактические токены
   - **Документация обновлена** — статус обновления документации

2. **Статусы для колонки "Документация обновлена":**
   - `Success` — все связанные документы успешно обновлены. Формат: `Success: [Doc1](путь), [Doc2](путь)`
   - `Need` — требуется обновление документации (если есть связанные документы, но они не были обновлены)
   - `N/A` — нет связанных документов для обновления

3. Пример строки в индексе:
   ```markdown
   | [FEAT-00001](FEAT-00001.md) | Добавить OAuth | 2026-01-15 | 2ч | 15k | Success: [Architecture-005](../../../general_docs/02_architecture/005_auth.md), [Plan-003](../../../general_docs/06_imp_plans/003_oauth.md) |
   ```

### 5. Отметить в задаче

Обновить секцию "Результат" → "Документация обновлена" в файле задачи:

```markdown
### Документация обновлена

- [x] Amy Santiago обновила связанные документы (2026-01-15)
- [Дискуссия 001](../../general_docs/01_discuss/001_auth_system.md) — добавлена ссылка на задачу в раздел "Связанные задачи"
- [Архитектура 005](../../general_docs/02_architecture/005_auth_flow.md) — обновлён раздел "История изменений", добавлена запись от 2026-01-15
- [План 003](../../general_docs/06_imp_plans/003_auth_implementation.md) — отмечены выполненные подзадачи
```

### 6. Создать отчёт

Сохранить детальный отчёт в:
- `llm_tasks/temp/amy-santiago/task_doc_{task_id}_{date}.md`

Формат отчёта:

```markdown
# Отчёт по документированию задачи {task_id}

**Дата:** {date}
**Задача:** [{task_id}](путь к задаче)

## Выполненные действия

### Обновлённые документы

1. **Дискуссия:** [001_auth_system.md](путь)
   - Добавлена ссылка на задачу в раздел "Связанные задачи"

2. **Архитектура:** [005_auth_flow.md](путь)
   - Обновлён раздел "История изменений"
   - Добавлена запись от {date}

3. **План:** [003_auth_implementation.md](путь)
   - Отмечены выполненные подзадачи: 2.1, 2.3

### Обновлённые индексы

- `general_docs/01_discuss/000_discuss.md`
- `general_docs/02_architecture/000_architecture.md`

## Результат

✓ Все связанные документы обновлены
✓ Индексы актуализированы
✓ Задача отмечена как задокументированная
```

## Выходные данные

- Обновлённые связанные документы
- Обновлённые индексы документации
- Обновлённая задача (с отметкой о документировании)
- Отчёт в `temp/task_doc_{task_id}_{date}.md`

## Обработка ошибок

### Если связанные документы не найдены

- Записать предупреждение в отчёт
- Не блокировать выполнение скилла
- Предложить создать документы

### Если документ не соответствует ожидаемой структуре

- Записать в отчёт
- Пропустить обновление этого документа
- Продолжить со следующими
