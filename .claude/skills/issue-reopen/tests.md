# Тесты: issue-reopen

## Smoke-тесты

### smoke-1: Проверка environment

**Команда:** `/issue-reopen #1 --dry-run`

**Ожидание:** Проверка gh авторизации

**Проверка:**
- [ ] Если gh не авторизован — ошибка с инструкцией
- [ ] Если авторизован — показан план

---

## Функциональные тесты

### func-1: Переоткрытие закрытого Issue

**Подготовка:**
1. Создать Issue: `gh issue create --title "Test reopen" --body "Test"`
2. Закрыть: `gh issue close {номер}`

**Команда:** `/issue-reopen #{номер}`

**Ожидание:** Issue переоткрыт

**Проверка:**
- [ ] Статус: ✅
- [ ] `gh issue view {номер}` показывает state: OPEN

**Очистка:** `gh issue close {номер}` и `gh issue delete {номер} --yes`

---

### func-2: Переоткрытие с комментарием

**Подготовка:**
1. Создать и закрыть Issue

**Команда:** `/issue-reopen #{номер} "Нужно доработать"`

**Ожидание:** Добавлен комментарий

**Проверка:**
- [ ] Issue переоткрыт
- [ ] Последний комментарий содержит "Нужно доработать"

---

### func-3: Переоткрытие с назначением

**Подготовка:**
1. Создать и закрыть Issue

**Команда:** `/issue-reopen #{номер} --assign`

**Ожидание:** Issue назначен на текущего пользователя

**Проверка:**
- [ ] Issue переоткрыт
- [ ] Assignee содержит текущего пользователя

---

## Edge cases

### edge-1: Issue уже открыт

**Подготовка:** Открытый Issue

**Команда:** `/issue-reopen #{номер}`

**Ожидание:** Информационное сообщение

**Проверка:**
- [ ] Вывод содержит ℹ️
- [ ] Текст: "уже открыт"
- [ ] Состояние не изменилось

---

### edge-2: Issue не существует

**Команда:** `/issue-reopen #999999`

**Ожидание:** Ошибка

**Проверка:**
- [ ] Вывод содержит ❌
- [ ] Причина: Issue не найден

---

### edge-3: Нет прав на переоткрытие

**Подготовка:** Issue в чужом репозитории без прав

**Команда:** `/issue-reopen #1`

**Ожидание:** Ошибка доступа

**Проверка:**
- [ ] Вывод содержит ❌
- [ ] Причина указывает на права доступа
