# /issue-reopen

Переоткрытие ранее закрытого GitHub Issue с добавлением комментария о причине.

## Триггеры

- `/issue-reopen #123` — переоткрыть Issue
- `/issue-reopen #123 "причина"` — переоткрыть с комментарием

## Параметры

| Параметр | Описание | Обязательный |
|----------|----------|--------------|
| `#номер` | Номер Issue | Да |
| `"причина"` | Комментарий о причине | Нет |
| `--assign` | Назначить на себя | Нет |
| `--dry-run` | Показать без изменений | Нет |
| `--json` | JSON формат вывода | Нет |

## Воркфлоу

### Шаг 0: Проверки

1. **environment-check:**
   - `gh` установлен
   - `gh auth status` — авторизован

2. **input-validate:**
   - Номер Issue — число

### Шаг 1: Получить информацию об Issue

```bash
gh issue view {номер} --json number,title,state,closedAt,stateReason,labels
```

### Шаг 2: Проверка статуса

**Если Issue открыт:**
```
ℹ️ Issue #{номер} уже открыт

Статус: OPEN
Действий не требуется.
```
→ Завершить

**Если Issue закрыт:**
→ Продолжить

### Шаг 3: Показать информацию

```
📋 Issue #{номер}

Заголовок: {title}
Статус: CLOSED
Закрыт: {closedAt}
Причина: {stateReason}
Метки: {labels}

Переоткрыть? [Y/n]
```

### Шаг 4: Переоткрытие

```bash
gh issue reopen {номер}
```

### Шаг 5: Добавление комментария (если указана причина)

```bash
gh issue comment {номер} --body "🔄 **Переоткрыт**

Причина: {причина}

---
*Переоткрыт через /issue-reopen*"
```

### Шаг 6: Назначение (если --assign)

```bash
gh issue edit {номер} --add-assignee @me
```

### Шаг 7: Результат

**Успех:**
```
✅ Issue #{номер} переоткрыт

Заголовок: {title}
Статус: OPEN
{Если --assign: Назначен: @me}

Следующие шаги:
- /issue-execute #{номер} — взять в работу
- /issue-update #{номер} — обновить описание
```

**Ошибка:**
```
❌ Ошибка: Не удалось переоткрыть Issue

Причина: {описание из gh}
Решение: {рекомендация}
```

## Примеры использования

### Простое переоткрытие

```
/issue-reopen #123
```

```
/issue-reopen #123 "Задача не была полностью выполнена"
```

### С назначением на себя

```
/issue-reopen #123 --assign
```

### Предварительный просмотр

```
/issue-reopen #123 --dry-run
```

## FAQ

### Когда нужно переоткрывать Issue?

- Задача была закрыта преждевременно
- Найдены новые требования
- Баг воспроизвёлся снова
- Решение оказалось неполным

### Сохраняется ли история?

Да. GitHub сохраняет всю историю: закрытие, переоткрытие, комментарии.

### Что если нет прав на переоткрытие?

Скилл выдаст ошибку с описанием от GitHub. Обратитесь к владельцу репозитория.
