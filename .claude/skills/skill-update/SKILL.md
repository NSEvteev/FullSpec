# Обновление скиллов

Команда для обновления существующих скиллов при добавлении нового. Анализирует, может ли новый скилл быть полезен для выполнения существующих, и предлагает интеграцию.

**Связанные скиллы:**
- [skill-create](/.claude/skills/skill-create/SKILL.md) — создание нового скилла
- [skill-delete](/.claude/skills/skill-delete/SKILL.md) — удаление скилла
- [context-update](/.claude/skills/context-update/SKILL.md) — распространение контекста

## Формат вызова

```
/skill-update <новый-скилл>
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `новый-скилл` | Путь к SKILL.md или имя скилла | Последний созданный |

**Примеры:**
- `/skill-update links-validate` — проверить интеграцию для links-validate
- `/skill-update .claude/skills/context-update/SKILL.md` — по полному пути

## Когда использовать

| Событие | Действие |
|---------|----------|
| Создан новый скилл | Автоматически вызывается из skill-create (Шаг 10) |
| Существенно изменён скилл | Проверить, нужно ли обновить связанные скиллы |
| Добавлена новая функциональность | Проверить, кто может её использовать |

## Правила

### Что считается "применимым"

Новый скилл применим к существующему, если:

1. **Дополняет воркфлоу** — может быть вызван на одном из шагов
2. **Улучшает качество** — добавляет проверки, валидацию, автоматизацию
3. **Связан по объекту** — работает с тем же типом объектов (скиллы, инструкции, документы)

### Что НЕ считается применимым

- Скилл из другой категории без логической связи
- Скилл, который дублирует существующую функциональность
- Скилл, который усложняет воркфлоу без явной пользы

### Куда добавлять

| Место | Когда |
|-------|-------|
| Воркфлоу (шаг) | Новый скилл выполняет действие, нужное на этом шаге |
| Связанные скиллы | Новый скилл семантически связан |
| Чек-лист | Новый скилл добавляет проверку |

## Воркфлоу

### Шаг 1: Получить новый скилл

1. Из аргумента или последний созданный
2. Прочитать SKILL.md нового скилла
3. Извлечь:
   - `name` — имя скилла
   - `description` — описание
   - `category` — категория
   - Воркфлоу — что делает скилл

### Шаг 2: Загрузить существующие скиллы

1. Найти все скиллы: `/.claude/skills/*/SKILL.md`
2. Исключить:
   - Сам новый скилл
   - Скилл `skill-update` (избежание циклических зависимостей)
   - Скиллы с пометкой `*(не создан)*`
3. Для каждого скилла прочитать:
   - Имя и описание
   - Категорию
   - Воркфлоу (шаги)
   - Связанные скиллы

### Шаг 3: Анализ применимости

Для каждого существующего скилла B:

**Алгоритм:**
```
1. Проверить категорию (для приоритизации, не для исключения):
   - Та же категория → проверить первым
   - Любая другая категория → тоже проверить

2. Проверить объект:
   - Новый скилл работает с тем же типом объектов → применим
   - Новый скилл может быть вспомогательным → проверить воркфлоу

3. Проверить воркфлоу B:
   - Есть ли шаг, где новый скилл логически уместен?
   - Есть ли TODO/placeholder, который новый скилл закрывает?
   - Упоминается ли похожая функциональность без реализации?

4. Результат:
   - Применим → добавить в список с указанием места
   - Не применим → пропустить
```

**Матрица применимости:**

| Новый скилл | Применим к | Место интеграции |
|-------------|-----------|------------------|
| `*-validate` | Скиллы с созданием/изменением | После создания (валидация) |
| `*-update` | Скиллы с созданием | После создания (синхронизация) |
| `context-*` | Все скиллы с .md файлами | После изменения документа |
| `links-*` | Скиллы, создающие .md | После создания/изменения |

### Шаг 4: Сформировать предложения

Для каждого применимого скилла:

1. Определить место интеграции (шаг воркфлоу)
2. Сформировать diff:
   ```diff
   ### Шаг N: {название}

   {существующее содержимое}

   + **Интеграция с {новый-скилл}:**
   + Вызвать [/{новый-скилл}](/.claude/skills/{новый-скилл}/SKILL.md) для {описание пользы}
   ```

3. Добавить в связанные скиллы (если ещё нет)

**Формат вывода:**

```
📋 Предложения по интеграции скилла {новый-скилл}

Проанализировано скиллов: {N}
Найдено применимых: {M}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 {существующий-скилл-1} (категория: {категория})
   Место: Шаг {X} — {название шага}
   Польза: {описание пользы}

   ```diff
   + Вызвать /{новый-скилл} для {действие}
   ```

   [1] Применить  [2] Пропустить  [3] Редактировать

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 {существующий-скилл-2} ...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Применить все? [Y/n/выборочно]
```

### Шаг 5: Применить изменения

После подтверждения пользователя:

1. Для каждого одобренного предложения:
   - Открыть SKILL.md существующего скилла
   - Добавить интеграцию в указанный шаг
   - Добавить в "Связанные скиллы" (если нет)
   - Обновить чек-лист (если есть)

2. Формат добавления в воркфлоу:
   ```markdown
   **Интеграция с {новый-скилл}:**
   Вызвать [/{новый-скилл}](/.claude/skills/{новый-скилл}/SKILL.md) для {описание}.
   ```

3. Формат добавления в чек-лист (если есть):
   ```markdown
   - [ ] **Шаг {N}:** Вызвал [/{новый-скилл}](/.claude/skills/{новый-скилл}/SKILL.md)
   ```

### Шаг 6: Результат

```
✅ Интеграция скилла {новый-скилл} завершена

Обновлено скиллов: {N}
- {скилл-1} — добавлен в Шаг {X}
- {скилл-2} — добавлен в Шаг {Y}
- {скилл-3} — добавлен в связанные скиллы

Пропущено: {M}
- {скилл-4} — пользователь отклонил

Следующий шаг: проверить изменения и протестировать интеграцию
```

## Критерии применимости

### По категориям

Новый скилл анализируется для **любой категории** существующих скиллов. Категория не является ограничением — важна логическая применимость.

**Приоритет анализа:**
1. Та же категория → проверить в первую очередь
2. Связанные категории → проверить во вторую очередь
3. Другие категории → проверить по объекту и воркфлоу

### По действиям

| Действие нового | Применим к действиям |
|-----------------|---------------------|
| `*-create` | — (создание не интегрируется) |
| `*-update` | `*-create` (после создания синхронизировать) |
| `*-validate` | `*-create`, `*-update` (после изменения проверить) |
| `*-delete` | — (удаление не интегрируется) |

### По объектам

| Объект нового | Применим к объектам |
|---------------|---------------------|
| `skill-*` | skill-*, context-* |
| `links-*` | все, работающие с .md |
| `context-*` | все, работающие с .md |
| `instruction-*` | instruction-*, context-*, links-* |

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Скилл не найден | Сообщить: "Скилл {имя} не найден" |
| Нет существующих скиллов | Сообщить: "Нет скиллов для анализа" |
| Нет применимых скиллов | Сообщить: "Интеграция не требуется" |
| Ошибка редактирования | Сообщить об ошибке, продолжить с остальными |
