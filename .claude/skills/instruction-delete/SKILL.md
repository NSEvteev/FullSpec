# Удаление инструкции

Команда для очистки ссылок и обновления статусов при удалении инструкции из `/.claude/instructions/`.

**Связанные скиллы:**
- [instruction-create](/.claude/skills/instruction-create/SKILL.md) — создание инструкций
- [instruction-update](/.claude/skills/instruction-update/SKILL.md) — проверка соответствия
- [context-delete](/.claude/skills/context-delete/SKILL.md) — очистка контекста (вызвать до удаления файлов)
- [context-update](/.claude/skills/context-update/SKILL.md) — распространение контекста (после изменений)

**Связанные инструкции:**
- [README.md](/.claude/instructions/README.md) — индекс инструкций, статусы заполнения
- [output-formats.md](/.claude/templates/output-formats.md) — форматы вывода (SSOT)

## Формат вызова

```
/instruction-delete <путь>
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь` | Путь к удалённой инструкции (относительно /.claude/instructions/) | — (обязательный) |

**Примеры:**
- `/instruction-delete src/api/deprecated.md` — очистить ссылки на удалённую инструкцию
- `/instruction-delete platform/old-feature.md` — очистить ссылки на удалённую инструкцию

## Когда использовать

| Событие | Действие |
|---------|----------|
| Удалена инструкция | Вызвать для очистки ссылок и сброса статусов |
| Переименована инструкция | Сначала `/instruction-delete старый-путь`, затем `/instruction-create новый-путь` |
| Объединены инструкции | Вызвать для удалённых инструкций |

## Правила

### Что удалять/обновлять

При удалении инструкции:

1. **Индекс инструкций** `README.md` — изменить статусы ✅ на ⬜ (строку НЕ удалять!)
2. **refactoring.md** — убрать ✓
3. **Связанные инструкции** — удалить из раздела "Связанные инструкции"
4. **Скиллы** — удалить ссылки или пометить `*(не создан)*`
5. **Frontmatter** — удалить из `related:` в других инструкциях

### Почему не удаляем из индекса

Индекс инструкций `README.md` содержит **полный перечень** всех инструкций проекта. При удалении инструкции:
- Строка остаётся в таблице
- Статусы "Создано" и "Заполнено" сбрасываются на ⬜
- Это позволяет видеть, какие инструкции ещё не реализованы

### Чего НЕ удалять

- Строки из индекса README.md — только сбрасывать статусы
- Упоминания в документации `/doc/` — отметить как устаревшие
- Ссылки с пометкой `*(не создан)*` — оставить

### Обязательный вывод напоминания

**Правило:** Раздел `⚠️ Проверьте вручную` выводится **ВСЕГДА** в итоговом отчёте, даже если автоматически обработаны все найденные ссылки.

Причина: документация в `/doc/` может содержать неявные упоминания, которые не находятся автоматическим поиском.

## Воркфлоу

### Шаг 1: Получить путь к инструкции

1. Из аргумента: `/instruction-delete src/api/deprecated.md`
2. Сформировать полный путь: `/.claude/instructions/src/api/deprecated.md`
3. Запомнить для поиска:
   - Имя файла: `deprecated.md`
   - Путь: `src/api/deprecated.md`
   - Полный путь: `/.claude/instructions/src/api/deprecated.md`

### Шаг 2: Проверить факт удаления

1. Проверить существование файла
2. Если файл существует:
   ```
   ⚠️ Инструкция src/api/deprecated.md ещё существует!

   Варианты:
   1. Удалить инструкцию и продолжить
   2. Отменить операцию

   Что выбрать?
   ```
3. Если выбрано "Удалить" — удалить файл
4. Если файл не существует — продолжить

### Шаг 3: Найти ссылки

**3.1. Поиск в инструкциях:**

```
Glob: /.claude/instructions/**/*.md
Grep: {путь} или {имя-файла}
```

**3.2. Поиск в индексе:**

```
Файл: /.claude/instructions/README.md
Grep: {имя-файла}
```

**3.3. Поиск в refactoring.md:**

```
Файл: /refactoring.md
Grep: {имя-файла}
```

**3.4. Поиск в скиллах:**

```
Glob: /.claude/skills/*/SKILL.md
Grep: {путь} или instructions/{путь}
```

**Паттерны поиска:**

| Паттерн | Тип ссылки | Действие |
|---------|------------|----------|
| `\[.*\]\(.*{путь}.*\)` | Markdown-ссылка | Удалить строку или пометить |
| `related:.*{путь}` | Frontmatter | Удалить из массива |
| `\| \[{имя}\].*✅` | Строка таблицы | Заменить ✅ на ⬜ |
| `{имя}.*←.*✓` | Статус в refactoring | Убрать ✓ |

   - Строку НЕ удалять

2. **refactoring.md:**
   - Убрать ✓ в конце строки

3. **Связанные инструкции:**

   - Удалить из массива `related:` во frontmatter

4. **Скиллы:**
   - Удалить ссылки на инструкцию
   - Или добавить пометку `*(не создан)*`

## Типы ссылок

### Строка таблицы в README.md

**Паттерн:**
```regex
\| \[{имя}\]\(.*\) \|.*\|.*\| ✅ \| ✅ \|
```

**Пример:**
```markdown
| [deprecated.md](./src/api/deprecated.md) | Устаревшее API | standard | ✅ | ✅ |
```

**Действие:** Заменить ✅ на ⬜ (строку НЕ удалять — инструкция остаётся в индексе)

### Связанные инструкции

**Паттерн:**
```regex
- \[.*\]\(.*{путь}.*\).*
```

**Пример:**
```markdown
- [deprecated](./deprecated.md) — устаревшие правила
```

**Действие:** Удалить строку

### Frontmatter related

**Паттерн:**
```regex
related:[\s\S]*?{путь}
```

**Пример:**
```yaml
related:
  - src/api/design.md
  - src/api/deprecated.md
```

**Действие:** Удалить элемент из массива

### Статус в refactoring.md

**Паттерн:**
```regex
{имя}.*←.*✓
```

**Пример:**
```
deprecated.md                    ← устаревшие правила ✓
```

**Действие:** Убрать ✓

### Ссылки в скиллах

**Паттерн:**
```regex
\[.*\]\(.*instructions/{путь}.*\)
```

**Пример:**
```markdown
- [instruction-delete](/.claude/skills/instruction-delete/SKILL.md) — удаление
```

**Действие:** Добавить `*(не создан)*` или удалить строку

## FAQ / Troubleshooting

### Почему инструкция не удаляется автоматически?

**Причины:**
1. Безопасность — инструкции критичны для проекта
2. Требуется ручное подтверждение
3. Нужно обновить связи в других файлах

После `/instruction-delete` удалите файл вручную:
```bash
rm /.claude/instructions/{path}.md
```

### Что происходит со статусом в README.md?

Статус инструкции сбрасывается на `⬜`, но строка НЕ удаляется. Это позволяет:
- Видеть историю инструкций
- Пересоздать инструкцию позже
- Отслеживать прогресс

### Как восстановить удалённую инструкцию?

```bash
git checkout -- /.claude/instructions/{path}.md
```

### Что если много ссылок на инструкцию?

Скилл покажет все ссылки и предложит:
- Применить все изменения
- Выборочно
- Экспорт списка

### Какие скиллы вызываются после удаления?

| Скилл | Зачем |
|-------|-------|
| `/context-update` | Обновление контекста в связанных файлах |
