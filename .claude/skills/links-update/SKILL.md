# Обновление ссылок

Команда для обновления ссылок в связанных документах при создании или изменении файла/папки репозитория.

**Связанные скиллы:**
- [links-create](/.claude/skills/links-create/SKILL.md) — создание ссылок в документе
- [links-delete](/.claude/skills/links-delete/SKILL.md) — пометка битых ссылок
- [context-update](/.claude/skills/context-update/SKILL.md) — распространение контекста (семантика)

## Формат вызова

```
/links-update [путь-к-файлу] [--old-name старое-имя]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь-к-файлу` | Файл или папка-источник | Последний созданный/изменённый |
| `--old-name` | Старое имя (при переименовании) | — |

**Примеры:**
- `/links-update .claude/skills/doc-create/SKILL.md` — обновить ссылки для нового скилла
- `/links-update .claude/skills/doc-create/SKILL.md --old-name file-create` — обновить после переименования
- `/links-update /src/utils/` — обновить ссылки на папку

## Когда использовать

| Событие | Действие |
|---------|----------|
| Создан новый скилл | Обновить индекс скиллов, связанные скиллы |
| Создан новый агент | Обновить индекс агентов |
| Создан/изменён файл | Обновить документы, которые должны ссылаться на него |
| Создана/изменена папка | Обновить документы, которые должны ссылаться на неё |
| Переименован файл/папка | Обновить все ссылки на старое имя |
| Удалён файл/папка | Пометить ссылки как битые или удалить |

## Воркфлоу

### Шаг 1: Определить источник

1. Получить путь к файлу/папке-источнику
2. Определить тип:
   - Скилл: `/.claude/skills/*/SKILL.md`
   - Агент: `/.claude/agents/*.md`
   - Инструкция: `/.claude/instructions/**/*.md`
   - Папка: путь заканчивается на `/`
   - Другой файл репозитория

### Шаг 2: Найти связанные документы

В зависимости от типа источника:

**Скилл:**
- [skills.md](/.claude/instructions/tools/skills.md) — индекс скиллов
- [agents.md](/.claude/instructions/tools/agents.md) — если назначен агенту
- Связанные скиллы: `{объект}-create`, `{объект}-update`, `{объект}-delete`

**Агент:**
- [agents.md](/.claude/instructions/tools/agents.md) — индекс агентов
- Скиллы, назначенные агенту

**Инструкция:**
- Документы, которые ссылаются на эту инструкцию
- Индексы в той же папке

**Другой файл/папка:**
- Все `.md` документы, которые упоминают этот файл/папку
- Поиск через скрипт [find_references.py](/.claude/scripts/find_references.py):
  ```bash
  python .claude/scripts/find_references.py /путь/к/файлу
  ```

**Fallback (если скрипт не существует):**
- Использовать Grep для поиска упоминаний:
  ```bash
  grep -r "имя-файла" --include="*.md" .
  ```
- Или вручную проверить:
  - Индексы в `/.claude/instructions/`
  - Документацию в `/doc/`
  - README файлы в папках проекта

### Шаг 3: Определить тип обновления

Для каждого связанного документа определить тип обновления:

| Тип | Описание | Действие |
|-----|----------|----------|
| `add` | Ссылки нет, нужно добавить | Добавить ссылку в нужное место |
| `update` | Ссылка есть, но устарела | Обновить текст или путь |
| `none` | Ссылка актуальна | Пропустить |

**Алгоритм определения типа:**

```
1. Поиск существующей ссылки:
   - Grep по документу: `\[.*\]\(.*{имя-источника}.*\)`
   - Если найдено → проверить актуальность
   - Если не найдено → тип = `add`

2. Проверка актуальности (если ссылка найдена):
   - Если указан --old-name и путь содержит старое имя → тип = `update`
   - Если путь ссылки не совпадает с актуальным → тип = `update`
   - Если описание в ссылке устарело → тип = `update`
   - Иначе → тип = `none`

3. Определение места для добавления (если тип = `add`):
   - Индекс (skills.md, agents.md): найти таблицу категории
   - Связанные скиллы: найти раздел "Связанные скиллы"
   - Агент: найти frontmatter `skills:`
   - Другой документ: найти контекст упоминания
```

### Шаг 4: Обновить документы

Для каждого документа с типом `add` или `update`:

1. Определить место для ссылки:
   - Индекс: добавить в таблицу категории
   - Связанные скиллы: добавить в раздел "Связанные скиллы"
   - Агент: добавить в `skills:` frontmatter

2. Выполнить обновление через `Edit`

3. Подтвердить изменение

### Шаг 5: Проверка по чек-листу

Перед завершением проверить выполнение всех пунктов чек-листа:

```
✅ Шаг 1: Определил источник
✅ Шаг 2: Нашёл связанные документы
✅ Шаг 3: Определил тип обновления
✅ Шаг 4: Обновил документы
```

### Шаг 6: Результат

Вывести отчёт:
```
✅ Ссылки обновлены для {источник}

Обновлено документов: {N}
- skills.md — добавлен в категорию documentation
- skill-update/SKILL.md — добавлена ссылка в "Связанные скиллы"
- skill-delete/SKILL.md — добавлена ссылка в "Связанные скиллы"

Без изменений: {M}
- agents.md — ссылка уже актуальна
```

## Типы связей

### Скилл → Индекс

**Файл:** [skills.md](/.claude/instructions/tools/skills.md)

**Место:** Таблица в категории скилла

**Формат:**
```markdown
| [skill-name](/.claude/skills/skill-name/SKILL.md) | Описание | Триггеры |
```

### Скилл → Связанные скиллы

**Файлы:** `/.claude/skills/{объект}-*/SKILL.md`

**Место:** Раздел "Связанные скиллы" в начале документа

**Формат:**
```markdown
**Связанные скиллы:**
- [skill-create](/.claude/skills/skill-create/SKILL.md) — создание
- [skill-update](/.claude/skills/skill-update/SKILL.md) — обновление
- [skill-delete](/.claude/skills/skill-delete/SKILL.md) — удаление
```

### Скилл → Агент

**Файл:** `/.claude/agents/{agent-name}.md`

**Место:** Frontmatter `skills:`

**Формат:**
```yaml
skills: [skill-create, skill-update, links-create]
```

### Агент → Индекс

**Файл:** [agents.md](/.claude/instructions/tools/agents.md)

**Место:** Таблица агентов

**Формат:**
```markdown
| [agent-name](/.claude/agents/agent-name.md) | Описание | Теги | Скиллы |
```

### Файл/папка → Документы

**Файлы:** Любые `.md` файлы, упоминающие этот файл/папку

**Место:** Контекст упоминания (текст, список, таблица)

**Формат:**
```markdown
[имя-файла](/путь/к/файлу)
[имя-папки/](/путь/к/папке/)
```

## Восстановление помеченных ссылок

При использовании [/links-delete](/.claude/skills/links-delete/SKILL.md) ссылки помечаются в формате:

```
текст *(ссылка удалена: /старый/путь)*
```

### Автоматический поиск пометок

При вызове `/links-update` с флагом `--old-name` автоматически ищутся пометки:

```
/links-update /src/new-module.ts --old-name /src/old-module.ts
```

**Паттерн поиска:**
```regex
(\S+)\s*\*\(ссылка удалена:\s*([^)]+)\)\*
```

### Восстановление

Для каждой найденной пометки предлагаются варианты:

```
📋 Найдены помеченные ссылки для восстановления

📄 /doc/architecture.md:45
   old-module *(ссылка удалена: /src/old-module.ts)*

   Варианты:
   [1] Восстановить как [old-module](/src/new-module.ts)
   [2] Удалить пометку (оставить текст: old-module)
   [3] Удалить строку целиком
   [4] Пропустить
```

### Формат восстановления

```diff
- old-module *(ссылка удалена: /src/old-module.ts)* — описание
+ [old-module](/src/new-module.ts) — описание
```

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Источник не найден | Сообщить: "Файл/папка {путь} не найден(а)" |
| Связанный документ не найден | Пропустить, добавить в отчёт |
| Не удалось определить место для ссылки | Спросить пользователя |
| Конфликт форматов | Показать оба варианта, спросить |
| Найдены пометки без --old-name | Показать пометки, спросить что делать |

## Примеры использования

### Пример 1: Создан новый скилл

**Контекст:** Только что создан скилл `doc-create` в категории `documentation`.

**Вызов:**
```
/links-update .claude/skills/doc-create/SKILL.md
```

**Вывод:**
```
✅ Ссылки обновлены для doc-create

Обновлено документов: 3
- skills.md — добавлен в категорию documentation
- doc-update/SKILL.md — добавлена ссылка в "Связанные скиллы"
- doc-delete/SKILL.md — добавлена ссылка в "Связанные скиллы"

Без изменений: 1
- agents.md — скилл не назначен агенту
```

### Пример 2: Переименован скилл

**Контекст:** Скилл `file-create` переименован в `doc-create`.

**Вызов:**
```
/links-update .claude/skills/doc-create/SKILL.md --old-name file-create
```

**Вывод:**
```
✅ Ссылки обновлены для doc-create (переименован из file-create)

Обновлено документов: 4
- skills.md — обновлено название и путь
- file-update/SKILL.md → doc-update/SKILL.md — обновлена ссылка
- file-delete/SKILL.md → doc-delete/SKILL.md — обновлена ссылка
- amy-santiago.md — обновлено в skills:
```

### Пример 3: Создан новый конфигурационный файл

**Контекст:** Создан файл `/config/redis.yaml`, который упоминается в документации.

**Вызов:**
```
/links-update /config/redis.yaml
```

**Вывод:**
```
✅ Ссылки обновлены для /config/redis.yaml

Обновлено документов: 2
- /doc/src/cache/README.md — добавлена ссылка [redis.yaml](/config/redis.yaml)
- /.claude/instructions/config/README.md — добавлена ссылка в список конфигов

Без изменений: 0
```

### Пример 4: Переименована папка

**Контекст:** Папка `/src/helpers/` переименована в `/src/utils/`.

**Вызов:**
```
/links-update /src/utils/ --old-name /src/helpers/
```

**Вывод:**
```
✅ Ссылки обновлены для /src/utils/ (переименовано из /src/helpers/)

Обновлено документов: 5
- /doc/src/README.md — обновлён путь
- /.claude/instructions/src/README.md — обновлён путь
- /src/api/handlers.ts — обновлён импорт
- /src/services/auth.ts — обновлён импорт
- /tests/unit/helpers.test.ts — обновлён путь
```

### Пример 5: Нет связанных документов

**Контекст:** Создан скилл `unique-tool-create`, нет других скиллов с тем же объектом.

**Вызов:**
```
/links-update .claude/skills/unique-tool-create/SKILL.md
```

**Вывод:**
```
✅ Ссылки обновлены для unique-tool-create

Обновлено документов: 1
- skills.md — добавлен в категорию tool-management

Связанные скиллы не найдены (unique-tool-update, unique-tool-delete не существуют)
```
