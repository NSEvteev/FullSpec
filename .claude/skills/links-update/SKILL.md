# Обновление ссылок

Команда для обновления ссылок в связанных документах при создании или изменении файла.

**Связанные скиллы:**
- [links-create](/.claude/skills/links-create/SKILL.md) — создание ссылок в документе
- [links-delete](/.claude/skills/links-delete/SKILL.md) — удаление битых ссылок *(не создан)*

## Формат вызова

```
/links-update [путь-к-файлу]
```

Если путь не указан — работает с последним созданным/изменённым .md файлом.

## Когда использовать

| Событие | Действие |
|---------|----------|
| Создан новый скилл | Обновить индекс скиллов, связанные скиллы |
| Создан новый агент | Обновить индекс агентов |
| Переименован файл | Обновить все ссылки на старое имя |
| Удалён файл | Пометить ссылки как битые или удалить |

## Воркфлоу

### Шаг 1: Определить источник

1. Получить путь к файлу-источнику
2. Определить тип файла:
   - Скилл: `/.claude/skills/*/SKILL.md`
   - Агент: `/.claude/agents/*.md`
   - Инструкция: `/.claude/instructions/**/*.md`
   - Другой документ

### Шаг 2: Найти связанные документы

В зависимости от типа источника:

**Скилл:**
- [skills.md](/.claude/instructions/tools/skills.md) — индекс скиллов
- [agents.md](/.claude/instructions/tools/agents.md) — если назначен агенту
- Связанные скиллы: `{объект}-create`, `{объект}-update`, `{объект}-delete`

**Агент:**
- [agents.md](/.claude/instructions/tools/agents.md) — индекс агентов
- Скиллы, назначенные агенту

**Инструкция:**
- Документы, которые ссылаются на эту инструкцию
- Индексы в той же папке

### Шаг 3: Определить тип обновления

Для каждого связанного документа определить:

| Тип | Описание | Действие |
|-----|----------|----------|
| `add` | Ссылки нет, нужно добавить | Добавить ссылку в нужное место |
| `update` | Ссылка есть, но устарела | Обновить текст или путь |
| `none` | Ссылка актуальна | Пропустить |

### Шаг 4: Обновить документы

Для каждого документа с типом `add` или `update`:

1. Определить место для ссылки:
   - Индекс: добавить в таблицу категории
   - Связанные скиллы: добавить в раздел "Связанные скиллы"
   - Агент: добавить в `skills:` frontmatter

2. Выполнить обновление через `Edit`

3. Подтвердить изменение

### Шаг 5: Результат

Вывести отчёт:
```
✅ Ссылки обновлены для {источник}

Обновлено документов: {N}
- skills.md — добавлен в категорию documentation
- skill-update/SKILL.md — добавлена ссылка в "Связанные скиллы"
- skill-delete/SKILL.md — добавлена ссылка в "Связанные скиллы"

Без изменений: {M}
- agents.md — ссылка уже актуальна
```

## Типы связей

### Скилл → Индекс

**Файл:** [skills.md](/.claude/instructions/tools/skills.md)

**Место:** Таблица в категории скилла

**Формат:**
```markdown
| [skill-name](/.claude/skills/skill-name/SKILL.md) | Описание | Триггеры |
```

### Скилл → Связанные скиллы

**Файлы:** `/.claude/skills/{объект}-*/SKILL.md`

**Место:** Раздел "Связанные скиллы" в начале документа

**Формат:**
```markdown
**Связанные скиллы:**
- [skill-create](/.claude/skills/skill-create/SKILL.md) — создание
- [skill-update](/.claude/skills/skill-update/SKILL.md) — обновление
- [skill-delete](/.claude/skills/skill-delete/SKILL.md) — удаление
```

### Скилл → Агент

**Файл:** `/.claude/agents/{agent-name}.md`

**Место:** Frontmatter `skills:`

**Формат:**
```yaml
skills: [skill-create, skill-update, links-create]
```

### Агент → Индекс

**Файл:** [agents.md](/.claude/instructions/tools/agents.md)

**Место:** Таблица агентов

**Формат:**
```markdown
| [agent-name](/.claude/agents/agent-name.md) | Описание | Теги | Скиллы |
```

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Источник не найден | Сообщить: "Файл {путь} не найден" |
| Связанный документ не найден | Пропустить, добавить в отчёт |
| Не удалось определить место для ссылки | Спросить пользователя |
| Конфликт форматов | Показать оба варианта, спросить |

## Примеры использования

### Пример 1: Создан новый скилл

**Контекст:** Только что создан скилл `doc-create` в категории `documentation`.

**Вызов:**
```
/links-update .claude/skills/doc-create/SKILL.md
```

**Вывод:**
```
✅ Ссылки обновлены для doc-create

Обновлено документов: 3
- skills.md — добавлен в категорию documentation
- doc-update/SKILL.md — добавлена ссылка в "Связанные скиллы"
- doc-delete/SKILL.md — добавлена ссылка в "Связанные скиллы"

Без изменений: 1
- agents.md — скилл не назначен агенту
```

### Пример 2: Переименован скилл

**Контекст:** Скилл `file-create` переименован в `doc-create`.

**Вызов:**
```
/links-update .claude/skills/doc-create/SKILL.md --old-name file-create
```

**Вывод:**
```
✅ Ссылки обновлены для doc-create (переименован из file-create)

Обновлено документов: 4
- skills.md — обновлено название и путь
- file-update/SKILL.md → doc-update/SKILL.md — обновлена ссылка
- file-delete/SKILL.md → doc-delete/SKILL.md — обновлена ссылка
- amy-santiago.md — обновлено в skills:
```

### Пример 3: Нет связанных документов

**Контекст:** Создан скилл `unique-tool-create`, нет других скиллов с тем же объектом.

**Вызов:**
```
/links-update .claude/skills/unique-tool-create/SKILL.md
```

**Вывод:**
```
✅ Ссылки обновлены для unique-tool-create

Обновлено документов: 1
- skills.md — добавлен в категорию tool-management

Связанные скиллы не найдены (unique-tool-update, unique-tool-delete не существуют)
```
