# Обработка кандидатов глоссария

Интерактивный просмотр и обработка терминов-кандидатов: добавить, отложить или удалить.

**Связанные скиллы:**
- [glossary-candidates](/.claude/skills/glossary-candidates/SKILL.md) — поиск кандидатов
- [glossary-link](/.claude/skills/glossary-link/SKILL.md) — расстановка ссылок

**Связанные инструкции:**
- [specs/glossary.md](/.claude/instructions/workflow/specs/glossary.md) — правила глоссария

## Формат вызова

```
/glossary-review [--all]
```

| Параметр | Описание |
|----------|----------|
| `--all` | Обработать все кандидаты без интерактива (добавить все) |

---

## Воркфлоу

### Шаг 1: Прочитать глоссарий

Загрузить `/specs/glossary.md`, найти секцию "Кандидаты".

### Шаг 2: Для каждого кандидата

Показать:
- Термин
- Источник (ссылка на документ)
- Контекст (цитата)
- Предложенное определение

Спросить действие.

### Шаг 3: Выполнить действие

| Действие | Результат |
|----------|-----------|
| Добавить | Переместить в соответствующую категорию |
| Отложить | Оставить в "Кандидаты", пометить "отложен" |
| Удалить | Удалить из "Кандидаты" |
| Изменить | Отредактировать определение, затем добавить |

### Шаг 4: Результат

```
✅ Обработка завершена

Добавлено: 3
- Access Token → Аутентификация
- Refresh Token → Аутентификация
- Circuit Breaker → Инфраструктура

Отложено: 1
- Token Rotation

Удалено: 1
- API (слишком общий)

Осталось кандидатов: 0
```

---

## Действия

### [1] Добавить

Переместить термин из "Кандидаты" в основную секцию:

```markdown
## Аутентификация

### Access Token
- **Определение:** Короткоживущий токен для доступа к API
- **Контекст:** auth сервис
- **Источник:** [ADR auth/002](/specs/services/auth/adr/002-jwt-tokens.md)
```

### [2] Отложить

Оставить в "Кандидаты" с пометкой:

```markdown
### Token Rotation
- **Источник:** [ADR auth/002]
- **Контекст:** "..."
- **Статус:** отложен (нужно уточнить)
```

### [3] Удалить

Удалить из секции "Кандидаты".

### [4] Изменить

Отредактировать определение или категорию, затем добавить.

## Примеры использования

### Пример 1: Интерактивный режим

```
/glossary-review

📋 Обработка кандидатов глоссария

Кандидатов: 3

─────────────────────────────────────
1/3: Access Token

Источник: [ADR auth/002](/specs/services/auth/adr/002-jwt-tokens.md)
Контекст: "Access Token выдаётся сроком на 1 час"
Определение: Короткоживущий токен для доступа к API
Категория: Аутентификация

[1] Добавить  [2] Отложить  [3] Удалить  [4] Изменить

> 1

✅ Добавлено: Access Token → Аутентификация

─────────────────────────────────────
2/3: Token Rotation
...
```

### Пример 2: Добавить все

```
/glossary-review --all

📋 Автоматическая обработка кандидатов

Кандидатов: 5

✅ Добавлено: 5 терминов
- Access Token → Аутентификация
- Refresh Token → Аутентификация
- Circuit Breaker → Инфраструктура
- Rate Limiting → Инфраструктура
- Session ID → Аутентификация

Следующий шаг: /glossary-link
```
