# Удаление ссылок

Команда для пометки битых ссылок в документах при удалении файлов или папок репозитория. Не удаляет контент, а помечает ссылки для последующего восстановления через links-update.

**Связанные скиллы:**
- [links-create](/.claude/skills/links-create/SKILL.md) — создание ссылок в документе
- [links-update](/.claude/skills/links-update/SKILL.md) — обновление и восстановление ссылок
- [context-delete](/.claude/skills/context-delete/SKILL.md) — очистка контекста (вызывать до удаления файла)

## Формат вызова

```
/links-delete <путь> [--dry-run] [--scan-all]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь` | Удалённый файл или папка | — (обязательный) |
| `--dry-run` | Показать изменения без применения | false |
| `--scan-all` | Найти все битые ссылки в проекте | false |

**Примеры:**
- `/links-delete /src/old-module.ts` — пометить ссылки на удалённый файл
- `/links-delete /src/legacy/` — пометить ссылки на удалённую папку
- `/links-delete --scan-all` — аудит всех битых ссылок
- `/links-delete /config/old.yaml --dry-run` — предпросмотр изменений

## Когда использовать

| Событие | Действие |
|---------|----------|
| Удалён файл/папка | Пометить все ссылки на него |
| Рефакторинг структуры | Массовая пометка перед перемещением |
| Аудит документации | `--scan-all` для поиска всех битых ссылок |
| Переименование | `/links-delete старый`, затем `/links-update новый` |

## Правила

### Принцип сохранения контента

**НЕ удаляем:**
- Строки целиком
- Ячейки таблиц
- Пункты списков

**Заменяем:**
- Ссылку `[текст](путь)` на пометку `текст *(ссылка удалена: путь)*`

### Зачем сохранять путь

Пометка включает исходный путь для:
- Понимания, что здесь было
- Восстановления ссылки через links-update
- Поиска всех пометок grep-ом

### Что помечать

| Тип ссылки | Пример | Действие |
|------------|--------|----------|
| Markdown-ссылка | `[file](path)` | Пометить |
| Ссылка в списке | `- [file](path)` | Пометить |
| Ссылка в таблице | `\| [file](path) \|` | Пометить |
| Ссылка в тексте | `см. [file](path)` | Пометить |
| Уже помеченная | `*(ссылка удалена)*` | Пропустить |

## Воркфлоу

### Шаг 1: Получить удалённый путь

1. Из аргумента: `/links-delete /src/old-module.ts`
2. Проверить, что файл/папка действительно не существует
3. Если существует — предупредить:
   ```
   ⚠️ Файл /src/old-module.ts существует!

   Варианты:
   1. Продолжить (пометить ссылки как битые)
   2. Отменить

   Что выбрать?
   ```

### Шаг 2: Найти ссылки

**Поиск по паттерну:**
```regex
\[([^\]]+)\]\(([^)]*{путь}[^)]*)\)
```

**Где искать:**
1. Все `.md` файлы в проекте
2. Приоритет: `/.claude/`, `/doc/`, корень проекта

**Алгоритм:**
```
1. Glob: **/*.md
2. Grep по каждому файлу: `\]\(.*{путь}.*\)`
3. Для каждого совпадения:
   - Извлечь текст ссылки
   - Извлечь полный путь
   - Определить контекст (список, таблица, текст)
4. Сформировать список изменений
```

### Шаг 3: Сформировать предложения

**Формат вывода:**

```

Найдено ссылок: {N} в {M} документах

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### Шаг 4: Применить пометки

После подтверждения:

1. Для каждой ссылки выполнить замену:
   ```
   [текст](путь) → текст *(ссылка удалена: путь)*
   ```

2. Использовать Edit tool для каждого файла

3. Подтвердить изменение

Вызвать [/context-update](/.claude/skills/context-update/SKILL.md) для изменённых документов:

```

## Формат пометки

### Структура

```
{текст ссылки} *(ссылка удалена: {исходный путь})*
```

### Примеры преобразования

**Простая ссылка:**
```diff
- [config.yaml](/config/config.yaml)
+ config.yaml *(ссылка удалена: /config/config.yaml)*
```

**Ссылка в списке:**
```diff
- - [API Design](/doc/api-design.md) — принципы проектирования
+ - API Design *(ссылка удалена: /doc/api-design.md)* — принципы проектирования
```

**Ссылка в тексте:**
```diff
- Подробнее см. [документацию](/doc/guide.md).
+ Подробнее см. документацию *(ссылка удалена: /doc/guide.md)*.
```

**Ссылка в таблице:**
```diff
- | [utils.ts](/src/utils.ts) | Утилиты |
+ | utils.ts *(ссылка удалена: /src/utils.ts)* | Утилиты |
```

### Regex для пометки

**Поиск ссылки:**
```regex
\[([^\]]+)\]\(([^)]+)\)
```

**Замена:**
```
$1 *(ссылка удалена: $2)*
```

## Интеграция с links-update

### Поиск пометок

**links-update** ищет паттерн:
```regex
(\S+)\s*\*\(ссылка удалена:\s*([^)]+)\)\*
```

### Восстановление ссылки

При вызове `/links-update` с новым путём:

```
📋 Найдены помеченные ссылки для восстановления

📄 /doc/architecture.md:45
   old-module *(ссылка удалена: /src/old-module.ts)*

   Варианты:
   1. Восстановить как [old-module](/src/new-module.ts)
   2. Удалить пометку (оставить только текст)
   3. Удалить строку целиком
   4. Пропустить
```

### Автоматическое восстановление

Если `/links-update` вызван с `--old-name`:

```
/links-update /src/new-module.ts --old-name /src/old-module.ts
```

Автоматически находит пометки с `old-module.ts` и предлагает заменить на новый путь.

## FAQ / Troubleshooting

### Почему скилл не удаляет строки целиком?

**Принцип сохранения контента:** links-delete только помечает ссылки, но не удаляет контент. Это позволяет:
- Сохранить контекст (что здесь было)
- Восстановить ссылку через links-update
- Найти все пометки grep-ом

### Как восстановить помеченную ссылку?

Если файл был перемещён/переименован:
```
/links-update новый-путь --old-name старый-путь
```

Скилл найдёт пометки и предложит восстановить.

### Что делать, если файл ещё существует?

Скилл предупредит и спросит подтверждение. Это защита от случайной пометки актуальных ссылок.

Если вы уверены — выберите "Продолжить".

### Как найти все помеченные ссылки в проекте?

```bash
grep -r "ссылка удалена:" --include="*.md" .
```

### Как удалить пометки навсегда?

Если файл удалён навсегда и пометки больше не нужны — удалите их вручную или используйте sed:

```bash
# Предпросмотр
grep -n "ссылка удалена:" файл.md

# Редактирование вручную
```

### Чем --scan-all отличается от обычного вызова?

| Режим | Что делает |
|-------|------------|
| `/links-delete путь` | Помечает ссылки на конкретный удалённый файл |
| `/links-delete --scan-all` | Находит ВСЕ битые ссылки в проекте |

Используйте `--scan-all` для аудита документации.

### Когда вызывать context-delete, а когда links-delete?

| Скилл | Когда | Что делает |
|-------|-------|------------|
| `/context-delete` | **До** удаления файла | Очищает семантический контекст |
| `/links-delete` | **После** удаления файла | Помечает битые ссылки |

**Рекомендуемый порядок:**
1. `/context-delete файл.md`
2. `rm файл.md`
3. `/links-delete файл.md`
