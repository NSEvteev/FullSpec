## Формат вызова

```
/doc-delete <путь-к-удалённому-файлу> [--no-issue] [--dry-run]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь` | Удалённый файл проекта (кроме исключённых) | — (обязательный) |
| `--no-issue` | Не создавать GitHub Issue | false |
| `--dry-run` | Показать изменения без применения | false |

**Исключённые пути:** `/doc/`, `/.claude/`, `/.git/`, `/tests/`, самоописывающиеся файлы (.md, .rst, .txt)

**Примеры:**
- `/doc-delete /src/auth/backend/old-handlers.ts`
- `/doc-delete /config/old-settings.yaml --no-issue`
- `/doc-delete /platform/docker/old-compose.yml`

## Правила

### Маппинг путей

**SSOT:** [doc-rules.md](/.claude/instructions/doc/rules.md#маппинг-путей)

**Правило:** Документация зеркалирует структуру проекта в `/doc/`.

```
/{any-path}/{file}.{ext} → /doc/{any-path}/{file}.md
```

| Удалённый файл | Документация |
|----------------|--------------|
| `/src/auth/backend/old-handlers.ts` | `/doc/src/auth/backend/old-handlers.md` |
| `/config/old-settings.yaml` | `/doc/config/old-settings.md` |
| `/platform/docker/old-compose.yml` | `/doc/platform/docker/old-compose.md` |

**Исключения (не документируются):** `/doc/`, `/.claude/`, `/.git/`, `/tests/`, `*.md`, `*.rst`, `*.txt`

**Связанная инструкция:** [doc/structure.md](/.claude/instructions/doc/structure.md)

### Что помечается

**Правило:** Документация НЕ удаляется автоматически — только помечается для ревью.

| Объект | Действие |
|--------|----------|
| Файл документации | Добавить пометку в начало файла |
| Ссылки на документацию | Найти и включить в Issue |
| Ссылки на исходный файл | Вызвать [/links-delete](/.claude/skills/links-delete/SKILL.md) |

**Причины:**
- Документация может быть актуальна для исторической справки
- Возможен рефакторинг (файл перемещён, а не удалён)
- Требуется ручное решение о судьбе документации

### Когда использовать

| Событие | Действие |
|---------|----------|
| Удалён файл из проекта | Пометить документацию, создать Issue |
| Удалена папка из проекта | Пометить все документы в папке |
| Рефакторинг (файл перемещён) | [/doc-delete](/.claude/skills/doc-delete/SKILL.md) + [/doc-create](/.claude/skills/doc-create/SKILL.md) |

**Исключённые пути:** Файлы в `/doc/`, `/.claude/`, `/.git/`, `/tests/` и самоописывающиеся (.md, .rst, .txt)

## Воркфлоу

### Шаг 1: Получить путь

1. Из аргумента: `/doc-delete /src/auth/backend/old-handlers.ts`
2. Или спросить: "Какой файл был удалён?"
3. Определить путь документации по маппингу

### Шаг 2: Проверить документацию

1. Проверить существование файла документации
2. Если документация не существует — сообщить:

```
ℹ️ Документация не найдена: /doc/src/auth/backend/old-handlers.md

Файл уже удалён или не был задокументирован.
Вызов /links-delete для исходного файла...
```

### Шаг 3: Пометить документацию

Добавить в начало файла документации (см. [Формат пометки](#формат-пометки)):

```markdown
> ⚠️ **ТРЕБУЕТ РЕВЬЮ:** Исходный файл `/src/auth/backend/old-handlers.ts` был удалён.
>
> - [ ] Удалить этот файл документации
> - [ ] Или обновить ссылку на новый файл
>
> См. Issue: #{issue-number}
```

### Шаг 4: Найти ссылки на документацию

Найти все ссылки на помечаемую документацию:

```
📋 Ссылки на /doc/src/auth/backend/old-handlers.md

Найдено: 3

1. /doc/src/auth/README.md:45
2. /.claude/instructions/src/README.md:23
3. /doc/src/auth/backend/api.md:12
```

### Шаг 5: Создать GitHub Issue

**Если не указан `--no-issue`:**

```bash
gh issue create \
  --label "docs" \
  --title "[DOCS] Ревью документации после удаления old-handlers.ts" \
  --body "..."
```

См. [GitHub Issue](#github-issue) для формата.

### Шаг 6: Вызвать links-delete

Пометить ссылки на удалённый исходный файл:

```
/links-delete /src/auth/backend/old-handlers.ts
```

### Шаг 7: Проверка по чек-листу

Перед завершением проверить выполнение всех пунктов:

```
✅ Шаг 1: Получил путь
✅ Шаг 2: Проверил документацию
✅ Шаг 3: Пометил документацию
✅ Шаг 4: Нашёл ссылки
✅ Шаг 5: Создал GitHub Issue
✅ Шаг 6: Вызвал /links-delete
```

### Шаг 8: Результат

Вывести полный отчёт:

```
✅ Документация помечена для ревью

Удалённый файл: /src/auth/backend/old-handlers.ts
Документация: /doc/src/auth/backend/old-handlers.md

Выполнено:
- Добавлена пометка в документацию
- Создан GitHub Issue #123
- Вызван /links-delete

GitHub Issue: https://github.com/user/repo/issues/123

Следующие шаги:
- Решить судьбу документации (удалить или обновить)
- Обновить или удалить ссылки
```

## Обработка ошибок

> **SSOT:** [error-handling.md](/.claude/instructions/skills/errors.md)

| Ошибка | Действие |
|--------|----------|
| Путь не указан | Спросить: "Какой файл был удалён?" |
| Путь в `/doc/` | Сообщить: "Нельзя пометить документацию" |
| Путь в `/.claude/` | Сообщить: "/.claude/ уже содержит документацию" |
| Путь в `/.git/` | Сообщить: "Служебные файлы Git не документируются" |
| Путь .md/.rst/.txt | Сообщить: "Файл уже является документацией" |
| Документация не существует | Сообщить, вызвать только `/links-delete` |
| Ошибка создания Issue | Сообщить, продолжить без Issue |
| Ошибка записи | Сообщить, откатить изменения |

**Откат при ошибке:**

```bash
# Откатить изменения в документации
git checkout -- /doc/src/{service}/{path}.md
```

## FAQ / Troubleshooting

### Почему документация не удаляется автоматически?

**Причины:**
1. Документация может быть актуальна для исторической справки
2. Файл мог быть перемещён, а не удалён
3. Требуется ручное решение о судьбе документации

Скилл только помечает документацию и создаёт Issue для отслеживания.

### Когда использовать --no-issue?

| Ситуация | Флаг |
|----------|------|
| Обычное удаление | Без флага (создаётся Issue) |
| Временное удаление | `--no-issue` |
| Локальная очистка | `--no-issue` |
| Рефакторинг (файл будет восстановлен) | `--no-issue` |

### Что делать при переименовании файла?

1. `/doc-delete старый-путь --no-issue`
2. `/doc-create новый-путь`
3. Удалить старую документацию вручную

Или используйте `/links-update` с `--old-name` для обновления ссылок.

### Как удалить документацию полностью?

После пометки:
1. Откройте Issue
2. Примите решение
3. Удалите файл: `rm /doc/{path}.md`
4. Закройте Issue

### Что если документация не существует?

Скилл сообщит, что документация не найдена, и вызовет только `/links-delete` для исходного файла.

### Как найти все помеченные документы?

```bash
grep -r "ТРЕБУЕТ РЕВЬЮ" --include="*.md" /doc/
```

## GitHub Issue

### Формат заголовка

```
[DOCS] Ревью документации после удаления {filename}
```

### Метки

- `docs` — документация

```markdown
## Описание

Файл `{исходный-путь}` был удалён.

- [ ] Удалить или обновить документацию
- [ ] Обновить ссылки в связанных документах:
  {список-ссылок}

## Связанные файлы

- Удалённый файл: `{исходный-путь}`
