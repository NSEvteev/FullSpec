## Формат вызова

```
/issue-create [--service <сервис>] [--type <тип>] [--priority <приоритет>] [--dry-run] "<заголовок>"
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--service` | Сервис (auth, notify, pay, users, gw, infra, docs) | Определяется из контекста |
| `--type` | Тип (feature, bug, enhancement) | feature |
| `--priority` | Приоритет (high, medium, low) | — |
| `--dry-run` | Показать план без создания | false |
| `--auto` | Автоматический режим (без подтверждения) | false |
| `заголовок` | Описание задачи | — (обязательный) |

**Примеры:**
- `/issue-create --service auth "Добавить OAuth авторизацию"`
- `/issue-create --service notify --type bug "Email не отправляется"`
- `/issue-create --service infra --priority high "Настроить CI"`
- `/issue-create --service pay --dry-run "Добавить Stripe"`
- `/issue-create --service auth --auto "Добавить OAuth"` — без подтверждения

### Режим --auto

В автоматическом режиме:
- Сервис определяется автоматически из контекста
- Issue создаётся без предпросмотра и подтверждения
- Описание минимальное (только обязательные поля)

```
/issue-create --service auth --auto "Добавить OAuth"

[Создание без подтверждения]

✅ Issue создан: #45
URL: https://github.com/user/repo/issues/45
```

**Режим --dry-run:**
```
/issue-create --service auth --type feature --dry-run "Добавить OAuth"

📋 Предварительный просмотр (--dry-run)

Будет создан Issue:
- Заголовок: [AUTH] Добавить OAuth
- Метки: service:auth, feature
- Тело: {сгенерированное описание}

Команда gh:
gh issue create --title "[AUTH] Добавить OAuth" --label "service:auth,feature" --body "..."

ℹ️ Issue НЕ создан (--dry-run)
```

## Правила

### Формат заголовка

**Правило:** Заголовок Issue должен иметь формат `[PREFIX] {описание}`.

```
[{PREFIX}] {заголовок}
```

**Примеры:**
- `[AUTH] Добавить OAuth авторизацию`
- `[NOTIFY] Исправить отправку email`
- `[INFRA] Настроить CI pipeline`

**Связанная инструкция:** [issues.md](/.claude/instructions/git/issues.md)

### Префиксы сервисов

| Сервис | Префикс | Label |
|--------|---------|-------|
| auth | `[AUTH]` | service:auth |
| notify | `[NOTIFY]` | service:notify |
| pay | `[PAY]` | service:payment |
| users | `[USERS]` | service:users |
| gw | `[GW]` | service:gateway |
| infra | `[INFRA]` | infra |
| docs | `[DOCS]` | docs |

### Правила меток

**Правило:** Каждый Issue должен иметь метки:

| Категория | Метки | Обязательно |
|-----------|-------|-------------|
| Сервис | `service:auth`, `service:notify`, `infra`, `docs` | Да |
| Тип | `feature`, `bug`, `enhancement` | Да |
| Приоритет | `priority:high`, `priority:medium`, `priority:low` | Нет |

### Шаблон тела Issue

**Правило:** Тело Issue должно содержать разделы:

```markdown
## Описание

{Что нужно сделать}

## Критерии готовности

- [ ] {Чек-лист выполнения}

## Связанные файлы

- {Ссылки на код/документацию}
```

## Воркфлоу

### Шаг 1: Определить сервис

1. Из параметра `--service`
2. Или из текущего контекста (открытый файл в `/src/{service}/`)
3. Или спросить у пользователя:

```
Для какого сервиса?
[1] auth
[2] notify
[3] pay
[4] users
[5] gw
[6] infra
[7] docs
```

### Шаг 2: Сформировать заголовок

1. Получить префикс по сервису
2. Сформировать: `[{PREFIX}] {заголовок}`

Пример: `[AUTH] Добавить OAuth авторизацию`

### Шаг 3: Определить метки

1. Метка сервиса: `service:auth`
2. Метка типа: `feature`, `bug`, `enhancement`
3. Метка приоритета (если указан): `priority:high`

### Шаг 4: Сформировать тело Issue

1. Использовать [Шаблон тела Issue](#шаблон-тела-issue)
2. Заполнить описание из контекста
3. Добавить связанные файлы (если известны)

### Шаг 5: Показать предпросмотр

```
📋 Создание Issue

Заголовок: [AUTH] Добавить OAuth авторизацию
Метки: service:auth, feature

Тело:
───────────────────────────────────────
## Описание

Добавить OAuth авторизацию через Google и GitHub.

## Критерии готовности

- [ ] Реализована авторизация через Google
- [ ] Реализована авторизация через GitHub
- [ ] Написаны тесты

## Связанные файлы

- /src/auth/backend/handlers.ts
───────────────────────────────────────

Создать? [Y/n/редактировать]
```

### Шаг 6: Создать Issue

```bash
gh issue create \
  --label "service:auth,feature" \
  --title "[AUTH] Добавить OAuth авторизацию" \
  --body "..."
```

### Шаг 7: Проверка по чек-листу

Перед завершением проверить выполнение всех пунктов:

```

✅ Шаг 5: Показал предпросмотр
```

## FAQ / Troubleshooting

### Как изменить сервис после создания Issue?

**Проблема:** Создан Issue с неправильным префиксом/сервисом.

**Решение:**
```
/issue-update {номер} --title "[НОВЫЙ_ПРЕФИКС] {заголовок}" --remove-label "service:старый" --add-label "service:новый"
```

### Можно ли создать Issue без указания сервиса?

**Ответ:** Нет, сервис обязателен для правильной маршрутизации задач.

**Что делать:**
1. Выбрать наиболее подходящий сервис из списка
2. Для общих задач использовать `infra` или `docs`
3. Для неопределённых — спросить в команде

### Что делать, если gh не установлен?

**Симптом:** `command not found: gh`

**Решение:**
1. Установить GitHub CLI: https://cli.github.com/
2. Авторизоваться: `gh auth login`
3. Проверить: `gh auth status`

### Как добавить несколько меток сразу при создании?

**Ответ:** При создании через `/issue-create` метки определяются автоматически по параметрам.

**Для дополнительных меток:**
```
/issue-update {номер} --add-label "метка1" --add-label "метка2"
```

### Критерии готовности не отображаются — что делать?

**Причина:** При создании не был заполнен раздел "Критерии готовности".

**Решение:**
```
/issue-update {номер} --body
```
Затем добавить раздел в формате:
```markdown
## Критерии готовности

- [ ] Пункт 1
- [ ] Пункт 2
```
