# Создание инструкции

Команда для создания новой инструкции в `/.claude/instructions/`.

**Связанные скиллы:**
- [instruction-update](/.claude/skills/instruction-update/SKILL.md) — проверка файлов на соответствие
- [instruction-delete](/.claude/skills/instruction-delete/SKILL.md) — удаление *(планируется)*
- [links-update](/.claude/skills/links-update/SKILL.md) — синхронизация ссылок
- [context-update](/.claude/skills/context-update/SKILL.md) — распространение контекста

## Формат вызова

```
/instruction-create <путь>
```

**Примеры:**
- `/instruction-create src/api/design.md`
- `/instruction-create git/commits.md`
- `/instruction-create platform/observability/metrics.md`

## Правила

### Расположение инструкций

Инструкции находятся в `/.claude/instructions/` и зеркалируют структуру проекта:

| Папка проекта | Инструкции |
|---------------|------------|
| `/src/` | `/.claude/instructions/src/` |
| `/platform/` | `/.claude/instructions/platform/` |
| `/tests/` | `/.claude/instructions/tests/` |
| `/doc/` | `/.claude/instructions/doc/` |
| `/shared/` | `/.claude/instructions/shared/` |
| `/config/` | `/.claude/instructions/config/` |

**Внутренние инструкции:**
- `/.claude/instructions/git/` — Git workflow
- `/.claude/instructions/tools/` — агенты и скиллы

### Формат названия

- Только латиница, цифры, дефис
- Расширение `.md`
- Примеры: `design.md`, `error-handling.md`, `feature-flags.md`

### Обновление статусов

При создании инструкции **обязательно** отметить её как созданную:
- В [README.md](/.claude/instructions/README.md) — значок ✓
- В [refactoring.md](/refactoring.md) — значок ✓

## Воркфлоу

### Шаг 1: Получить путь

1. Из аргумента: `/instruction-create src/api/design.md`
2. Или спросить: "Какую инструкцию создать? (путь относительно /.claude/instructions/)"

**Валидация:**
- Путь начинается с одной из папок: `src/`, `platform/`, `tests/`, `doc/`, `shared/`, `config/`, `git/`, `tools/`
- Файл имеет расширение `.md`
- Путь соответствует структуре из [refactoring.md](/refactoring.md)

### Шаг 2: Проверка существования

```bash
ls .claude/instructions/{путь}
```

Если файл существует — сообщить и спросить:
- Перезаписать?
- Открыть для редактирования?
- Отменить?

### Шаг 3: Сгенерировать содержимое

**3.1. Найти описание в refactoring.md:**

1. Открыть [refactoring.md](/refactoring.md)
2. Найти строку с именем файла инструкции в разделе `/.claude/instructions/`
3. Извлечь описание из комментария после `←`

**Алгоритм поиска:**
1. Grep по refactoring.md: `{имя-файла}\s+←`
2. Извлечь текст после `←` до конца строки
3. Если не найдено — спросить описание у пользователя

**Пример поиска:**
```
/.claude/instructions/
  /src/
    /api/
      design.md                  ← URL, методы, статусы
```
Описание: "URL, методы, статусы"

**3.2. Определить тип инструкции:**

1. Открыть [README.md](/.claude/instructions/README.md) — единый индекс всех инструкций
2. Найти строку с инструкцией в таблице соответствующего раздела
3. Прочитать значение столбца "Тип"

**Типы инструкций:**

| Тип | Назначение | Когда использовать |
|-----|------------|-------------------|
| `standard` | Требования, КАК делать | Стандарты качества, применимы к любому проекту |
| `project` | Специфика, ЧТО есть | Описывает конкретный проект, структуру, конфигурации |

**Пример поиска типа:**
```
| [design.md](./src/api/design.md) | URL naming... | standard | ⬜ | ⬜ |
```
Тип: `standard`

**3.3. Прочитать контекст:**

1. В том же [README.md](/.claude/instructions/README.md) найти раздел папки назначения
2. Понять общий контекст и стиль инструкций в этой категории
3. Посмотреть существующие инструкции как примеры

**3.4. Сгенерировать содержимое:**

| Раздел | Источник |
|--------|----------|
| Заголовок | Имя файла → человекочитаемое название |
| Описание | Комментарий из refactoring.md |
| Правила | Анализ предметной области + существующие паттерны |
| Примеры | Конкретные примеры кода/конфигов из проекта |
| Связанные | Другие инструкции в той же папке |

**Структура инструкции:**
- Заголовок
- Краткое описание (из комментария в refactoring.md)
- Основные правила (2-5 правил)
- Примеры (минимум 1-2)
- Связанные инструкции

### Шаг 4: Создать файл

```bash
# Если нужна подпапка
mkdir -p .claude/instructions/{папка}
```

Записать содержимое в файл.

### Шаг 5: Ревью

Провести ревью созданной инструкции:

**Формат вывода:**
```
📋 Ревью инструкции {путь}

✅ Сильные стороны:
- {что хорошо в инструкции}

💡 Предложения по улучшению:
1. {улучшение 1}
2. {улучшение 2}

Применить улучшения? [Y/n/выборочно]
```

**Что проверять:**
- Соответствие шаблону (frontmatter, разделы)
- Полнота правил (достаточно ли для понимания?)
- Качество примеров (конкретные, из проекта)
- Связанные инструкции (все ли указаны?)

**Ревью блокирует!** Нельзя переходить к следующему шагу, пока не применены улучшения или пользователь явно отказался.

### Шаг 6: Обновить статусы

1. **Индекс инструкций** [README.md](/.claude/instructions/README.md):
   - Найти соответствующую таблицу (раздел папки)
   - Если инструкция уже в таблице → изменить статус с ⬜ на ✅
   - Если инструкции нет в таблице → добавить новую строку со ссылкой

   **Формат строки в таблице:**
   ```markdown
   | [design.md](./src/api/design.md) | URL naming, HTTP методы, статус-коды | ✅ |
   ```

2. **Файл рефакторинга** [refactoring.md](/refactoring.md):
   - Найти файл в структуре `/.claude/`
   - Добавить ✓ в конце строки

**Формат отметки в refactoring.md:**
```
design.md                         ← URL, методы, статусы ✓
```

> ⚠️ **Важно:** Ссылка в README.md обязательна! Формат: `[имя-файла.md](./путь/к/файлу.md)`

### Шаг 7: Синхронизация ссылок

Вызвать [/links-update](/.claude/skills/links-update/SKILL.md) для обновления ссылок в связанных документах:

```
/links-update .claude/instructions/{путь}
```

**Что делает links-update:**
1. Находит документы, которые должны ссылаться на новую инструкцию
2. Добавляет ссылки в разделы "Связанные инструкции"
3. Обновляет индексы папок

### Шаг 8: Обновить контекст

Вызвать [/context-update](/.claude/skills/context-update/SKILL.md) для распространения контекста:

```
/context-update .claude/instructions/{путь}
```

**Что делает context-update:**
1. Находит связанные документы по графу зависимостей
2. Обновляет их контекст с учётом новой инструкции
3. Синхронизирует описания и ссылки

### Шаг 9: Проверка по чек-листу

Перед завершением проверить выполнение всех пунктов чек-листа:

```
✅ Шаг 1: Получил путь
✅ Шаг 2: Проверил существование
✅ Шаг 3: Сгенерировал содержимое
✅ Шаг 4: Создал файл
✅ Шаг 5: Провёл ревью
✅ Шаг 6: Обновил статусы
✅ Шаг 7: Вызвал links-update
✅ Шаг 8: Вызвал context-update
```

### Шаг 10: Результат

```
✅ Инструкция создана

Файл: /.claude/instructions/{путь}
Описание: {краткое описание}

Статусы обновлены:
- /.claude/instructions/README.md
- /refactoring.md

Ссылки синхронизированы:
- {N} документов обновлено через /links-update

Контекст обновлён:
- {M} документов обновлено через /context-update
```

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Неверный путь | Показать список допустимых папок |
| Файл уже существует | Спросить: перезаписать / редактировать / отменить |
| Папка не существует | Создать папку автоматически |
| Не найдено описание в refactoring.md | Спросить описание у пользователя |

## Примеры использования

### Пример 1: Создание инструкции API design

**Вызов:**
```
/instruction-create src/api/design.md
```

**Результат:**
```
✅ Инструкция создана

Файл: /.claude/instructions/src/api/design.md
Описание: URL, методы, статусы, частичное обновление

Статусы обновлены:
- /.claude/instructions/README.md
- /refactoring.md
```

### Пример 2: Создание инструкции для Git commits

**Вызов:**
```
/instruction-create git/commits.md
```

**Результат:**
```
✅ Инструкция создана

Файл: /.claude/instructions/git/commits.md
Описание: Коммиты (conventional, changelog)

Статусы обновлены:
- /.claude/instructions/README.md
- /refactoring.md
```

### Пример 3: Инструкция уже существует

**Вызов:**
```
/instruction-create src/api/design.md
```

**Ответ:**
```
Файл /.claude/instructions/src/api/design.md уже существует.

Варианты:
1. Перезаписать (содержимое будет потеряно)
2. Открыть для редактирования
3. Отменить

Что выбрать?
```

### Пример 4: Неверный путь

**Вызов:**
```
/instruction-create random/folder/file.md
```

**Ответ:**
```
❌ Неверный путь: random/folder/file.md

Путь должен начинаться с одной из допустимых папок:
- src/       — инструкции для исходного кода
- platform/  — инструкции для инфраструктуры
- tests/     — инструкции для тестов
- doc/       — инструкции для документации
- shared/    — инструкции для общего кода
- config/    — инструкции для конфигураций
- git/       — инструкции для Git workflow
- tools/     — инструкции для агентов и скиллов

Пример: /instruction-create src/api/design.md
```

### Пример 5: Описание не найдено в refactoring.md

**Вызов:**
```
/instruction-create src/api/new-feature.md
```

**Ответ:**
```
⚠️ Описание для src/api/new-feature.md не найдено в refactoring.md

Введите краткое описание инструкции:
> _
```

**После ввода пользователем:**
```
Описание: Правила разработки новых API-эндпоинтов

Создать инструкцию? [Y/n]
```
