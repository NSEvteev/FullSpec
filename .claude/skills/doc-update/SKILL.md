# Обновление документации

Команда для синхронизации документации с изменениями в исходном коде `/src/`.

**Связанная инструкция:** [/.claude/instructions/src/documentation.md](/.claude/instructions/src/documentation.md)

**Связанные скиллы:**
- [doc-create](/.claude/skills/doc-create/SKILL.md) — создание документации
- [doc-delete](/.claude/skills/doc-delete/SKILL.md) — пометка документации при удалении

## Формат вызова

```
/doc-update <путь-к-файлу-в-src> [--diff] [--auto]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь` | Файл в /src/ | — (обязательный) |
| `--diff` | Показать изменения без применения | false |
| `--auto` | Применить изменения автоматически | false |

**Примеры:**
- `/doc-update /src/auth/backend/handlers.ts`
- `/doc-update /src/auth/backend/handlers.ts --diff`

---

## Когда использовать

| Событие | Действие |
|---------|----------|
| Добавлена новая функция | Добавить в раздел API |
| Изменена сигнатура | Обновить сигнатуру в документации |
| Удалена функция | Удалить из документации |
| Изменены зависимости | Обновить раздел Зависимости |
| Переименован файл | Вызвать `/doc-delete` + `/doc-create` |

## Воркфлоу

### Шаг 1: Получить путь к файлу

1. Из аргумента: `/doc-update /src/auth/backend/handlers.ts`
2. Проверить, что файл существует
3. Определить путь документации

### Шаг 2: Проверить документацию

1. Если документация не существует:
   ```
   ⚠️ Документация не найдена: /doc/src/auth/backend/handlers.md

   Варианты:
   1. Создать документацию (/doc-create)
   2. Отменить
   ```

### Шаг 3: Сравнить API

1. Извлечь API из исходного файла (функции, классы)
2. Извлечь API из документации
3. Найти различия:
   - Новые функции (в коде есть, в документации нет)
   - Удалённые функции (в документации есть, в коде нет)
   - Изменённые сигнатуры

### Шаг 4: Сформировать предложения

```
📋 Обновление документации /doc/src/auth/backend/handlers.md

Найдено изменений: {N}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

➕ Новые функции (добавить в документацию):

1. validateToken(token: string): boolean
   - Добавить в раздел API

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔄 Изменённые сигнатуры:

1. login
   Было:   login(email: string, password: string): AuthResponse
   Стало:  login(credentials: LoginCredentials): Promise<AuthResponse>

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

➖ Удалённые функции (удалить из документации):

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```

### Шаг 5: Применить изменения

После подтверждения:
1. Добавить новые функции в раздел API
2. Обновить изменённые сигнатуры
3. Удалить документацию удалённых функций

### Шаг 6: Проверить зависимости

1. Сравнить импорты в коде с разделом "Зависимости"
2. Предложить обновления если есть различия

## Типы изменений

### Добавление функции

**В документацию добавляется:**
```markdown
### {functionName}

{Описание — заполнить вручную}

**Сигнатура:**
\`\`\`{language}
{signature}
\`\`\`
```

### Изменение сигнатуры

**Обновляется блок сигнатуры:**
```diff
- \`\`\`typescript
- function login(email: string, password: string): AuthResponse
- \`\`\`
+ \`\`\`typescript
+ function login(credentials: LoginCredentials): Promise<AuthResponse>
+ \`\`\`
```

### Удаление функции

**Удаляется весь блок функции из документации.**

## Примеры

### Пример 1: Добавлена новая функция

**Контекст:** В `handlers.ts` добавлена функция `validateToken`.

**Вызов:**
```
/doc-update /src/auth/backend/handlers.ts
```

**Вывод:**
```
📋 Обновление документации /doc/src/auth/backend/handlers.md

Найдено изменений: 1

➕ Новые функции:
1. validateToken(token: string): boolean

Применить? [Y/n]
> Y

✅ Документация обновлена

Добавлено функций: 1
```

### Пример 2: Режим diff

**Вызов:**
```
/doc-update /src/auth/backend/handlers.ts --diff
```

**Вывод:**
```
📋 Предпросмотр изменений (--diff)

Файл: /doc/src/auth/backend/handlers.md

```diff
 ## API

 ### login
+
+### validateToken
+
+{Описание}
+
+**Сигнатура:**
+\`\`\`typescript
+function validateToken(token: string): boolean
+\`\`\`
```

Изменения НЕ применены (--diff)
```

### Пример 3: Документация не существует

**Вызов:**
```
/doc-update /src/auth/backend/new-handlers.ts
```

**Вывод:**
```
⚠️ Документация не найдена: /doc/src/auth/backend/new-handlers.md

Варианты:
[1] Создать документацию (/doc-create)
[2] Отменить

> 1

Запуск /doc-create /src/auth/backend/new-handlers.ts...
```

### Пример 4: Нет изменений

**Вызов:**
```
/doc-update /src/auth/backend/handlers.ts
```

**Вывод:**
```
✅ Документация актуальна

Файл: /doc/src/auth/backend/handlers.md

Изменений не обнаружено.
API в коде соответствует документации.
```
