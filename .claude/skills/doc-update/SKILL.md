## Формат вызова

```
/doc-update <путь-к-файлу> [--diff] [--auto]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь` | Любой файл проекта (кроме исключённых) | — (обязательный) |
| `--diff` | Показать изменения без применения | false |
| `--auto` | Применить изменения автоматически | false |

**Исключённые пути:** `/doc/`, `/.claude/`, `/.git/`, `/tests/`, самоописывающиеся файлы (.md, .rst, .txt)

**Примеры:**
- `/doc-update /src/auth/backend/handlers.ts`
- `/doc-update /config/settings.yaml --diff`
- `/doc-update /platform/docker/Dockerfile --auto`

## Правила

### Маппинг путей

**SSOT:** [doc-rules.md](/.claude/templates/doc-rules.md#маппинг-путей)

**Правило:** Документация зеркалирует структуру проекта в `/doc/`.

```
/{any-path}/{file}.{ext} → /doc/{any-path}/{file}.md
```

| Исходный файл | Документация |
|---------------|--------------|
| `/src/auth/backend/handlers.ts` | `/doc/src/auth/backend/handlers.md` |
| `/config/settings.yaml` | `/doc/config/settings.md` |
| `/platform/docker/compose.yml` | `/doc/platform/docker/compose.md` |

**Исключения (не документируются):** `/doc/`, `/.claude/`, `/.git/`, `/tests/`, `*.md`, `*.rst`, `*.txt`

**Связанная инструкция:** [documentation.md](/.claude/instructions/tools/documentation.md)

### Типы изменений

| Тип изменения | Действие в документации |
|---------------|------------------------|
| Добавлена функция | Добавить в раздел API |
| Изменена сигнатура | Обновить блок сигнатуры |
| Удалена функция | Удалить из раздела API |
| Изменены импорты | Обновить раздел Зависимости |

**Добавление функции:**
```markdown
### {functionName}

{Описание — заполнить вручную}

**Сигнатура:**
\`\`\`{language}
{signature}
\`\`\`
```

**Изменение сигнатуры:**
```diff
- function login(email: string, password: string): AuthResponse
+ function login(credentials: LoginCredentials): Promise<AuthResponse>
```

**Удаление функции:** удаляется весь блок функции.

### Когда использовать

| Событие | Действие |
|---------|----------|
| Добавлена новая функция | Добавить в раздел API |
| Изменена сигнатура | Обновить сигнатуру в документации |
| Удалена функция | Удалить из документации |
| Изменены зависимости | Обновить раздел Зависимости |
| Переименован файл | Вызвать [/doc-delete](/.claude/skills/doc-delete/SKILL.md) + [/doc-create](/.claude/skills/doc-create/SKILL.md) |

## Воркфлоу

### Шаг 1: Получить путь к файлу

1. Из аргумента: `/doc-update /config/settings.yaml`
2. Или спросить: "Какой файл обновить?"
3. Валидация (блокирующая):
   - Файл существует
   - Путь НЕ в `/doc/`, `/.claude/`, `/.git/`, `/tests/`
   - Файл НЕ самоописывающийся (.md, .rst, .txt)
   - Файл не является директорией

### Шаг 2: Проверить документацию

1. Определить путь документации по маппингу
2. Если документация не существует — спросить:

```
⚠️ Документация не найдена: /doc/src/auth/backend/handlers.md

Варианты:
[1] Создать документацию (/doc-create)
[2] Отменить

Что выбрать?
```

### Шаг 3: Сравнить API

1. Извлечь API из исходного файла (функции, классы)
2. Извлечь API из документации
3. Найти различия:
   - Новые функции (в коде есть, в документации нет)
   - Удалённые функции (в документации есть, в коде нет)
   - Изменённые сигнатуры

### Шаг 4: Сформировать предложения

```

Найдено изменений: {N}

➕ Новые функции (добавить в документацию):

🔄 Изменённые сигнатуры:

1. login
   Было:   login(email: string, password: string): AuthResponse
   Стало:  login(credentials: LoginCredentials): Promise<AuthResponse>

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### Шаг 5: Применить изменения

После подтверждения:
1. Добавить новые функции в раздел API
2. Обновить изменённые сигнатуры
3. Удалить документацию удалённых функций

### Шаг 6: Проверить зависимости

1. Сравнить импорты в коде с разделом "Зависимости"
2. Предложить обновления если есть различия

### Шаг 7: Синхронизация ссылок

**Что обновляется:**
- README.md сервиса — если изменился API
- Связанные документы — если упоминают изменённые функции

## Обработка ошибок

> **SSOT:** [error-handling.md](/.claude/templates/error-handling.md)

| Ошибка | Действие |
|--------|----------|
| Файл не существует | Сообщить: "Файл {путь} не найден" |
| Файл в `/doc/` | Сообщить: "Нельзя документировать документацию" |
| Файл в `/.claude/` | Сообщить: "/.claude/ уже содержит документацию" |
| Файл в `/.git/` | Сообщить: "Служебные файлы Git не документируются" |
| Файл .md/.rst/.txt | Сообщить: "Файл уже является документацией" |
| Документация не существует | Предложить: создать (/doc-create) или отменить |
| Нет изменений | Сообщить: "Документация актуальна" |
| Не удалось извлечь API | Показать предупреждение, предложить ручное обновление |
| Ошибка записи | Сообщить, откатить изменения |

**Откат при ошибке:**

```bash
# Откатить изменения в документации
git checkout -- /doc/src/{service}/{path}.md
```

## FAQ / Troubleshooting

### Когда использовать doc-update, а когда doc-create?

| Ситуация | Скилл |
|----------|-------|
| Файл новый, документации нет | `/doc-create` |
| Документация есть, код изменился | `/doc-update` |
| Файл переименован/перемещён | `/doc-delete` + `/doc-create` |

### Что делать, если документация не найдена?

Скилл предложит создать документацию через `/doc-create`. Выберите:
1. **Создать** — вызовет `/doc-create`
2. **Отменить** — прервёт операцию

### Как применить изменения выборочно?

При запросе подтверждения выберите "выборочно" и укажите номера изменений:

```
Применить изменения? [Y/n/выборочно]
> выборочно

Введите номера через запятую: 1, 3
```

### Что означает --diff и --auto?

| Флаг | Действие |
|------|----------|
| `--diff` | Показать изменения без применения (предпросмотр) |
| `--auto` | Применить все изменения автоматически |

### Почему не обнаружены изменения?

**Возможные причины:**
- API в коде соответствует документации
- Изменения касаются только тела функций (не сигнатуры)
- Функция приватная и не документируется

### Как откатить изменения?

```bash
git checkout -- /doc/{path}.md
```
