# Очистка контекста

Команда для очистки семантического контекста в связанных документах при удалении .md файла из проекта.

**Связанные скиллы:**
- [context-update](/.claude/skills/context-update/SKILL.md) — распространение контекста (обратная операция)
- [links-delete](/.claude/skills/links-delete/SKILL.md) — пометка битых ссылок (синтаксис)

## Формат вызова

```
/context-delete <файл.md>
```

| Параметр | Описание | Обязательный |
|----------|----------|--------------|
| `файл.md` | Путь к удаляемому файлу | Да |

**Важно:** Вызывать **до** физического удаления файла, чтобы можно было прочитать его содержимое для анализа контекста.

## Отличие от links-delete

| Аспект | links-delete | context-delete |
|--------|--------------|----------------|
| Цель | Пометить **битые ссылки** | Очистить **семантический контекст** |
| Что анализирует | Синтаксис `[text](path)` | Описания, упоминания, зависимости |
| Что меняет | Помечает ссылки как `*(не существует)*` | Предлагает удалить/переписать разделы |
| Решение | Автоматическое | Предложение пользователю |
| Когда | После удаления файла | До удаления файла |

**Типичный порядок:**
1. `/context-delete файл.md` — очистить семантический контекст
2. Удалить файл
3. `/links-delete файл.md` — пометить оставшиеся битые ссылки

## Когда использовать

| Событие | Действие |
|---------|----------|
| Удаление скилла | Убрать упоминания из связанных скиллов |
| Удаление инструкции | Обновить индексы и ссылающиеся документы |
| Удаление документации | Очистить контекст в родительских документах |
| Удаление раздела документа | Обновить ссылки на этот раздел |

## Правила

### Обязательный вывод напоминания

**Правило:** Раздел `⚠️ Проверьте вручную` выводится **ВСЕГДА** в итоговом отчёте, даже если автоматически обработаны все найденные ссылки.

Причина: документация в `/doc/` может содержать неявные упоминания, которые не находятся автоматическим поиском.

## Воркфлоу

### Шаг 1: Проверить существование файла

1. Проверить, существует ли файл:
   ```bash
   test -f "файл.md" && echo "exists" || echo "not found"
   ```
2. Если файл не существует — предупредить:
   ```
   ⚠️ Файл {путь} не найден.

   Варианты:
   1. Файл уже удалён — используйте /links-delete для пометки битых ссылок
   2. Путь указан неверно — проверьте путь
   ```

### Шаг 2: Сохранить контекст удаляемого файла

1. Прочитать содержимое файла
2. Извлечь ключевую информацию:
   - Название/заголовок
   - Описание (из frontmatter или первого параграфа)
   - Ключевые термины и понятия
   - Исходящие ссылки (на что ссылается)

**Результат:**
```
Контекст удаляемого файла: {путь}

Название: {заголовок}
Описание: {описание}
Ключевые термины: {термин1}, {термин2}, ...
Исходящие ссылки: {N} файлов
```

### Шаг 3: Найти входящие связи

1. Поиск прямых ссылок на файл:
   ```bash
   grep -r "имя-файла" --include="*.md" .
   ```

2. Использовать скрипт (если существует):
   ```bash
   python .claude/scripts/find_references.py файл.md --format json
   ```

3. Проверить типичные места:
   - Индексы: `skills.md`, `agents.md`, `README.md`
   - Связанные скиллы: `{объект}-create`, `{объект}-update`
   - Frontmatter `related:` в других файлах

**Результат:** Список файлов, ссылающихся на удаляемый

### Шаг 4: Анализ семантического контекста

Для каждого файла B, ссылающегося на удаляемый файл A:

1. Прочитать B
2. Найти все упоминания A:
   - Прямые ссылки `[текст](путь-к-A)`
   - Упоминания названия A в тексте
   - Описания, связанные с A
3. Классифицировать тип зависимости:

| Тип | Описание | Действие |
|-----|----------|----------|
| **Ссылка в списке** | A в списке связанных | Удалить строку |
| **Ссылка в таблице** | A в строке таблицы | Удалить строку или пометить |
| **Ссылка в тексте** | A упоминается в параграфе | Переписать параграф |
| **Зависимость** | B зависит от функционала A | Предупредить, требует ручной обработки |
| **Пример** | A используется как пример | Убрать пример или заменить |

### Шаг 5: Сформировать предложения

Для каждой зависимости сформировать:

1. **Для простых случаев (ссылка в списке/таблице):**
   ```
   Файл: skills.md
   Тип: Ссылка в таблице
   Предложение: Удалить строку из таблицы

   ```diff
   - | [context-delete](/.claude/skills/context-delete/SKILL.md) | Очистка контекста | `/context-delete` |
   ```
   ```

2. **Для сложных случаев (текст, зависимость):**
   ```
   Файл: context-update/SKILL.md
   Тип: Ссылка в разделе "Связанные скиллы"
   Предложение: Удалить из списка связанных

   ```diff
   - - [context-delete](/.claude/skills/context-delete/SKILL.md) — очистка контекста *(не создан)*
   ```

   ⚠️ Также: упоминание в разделе "Когда использовать" — требует ручной проверки
   ```

## Типы контекстных зависимостей

### Структурные

| Тип | Пример | Автоматизация |
|-----|--------|---------------|
| Индекс → Элемент | skills.md → skill-create | Удалить строку |
| Frontmatter related | `related: [file.md]` | Удалить из массива |
| Связанные скиллы | Раздел "Связанные скиллы" | Удалить строку |

### Семантические

| Тип | Пример | Автоматизация |
|-----|--------|---------------|
| Упоминание в тексте | "Используйте skill-create для..." | Ручная проверка |
| Пример использования | Код/команды с упоминанием | Ручная проверка |
| Зависимость | "После выполнения skill-create" | Ручная проверка |

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Файл не существует | Предложить использовать links-delete |
| Нет входящих ссылок | Сообщить: "Зависимостей не найдено, можно удалять" |
| Слишком много зависимостей (>20) | Показать summary, предложить фильтрацию |
| Пользователь отменил | Не применять изменения, вывести итог |
