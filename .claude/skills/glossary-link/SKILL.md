# Добавление ссылок на глоссарий

Автоматическая расстановка ссылок `[📖 Термин](/specs/glossary.md#термин)` в документах проекта.

**Связанные скиллы:**
- [glossary-candidates](/.claude/skills/glossary-candidates/SKILL.md) — поиск кандидатов
- [glossary-review](/.claude/skills/glossary-review/SKILL.md) — обработка кандидатов

**Связанные инструкции:**
- [specs/glossary.md](/.claude/instructions/specs/glossary.md) — правила глоссария

## Формат вызова

```
/glossary-link [path] [--dry-run]
```

| Параметр | Описание |
|----------|----------|
| `path` | Путь к файлу или папке (по умолчанию `/specs/`) |
| `--dry-run` | Показать изменения без применения |

---

## Правила

### Что заменять

- Первое вхождение термина в каждом разделе (после заголовка ##)
- Точное совпадение (с учётом регистра)

### Что НЕ заменять

| Исключение | Причина |
|------------|---------|
| Заголовки (`#`, `##`, `###`) | Ломает навигацию |
| Блоки кода (```) | Ломает синтаксис |
| Уже существующие ссылки | Избежать дублей |
| Повторные упоминания в разделе | Читаемость |
| Сам файл глоссария | Циклические ссылки |

### Формат ссылки

```markdown
[📖 Access Token](/specs/glossary.md#access-token)
```

- Эмодзи `📖` — визуально отличает от обычных ссылок
- Относительный путь от корня репозитория
- Якорь в lowercase с дефисами

## Воркфлоу

### Шаг 1: Загрузить глоссарий

Получить список всех терминов из `/specs/glossary.md`.

### Шаг 2: Найти файлы для обработки

```bash
find {path} -name "*.md" -not -name "glossary.md"
```

### Шаг 3: Для каждого файла

1. Прочитать содержимое
2. Разбить на разделы (по ##)
3. В каждом разделе найти первое вхождение термина
4. Заменить на ссылку (если не исключение)
5. Сохранить файл

### Шаг 4: Результат

```
✅ Ссылки на глоссарий добавлены

Обработано файлов: 12
Добавлено ссылок: 28

По файлам:
- /specs/discussions/001-auth.md: 5 ссылок
- /specs/impact/001-auth.md: 8 ссылок
- /specs/services/auth/adr/002-jwt.md: 15 ссылок
```

## Примеры использования

### Пример 1: Обработать всю папку /specs/

```
/glossary-link

📋 Добавление ссылок на глоссарий

Термины в глоссарии: 14
Файлов для обработки: 12

Обработка...

✅ Завершено

Добавлено ссылок: 28
Пропущено (уже есть): 5
Файлов изменено: 8
```

### Пример 2: Конкретный файл

```
/glossary-link /specs/services/auth/adr/002-jwt-tokens.md

📋 Добавление ссылок в файл

Найдено терминов: 6
- Access Token (строка 45)
- Refresh Token (строка 52)
- JWT (строка 23)
- Redis (строка 67)
- Session (строка 78)
- Rate Limiting (строка 89)

Добавить ссылки? [Y/n] Y

✅ Добавлено: 6 ссылок
```

### Пример 3: Dry-run

```
/glossary-link --dry-run

📋 Предварительный просмотр (--dry-run)

Будет изменено файлов: 8
Будет добавлено ссылок: 28

Детали:
- discussions/001-auth.md: +5 ссылок
  - Access Token (строка 34)
  - Refresh Token (строка 45)
  ...

ℹ️ Изменения НЕ применены (--dry-run)
```

### Пример 4: Результат в документе

**До:**
```markdown
## Аутентификация

Пользователь получает Access Token при логине. Access Token используется для запросов к API.
```

**После:**
```markdown
## Аутентификация

Пользователь получает [📖 Access Token](/specs/glossary.md#access-token) при логине. Access Token используется для запросов к API.
```

> Обратите внимание: только первое вхождение в разделе становится ссылкой.
