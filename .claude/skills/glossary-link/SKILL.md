# Добавление ссылок на глоссарий

Команда проходит по всем .md файлам и добавляет ссылки на термины из глоссария.

## Формат ссылки на глоссарий

```markdown
[📖 Термин](../../general_docs/glossary.md#термин)
```

**Эмодзи `📖`** — визуально отличает ссылки на глоссарий от обычных ссылок.

## Инструкции

### Шаг 1: Прочитать глоссарий

Прочитать `general_docs/glossary.md` и собрать список всех терминов из раздела "## Термины".

Для каждого термина определить:
- Точное написание (с учётом регистра)
- Якорь для ссылки (lowercase, дефисы вместо пробелов)

### Шаг 2: Найти все .md файлы

Использовать Glob для поиска всех `.md` файлов:

```
**/*.md
```

**Исключить:**
- `general_docs/glossary.md` — сам глоссарий
- `node_modules/`
- `.git/`

### Шаг 3: Обработать каждый файл

Для каждого .md файла:

#### 3.1: Найти вхождения терминов

Использовать Grep или Read для поиска терминов.

**Правила поиска:**
- Искать точные совпадения слов (не части слов)
- Учитывать различные формы: "Термин", "термин", "термины"
- Игнорировать термины в code blocks (``` ... ```)
- Игнорировать термины, которые уже являются ссылками

#### 3.2: Заменить на ссылки

**Заменять только первое вхождение** термина в каждом разделе (## или ###).

Формат замены:
```
Термин → [📖 Термин](путь/к/glossary.md#термин)
```

**Вычисление относительного пути:**
- Из `general_docs/architecture/doc.md` → `../glossary.md`
- Из `llm_instructions/file.md` → `../general_docs/glossary.md`
- Из `README.md` → `general_docs/glossary.md`

#### 3.3: Сохранить изменения

Использовать Edit tool для обновления файла.

### Шаг 4: Вывести результат

```
Обработано файлов: X
Добавлено ссылок: Y

Файлы с изменениями:
- general_docs/architecture/01_overview.md: 5 ссылок
- llm_instructions/instructions_general_docs.md: 3 ссылки
- README.md: 2 ссылки

✓ Ссылки на глоссарий добавлены
```

## Важные правила

1. **Не заменять в заголовках** — термины в `#`, `##`, `###` не трогать
2. **Не заменять в code blocks** — внутри ``` ... ``` не трогать
3. **Не заменять уже существующие ссылки** — `[текст](url)` не трогать
4. **Первое вхождение в разделе** — повторные упоминания не делать ссылками
5. **Относительные пути** — всегда использовать относительные пути к glossary.md

## Откат изменений

Если нужно откатить:
```bash
git checkout -- **/*.md
```
