# /links-validate

Проверка всех ссылок в markdown-файлах проекта на валидность. Находит битые ссылки, несуществующие файлы и папки.

## Триггеры

- `/links-validate` — проверить все ссылки
- `/links-validate <путь>` — проверить ссылки в конкретном файле/папке
- `/links-validate --fix` — предложить исправления

## Параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `<путь>` | Файл или папка для проверки | Весь проект |
| `--fix` | Предложить исправления | false |
| `--dry-run` | Показать без изменений | false |
| `--json` | JSON формат вывода | false |
| `--verbose` | Подробный вывод | false |

## Воркфлоу

### Шаг 0: Проверка входных данных

1. Если указан путь — проверить существование
2. Определить scope:
   - Файл → проверить один файл
   - Папка → проверить все .md файлы рекурсивно
   - Не указано → весь проект

### Шаг 1: Поиск markdown-файлов

1. Найти все .md файлы в scope:
   - `/.claude/**/*.md`
   - `/doc/**/*.md`
   - `README.md`, `CLAUDE.md`, `CONTRIBUTING.md`, `SECURITY.md`

2. Исключить:
   - `node_modules/`
   - `.git/`

### Шаг 2: Извлечение ссылок

Для каждого файла извлечь:

```regex
\[([^\]]+)\]\(([^)]+)\)
```

Типы ссылок:
- **Относительные:** `./file.md`, `../folder/file.md`
- **Абсолютные от корня:** `/path/to/file.md`
- **Якоря:** `#section`, `file.md#section`
- **Внешние:** `https://...` (пропускаем)

### Шаг 3: Валидация ссылок

Для каждой ссылки:

1. **Внешние (http/https)** — пропустить (не проверяем)
2. **Файловые:**
   - Разрешить путь относительно текущего файла
   - Проверить существование файла/папки
   - Если якорь — проверить наличие заголовка

### Шаг 4: Формирование отчёта

```
📋 Валидация ссылок

Проверено:
- Файлов: {N}
- Ссылок: {M}

Результат:
- ✅ Валидных: {K}
- ❌ Битых: {L}

{Если есть битые:}
Битые ссылки:

| Файл | Строка | Ссылка | Причина |
|------|--------|--------|---------|
| {файл} | {строка} | {ссылка} | {причина} |
```

### Шаг 5: Предложение исправлений (--fix)

Если `--fix`:

1. Для каждой битой ссылки:
   - Поиск похожих файлов (fuzzy match)
   - Проверка переименования (git history)

2. Вывести предложения:
   ```
   💡 Предложения по исправлению:

   1. {файл}:{строка}
      Было: {старая ссылка}
      Предложение: {новая ссылка}

   Применить исправления? [Y/n/выборочно]
   ```

3. При подтверждении — применить через Edit

### Шаг 6: Результат

**Успех (нет битых):**
```
✅ Все ссылки валидны

Проверено: {N} файлов, {M} ссылок
```

**Есть битые:**
```
⚠️ Найдено {L} битых ссылок

Используйте /links-validate --fix для исправления
```

## Примеры использования

### Проверка всего проекта

```
/links-validate
```

**Результат:**
```
📋 Валидация ссылок

Проверено:
- Файлов: 87
- Ссылок: 342

Результат:
- ✅ Валидных: 340
- ❌ Битых: 2

Битые ссылки:

| Файл | Строка | Ссылка | Причина |
|------|--------|--------|---------|
| /.claude/skills/old-skill/SKILL.md | 15 | [test](./test.md) | Файл не существует |
| /doc/api.md | 42 | [auth](../src/auth/README.md) | Файл не существует |
```

### Проверка конкретной папки

```
/links-validate /.claude/instructions/
```

### Исправление битых ссылок

```
/links-validate --fix
```

**Результат:**
```
💡 Предложения по исправлению:

1. /.claude/skills/old-skill/SKILL.md:15
   Было: [test](./test.md)
   Предложение: [test](./tests.md)

2. /doc/api.md:42
   Было: [auth](../src/auth/README.md)
   Предложение: [auth](/src/auth/README.md)

Применить исправления? [Y/n/выборочно]
```

## FAQ

### Проверяются ли внешние ссылки?

Нет. Внешние ссылки (http/https) пропускаются, так как требуют сетевых запросов и могут быть временно недоступны.

### Как проверить только один файл?

```
/links-validate path/to/file.md
```

### Что делать с битыми ссылками?

1. Использовать `--fix` для автоматических предложений
2. Или вручную исправить через Edit
3. Или удалить ссылку, если файл больше не нужен

## Автоматизация

Рекомендуется добавить в CI:

```yaml
# .github/workflows/links.yml
jobs:
  validate-links:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Validate links
        run: claude /links-validate --json
```
