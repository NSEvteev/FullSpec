# Тесты: health-check

## Smoke-тесты

### smoke-1: Базовый запуск

**Команда:** `/health-check`

**Ожидание:** Отчёт о состоянии

**Проверка:**
- [ ] Показана таблица компонентов
- [ ] Указан общий статус
- [ ] Перечислены проблемы (если есть)

---

### smoke-2: Быстрая проверка

**Команда:** `/health-check --quick`

**Ожидание:** Быстрый отчёт без тестов

**Проверка:**
- [ ] Время выполнения короткое
- [ ] Тесты не запускались

---

## Функциональные тесты

### func-1: Обнаружение отсутствующего SKILL.md

**Подготовка:**
1. Создать папку `/.claude/skills/broken-skill/`
2. НЕ создавать SKILL.md

**Команда:** `/health-check`

**Ожидание:** Обнаружена проблема

**Проверка:**
- [ ] Статус скиллов: ⚠️ или ❌
- [ ] В списке проблем: "SKILL.md отсутствует"

**Очистка:** Удалить папку

---

### func-2: Обнаружение незарегистрированного скилла

**Подготовка:**
1. Создать скилл вручную (не через /skill-create)
2. НЕ добавлять в skills.md

**Команда:** `/health-check`

**Ожидание:** Обнаружено отсутствие в индексе

**Проверка:**
- [ ] В списке проблем: "Скилл не в skills.md"

---

### func-3: Интеграция с links-validate

**Команда:** `/health-check --verbose`

**Ожидание:** Вызван links-validate

**Проверка:**
- [ ] В отчёте есть раздел "Ссылки"
- [ ] Показано количество проверенных ссылок

---

### func-4: JSON формат

**Команда:** `/health-check --json`

**Ожидание:** Валидный JSON

**Проверка:**
- [ ] Вывод — валидный JSON
- [ ] Содержит поля: status, components, issues

---

## Edge cases

### edge-1: Пустой проект

**Подготовка:** Удалить все скиллы и инструкции

**Команда:** `/health-check`

**Ожидание:** Корректная обработка

**Проверка:**
- [ ] Нет ошибок
- [ ] Статус: warning (нет скиллов)

---

### edge-2: Все проверки пройдены

**Подготовка:** Корректный проект

**Команда:** `/health-check`

**Ожидание:** Успех

**Проверка:**
- [ ] Статус: ✅
- [ ] Текст: "Все проверки пройдены"

---

### edge-3: Режим --fix

**Подготовка:** Проект с исправимой проблемой

**Команда:** `/health-check --fix --dry-run`

**Ожидание:** План исправлений

**Проверка:**
- [ ] Показан план исправлений
- [ ] Изменения НЕ применены (--dry-run)
