# Создание агента

Команда для создания нового агента по шаблону.

**Связанные скиллы:**
- [skill-create](/.claude/skills/skill-create/SKILL.md) — создание скиллов для агента
- [links-update](/.claude/skills/links-update/SKILL.md) — синхронизация ссылок

**Связанные инструкции:**
- [agents/README.md](/.claude/agents/README.md) — индекс агентов, теги
- [skills/README.md](/.claude/skills/README.md) — индекс скиллов, категории

**Utility-скиллы:**
- [input-validate](/.claude/skills/input-validate/SKILL.md) — валидация названия агента
- [environment-check](/.claude/skills/environment-check/SKILL.md) — проверка git

## Формат вызова

```
/agent-create [название] [--dry-run] [--auto]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `название` | Имя агента (латиница, дефисы) | Спросить |
| `--dry-run` | Показать план без создания | false |
| `--auto` | Автоматический режим без подтверждений | false |

---

## Правила

### Формат названия

```
{domain}-agent
```

- Только латиница, цифры, дефис
- Суффикс `-agent` опционален
- Примеры: `docs-agent`, `git-helper`, `code-reviewer`

### Расположение

```
/.claude/agents/{название}.md
```

### Связь с скиллами

- Теги агента соответствуют категориям скиллов
- При создании агента — выбираются скиллы из соответствующих категорий
- Скиллы указываются в `skills:` frontmatter агента

## Воркфлоу

### Шаг 0: Fail-fast проверки

| Проверка | При ошибке |
|----------|------------|
| Папка `/.claude/agents/` существует | Создать |
| Git status чистый | Предупредить |
| Агент с таким именем не существует | Спросить: перезаписать / отменить |

### Шаг 1: Получить название

1. Из аргумента: `/agent-create docs-agent`
2. Или спросить: "Как назвать агента?"
3. Валидация формата

### Шаг 2: Описание и теги

1. Спросить описание агента (1 строка)
2. Показать список доступных тегов из [agents.md](/.claude/agents/README.md)
3. Выбрать подходящие теги

**Доступные теги:**

| Тег | Категории скиллов |
|-----|-------------------|
| `skill-management` | skill-management |
| `agent-management` | agent-management |
| `instruction-management` | instruction-management |
| `documentation` | documentation |
| `git` | git |
| `code` | code |
| `testing` | testing |
| `infrastructure` | infrastructure |

### Шаг 3: Выбор скиллов

1. Для каждого тега — показать скиллы этой категории из [skills.md](/.claude/skills/README.md)
2. Предложить выбрать скиллы для агента
3. В режиме `--auto` — добавить все скиллы из выбранных категорий

### Шаг 4: Создать файл агента

```bash
# Создать файл
touch .claude/agents/{название}.md
```

Записать содержимое по шаблону.

### Шаг 5: Обновить индекс агентов

1. Открыть [agents.md](/.claude/agents/README.md)
2. Добавить строку в таблицу "Список агентов"

**Формат строки:**
```markdown
| [{название}](/.claude/agents/{название}.md) | {описание} | {теги} | {скиллы} |
```

### Шаг 6: Синхронизация

Вызвать [/links-update](/.claude/skills/links-update/SKILL.md):

```
/links-update .claude/agents/{название}.md
```

### Шаг 7: Результат

```
✅ Агент создан

Файл: /.claude/agents/{название}.md
Описание: {описание}
Теги: {список тегов}
Скиллы: {список скиллов}

Индекс обновлён: /.claude/agents/README.md

Следующие шаги:
- Дополнить инструкции агента в файле
- Создать дополнительные скиллы при необходимости (/skill-create)
```

## Шаблон агента

```markdown
---
name: {название}
description: {описание}
tags: [{тег1}, {тег2}]
skills:
  - {скилл1}
  - {скилл2}
---

# {Название агента}

{Описание агента — что он делает, когда использовать.}

## Когда использовать

- {сценарий 1}
- {сценарий 2}

## Возможности

- {возможность 1}
- {возможность 2}

## Пример использования

{Пример диалога или команды}
```

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Агент уже существует | Спросить: перезаписать / отменить |
| Невалидное название | Показать правила именования |
| Нет подходящих скиллов | Предложить создать через `/skill-create` |

**Откат при ошибке:**

```bash
# Удалить файл агента
rm -f .claude/agents/{название}.md

# Откатить изменения в индексе
git checkout -- .claude/agents/README.md
```

## FAQ

### Как добавить скилл к существующему агенту?

1. Откройте `/.claude/agents/{agent}.md`
2. Добавьте скилл в `skills:` frontmatter
3. Или используйте `/agent-update` (планируется)

### Как удалить агента?

```bash
rm /.claude/agents/{название}.md
# Откатить изменения в индексе
git checkout -- /.claude/agents/README.md
```

Или используйте `/agent-delete` (планируется).

### Какие теги выбрать?

Выбирайте теги, соответствующие задачам агента:
- Работа с документами → `documentation`
- Git операции → `git`
- Проверка кода → `code`, `testing`
- Инфраструктура → `infrastructure`
