# Поиск терминов-кандидатов

Анализ документа на наличие терминов, которые стоит добавить в глоссарий.

**Связанные скиллы:**
- [glossary-review](/.claude/skills/glossary-review/SKILL.md) — обработка кандидатов
- [glossary-link](/.claude/skills/glossary-link/SKILL.md) — расстановка ссылок

**Связанные инструкции:**
- [specs/glossary.md](/.claude/instructions/workflow/specs/glossary.md) — правила глоссария

## Формат вызова

```
/glossary-candidates <file> [--auto]
```

| Параметр | Описание |
|----------|----------|
| `file` | Путь к документу для анализа |
| `--auto` | Автоматически добавить в "Кандидаты" |

## Воркфлоу

### Шаг 1: Прочитать документ

Загрузить содержимое файла.

### Шаг 2: Прочитать глоссарий

Загрузить `/specs/glossary.md` — список существующих терминов.

### Шаг 3: Извлечь потенциальные термины

Искать:
- Существительные в кавычках: `"Access Token"`
- Термины с заглавных букв: `JWT`, `OAuth2`
- Определения: `X — это...`, `X означает...`
- Сокращения: `ADR`, `API`, `SSO`

### Шаг 4: Отфильтровать

Исключить:
- Общеизвестные термины (HTTP, JSON, API)
- Уже существующие в глоссарии
- Слишком общие слова

### Шаг 5: Сформировать список кандидатов

```
📋 Найдены кандидаты в глоссарий

Документ: /specs/services/auth/adr/002-jwt-tokens.md

Новые термины (5):

1. Access Token
   Контекст: "Access Token выдаётся на 1 час"
   Предложенное определение: Короткоживущий токен для доступа к API

2. Refresh Token
   Контекст: "Refresh Token хранится в Redis"
   Предложенное определение: Долгоживущий токен для обновления Access Token

3. Token Rotation
   Контекст: "При каждом refresh выполняется token rotation"
   Предложенное определение: Механизм выдачи нового refresh token при каждом обновлении

Уже в глоссарии (2):
- JWT
- Redis

Добавить в "Кандидаты"? [все / выборочно / пропустить]
```

### Шаг 6: Добавить в глоссарий

В секцию "Кандидаты" файла `/specs/glossary.md`:

```markdown
## Кандидаты

### Access Token
- **Источник:** [ADR auth/002](/specs/services/auth/adr/002-jwt-tokens.md)
- **Контекст:** "Access Token выдаётся на 1 час"
- **Статус:** ожидает рассмотрения
```

## Правила поиска

### Что считается термином

| Критерий | Пример |
|----------|--------|
| Специфичен для проекта | Access Token, Session ID |
| Может быть понят неоднозначно | Token (какой именно?) |
| Используется в нескольких документах | — |
| Имеет техническое значение | Circuit Breaker |

### Что НЕ является термином

| Исключение | Пример |
|------------|--------|
| Общеизвестные | HTTP, JSON, REST |
| Слишком общие | User, Request, Response |
| Имена файлов | `auth.py`, `config.yaml` |
| Команды | `git commit`, `npm install` |

## Примеры использования

### Пример 1: Анализ ADR

```
/glossary-candidates /specs/services/auth/adr/002-jwt-tokens.md

📋 Найдены кандидаты в глоссарий

Документ: /specs/services/auth/adr/002-jwt-tokens.md

Новые термины (3):
1. Access Token — Короткоживущий токен для доступа к API
2. Refresh Token — Долгоживущий токен для обновления
3. Token Rotation — Механизм обновления токенов

Добавить в "Кандидаты"? [все / выборочно / пропустить] все

✅ Добавлено в глоссарий: 3 кандидата
Файл: /specs/glossary.md

Следующий шаг: /glossary-review
```

### Пример 2: Автоматический режим

```
/glossary-candidates /specs/impact/001-auth.md --auto

✅ Автоматически добавлено: 2 кандидата
- Session ID
- Authentication Flow

Следующий шаг: /glossary-review
```
