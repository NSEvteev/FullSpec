# Создание ссылок

Команда для автоматического оформления ссылок на `.md` файлы в документе.

**Связанные скиллы:**
- [links-update](/.claude/skills/links-update/SKILL.md) — обновление ссылок в связанных документах
- [links-delete](/.claude/skills/links-delete/SKILL.md) — удаление битых ссылок *(не создан)*

## Формат вызова

```
/links-create [путь-к-файлу]
```

Если путь не указан — работает с текущим документом (последний редактируемый .md файл).

## Правила

### Что считается упоминанием .md файла

**Нужно оформить как ссылку:**
- Путь к файлу: `/.claude/instructions/tools/skills.md`
- Имя файла с расширением: `skills.md`, `SKILL.md`
- Путь в обратных кавычках: `` `/.claude/skills/foo/SKILL.md` ``

**Не трогать:**
- Уже оформленные ссылки: `[skills.md](/.claude/instructions/tools/skills.md)`
- Внутри блоков кода (``` ... ```)
- В URL и внешних ссылках
- В примерах вывода команд

### Формат ссылки

```markdown
[имя-файла](полный-путь)
```

**Примеры:**
- `skills.md` → `[skills.md](/.claude/instructions/tools/skills.md)`
- `/.claude/agents/amy.md` → `[amy.md](/.claude/agents/amy.md)`

## Воркфлоу

### Шаг 1: Получить документ

1. Из аргумента: `/links-create path/to/file.md`
2. Или использовать текущий документ
3. Прочитать содержимое файла

### Шаг 2: Найти упоминания

Найти все упоминания `.md` файлов, которые НЕ оформлены как ссылки:

**Паттерны поиска:**
- `[^[\]](/path/to/file.md)` — путь не внутри ссылки
- `[^[\]]file.md` — имя файла не внутри ссылки
- Исключить содержимое блоков кода

**Результат:** список `{текст, строка, позиция}`

### Шаг 3: Проверить файлы

Для каждого найденного упоминания:

1. Определить полный путь к файлу
2. Проверить существование файла через `Glob`
3. Если файл не найден — пометить как "не найден"

**Результат:** список `{текст, путь, существует: да/нет}`

### Шаг 4: Создать ссылки

Для каждого существующего файла:

1. Сформировать ссылку: `[имя](путь)`
2. Заменить упоминание на ссылку через `Edit`

**Не заменять:**
- Файлы, которые не найдены
- Упоминания внутри блоков кода

### Шаг 5: Результат

Вывести отчёт:
```
✅ Ссылки созданы в {файл}

Оформлено ссылок: {N}
- строка {X}: skills.md → [skills.md](/.claude/instructions/tools/skills.md)
- строка {Y}: agents.md → [agents.md](/.claude/instructions/tools/agents.md)

Не найдено файлов: {M}
- строка {Z}: unknown.md — файл не существует

Пропущено (внутри блоков кода): {K}
```

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Файл не указан и нет текущего | Спросить путь к файлу |
| Файл не существует | Сообщить: "Файл {путь} не найден" |
| Нет упоминаний .md | Сообщить: "Упоминания .md файлов не найдены" |
| Файл из упоминания не найден | Пропустить, добавить в отчёт |

## Примеры использования

### Пример 1: Оформление ссылок в новом документе

**Вызов:**
```
/links-create .claude/skills/foo-create/SKILL.md
```

**До:**
```markdown
1. Прочитать индекс: /.claude/instructions/tools/skills.md
2. Проверить агентов в agents.md
```

**После:**
```markdown
1. Прочитать индекс: [skills.md](/.claude/instructions/tools/skills.md)
2. Проверить агентов в [agents.md](/.claude/instructions/tools/agents.md)
```

**Вывод:**
```
✅ Ссылки созданы в .claude/skills/foo-create/SKILL.md

Оформлено ссылок: 2
- строка 1: skills.md → [skills.md](/.claude/instructions/tools/skills.md)
- строка 2: agents.md → [agents.md](/.claude/instructions/tools/agents.md)
```

### Пример 2: Документ с битыми ссылками

**Вызов:**
```
/links-create README.md
```

**Вывод:**
```
✅ Ссылки созданы в README.md

Оформлено ссылок: 3
- строка 5: CONTRIBUTING.md → [CONTRIBUTING.md](/CONTRIBUTING.md)
- строка 8: SECURITY.md → [SECURITY.md](/SECURITY.md)
- строка 12: CLAUDE.md → [CLAUDE.md](/CLAUDE.md)

Не найдено файлов: 1
- строка 15: old-docs.md — файл не существует
```

### Пример 3: Документ без упоминаний

**Вызов:**
```
/links-create notes.md
```

**Вывод:**
```
Упоминания .md файлов не найдены в notes.md
```
