## Формат вызова

```
/links-create [путь-к-файлу] [--dry-run]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь-к-файлу` | Файл для обработки | Последний редактируемый .md |
| `--dry-run` | Показать план без изменений | false |

Если путь не указан — работает с текущим документом (последний редактируемый .md файл).

**Режим --dry-run:**
```
/links-create README.md --dry-run

📋 Предварительный просмотр (--dry-run)

Найдено упоминаний для оформления: 5

1. строка 15: skills.md → [skills.md](/.claude/skills/README.md)
2. строка 23: /src/auth/ → [/src/auth/](/src/auth/)
3. строка 45: package.json → [package.json](/package.json)
4. строка 67: .gitignore → [.gitignore](/.gitignore)
5. строка 89: config/ → [config/](/config/)

ℹ️ Изменения НЕ применены (--dry-run)
```

## Правила

### Что считается упоминанием

**Нужно оформить как ссылку:**
- Путь к файлу: `/.claude/skills/README.md`, `/config/settings.json`
- Имя файла с расширением: `skills.md`, `package.json`, `.gitignore`
- Путь к папке: `/.claude/scripts/`, `.claude/skills/`
- Путь в обратных кавычках: `` `/.claude/skills/foo/SKILL.md` ``

**Условие:** Файл или папка должны существовать в репозитории.

**Не трогать:**
- Уже оформленные ссылки: `[skills.md](/.claude/skills/README.md)`
- Внутри блоков кода (``` ... ```)
- В URL и внешних ссылках
- В примерах вывода команд
- В командах bash (mkdir, rm, cd и т.д.)
- Файлы/папки, которых нет в репозитории

### Формат ссылки

**Файлы:**
```markdown
[имя-файла](полный-путь)
```

**Папки:**
```markdown
[имя-папки/](полный-путь/)
```

**Примеры:**
- `skills.md` → `[skills.md](/.claude/skills/README.md)`
- `/.claude/agents/amy.md` → `[amy.md](/.claude/agents/amy.md)`
- `/.claude/scripts/` → `[scripts/](/.claude/scripts/)`
- `package.json` → `[package.json](/package.json)`
- `.gitignore` → `[.gitignore](/.gitignore)`

## Воркфлоу

### Шаг 1: Получить документ

1. Из аргумента: `/links-create path/to/file.md`
2. Или использовать текущий документ
3. Прочитать содержимое файла

### Шаг 2: Найти упоминания

Найти все упоминания файлов и папок, которые НЕ оформлены как ссылки:

**Паттерны поиска (регулярные выражения):**

| Тип | Паттерн | Примеры |
|-----|---------|---------|
| Абсолютный путь к файлу | `/[\w\-./]+\.\w+` | `/config/settings.json`, `/.claude/skills/SKILL.md` |
| Относительный путь к файлу | `\.?/[\w\-./]+\.\w+` | `./src/index.ts`, `../config.json` |
| Имя файла с расширением | `[\w\-]+\.\w+` | `package.json`, `.gitignore`, `README.md` |
| Путь к папке | `/[\w\-./]+/` | `/.claude/scripts/`, `/src/api/` |

**Исключения (не искать внутри):**
- Блоки кода: между ` ``` ` и ` ``` `
- Уже оформленные ссылки: `[текст](путь)`
- Bash-команды: после `mkdir`, `rm`, `cd`, `ls`, `cp`, `mv`
- URL: начинается с `http://` или `https://`

**Алгоритм:**
1. Разбить документ на сегменты: текст / блок кода
2. В текстовых сегментах применить паттерны
3. Для каждого совпадения проверить, не внутри ссылки/команды
4. Собрать список: `{текст, строка, позиция, тип: файл|папка}`

**Результат:** список `{текст, строка, позиция, тип: файл|папка}`

### Шаг 3: Проверить файлы

Для каждого найденного упоминания:

1. Определить полный путь к файлу
2. Проверить существование файла через `Glob`
3. Если файл не найден — пометить как "не найден"

**Результат:** список `{текст, путь, существует: да/нет}`

### Шаг 4: Создать ссылки

Для каждого существующего файла:

1. Сформировать ссылку: `[имя](путь)`
2. Заменить упоминание на ссылку через `Edit`

**Не заменять:**
- Файлы, которые не найдены
- Упоминания внутри блоков кода

### Шаг 5: Проверка по чек-листу

Перед завершением проверить выполнение всех пунктов чек-листа:

```
✅ Шаг 1: Получил документ
✅ Шаг 2: Нашёл упоминания файлов/папок
✅ Шаг 3: Проверил существование
✅ Шаг 4: Создал ссылки
```

### Шаг 6: Результат

Вывести отчёт:
```
✅ Ссылки созданы в {файл}

Оформлено ссылок: {N}
- строка {X}: skills.md → [skills.md](/.claude/skills/README.md)
- строка {Y}: agents.md → [agents.md](/.claude/agents/README.md)

Не найдено файлов: {M}
- строка {Z}: unknown.md — файл не существует

Пропущено (внутри блоков кода): {K}
```

| Ошибка | Действие |
|--------|----------|
| Файл не указан и нет текущего | Спросить путь к файлу |
| Файл не существует | Сообщить: "Файл {путь} не найден" |

| Файл из упоминания не найден | Пропустить, добавить в отчёт |

## FAQ / Troubleshooting

### Почему ссылка не создалась?

**Возможные причины:**

| Причина | Решение |
|---------|---------|
| Файл не существует | Проверьте путь, создайте файл |
| Упоминание внутри блока кода | Это ожидаемое поведение — код не трогаем |
| Уже оформлена как ссылка | Скилл не дублирует существующие ссылки |
| Часть bash-команды | Команды типа `mkdir`, `rm` игнорируются |

### Как обработать файл с нестандартным расширением?

Скилл ищет стандартные паттерны (`.md`, `.ts`, `.json` и т.д.). Для нестандартных:

1. Укажите полный путь: `/path/to/file.custom`
2. Или используйте `--dry-run` чтобы увидеть, что находит скилл

### Можно ли создать ссылки на внешние файлы?

Нет. links-create работает только с файлами внутри репозитория. Для внешних ссылок используйте стандартный markdown вручную.

### Как отменить созданные ссылки?

```bash
git diff  # посмотреть изменения
git checkout -- файл.md  # отменить конкретный файл
```

### Что делать, если найдено слишком много упоминаний?

Используйте `--dry-run` для предпросмотра:

```
/links-create файл.md --dry-run
```

Затем примените выборочно через ручное редактирование.

## Примеры использования

### Пример 1: Оформление ссылок в новом документе

**Вызов:**
```
/links-create .claude/skills/foo-create/SKILL.md
```

**До:**
```markdown
1. Прочитать индекс: /.claude/skills/README.md
2. Проверить агентов в agents.md
```

**После:**
```markdown
1. Прочитать индекс: [skills.md](/.claude/skills/README.md)
2. Проверить агентов в [agents.md](/.claude/agents/README.md)
```

**Вывод:**
```
✅ Ссылки созданы в .claude/skills/foo-create/SKILL.md

Оформлено ссылок: 2
- строка 1: skills.md → [skills.md](/.claude/skills/README.md)
- строка 2: agents.md → [agents.md](/.claude/agents/README.md)
```

### Пример 2: Документ с несуществующими файлами

**Вызов:**
```
/links-create README.md
```

**Вывод:**
```
✅ Ссылки созданы в README.md

Оформлено ссылок: 3
- строка 5: CONTRIBUTING.md → [CONTRIBUTING.md](/CONTRIBUTING.md)
- строка 8: SECURITY.md → [SECURITY.md](/SECURITY.md)
- строка 12: CLAUDE.md → [CLAUDE.md](/CLAUDE.md)

Не найдено файлов: 1
- строка 15: old-docs.md — файл не существует
```

### Пример 3: Оформление ссылок на папки

**Вызов:**
```
/links-create .claude/skills/context-update/SKILL.md
```

**До:**
```markdown
Скрипты находятся в /.claude/scripts/
Конфигурации — в /config/
```

**После:**
```markdown
Скрипты находятся в [scripts/](/.claude/scripts/)
Конфигурации — в [config/](/config/)
```

**Вывод:**
```
✅ Ссылки созданы в .claude/skills/context-update/SKILL.md

Оформлено ссылок: 2
- строка 1: /.claude/scripts/ → [scripts/](/.claude/scripts/)
- строка 2: /config/ → [config/](/config/)
```

### Пример 4: Документ без упоминаний

**Вызов:**
```
/links-create notes.md
```

**Вывод:**
```
Упоминания файлов и папок не найдены в notes.md
```
