# /doc-reindex

Полная переиндексация документации: обновление всех ссылок, синхронизация README-файлов с реальной структурой, обновление индексов.

## Триггеры

- `/doc-reindex` — переиндексировать всю документацию
- `/doc-reindex <путь>` — переиндексировать конкретную папку
- `/doc-reindex --check` — только проверка без изменений

## Параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `<путь>` | Папка для переиндексации | Весь проект |
| `--check` | Только проверка | false |
| `--dry-run` | Показать план без изменений | false |
| `--json` | JSON формат вывода | false |
| `--verbose` | Подробный вывод | false |

## Воркфлоу

### Шаг 0: Определение scope

1. Если путь указан — работать с папкой
2. Иначе — весь проект:
   - `/.claude/instructions/`
   - `/.claude/skills/`
   - `/doc/`

### Шаг 1: Сканирование структуры

1. Построить дерево файлов:
   ```
   - Все .md файлы
   - Все папки с README.md
   - Индексные файлы (README.md, index.md)
   ```

2. Определить несоответствия:
   - README.md не соответствует содержимому папки
   - Отсутствующие файлы в индексах
   - Лишние записи в индексах

### Шаг 2: Валидация ссылок

1. Вызвать `/links-validate {scope}`
2. Собрать список битых ссылок

### Шаг 3: Анализ README-файлов

Для каждого README.md:

1. Извлечь список файлов из README
2. Сравнить с реальным содержимым папки
3. Найти:
   - Отсутствующие в README (новые файлы)
   - Отсутствующие на диске (удалённые файлы)
   - Неправильные пути

### Шаг 4: Формирование плана

```
📋 План переиндексации

Валидация ссылок:
- ✅ Валидных: {N}
- ❌ Битых: {M}

README файлы:
| README | Добавить | Удалить | Исправить |
|--------|----------|---------|-----------|
| /.claude/instructions/README.md | 2 | 1 | 0 |
| /doc/README.md | 0 | 0 | 3 |

Всего изменений: {K}
```

### Шаг 5: Подтверждение

```
⚠️ Будет изменено {K} файлов

Просмотреть детали? [Y/n]
```

При `Y` — показать детали каждого изменения.

### Шаг 6: Применение изменений

1. Для каждого README с изменениями:
   - Добавить новые записи
   - Удалить устаревшие записи
   - Исправить пути

2. Для битых ссылок:
   - Вызвать `/links-validate --fix`
   - Или пометить для ручного исправления

### Шаг 7: Результат

**Успех:**
```
✅ Переиндексация завершена

Статистика:
| Метрика | Значение |
|---------|----------|
| Проверено файлов | {N} |
| Обновлено README | {M} |
| Исправлено ссылок | {K} |

Время: {X} сек
```

**Есть проблемы:**
```
⚠️ Переиндексация завершена с предупреждениями

Выполнено:
- ✅ Обновлено README: {M}
- ✅ Исправлено ссылок: {K}

Требует ручного исправления:
- ❌ {файл}:{строка} — {причина}

Используйте /links-validate --fix для исправления
```

## Примеры использования

### Переиндексация всего проекта

```
/doc-reindex
```

### Проверка без изменений

```
/doc-reindex --check
```

**Результат:**
```
📋 Отчёт о состоянии документации

Ссылки:
- ✅ Валидных: 340
- ❌ Битых: 2

README соответствие:

Используйте /doc-reindex для исправления
```

### Переиндексация конкретной папки

```

```

## FAQ

### Когда запускать переиндексацию?

- После массового переименования файлов
- После импорта внешней документации
- При подозрении на рассинхронизацию
- В CI для проверки (--check)

### Безопасно ли это?

Да. Перед изменениями показывается план, требуется подтверждение. Используйте `--dry-run` для безопасного предпросмотра.

### Что если много изменений?

Можно работать поэтапно — указывать конкретные папки для переиндексации.
