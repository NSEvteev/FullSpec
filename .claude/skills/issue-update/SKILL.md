## Формат вызова

```
/issue-update <номер> [--title "<заголовок>"] [--add-label <метка>] [--remove-label <метка>] [--body] [--comment "<текст>"]
```

| Параметр | Описание |
|----------|----------|
| `номер` | Номер Issue (обязательный) |
| `--title` | Новый заголовок |
| `--add-label` | Добавить метку |
| `--remove-label` | Удалить метку |
| `--body` | Редактировать тело Issue |
| `--comment` | Добавить комментарий |

**Примеры:**
- `/issue-update 123 --add-label "in-progress"`
- `/issue-update 123 --title "[AUTH] Новый заголовок"`
- `/issue-update 123 --comment "Начал работу"`

## Правила

### Типы обновлений

**Правило:** Обновления применяются через `gh issue edit`.

| Тип | Команда gh |
|-----|------------|
| Изменить заголовок | `gh issue edit {num} --title "..."` |
| Добавить метку | `gh issue edit {num} --add-label "..."` |
| Удалить метку | `gh issue edit {num} --remove-label "..."` |
| Изменить тело | `gh issue edit {num} --body "..."` |
| Добавить комментарий | `gh issue comment {num} --body "..."` |

**Связанная инструкция:** [issues.md](/.claude/instructions/git/issues.md)

### Доступные метки

| Категория | Метки |
|-----------|-------|
| Статус | `in-progress`, `blocked`, `needs-review` |
| Приоритет | `priority:high`, `priority:medium`, `priority:low` |
| Сервис | `service:auth`, `service:notify`, `service:payment`, `service:users`, `service:gateway` |
| Тип | `feature`, `bug`, `enhancement` |
| Другое | `docs`, `infra` |

## Воркфлоу

### Шаг 1: Получить номер Issue

1. Из аргумента: `/issue-update 123`
2. Проверить существование: `gh issue view 123`

### Шаг 2: Показать текущее состояние

```
📋 Issue #123

Заголовок: [AUTH] Добавить OAuth авторизацию
Статус: open
Метки: service:auth, feature
Назначен: —

Тело:
───────────────────────────────────────
## Описание
...
───────────────────────────────────────
```

### Шаг 3: Определить изменения

1. Из параметров командной строки
2. Или интерактивно:

```
Что обновить?
[1] Заголовок
[2] Добавить метку
[3] Удалить метку
[4] Редактировать тело
[5] Добавить комментарий
[6] Отмена
```

### Шаг 4: Применить изменения

```bash
gh issue edit 123 --add-label "in-progress"
```

### Шаг 5: Проверка по чек-листу

Перед завершением проверить выполнение всех пунктов:

```
✅ Шаг 1: Получил номер Issue
✅ Шаг 2: Показал текущее состояние
✅ Шаг 3: Определил изменения
✅ Шаг 4: Применил изменения
```

### Шаг 6: Результат

Вывести полный отчёт:

```
✅ Issue #123 обновлён

Изменения:
- Добавлена метка: in-progress

Текущее состояние:
- Метки: service:auth, feature, in-progress

URL: https://github.com/user/repo/issues/123
```

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Номер не указан | Спросить: "Какой Issue обновить?" |
| Issue не существует | Сообщить: "Issue #{num} не найден" |
| Issue закрыт | Предупредить, предложить переоткрыть |
| Метка не существует | Сообщить: "Метка {X} не существует" |
| gh не авторизован | Сообщить: "Выполните: gh auth login" |
| Ошибка обновления | Показать ошибку gh |

**Нет отката:** Изменения применяются атомарно через `gh issue edit`.
