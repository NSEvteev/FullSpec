## Формат вызова

```
/issue-delete <номер> [--reason <причина>] [--comment "<комментарий>"]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `номер` | Номер Issue (обязательный) | — |
| `--reason` | Причина: `not_planned`, `duplicate` | not_planned |
| `--comment` | Комментарий при закрытии | — |

**Примеры:**
- `/issue-delete 123 --comment "Неактуально после обсуждения"`
- `/issue-delete 123 --reason not_planned --comment "Неактуально"`
- `/issue-delete 123 --reason duplicate --comment "Дубликат #100"`

## Правила

### Причины закрытия

**Правило:** Причина закрытия определяет, как Issue отображается в истории.

| Причина | Описание | Команда gh |
|---------|----------|------------|
| `not_planned` | Неактуально / отменено | `gh issue close {num} --reason "not planned"` |
| `duplicate` | Дубликат другого Issue | `gh issue close {num} --reason "not planned"` + комментарий |

**Связанная инструкция:** [issues.md](/.claude/instructions/git/issues.md)

### Формат комментария

**Правило:** При закрытии рекомендуется добавить комментарий с пояснением.

| Причина | Рекомендуемый комментарий |
|---------|--------------------------|
| `not_planned` | "Причина: {описание}" |
| `duplicate` | "Дубликат #{num}" |

## Воркфлоу

### Шаг 1: Получить номер Issue

1. Из аргумента: `/issue-delete 123`
2. Проверить существование и статус: `gh issue view 123`

### Шаг 2: Показать текущее состояние

```
📋 Issue #123

Заголовок: [AUTH] Добавить OAuth авторизацию
Статус: open
Метки: service:auth, feature, in-progress
```

### Шаг 3: Определить причину

1. Из параметра `--reason`
2. Или спросить:

```
Причина закрытия?
[1] Неактуально (not_planned)
[2] Дубликат (duplicate)
```

### Шаг 4: Добавить комментарий

**Если указан `--comment`:**

```bash
gh issue comment 123 --body "Причина: требования изменились"
```

### Шаг 5: Закрыть Issue

```bash
gh issue close 123 --reason "not planned"
```

### Шаг 6: Проверка по чек-листу

Перед завершением проверить выполнение всех пунктов:

```
✅ Шаг 1: Получил номер Issue
✅ Шаг 2: Показал текущее состояние
✅ Шаг 3: Определил причину закрытия
✅ Шаг 4: Добавил комментарий
✅ Шаг 5: Закрыл Issue
```

### Шаг 7: Результат

Вывести полный отчёт:

```
✅ Issue #123 закрыт

Причина: not_planned
Комментарий: Требования изменились

URL: https://github.com/user/repo/issues/123
```

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Номер не указан | Спросить: "Какой Issue закрыть?" |
| Issue не существует | Сообщить: "Issue #{num} не найден" |
| Issue уже закрыт | Показать информацию, предложить переоткрыть |
| gh не авторизован | Сообщить: "Выполните: gh auth login" |
| Ошибка закрытия | Показать ошибку gh |

**Нет отката:** Закрытие Issue можно отменить переоткрытием через `gh issue reopen`.
