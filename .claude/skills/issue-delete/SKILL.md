## Формат вызова

```
/issue-delete <номер> [--reason <причина>] [--comment "<комментарий>"]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `номер` | Номер Issue (обязательный) | — |
| `--reason` | Причина: `not_planned`, `duplicate` | not_planned |
| `--comment` | Комментарий при закрытии | — |

**Примеры:**
- `/issue-delete 123 --comment "Неактуально после обсуждения"`
- `/issue-delete 123 --reason not_planned --comment "Неактуально"`
- `/issue-delete 123 --reason duplicate --comment "Дубликат #100"`

## Правила

### Причины закрытия

**Правило:** Причина закрытия определяет, как Issue отображается в истории.

| Причина | Описание | Команда gh |
|---------|----------|------------|
| `not_planned` | Неактуально / отменено | `gh issue close {num} --reason "not planned"` |
| `duplicate` | Дубликат другого Issue | `gh issue close {num} --reason "not planned"` + комментарий |

**Связанная инструкция:** [issues.md](/.claude/instructions/git/issues.md)

### Формат комментария

**Правило:** При закрытии рекомендуется добавить комментарий с пояснением.

| Причина | Рекомендуемый комментарий |
|---------|--------------------------|
| `not_planned` | "Причина: {описание}" |
| `duplicate` | "Дубликат #{num}" |

1. Из аргумента: `/issue-delete 123`

```bash
gh issue close 123 --reason "not planned"
```

## FAQ / Troubleshooting

### Чем /issue-delete отличается от /issue-complete?

**Различия:**

| Аспект | /issue-delete | /issue-complete |
|--------|---------------|-----------------|
| Причина | `not_planned` | `completed` |
| Когда | Задача неактуальна | Задача выполнена |
| Статистика | Не считается выполненной | Считается выполненной |
| PR | Не требуется | Рекомендуется |

**Правило:** Используй `/issue-delete` только если задача НЕ будет выполнена.

### Можно ли восстановить закрытый Issue?

**Ответ:** Да, любой закрытый Issue можно переоткрыть.

**Команда:**
```bash
gh issue reopen {номер}
```

После переоткрытия Issue снова становится `OPEN`.

### Как закрыть как дубликат?

**Команда:**
```
/issue-delete {номер} --reason duplicate --comment "Дубликат #{номер_оригинала}"
```

**Что произойдёт:**
1. Добавляется комментарий со ссылкой на оригинал
2. Issue закрывается как `not_planned`
3. В истории видна связь с оригиналом

### Issue в работе — можно ли удалить?

**Ответ:** Да, но появится предупреждение.

**Что делать:**
1. Скилл покажет, что Issue имеет метку `in-progress`
2. Предложит снять метку и удалить назначение
3. После подтверждения — закроет Issue

### Как массово закрыть неактуальные Issue?

**Ответ:** Используйте gh CLI напрямую:

```bash
# Закрыть все Issue с меткой "wontfix"
gh issue list --label "wontfix" --json number -q '.[].number' | \
  xargs -I {} gh issue close {} --reason "not planned"
```

### Удалённый Issue не пропадает из списка — почему?

**Ответ:** Issue не удаляется, а закрывается.

**Для просмотра только открытых:**
```bash
gh issue list --state open
```

**Для скрытия закрытых в веб-интерфейсе:** фильтр `is:open`
