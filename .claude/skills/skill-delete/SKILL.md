# Удаление скилла

Команда для обновления существующих скиллов при удалении скилла. Находит все ссылки на удаляемый скилл и корректирует воркфлоу связанных скиллов.

**Связанные скиллы:**
- [skill-create](/.claude/skills/skill-create/SKILL.md) — создание нового скилла
- [skill-update](/.claude/skills/skill-update/SKILL.md) — обновление скиллов при добавлении нового
- [context-delete](/.claude/skills/context-delete/SKILL.md) — очистка контекста (вызвать до удаления файлов)
- [context-update](/.claude/skills/context-update/SKILL.md) — распространение контекста (после изменений)

**Связанные инструкции:**
- [skills/README.md](/.claude/skills/README.md) — индекс скиллов, категории
- [output-formats.md](/.claude/instructions/skills/output.md) — форматы вывода (SSOT)

**Utility-скиллы:**
- [environment-check](/.claude/skills/environment-check/SKILL.md) — проверка gh/git перед выполнением
- [input-validate](/.claude/skills/input-validate/SKILL.md) — валидация входных данных

## Формат вызова

```
/skill-delete <имя-скилла>
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `имя-скилла` | Имя удалённого скилла | — (обязательный) |

**Примеры:**
- `/skill-delete links-validate` — очистить ссылки на удалённый links-validate
- `/skill-delete context-sync` — очистить ссылки на удалённый context-sync

## Когда использовать

| Событие | Действие |
|---------|----------|
| Удалён скилл | Вызвать для очистки ссылок |
| Переименован скилл | Сначала `/skill-delete старое-имя`, затем `/skill-update новое-имя` |
| Объединены скиллы | Вызвать для удалённых скиллов |

## Правила

### Что удалять

При удалении скилла из существующих скиллов удаляются:

1. **Вызовы в воркфлоу** — строки вида `Вызвать /{скилл}` и блоки `**Интеграция с {скилл}:**`
2. **Связанные скиллы** — строки в разделе "Связанные скиллы"
3. **Чек-лист** — пункты, упоминающие удалённый скилл
4. **Frontmatter** — записи в `skills:` (для агентов)

### Чего НЕ удалять

- Упоминания в примерах и документации (отметить как устаревшие)
- Ссылки в комментариях `*(не создан)*` — оставить или обновить статус

### Корректировка логики

После удаления ссылки проверить:

1. Остался ли шаг осмысленным без вызова скилла?
2. Нужно ли удалить весь блок или только ссылку?
3. Нужно ли обновить нумерацию шагов?

### Обязательный вывод напоминания

**Правило:** Раздел `⚠️ Проверьте вручную` выводится **ВСЕГДА** в итоговом отчёте, даже если автоматически обработаны все найденные ссылки.

Причина: документация в `/doc/` может содержать неявные упоминания, которые не находятся автоматическим поиском.

## Воркфлоу

### Шаг 1: Получить удаляемый скилл

1. Из аргумента: `/skill-delete links-validate`
2. Запомнить имя скилла для поиска

**Информация для поиска:**
- Имя скилла: `{имя}`
- Путь: `/.claude/skills/{имя}/SKILL.md`
- Паттерны поиска:
  - `[{имя}]` — ссылки
  - `/{имя}` — вызовы
  - `{имя}/SKILL.md` — полные пути
  - `skills:.*{имя}` — frontmatter

### Шаг 2: Проверить факт удаления

1. Проверить существование папки `/.claude/skills/{имя}/`
2. Если папка существует:
   ```
   ⚠️ Скилл {имя} ещё существует!

   Варианты:
   1. Удалить скилл и продолжить
   2. Отменить операцию

   Что выбрать?
   ```
3. Если папка не существует — продолжить

### Шаг 3: Найти ссылки в существующих скиллах

**3.1. Поиск в скиллах:**

1. Найти все скиллы: `/.claude/skills/*/SKILL.md`
2. Исключить:
   - Скилл `skill-delete` (избежание циклических зависимостей)
3. Для каждого скилла найти упоминания

**3.2. Поиск в агентах:**

1. Найти все агенты: `/.claude/agents/*.md`
2. Для каждого агента проверить frontmatter `skills:`
3. Если содержит удаляемый скилл → добавить в список изменений

**3.3. Поиск в индексах:**

1. Проверить индекс скиллов: `/.claude/skills/README.md`
2. Проверить индекс агентов: `/.claude/agents/README.md`
3. Если есть упоминания → добавить в список изменений для автоматического удаления

**Поиск по паттернам:**

| Паттерн | Тип ссылки | Действие |
|---------|------------|----------|
| `\[/{имя}\]` | Вызов в тексте | Удалить блок интеграции |
| `\[{имя}\]\(.*\)` | Markdown-ссылка | Удалить строку |
| `**Интеграция с {имя}:**` | Блок интеграции | Удалить весь блок |
| `Вызвать.*/{имя}` | Инструкция вызова | Удалить строку/блок |
| `- \[.*{имя}.*\]` | Связанные скиллы | Удалить строку |
| `skills:.*{имя}` | Frontmatter агента | Удалить из массива |
| `\| \[{имя}\]\(.*\).*\|` | Строка таблицы индекса | Удалить строку из skills.md |

**Алгоритм:**
```
1. Grep по /.claude/skills/*/SKILL.md: `{имя}`
2. Grep по /.claude/agents/*.md: `{имя}`
3. Grep по /.claude/instructions/tools/*.md: `{имя}`
4. Для каждого найденного файла:
   - Прочитать содержимое
   - Найти все строки с упоминанием
   - Определить тип ссылки
   - Определить границы удаления (строка / блок / элемент массива)
5. Сформировать список изменений
```

### Шаг 4: Сформировать предложения

Для каждого скилла со ссылками:

1. Показать найденные ссылки
2. Предложить удаление

**Формат вывода:**

```
📋 Очистка ссылок на удалённый скилл {имя}

Проверено скиллов: {N}
Найдено ссылок: {M} в {K} скиллах

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 {скилл-1} (категория: {категория})

   Найдено ссылок: {X}

   1. Связанные скиллы (строка 28):
      ```diff
      - - [links-validate](/.claude/skills/links-validate/SKILL.md) — проверка ссылок
      ```

   2. Шаг 7 — Валидация (строки 156-158):
      ```diff
      - **Интеграция с links-validate:**
      - Вызвать [/links-validate](/.claude/skills/links-validate/SKILL.md) для проверки.
      ```

   3. Чек-лист (строка 203):
      ```diff
      - - [ ] **Шаг 7:** Вызвал /links-validate
      ```

   [1] Удалить все  [2] Выборочно  [3] Пропустить

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 {скилл-2} ...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Удалить все ссылки? [Y/n/выборочно]
```

### Шаг 5: Применить изменения

После подтверждения пользователя:

1. Для каждого одобренного изменения:
   - Открыть SKILL.md существующего скилла
   - Удалить найденные ссылки
   - Проверить корректность оставшегося содержимого

2. **Удаление блока интеграции:**
   ```diff
   ### Шаг 7: Валидация

   Проверить созданный файл:
   - Все ссылки корректны
   - YAML валиден

   - **Интеграция с links-validate:**
   - Вызвать [/links-validate](/.claude/skills/links-validate/SKILL.md) для автоматической проверки.
   ```

3. **Удаление из связанных скиллов:**
   ```diff
   **Связанные скиллы:**
   - [skill-create](/.claude/skills/skill-create/SKILL.md) — создание
   - [skill-update](/.claude/skills/skill-update/SKILL.md) — обновление
   - - [links-validate](/.claude/skills/links-validate/SKILL.md) — проверка ссылок
   ```

4. **Удаление из чек-листа:**
   ```diff
   - [ ] **Шаг 6:** Провёл ревью
   - [ ] **Шаг 7:** Проверил валидацию
   - - [ ] **Шаг 7:** Вызвал /links-validate
   - [ ] **Шаг 8:** Обновил связанные скиллы
   ```

5. **Проверка после удаления:**
   - Если шаг стал пустым → предложить удалить шаг
   - Если нумерация сбилась → исправить
   - Если остались битые ссылки → предупредить

6. **Удаление из индекса skills.md:**

   Если скилл найден в `/.claude/skills/README.md`:

   ```diff
   - | [links-validate](/.claude/skills/links-validate/SKILL.md) | Проверка ссылок | `/links-validate`, "проверь ссылки" |
   ```

   **Алгоритм:**
   1. Найти строку таблицы, содержащую `[{имя}]`
   2. Удалить всю строку
   3. Проверить, что таблица осталась корректной (заголовок + минимум 1 строка данных)

   **Если таблица категории стала пустой:**
   - Оставить строку-заглушку: `| — | — | — |`

### Шаг 6: Обновить контекст

После применения изменений вызвать [/context-update](/.claude/skills/context-update/SKILL.md) для обновления контекста:

```
/context-update <список-изменённых-файлов>
```

**Какие файлы передавать:**
- Все изменённые SKILL.md файлы
- Индексы скиллов (если обновлялись)

**Пример:**
```
/context-update .claude/skills/skill-create/SKILL.md .claude/skills/instruction-create/SKILL.md
```

**Зачем:**
- Распространить изменения на связанные документы
- Обновить индексы и README файлы
- Синхронизировать контекст после массовых изменений

### Шаг 7: Проверка по чек-листу

Перед завершением проверить выполнение всех пунктов чек-листа:

```
✅ Шаг 1: Получил имя удаляемого скилла
✅ Шаг 2: Проверил факт удаления

✅ Шаг 4: Сформировал предложения
✅ Шаг 5: Применил изменения
✅ Шаг 6: Вызвал context-update
```

  - Удалён блок интеграции (Шаг 7)
  - Удалён пункт чек-листа

  - Удалён из связанных скиллов

Пропущено: {M}

- Вызван /context-update для {N} изменённых файлов

- Папка скилла (если ещё существует)

## Типы ссылок

### Связанные скиллы

**Паттерн:**
```regex
- \[{имя}\]\(.*\).*
```

**Пример:**
```markdown
- [links-validate](/.claude/skills/links-validate/SKILL.md) — проверка ссылок
```

**Действие:** Удалить строку целиком

### Блок интеграции

**Паттерн:**
```regex
\*\*Интеграция с {имя}:\*\*[\s\S]*?(?=\n\n|\n###|\n\*\*|$)
```

**Пример:**
```markdown
**Интеграция с links-validate:**
Вызвать [/links-validate](/.claude/skills/links-validate/SKILL.md) для проверки ссылок.
```

**Действие:** Удалить весь блок (от заголовка до пустой строки или следующего блока)

### Вызов в тексте

**Паттерн:**
```regex
Вызвать.*\[?/?{имя}\]?.*
```

**Пример:**
```markdown
Вызвать /links-validate для проверки.
```

**Действие:** Удалить строку или предложение

### Чек-лист

**Паттерн:**
```regex
- \[ \].*{имя}.*
```

**Пример:**
```markdown
- [ ] **Шаг 7:** Вызвал /links-validate
```

**Действие:** Удалить строку

### Frontmatter (агенты)

**Паттерн:**
```regex
skills:.*{имя}
```

**Пример до:**
```yaml
skills: [skill-create, skill-update, links-validate]
```

**Пример после:**
```yaml
skills: [skill-create, skill-update]
```

**Алгоритм удаления из массива YAML:**
1. Найти строку `skills: [...]`
2. Распарсить массив
3. Удалить элемент `{имя}`
4. Если массив пустой → удалить всю строку `skills:`
5. Если один элемент → оставить `skills: [{элемент}]`
6. Собрать обратно с правильными запятыми

**Варианты форматов:**
```yaml
# Inline формат
skills: [skill-create, links-validate]

# Многострочный формат
skills:
  - skill-create
  - links-validate
```

## FAQ / Troubleshooting

### Почему скилл не удаляет папку автоматически?

**Причины:**
1. Безопасность — ручное удаление снижает риск ошибки
2. Папка может содержать нужные файлы
3. Требуется подтверждение пользователя

После вызова `/skill-delete` удалите папку вручную:
```bash
rm -rf /.claude/skills/{skill-name}/
```

### Что если скилл критичный?

Критичные скиллы (`skill-*`, `instruction-*`, `issue-*`):
- Нельзя удалить единственный тест
- При failed тесте — предложение создать Issue

Скилл предупредит перед удалением.

### Как восстановить удалённый скилл?

Если папка ещё не удалена:
```bash
git checkout -- /.claude/skills/{skill-name}/
```

Если папка удалена:
```bash
git checkout HEAD~1 -- /.claude/skills/{skill-name}/
```

### Что делать, если много ссылок?

При >10 ссылок скилл покажет summary и предложит:
1. Применить все изменения
2. Выборочно
3. Экспорт списка

### Как найти все ссылки на скилл?

```bash
grep -r "skill-name" --include="*.md" /.claude/
```
