# Распространение контекста

Команда для распространения изменений контекста по графу связанных документов после работы с .md файлом.

**Связанные скиллы:**
- [links-update](/.claude/skills/links-update/SKILL.md) — обновление ссылок (синтаксис)
- [links-create](/.claude/skills/links-create/SKILL.md) — создание ссылок в документе
- [context-delete](/.claude/skills/context-delete/SKILL.md) — очистка контекста при удалении *(не создан)*

## Формат вызова

```
/context-update <файл.md> [--depth N]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `файл.md` | Файл, в котором произошли изменения | — |
| `--depth N` | Глубина обхода графа связей | 2 |

## Отличие от links-update

| Аспект | links-update | context-update |
|--------|--------------|----------------|
| Цель | Обновить **ссылки** (синтаксис) | Обновить **контекст** (семантика) |
| Что меняет | Добавляет/исправляет `[text](path)` | Добавляет/изменяет описания, связи |
| Глубина | Прямые ссылки | Транзитивные связи (A → B → C) |
| Решение | Автоматическое | Предложение пользователю |
| Когда | После создания файла | После изменения содержимого |

## Когда использовать

| Событие | Действие |
|---------|----------|
| Добавлен новый раздел в документ | Проверить, нужно ли обновить связанные документы |
| Изменена логика/архитектура | Распространить изменения по зависимостям |
| Добавлено новое правило | Обновить документы, которые должны следовать правилу |
| Удалён раздел | Проверить, не сломались ли ссылки на него |
| Завершена работа над документом | Синхронизировать контекст со связанными файлами |

## Воркфлоу

### Шаг 1: Определить инкремент контекста

1. Прочитать файл A
2. Если есть git history — сравнить с предыдущей версией:
   ```bash
   git diff HEAD~1 -- файл.md
   ```
3. Если нет — анализировать весь файл как новый контекст

**Определить:**
- Что добавлено (новые разделы, правила, описания)
- Что изменено (переформулировки, уточнения)
- Что удалено (убранные разделы, устаревшая информация)

**Результат:**
```
Инкремент контекста в {файл}:

Добавлено:
- Раздел "Правила ссылок на папки"
- Новый тип связи "Файл/папка → Документы"

Изменено:
- Расширена область применения (не только .md файлы)

Удалено:
- (ничего)
```

### Шаг 2: Найти прямые связи (уровень 1)

1. Найти все ссылки на .md файлы в документе A
2. Использовать скрипт [find_references.py](/.claude/scripts/find_references.py) для поиска обратных ссылок:
   ```bash
   python .claude/scripts/find_references.py файл.md --format json
   ```

**Результат:** Список файлов B1, B2, ... Bn, связанных с A

### Шаг 3: Анализ релевантности

Для каждого файла B из списка:

1. Прочитать B, определить его область ответственности
2. Сравнить инкремент A с областью B:

| Соотношение | Описание | Действие |
|-------------|----------|----------|
| A ⊂ B (A уже B) | Изменения A входят в область B | B может нуждаться в обновлении описаний |
| A ⊃ B (A шире B) | Изменения A выходят за область B | Проверить, нужно ли B знать об этом |
| A ∩ B (пересечение) | Частичное совпадение | Обновить пересекающуюся часть |
| A ∩ B = ∅ (нет связи) | Нет смысловой связи | Пропустить, не идти глубже |

**Результат:** Список {B, тип_связи, релевантность: высокая/средняя/низкая}

### Шаг 4: Транзитивные связи (уровень 2+)

Для каждого B с релевантностью "высокая" или "средняя":

1. Найти ссылки на .md файлы в B
2. Для каждого файла C:
   - Проверить связь C с инкрементом A
   - Если есть связь — добавить в список
3. Повторить до достижения `--depth`

**Фильтрация:** На следующий уровень идём только если:
- Текущий файл релевантен (высокая/средняя)
- Не достигнута максимальная глубина
- Файл ещё не был обработан (избежание циклов)

### Шаг 5: Сформировать предложения

Для каждого релевантного файла сформировать:

1. **Текстовое описание:**
   ```
   Файл: skills.md
   Связь: A ⊂ B (skill-create входит в индекс скиллов)
   Предложение: Обновить описание skill-create в таблице категории
   ```

2. **Готовый diff:**
   ```diff
   - | skill-create | Создание скилла | /skill-create |
   + | skill-create | Создание скилла по шаблону с валидацией | /skill-create |
   ```

### Шаг 6: Вывод и подтверждение

Вывести пользователю:

```

Источник: skill-create/SKILL.md
Инкремент: Добавлены правила ссылок на папки

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   Связь: индекс содержит описание skill-create

   ```diff
   - | skill-create | Создание нового скилла |

   ```

   [1] Применить  [2] Пропустить  [3] Редактировать

Применить все? [Y/n/выборочно]
```

### Шаг 7: Применить изменения

После подтверждения пользователя:

1. Применить одобренные изменения через `Edit`
2. Пропустить отклонённые
3. Для "Редактировать" — показать текущий текст, принять правки пользователя

**Результат:**
```
✅ Контекст обновлён

Применено изменений: 3
- skills.md — обновлено описание
- CLAUDE.md — добавлено упоминание
- links-create/SKILL.md — добавлена связь

Пропущено: 1
- agents.md — пользователь отклонил

Следующий шаг: проверить изменения, при необходимости запустить /context-update для изменённых файлов
```

## Типы связей контекста

### Иерархические

| Тип | Описание | Пример |
|-----|----------|--------|
| Индекс → Элемент | Индекс содержит ссылку на элемент | skills.md → skill-create |
| Родитель → Потомок | Общий документ ссылается на частный | CLAUDE.md → skills.md |

### Семантические

| Тип | Описание | Пример |
|-----|----------|--------|
| Правило → Применение | Правило используется в другом месте | skill-create (правила) → links-create (использует) |
| Зависимость | Один документ зависит от другого | context-update → find_references.py |

### По направлению

| Направление | Описание |
|-------------|----------|
| A → B | A ссылается на B (исходящая) |
| A ← B | B ссылается на A (входящая) |
| A ↔ B | Взаимные ссылки |

---

## Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Файл не найден | Сообщить: "Файл {путь} не найден" |
| Нет изменений (чистый git status) | Анализировать весь файл как контекст |
| Циклическая зависимость | Пометить файл как обработанный, не заходить повторно |
| Слишком много связей (>20) | Предупредить, предложить увеличить фильтрацию |
| Пользователь отменил | Не применять изменения, вывести итог |
