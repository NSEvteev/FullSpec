## Чек-лист smoke test

### Вызов по триггерам

- [ ] Вызывается по команде `/specs-sync`
- [ ] Вызывается по фразе "синхронизируй статусы"
- [ ] Вызывается по фразе "пересчитай статусы"
- [ ] Вызывается по фразе "обнови статусы specs"
- [ ] Вызывается по фразе "sync statuses" (EN)
- [ ] Вызывается по фразе "recalculate statuses" (EN)

### Базовая функциональность

- [ ] Не падает с ошибкой при базовом вызове
- [ ] Корректно работает флаг `--dry-run`
- [ ] Выводит количество проверенных цепочек
- [ ] Выводит количество найденных расхождений
- [ ] Показывает детали каждого расхождения (текущий/ожидаемый статус, причина)

### Правила синхронизации

- [ ] Impact -> APPROVED когда ВСЕ ADR в APPROVED
- [ ] ADR -> RUNNING, Impact -> RUNNING когда ВСЕ планы Impact в APPROVED
- [ ] Discussion -> RUNNING когда первый Plan -> RUNNING
- [ ] Impact -> DONE когда ВСЕ ADR в финальном + минимум 1 DONE
- [ ] Discussion -> DONE когда Impact -> DONE

### Интеграция

- [ ] Вызывает `/spec-status` для применения изменений
- [ ] Ожидает подтверждения перед применением (блокирующее)
