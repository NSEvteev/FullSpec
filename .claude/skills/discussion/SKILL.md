# Создание дискуссии

Скилл для автоматической обработки начала новой дискуссии.

## Связанная документация

| Документ | Назначение |
|----------|------------|
| [general_docs.md](../../../llm_instructions/general_docs.md#дискуссии-discuss) | Правила ведения дискуссий |
| [templates/discuss.md](../../../llm_instructions/templates/discuss.md) | Шаблон дискуссии |
| [000_discuss.md](../../../general_docs/01_discuss/000_discuss.md) | Индекс дискуссий |

## Когда использовать

**Триггеры:**
- Сообщение начинается с "Дискуссия:" (регистронезависимо)
- Команда `/discussion`

**Примеры триггеров:**
- "Дискуссия: какой OAuth-провайдер выбрать?"
- "дискуссия: нужен ли нам Redis для кэширования?"
- `/discussion Авторизация в микросервисах`

## Workflow статусов дискуссии

```
🟡 draft        → Файл создан, варианты НЕ предложены
      ↓
🔵 in_progress  → Варианты предложены, идёт обсуждение с пользователем
      ↓
🟣 review       → Пользователь выбрал вариант, проводится ревью (/discussion-review)
      ↓
🟢 approved     → Решение принято после ревью
      ↓
⚪ final        → Созданы документы архитектуры
```

**ВАЖНО:** При финализации создаются **только файлы `02_architecture/`**, не ресурсы и не планы.

**Переход in_progress → review:** При выборе варианта запускается `/discussion-review` для анализа последствий.

## Инструкции

### Шаг 1: Извлечь тему дискуссии

Из сообщения пользователя извлечь:
- **Тема** — что обсуждается (текст после "Дискуссия:")
- **Исходный запрос** — полный текст сообщения пользователя

### Шаг 2: Проверить индекс дискуссий

```bash
cat general_docs/01_discuss/000_discuss.md
```

Проверить таблицу "Индекс документов" на похожие темы:
- Совпадение ключевых слов в названии
- Схожая тематика

### Шаг 3: Если найдены похожие дискуссии

Показать пользователю и спросить через AskUserQuestion:
- Продолжить существующую дискуссию
- Создать новую дискуссию

### Шаг 4: Создать дискуссию через скрипт

**ВАЖНО:** Использовать скрипт `discuss_new.py` для создания:

```bash
python scripts/discuss_new.py "Тема дискуссии"
```

Скрипт автоматически:
- Генерирует следующий ID (001, 002, ...)
- Создаёт файл по шаблону
- Обновляет индекс `000_discuss.md`
- Устанавливает статус `🟡 draft`

### Шаг 5: Заполнить дискуссию

После создания файла скриптом:

1. **Прочитать созданный файл**
2. **Заполнить "Анализ цели и контекст":**
   - Проблема
   - Область обсуждения
   - Что вне области
3. **Предложить варианты в таблице "Предложения"**
4. **Заполнить критерии оценки**
5. **Заполнить сравнительную таблицу** (⭐ рейтинг)
6. **Заполнить текстовое сравнение вариантов**
7. **Заполнить рекомендуемые варианты** (топ 3-4)
8. **Изменить статус:** `🟡 draft` → `🔵 in_progress`
9. **Запустить `/doc-review`** для критического анализа

### Шаг 6: Предложить решение пользователю

После заполнения спросить пользователя через AskUserQuestion:
- Выбрать один из рекомендуемых вариантов
- Продолжить обсуждение (свой вариант)

**Если пользователь выбрал вариант:**
1. Изменить статус: `🔵 in_progress` → `🟣 review`
2. **Запустить `/discussion-review`** для анализа последствий выбора
3. После ревью — если подтверждено — заполнить "Принятое решение" и перейти в `🟢 approved`

**Если пользователь продолжает обсуждение:**
1. Оставить статус `🔵 in_progress`
2. Добавить комментарий в "История обсуждения"

## Структура документа дискуссии

Согласно [шаблону](../../../llm_instructions/templates/discuss.md):

1. Метаданные (статус, дата)
2. Исходный запрос пользователя
3. Анализ цели и контекст
4. Предложения (таблица вариантов)
5. Критерии оценки
6. Сравнение вариантов по критериям (таблица ⭐)
7. **Текстовое сравнение вариантов** (анализ отличий)
8. **Рекомендуемые варианты** (топ 3-4)
9. **Принятое решение** (с аргументацией внутри!)
10. Дискуссия и анализ
11. Связанная архитектура
12. Добавления к глоссарию

## Команды

```bash
# Создать дискуссию
python scripts/discuss_new.py "Тема"
python scripts/discuss_new.py -t "Тема" -d "Описание" -q "Полный запрос"
python scripts/discuss_new.py -i  # интерактивно

# Удалить дискуссию
python scripts/discuss_delete.py 001           # с подтверждением
python scripts/discuss_delete.py 001 --force   # без подтверждения

# Makefile
make discuss-new              # интерактивно
make discuss-new-topic TOPIC="Тема"
make discuss-delete ID="001"  # удалить дискуссию
make discuss-index            # показать индекс
```

## Связанные файлы

- `scripts/discuss_new.py` — скрипт создания дискуссии
- `scripts/discuss_delete.py` — скрипт удаления дискуссии
- `general_docs/01_discuss/000_discuss.md` — индекс дискуссий
- `general_docs/.doc_counter` — единый счётчик ID для всех документов
- `llm_instructions/templates/discuss.md` — шаблон
- `llm_instructions/general_docs.md` — правила ведения документации

## Связанные скиллы

- `/doc-review` — ревью созданной дискуссии (вызывать после заполнения)
- `/discussion-review` — ревью выбранного решения (что сможет/не сможет сделать)
- `/glossary-candidates` — поиск терминов для глоссария
