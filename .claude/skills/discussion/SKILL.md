# Управление дискуссиями

Скилл для работы с дискуссиями: создание, изменение статусов, удаление.

## ВАЖНОЕ ПРАВИЛО

**При ЛЮБЫХ действиях с дискуссиями — сначала прочитать этот скилл!**

Это включает:
- Создание новой дискуссии
- Изменение существующей дискуссии
- Изменение статуса дискуссии
- Удаление дискуссии
- Работу с индексом `000_discuss.md`

**Почему:** Скрипты автоматически обновляют счётчики и индексы. Ручные операции (`rm`, прямое редактирование) ломают синхронизацию.

## Создание

### Когда использовать

**Триггеры:**
- Сообщение начинается с "Дискуссия:" (регистронезависимо)
- Команда `/discussion`

**Примеры:**
- "Дискуссия: какой OAuth-провайдер выбрать?"
- `/discussion Авторизация в микросервисах`

### Шаги создания

#### Шаг 1: Проверить индекс дискуссий

Прочитать `general_docs/01_discuss/000_discuss.md` и проверить:
- Нет ли похожих тем
- Если есть — спросить пользователя: продолжить существующую или создать новую

#### Шаг 2: Создать через скрипт

**ОБЯЗАТЕЛЬНО использовать скрипт:**

```bash
python scripts/discuss_new.py "Тема дискуссии"
# или с параметрами:
python scripts/discuss_new.py -t "Тема" -d "Описание" -q "Полный запрос"
```

**Скрипт автоматически:**
- Генерирует следующий ID (001, 002, ...)
- Создаёт файл по шаблону
- Обновляет индекс `000_discuss.md`
- Обновляет счётчик `.doc_counter`
- Устанавливает статус `🟡 draft`

**ЗАПРЕЩЕНО:** Создавать файлы вручную через Write tool — это сломает счётчик!

#### Шаг 3: Заполнить дискуссию

После создания файла скриптом:

1. Прочитать созданный файл
2. Заполнить "Анализ цели и контекст"
3. Предложить варианты в таблице "Предложения"
4. Заполнить критерии оценки
5. Заполнить сравнительную таблицу (⭐ рейтинг)
6. Заполнить текстовое сравнение вариантов
7. Заполнить рекомендуемые варианты (топ 3-4)
8. **Изменить статус:** `🟡 draft` → `🔵 in_progress`
9. Запустить `/doc-review` для критического анализа

#### Шаг 4: Предложить решение

Спросить пользователя через AskUserQuestion:
- Выбрать один из рекомендуемых вариантов
- Продолжить обсуждение

## Изменение

### Изменение статуса

**Workflow статусов:**

```
🟡 draft        → Файл создан, варианты НЕ предложены
      ↓
🔵 in_progress  → Варианты предложены, идёт обсуждение
      ↓
🟣 review       → Пользователь выбрал вариант, ревью (/discussion-review)
      ↓
🟢 approved     → Решение принято после ревью
      ↓
⚪ final        → Созданы документы архитектуры
```

**Переходы:**

| Из | В | Когда | Действия |
|----|---|-------|----------|
| `🟡 draft` | `🔵 in_progress` | После заполнения вариантов | Edit статус в файле |
| `🔵 in_progress` | `🟣 review` | Пользователь выбрал вариант | Запустить `/discussion-review` |
| `🟣 review` | `🟢 approved` | Решение подтверждено | Заполнить "Принятое решение" |
| `🟢 approved` | `⚪ final` | Создана архитектура | Добавить ссылки в "Связанная архитектура" |

**Как изменить статус:**

```markdown
# Было:
**Статус:** 🟡 draft

# Стало:
**Статус:** 🔵 in_progress
```

**После изменения статуса:** Обновить индекс `000_discuss.md` (статистику и таблицу)

### Изменение контента

Редактировать файл дискуссии через Edit tool:
- Добавление/изменение вариантов
- Обновление критериев
- Добавление записей в секцию "История обсуждения"
- Заполнение "Принятое решение"

## Удаление

### ВАЖНО

**ОБЯЗАТЕЛЬНО использовать скрипт:**

```bash
python scripts/discuss_delete.py 001           # с подтверждением
python scripts/discuss_delete.py 001 --force   # без подтверждения
```

**Скрипт автоматически:**
- Удаляет файл дискуссии
- Обновляет индекс `000_discuss.md`
- Декрементирует счётчик (если это была последняя дискуссия)

**ЗАПРЕЩЕНО:** Удалять файлы через `rm` или Bash — это сломает счётчик!

### Когда удалять

- Дискуссия создана по ошибке
- Тема дублирует существующую
- Пользователь явно просит удалить

## Команды

### Скрипты

```bash
# Создать дискуссию
python scripts/discuss_new.py "Тема"
python scripts/discuss_new.py -t "Тема" -d "Описание" -q "Полный запрос"
python scripts/discuss_new.py -i  # интерактивно

# Удалить дискуссию
python scripts/discuss_delete.py 001           # с подтверждением
python scripts/discuss_delete.py 001 --force   # без подтверждения
```

### Makefile

```bash
make discuss-new              # интерактивно
make discuss-new-topic TOPIC="Тема"
make discuss-delete ID="001"  # удалить дискуссию
make discuss-index            # показать индекс
```
