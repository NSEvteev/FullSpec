# Создание дискуссии

Скилл для автоматической обработки начала новой дискуссии.

## Когда использовать

**Триггеры:**
- Сообщение начинается с "Дискуссия:" (регистронезависимо)
- Команда `/discussion`

**Примеры триггеров:**
- "Дискуссия: какой OAuth-провайдер выбрать?"
- "дискуссия: нужен ли нам Redis для кэширования?"
- `/discussion Авторизация в микросервисах`

## Инструкции

### Шаг 1: Извлечь тему дискуссии

Из сообщения пользователя извлечь:
- **Тема** — что обсуждается (текст после "Дискуссия:")
- **Исходный запрос** — полный текст сообщения пользователя

### Шаг 2: Проверить индекс дискуссий

```bash
# Прочитать индекс
cat general_docs/01_discuss/000_discuss.md
```

Проверить таблицу "Индекс документов" на похожие темы:
- Совпадение ключевых слов в названии
- Схожая тематика

### Шаг 3: Если найдены похожие дискуссии

Показать пользователю и спросить через AskUserQuestion:
- Продолжить существующую дискуссию
- Создать новую дискуссию

### Шаг 4: Создать дискуссию через скрипт

**ВАЖНО:** Использовать скрипт `discuss_new.py` для создания:

```bash
python scripts/discuss_new.py "Тема дискуссии"
```

Скрипт автоматически:
- Генерирует следующий ID (001, 002, ...)
- Создаёт файл по шаблону
- Обновляет индекс `000_discuss.md`

### Шаг 5: Заполнить дискуссию

После создания файла скриптом:
1. Прочитать созданный файл
2. Заполнить раздел "Анализ цели и контекст"
3. Предложить варианты в таблице "Предложения"
4. Показать пользователю и спросить, какие варианты рассмотреть

## Формат ответа

```markdown
## Создана дискуссия: [ID]_[название].md

**Файл:** `general_docs/01_discuss/[ID]_[название].md`
**Статус:** draft

### Ваш запрос
> [цитата]

### Предлагаемые варианты

| # | Вариант | Краткое описание |
|---|---------|------------------|
| 1 | [Вариант 1] | [Описание] |
| 2 | [Вариант 2] | [Описание] |

Какие варианты рассмотреть подробнее?
```

## Примеры использования

### Пример 1: Новая дискуссия

**Ввод:**
```
Дискуссия: какой UI-фреймворк выбрать?
```

**Действия:**
```bash
# 1. Проверить индекс на похожие
cat general_docs/01_discuss/000_discuss.md

# 2. Создать дискуссию
python scripts/discuss_new.py "UI-фреймворк"

# 3. Заполнить варианты в созданном файле
# 4. Показать пользователю
```

### Пример 2: Найдена похожая дискуссия

**Ввод:**
```
Дискуссия: выбор стилей для UI
```

**Действия:**
1. Найдена похожая: `001_ui_styleguide.md`
2. Спросить пользователя: продолжить или создать новую
3. Если продолжить — открыть существующую
4. Если создать — запустить скрипт

## Команды

```bash
# Создать дискуссию
python scripts/discuss_new.py "Тема"
python scripts/discuss_new.py -t "Тема" -d "Описание" -q "Полный запрос"
python scripts/discuss_new.py -i  # интерактивно

# Удалить дискуссию
python scripts/discuss_delete.py 001           # с подтверждением
python scripts/discuss_delete.py 001 --force   # без подтверждения

# Makefile
make discuss-new              # интерактивно
make discuss-new-topic TOPIC="Тема"
make discuss-delete ID="001"  # удалить дискуссию
make discuss-index            # показать индекс
```

## Связанные файлы

- `scripts/discuss_new.py` — скрипт создания дискуссии
- `scripts/discuss_delete.py` — скрипт удаления дискуссии
- `general_docs/01_discuss/000_discuss.md` — индекс дискуссий
- `general_docs/.doc_counter` — единый счётчик ID для всех документов
- `llm_instructions/templates/discuss.md` — шаблон

## Связанные скиллы

- `/doc-review` — ревью созданной дискуссии
- `/glossary-candidates` — поиск терминов для глоссария
