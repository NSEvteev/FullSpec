# Создание дискуссии

Скилл для автоматической обработки начала новой дискуссии.

## Когда использовать

**Триггеры:**
- Сообщение начинается с "Дискуссия:" (регистронезависимо)
- Команда `/discussion`

**Примеры триггеров:**
- "Дискуссия: какой OAuth-провайдер выбрать?"
- "дискуссия: нужен ли нам Redis для кэширования?"
- `/discussion Авторизация в микросервисах`

## Инструкции

### Шаг 1: Извлечь тему дискуссии

Из сообщения пользователя извлечь:
- **Тема** — что обсуждается (текст после "Дискуссия:")
- **Ключевые слова** — для поиска похожих дискуссий

### Шаг 2: Проверить индекс дискуссий

1. Прочитать индекс `general_docs/01_discuss/000_discuss.md`
2. Извлечь список существующих дискуссий (ID, название, статус)
3. Для каждой существующей дискуссии проверить релевантность по:
   - Совпадение ключевых слов в названии
   - Схожая тематика (авторизация, кэширование, архитектура и т.д.)

### Шаг 3: Если найдены похожие дискуссии

Показать пользователю список похожих дискуссий:

```markdown
## Найдены похожие дискуссии

| ID | Название | Статус | Релевантность |
|----|----------|--------|---------------|
| 001 | [Название](путь) | draft | Высокая — [причина] |
| 003 | [Название](путь) | final | Средняя — [причина] |

**Варианты:**
1. Продолжить существующую дискуссию (указать ID)
2. Создать новую дискуссию (темы отличаются)
```

Спросить пользователя через AskUserQuestion:
- Продолжить дискуссию [ID]
- Создать новую дискуссию

### Шаг 4: Если похожих дискуссий нет

Сообщить пользователю и перейти к созданию:

```markdown
Похожих дискуссий не найдено. Создаю новую дискуссию.
```

### Шаг 5: Создать новую дискуссию

1. **Определить ID** — следующий порядковый номер (001, 002, ...)
2. **Сформировать имя файла** — `[ID]_краткое_описание.md`
3. **Создать файл** по шаблону `llm_instructions/templates/discuss.md`
4. **Заполнить начальные данные:**
   - Статус: `draft`
   - Исходный запрос пользователя: дословная цитата
   - Анализ цели: кратко сформулировать проблему/вопрос

### Шаг 6: Обновить индекс

1. Добавить новую дискуссию в таблицу `000_discuss.md`
2. Обновить статистику (увеличить счётчик draft)
3. Обновить дату последнего обновления

### Шаг 7: Начать обсуждение

После создания файла:
1. Показать пользователю путь к созданному файлу
2. Предложить варианты решения по теме дискуссии
3. Спросить, какие варианты рассмотреть подробнее

## Формат ответа при создании

```markdown
## Создана дискуссия: [ID]_[название].md

**Файл:** `general_docs/01_discuss/[ID]_[название].md`
**Статус:** draft

### Ваш запрос
> [цитата]

### Предварительный анализ
[Краткое описание проблемы/вопроса]

### Предлагаемые варианты

| # | Вариант | Краткое описание |
|---|---------|------------------|
| 1 | [Вариант 1] | [Описание] |
| 2 | [Вариант 2] | [Описание] |
| 3 | [Вариант 3] | [Описание] |

Какие варианты рассмотреть подробнее?
```

## Примеры использования

### Пример 1: Новая дискуссия без похожих

**Сообщение пользователя:**
```
Дискуссия: какой формат хранения сессий выбрать — Redis или PostgreSQL?
```

**Действия:**
1. Извлечь тему: "формат хранения сессий — Redis или PostgreSQL"
2. Прочитать индекс дискуссий
3. Похожих не найдено
4. Создать `001_session_storage.md`
5. Обновить индекс
6. Предложить варианты: Redis, PostgreSQL, JWT без хранения

### Пример 2: Найдена похожая дискуссия

**Сообщение пользователя:**
```
Дискуссия: нужно выбрать способ аутентификации пользователей
```

**Действия:**
1. Извлечь тему: "способ аутентификации пользователей"
2. Прочитать индекс дискуссий
3. Найдена похожая: `002_user_authentication.md` (статус: in_progress)
4. Показать пользователю:
   - Найдена дискуссия `002_user_authentication.md` — "Аутентификация пользователей"
   - Статус: in_progress
   - Варианты: продолжить или создать новую
5. Дождаться выбора пользователя

### Пример 3: Продолжение существующей дискуссии

**Пользователь выбрал:** Продолжить дискуссию 002

**Действия:**
1. Прочитать файл `002_user_authentication.md`
2. Показать текущее состояние дискуссии
3. Предложить следующие шаги:
   - Добавить новый вариант
   - Обсудить существующие варианты
   - Принять решение
