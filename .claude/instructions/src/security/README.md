## 1. Auth

**Файл:** [auth.md](./auth.md)

**Тип:** standard

**Описание:** Правила аутентификации и авторизации при межсервисном взаимодействии.

**Ключевые правила:**
- Zero Trust: каждый запрос проверяется, даже внутри периметра
- JWT токены с коротким TTL (15 минут максимум)
- Asymmetric keys: RS256/ES256 для подписи
- Service accounts с минимальными правами (Least Privilege)

**Структура JWT:**
```json
{
  "iss": "auth-service",
  "sub": "svc-payment",
  "aud": ["users-service"],
  "exp": 1705320900,
  "scope": ["read:users"]
}
```

- [../../platform/security.md](../../platform/security.md) -- инфраструктура безопасности
- [../runtime/resilience.md](../runtime/resilience.md) -- retry при сбоях auth

## 2. Audit

**Файл:** [audit.md](./audit.md)

**Тип:** standard

**Описание:** Правила ведения аудит-логов, защиты персональных данных и соответствия GDPR.

**Ключевые правила:**
- Принцип: Кто / Что / Когда -- actor, action, timestamp
- Структура аудит-записи: event_id, actor, action, resource, changes, result
- PII: маскирование, шифрование, минимизация
- GDPR: право на удаление, экспорт данных, согласие

**Data Retention:**
| Тип данных | Срок хранения |
|------------|---------------|
| Аудит-логи | 7 лет |
| Операционные логи | 90 дней |
| PII | До отзыва согласия |

**Связанные инструкции:**
- [auth.md](./auth.md) -- аудит аутентификации
- [../data/logging.md](../data/logging.md) -- формат логирования
- [../../platform/observability/logging.md](../../platform/observability/logging.md) -- инфраструктура логирования

## Принципы безопасности

| Принцип | Описание | Реализация |
|---------|----------|------------|
| **Zero Trust** | Не доверяем никому по умолчанию | JWT проверка на каждом сервисе |
| **Least Privilege** | Минимальные необходимые права | Scope в JWT токене |
| **Defense in Depth** | Несколько уровней защиты | Auth + Audit + Encryption |
| **Fail Secure** | При сбое -- отказ в доступе | 401/403 при невалидном токене |
