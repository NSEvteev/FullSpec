## Ошибки валидации

### Путь в исключённой директории

**Симптом:**
```
❌ Путь /doc/src/auth/handlers.md находится в исключённой директории
```

**Причина:** Попытка создать документацию для файла в `/doc/`, `/.claude/`, `/tests/`.

**Решение:** Документация создаётся только для файлов в `/src/`, `/shared/`, `/platform/`.

---

### Файл не найден

**Симптом:**
```
❌ Файл /src/auth/handlers.ts не найден
```

**Причина:** Исходный файл не существует.

**Решение:**
1. Проверить путь: `ls /src/auth/`
2. Создать файл перед документированием
3. Для удалённых файлов использовать `/docs-delete`

---

### Самоописывающийся файл

**Симптом:**
```
❌ Файл /src/auth/README.md уже является документацией
```

**Причина:** Файлы `.md`, `.rst`, `.txt`, `.adoc` не требуют документации.

**Решение:** Эти файлы сами являются документацией, их не нужно документировать.

---

### Документация уже существует

**Симптом:**
```
⚠️ Документация уже существует: /doc/src/auth/handlers.md
```

**Решение:**
- Для обновления: `/docs-update /src/auth/handlers.ts`
- Для перезаписи: `/docs-create /src/auth/handlers.ts --force`

---

## Ошибки файловой системы

### Нет прав на запись

**Симптом:**
```
❌ Нет прав на запись: /doc/src/auth/handlers.md
```

**Решение:**
```bash
# Проверить права
ls -la /doc/src/auth/

# Изменить права (если нужно)
chmod 644 /doc/src/auth/handlers.md
```

---

### Директория не существует

**Симптом:**
```
❌ Директория /doc/src/auth/ не существует
```

**Решение:** Скилл создаёт директории автоматически. Если ошибка возникает:
```bash
mkdir -p /doc/src/auth/
```

---

### Диск заполнен

**Симптом:**
```
❌ Не удалось записать файл: No space left on device
```

**Решение:**
1. Проверить место: `df -h`
2. Очистить временные файлы
3. Повторить операцию

---

## Ошибки шаблонов

### Шаблон не найден

**Симптом:**
```
❌ Шаблон backend-template.md не найден
```

**Причина:** Шаблоны должны находиться в `/.claude/templates/docs/`.

**Решение:**
1. Проверить наличие: `ls /.claude/templates/docs/`
2. Восстановить из git: `git checkout HEAD -- .claude/templates/docs/`

---

### Неподдерживаемый тип файла

**Симптом:**
```
⚠️ Неизвестное расширение .xyz — будет использован minimal-template.md
```

**Это предупреждение, не ошибка.** Скилл использует минимальный шаблон.

**Чтобы добавить поддержку:**
1. Обновить [rules.md](./rules.md) — добавить расширение
2. Создать шаблон (если нужен специфический)

## Ошибки анализа кода

### Не удалось распарсить файл

**Симптом:**
```
⚠️ Не удалось извлечь API из /src/auth/handlers.ts
```

**Причина:** Синтаксические ошибки в файле или неподдерживаемый синтаксис.

**Решение:**
1. Проверить синтаксис файла
2. Скилл создаст документацию с пустой секцией API
3. Заполнить API вручную

---

### Пустой файл

**Симптом:**
```
⚠️ Файл /src/auth/handlers.ts пуст
```

**Решение:** Сначала добавьте код, затем документируйте.

## Ошибки цепочек

### links-update не вызван

**Симптом:**
```
⚠️ Не удалось обновить ссылки
```

**Причина:** Ошибка при вызове `/links-update` после создания/обновления.

**Решение:**
```
/links-update /doc/src/auth/handlers.md
```

---

### issue-create не вызван

**Симптом (при docs-delete):**
```
⚠️ Не удалось создать Issue для ревью
```

**Причина:** Ошибка GitHub API или отсутствие прав.

**Решение:**
1. Проверить авторизацию: `gh auth status`
2. Создать Issue вручную:
```bash
gh issue create --title "[DOCS] Ревью документации /doc/src/auth/handlers.md" --label "docs"
```

## Откат изменений

### Откат docs-create

Если создание документации прошло с ошибками:

```bash
# Удалить созданный файл
rm /doc/src/auth/handlers.md

# Удалить ссылку из исходного файла (если добавлена)
# Вручную или через git
git checkout HEAD -- /src/auth/handlers.ts
```

### Откат docs-update

```bash
# Вернуть предыдущую версию документации
git checkout HEAD~1 -- /doc/src/auth/handlers.md
```

### Откат docs-delete

```bash
# Восстановить файл
git checkout HEAD~1 -- /doc/src/auth/handlers.md

# Убрать пометку о ревью (если добавлена)
# Вручную отредактировать файл
```

## Формат сообщений об ошибках

### Структура

```
❌ Ошибка: {краткое описание}

Путь: {путь к файлу}
Причина: {детали}

Решение:
{конкретные шаги}

Документация: {ссылка}
```

### Пример

```
❌ Ошибка: Не удалось создать документацию

Путь: /src/auth/handlers.ts
Причина: Документация уже существует

Решение:
1. Для обновления: /docs-update /src/auth/handlers.ts
2. Для перезаписи: /docs-create /src/auth/handlers.ts --force

Документация: /.claude/instructions/docs/workflow.md
```

## Предупреждения vs Ошибки

| Тип | Символ | Действие |
|-----|--------|----------|
| Ошибка (блокирующая) | ❌ | Остановить выполнение |
| Предупреждение | ⚠️ | Продолжить с ограничениями |
| Информация | ℹ️ | Продолжить |
| Успех | ✅ | Завершено |

### Блокирующие ошибки

- Файл не найден
- Путь в исключённой директории
- Нет прав на запись
- Шаблон не найден

### Предупреждения (не блокируют)

- Документация уже существует
- Неизвестное расширение
- Не удалось извлечь API
- Не удалось обновить ссылки
