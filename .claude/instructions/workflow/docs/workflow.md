## docs-create

Создание документации для нового файла в `/src/`.

### Формат вызова

```
/docs-create <путь-к-файлу> [--dry-run] [--no-link]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь-к-файлу` | Путь к исходному файлу | — (обязательный) |
| `--dry-run` | Предварительный просмотр без изменений | false |
| `--no-link` | Не добавлять ссылку в исходный файл | false |

### Воркфлоу (9 шагов)

```
Шаг 1: Валидация пути
    ↓
Шаг 2: Определение типа файла
    ↓
Шаг 3: Выбор шаблона
    ↓
Шаг 4: Анализ исходного файла
    ↓
Шаг 5: Генерация документации
    ↓
Шаг 6: Создание директорий
    ↓
Шаг 7: Запись файла документации
    ↓
Шаг 8: Добавление ссылки в исходный файл
    ↓
Шаг 9: Вызов /links-update
```

### Шаги подробно

**Шаг 1: Валидация пути**
- Проверить, что файл существует
- Проверить, что путь не исключён (не в `/doc/`, `/.claude/`, `/tests/`)
- Проверить, что файл не самоописывающийся (`.md`, `.rst`)

**Шаг 2: Определение типа файла**
- По расширению: `.ts`, `.py`, `.sql`, `.tsx` и др.
- По пути: `handlers`, `services`, `schema`, `migrations`

**Шаг 3: Выбор шаблона**
| Тип | Шаблон |
|-----|--------|
| Backend handlers/services | `backend-template.md` |
| Database schema/migrations | `database-template.md` |
| Frontend components/pages | `frontend-template.md` |
| Прочее | `minimal-template.md` |

**Шаг 4: Анализ исходного файла**
- Извлечь экспортируемые функции/классы
- Извлечь типы и интерфейсы
- Извлечь зависимости (imports)

**Шаг 5: Генерация документации**
- Применить шаблон
- Заполнить секции API
- Добавить ссылку на исходный файл

**Шаг 6-7: Создание файлов**
- Создать директории по пути
- Записать файл документации

**Шаг 8: Добавление ссылки**
- В начало исходного файла добавить комментарий со ссылкой на документацию

**Шаг 9: Обновление ссылок**
- Вызвать `/links-update` для обновления связей

### Режим --dry-run

```
/docs-create /src/auth/handlers.ts --dry-run
```

Выводит:
```
📋 Предварительный просмотр (--dry-run)

Исходный файл: /src/auth/handlers.ts
Документация: /doc/src/auth/handlers.md
Шаблон: backend-template.md

Будет создано:
───────────────────────────────────────
# Auth Handlers

> Исходный код: [handlers.ts](/src/auth/handlers.ts)

## API

### login
...
───────────────────────────────────────

ℹ️ Изменения НЕ применены (--dry-run)
```

### Откат при ошибке

Если ошибка на шаге 7-8:
1. Удалить созданный файл документации
2. Сообщить об ошибке
3. Не вызывать `/links-update`

## docs-update

Обновление документации при изменении исходного файла.

### Формат вызова

```
/docs-update [путь] [--diff] [--auto] [--force]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь` | Путь к файлу (или автоопределение) | Изменённые файлы |
| `--diff` | Показать изменения без применения | false |
| `--auto` | Применить без подтверждения | false |
| `--force` | Перезаписать даже если нет изменений | false |

### Воркфлоу

```
Шаг 1: Определить изменённые файлы
    ↓
Шаг 2: Найти соответствующую документацию
    ↓
Шаг 3: Проанализировать изменения в исходном файле
    ↓
Шаг 4: Определить тип изменений
    ↓
Шаг 5: Сгенерировать обновления
    ↓
Шаг 6: Показать diff / применить
    ↓
Шаг 7: Вызов /links-update
```

### Типы изменений API

| Тип | Описание | Действие |
|-----|----------|----------|
| Добавлена функция | Новый export | Добавить в секцию API |
| Удалена функция | Удалён export | Удалить из документации |
| Изменена сигнатура | Параметры/возврат | Обновить сигнатуру |
| Изменены зависимости | Imports | Обновить секцию Dependencies |

### Режим --diff

```
/docs-update /src/auth/handlers.ts --diff
```

Выводит:
```
📋 Изменения в документации

Файл: /doc/src/auth/handlers.md

@@ -15,6 +15,12 @@ API
 ### login
 ...

+### logout
+
+Выход из системы.
+
+**Сигнатура:**
+```typescript
+export function logout(token: string): Promise<void>
+```

Применить изменения? [Y/n]
```

### Режим --auto

```
/docs-update --auto
```

- Автоматически находит все изменённые файлы в `/src/`
- Обновляет документацию без подтверждения
- Выводит сводку изменений

## docs-delete

Обработка удаления исходного файла.

### Формат вызова

```
/docs-delete <путь-к-файлу> [--no-issue] [--remove]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `путь-к-файлу` | Путь к удалённому файлу | — (обязательный) |
| `--no-issue` | Не создавать GitHub Issue | false |
| `--remove` | Удалить документацию сразу | false |

### Воркфлоу

```
Шаг 1: Найти документацию для удалённого файла
    ↓
Шаг 2: Добавить пометку о требовании ревью
    ↓
Шаг 3: Создать GitHub Issue (если не --no-issue)
    ↓
Шаг 4: Вызов /links-delete для обновления ссылок
```

### Формат пометки в документации

```markdown
> ⚠️ **ТРЕБУЕТ РЕВЬЮ:** Исходный файл `/src/auth/handlers.ts` был удалён.
>
> - [ ] Удалить этот файл документации
> - [ ] Или обновить ссылку на новый файл
>
> См. Issue: #123
```

### Формат GitHub Issue

```markdown
## Описание

Файл `/src/auth/handlers.ts` был удалён.
Документация `/doc/src/auth/handlers.md` требует ревью.

## Действия

- [ ] Удалить документацию
- [ ] Или обновить ссылки на новый файл
- [ ] Обновить связанные документы

## Связанные файлы

- /doc/src/auth/handlers.md
```

### Режим --remove

```
/docs-delete /src/auth/handlers.ts --remove
```

- Сразу удаляет файл документации
- Не создаёт Issue
- Вызывает `/links-delete` для очистки ссылок

**Использовать осторожно** — теряется история документации.

## docs-reindex

Полная переиндексация документации проекта.

### Формат вызова

```
/docs-reindex [--check] [--json] [--verbose] [--fix]
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--check` | Только проверка, без изменений | false |
| `--json` | JSON формат вывода | false |
| `--verbose` | Подробный вывод | false |
| `--fix` | Автоматическое исправление проблем | false |

### Воркфлоу

```
Шаг 1: Сканировать /src/ на наличие файлов
    ↓
Шаг 2: Сканировать /doc/src/ на наличие документации
    ↓
Шаг 3: Построить карту соответствий
    ↓
Шаг 4: Выявить проблемы
    ↓
Шаг 5: Вывести отчёт / исправить
```

### Типы проблем

| Проблема | Описание | Исправление |
|----------|----------|-------------|
| Missing | Файл есть, документации нет | `/docs-create` |
| Orphan | Документация есть, файла нет | `/docs-delete` |
| Outdated | Документация устарела | `/docs-update` |
| Broken link | Битая ссылка в документации | `/links-update` |

### Режим --check

```
/docs-reindex --check
```

Выводит:
```
📋 Проверка документации

✅ Синхронизировано: 45 файлов
⚠️ Проблемы: 3

Missing (нет документации):
- /src/auth/2fa.ts

Orphan (нет исходного файла):
- /doc/src/auth/old-handlers.md

Outdated (требует обновления):
- /doc/src/notify/email.md

Для исправления: /docs-reindex --fix
```

### Режим --fix

```
/docs-reindex --fix
```

- Создаёт недостающую документацию
- Помечает orphan-документы
- Обновляет устаревшую документацию

## Цепочки вызовов

### docs-create → links-update

```
/docs-create /src/auth/handlers.ts
    ↓
/links-update  # Автоматически
```

### docs-update → links-update

```
/docs-update /src/auth/handlers.ts
    ↓
/links-update  # Автоматически
```

### docs-delete → issue-create → links-delete

```
/docs-delete /src/auth/handlers.ts
    ↓
/issue-create  # Создать Issue для ревью
    ↓
/links-delete  # Обновить ссылки
```

### docs-reindex → docs-*

```
/docs-reindex --fix
    ↓
/docs-create   # Для missing
    ↓
/docs-delete   # Для orphan
    ↓
/docs-update   # Для outdated
```
