## Маппинг путей

### Правило

```
/{any-path}/{file}.{ext} → /doc/{any-path}/{file}.md
```

**Принцип:** Документация зеркалирует структуру проекта в папке `/doc/`.

### Примеры

| Исходный файл | Документация |
|---------------|--------------|
| `/src/auth/backend/handlers.ts` | `/doc/src/auth/backend/handlers.md` |
| `/config/settings.yaml` | `/doc/config/settings.md` |
| `/platform/docker/compose.yml` | `/doc/platform/docker/compose.md` |
| `/shared/utils/helpers.ts` | `/doc/shared/utils/helpers.md` |
| `/Makefile` | `/doc/Makefile.md` |

### Исключённые пути

| Путь | Причина |
|------|---------|
| `/doc/**` | Целевая папка документации |
| `/.claude/**` | Уже содержит документацию (SKILL.md, инструкции) |
| `/.git/**` | Служебные файлы Git |
| `/tests/**` | Тесты (описываются в инструкциях) |
| `/node_modules/**` | Внешние зависимости |
| `/.venv/**`, `/venv/**` | Виртуальные окружения Python |

### Самоописывающиеся файлы (исключены)

| Расширение | Тип файла |
|------------|-----------|
| `.md` | Markdown — уже документация |
| `.rst` | reStructuredText — уже документация |
| `.txt` | Текстовые файлы |
| `.adoc` | AsciiDoc — уже документация |

## Валидация путей

### Блокирующие проверки

| Проверка | Ошибка |
|----------|--------|
| Путь НЕ в `/doc/**` | "Нельзя документировать документацию" |
| Путь НЕ в `/.claude/**` | "/.claude/ уже содержит документацию" |
| Путь НЕ в `/.git/**` | "Служебные файлы Git не документируются" |
| Файл НЕ самоописывающийся | "Файл уже является документацией" |
| Файл существует | "Файл {путь} не найден" (для doc-create, doc-update) |

### Предупреждения

| Проверка | Предупреждение |
|----------|----------------|
| Файл уже задокументирован | "Документация существует: {путь}" |
| Неизвестное расширение | "Будет создан минимальный шаблон" |
| Путь в `/node_modules/` | "Не рекомендуется" |

## Поддерживаемые языки

### Языки программирования

| Расширение | Язык | Как извлекать API |
|------------|------|-------------------|
| `.ts`, `.tsx` | TypeScript | `export function`, `export class` |
| `.js`, `.jsx` | JavaScript | `export`, `module.exports` |
| `.py` | Python | `def`, `class` без `_` префикса |
| `.go` | Go | Функции и типы с заглавной буквы |
| `.sql` | SQL | `CREATE TABLE`, `CREATE FUNCTION` |
| `.rs` | Rust | `pub fn`, `pub struct` |
| `.java` | Java | `public class`, `public` методы |

### Конфигурационные файлы

| Расширение | Что документировать |
|------------|---------------------|
| `.yaml`, `.yml` | Структура, ключевые поля |
| `.json` | Схема, обязательные поля |
| `.toml` | Секции, параметры |
| `.env` | Переменные окружения (без значений!) |
| `Dockerfile` | Этапы сборки, порты |
| `docker-compose.yml` | Сервисы, сети, volumes |
| `Makefile` | Targets, переменные |

## Шаблон документации

> **SSOT:** Шаблоны находятся в [templates.md](./templates.md)

### Базовая структура

```markdown
# {Название модуля}

> Исходный код: [{filename}](/{path-to-source})

{Краткое описание}

## API

### {FunctionName}

{Описание}

**Сигнатура:**
\`\`\`{language}
{signature}
\`\`\`

---

## Зависимости

- [{dependency}](/{path}) — {description}
```

## Формат пометки (doc-delete)

Когда исходный файл удалён:

```markdown
> ⚠️ **ТРЕБУЕТ РЕВЬЮ:** Исходный файл `{путь}` был удалён.
>
> - [ ] Удалить этот файл документации
> - [ ] Или обновить ссылку на новый файл
>
> См. Issue: #{issue-number}
```

## Связанные скиллы

| Событие | Скилл |
|---------|-------|
| Создан новый файл | `/docs-create` |
| Изменён файл | `/docs-update` |
| Удалён файл | `/docs-delete` |
| Перемещён файл | `/docs-delete` + `/docs-create` |

**Цепочки вызовов:**
```
doc-create → links-update
doc-update → links-update
doc-delete → issue-create → links-delete
```
