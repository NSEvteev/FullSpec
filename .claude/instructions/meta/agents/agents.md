## Назначение

Папка `.claude/agents/` содержит конфигурации специализированных AI-агентов для автоматизации задач в проекте.

## Структура

```
.claude/
└── agents/
    ├── [имя-агента].md     # Конфигурация агента
    └── [другие агенты]
```

## Что такое агент

**[📖 Агент](../general_docs/glossary.md#агент)** — специализированный AI-помощник, работающий в отдельном контексте со своим набором инструментов, system prompt и разрешениями.

### Отличия от [📖 скиллов](../general_docs/glossary.md#скилл)

| Аспект | [📖 Агенты](../general_docs/glossary.md#агент) | [📖 Скиллы](../general_docs/glossary.md#скилл) |
|--------|--------|--------|
| Контекст | Отдельный (изолированный) | Основная беседа |
| Инструменты | Собственный набор | Наследуют от основной беседы |
| Применение | Сложные многошаговые задачи | Методологии и стандарты |

## Формат файла агента

```yaml
---
name: имя-агента
description: Описание агента и когда его использовать.
tools: Read, Grep, Glob, Bash
model: sonnet
skills: skill-1, skill-2
---

Системный промпт агента.

Инструкции, чек-листы, правила работы.
```

### Обязательные поля

| Поле | Описание |
|------|----------|
| `name` | Уникальное имя (строчные буквы, дефисы, до 64 символов) |
| `description` | Описание назначения и условий использования |

### Опциональные поля

| Поле | Описание |
|------|----------|
| `tools` | Доступные инструменты: Read, Write, Edit, Bash, Grep, Glob и др. |
| `model` | Модель: `haiku`, `sonnet`, `opus` |
| `skills` | Список [📖 скиллов](../general_docs/glossary.md#скилл) через запятую |

## Workflow управления задачами агентов

**См. подробнее:** [tasks.md](tasks.md#задачи-агентов)

### Структура файлов агента

```
llm_tasks/agents/[имя-агента]/
├── current_tasks.md      # Текущие задачи
├── future_tasks.md       # Бэклог
├── completed_tasks.md    # Архив завершённых задач
└── temp/                 # Временные файлы
```

### Ключевые правила:

1. **При запуске:** Показать "Что было сделано в прошлый раз"
2. **При начале новой задачи:** Очистить `temp/`
3. **Во время работы:** Сохранять временные файлы в `temp/`
4. **При завершении:** Переместить задачу в `completed_tasks.md` (вверх файла)

## Добавление нового агента

При создании нового агента:

1. Создать файл `.claude/agents/[имя-агента].md`
2. Заполнить frontmatter с обязательными полями
3. Написать системный промпт с инструкциями
4. Указать необходимые инструменты (минимально достаточный набор)
5. Привязать [📖 скиллы](../general_docs/glossary.md#скилл) через поле `skills` (если нужно)
6. **Создать структуру задач:**
   - `llm_tasks/agents/[имя-агента]/current_tasks.md`
   - `llm_tasks/agents/[имя-агента]/future_tasks.md`
   - `llm_tasks/agents/[имя-агента]/completed_tasks.md`
   - `llm_tasks/agents/[имя-агента]/temp/` (пустая папка)
7. **Добавить в конфигурацию агента** секцию об управлении задачами и `temp/`
8. Добавить описание агента в этот файл
