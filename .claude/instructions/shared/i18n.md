## Правила

### Один язык — одна папка

**Правило:** Каждая локаль в отдельной папке.

```
/i18n/
  /ru/              # Русский
  /en/              # Английский
  /kk/              # Казахский (если нужен)
```

### Русский — базовый язык

**Правило:** Все строки сначала на русском, затем переводятся.

**Причина:** Проект ориентирован на русскоязычный рынок. Русский — source of truth.

### Не хардкодить строки

**Правило:** Все пользовательские строки через i18n.

```typescript
// Правильно
t('common.save')

// Неправильно
'Сохранить'
```

### Ключи — на английском

**Правило:** Ключи пишутся на английском в dot-notation.

```json
{
  "common.save": "Сохранить",
  "common.cancel": "Отмена"
}
```

## Формат ключей

### Структура ключа

```
{namespace}.{section}.{element}[.{modifier}]
```

| Компонент | Описание | Пример |
|-----------|----------|--------|
| `namespace` | Файл перевода | `common`, `errors`, `auth` |
| `section` | Логическая группа | `button`, `form`, `message` |
| `element` | Конкретный элемент | `save`, `email`, `success` |
| `modifier` | Модификатор (опционально) | `plural`, `short` |

### Примеры ключей

| Ключ | Значение |
|------|----------|
| `common.button.save` | Сохранить |
| `common.button.cancel` | Отмена |
| `common.button.delete` | Удалить |
| `auth.form.email` | Email |
| `auth.form.password` | Пароль |
| `auth.message.login_success` | Вы успешно вошли |
| `errors.network.timeout` | Превышено время ожидания |
| `validation.email.invalid` | Некорректный email |

### Правила именования

| Правило | Пример |
|---------|--------|
| snake_case для ключей | `login_success`, не `loginSuccess` |
| Множественное число для коллекций | `items`, не `item` |
| Глаголы в инфинитиве | `save`, не `saving` |
| Прилагательные для состояний | `empty`, `loading`, `error` |

## Плюрализация

### Формат

```json
{
  "items.count_one": "{{count}} элемент",
  "items.count_few": "{{count}} элемента",
  "items.count_many": "{{count}} элементов"
}
```

### Правила для русского

| Число | Суффикс | Пример |
|-------|---------|--------|
| 1, 21, 31... | `_one` | 1 элемент |
| 2-4, 22-24... | `_few` | 2 элемента |
| 0, 5-20, 25-30... | `_many` | 5 элементов |

### Использование

```typescript
// i18next
t('items.count', { count: 1 });  // "1 элемент"
t('items.count', { count: 3 });  // "3 элемента"
t('items.count', { count: 5 });  // "5 элементов"
```

### Примеры плюрализации

```json
{
  "users.online_one": "{{count}} пользователь онлайн",
  "users.online_few": "{{count}} пользователя онлайн",
  "users.online_many": "{{count}} пользователей онлайн",

  "files.selected_one": "Выбран {{count}} файл",
  "files.selected_few": "Выбрано {{count}} файла",
  "files.selected_many": "Выбрано {{count}} файлов",

  "days.remaining_one": "Остался {{count}} день",

}
```

## Форматирование

### Интерполяция переменных

```json
{
  "greeting": "Привет, {{name}}!",
  "welcome_back": "С возвращением, {{name}}! Последний визит: {{date}}"
}
```

```typescript
t('greeting', { name: 'Иван' });  // "Привет, Иван!"
```

### Форматирование дат

```typescript
// Использовать Intl.DateTimeFormat
const date = new Date();

const formatted = new Intl.DateTimeFormat('ru-RU', {
  year: 'numeric',
  month: 'long',
  day: 'numeric'
}).format(date);
// "15 января 2024 г."
```

### Форматирование чисел

```typescript
// Использовать Intl.NumberFormat
const number = 1234567.89;

// Числа
new Intl.NumberFormat('ru-RU').format(number);
// "1 234 567,89"

// Валюта
new Intl.NumberFormat('ru-RU', {
  style: 'currency',
  currency: 'RUB'
}).format(number);
// "1 234 567,89 ₽"

// Проценты
new Intl.NumberFormat('ru-RU', {
  style: 'percent',
  minimumFractionDigits: 1
}).format(0.1234);
// "12,3 %"
```

## Процесс локализации

### Добавление новой строки

1. Добавить ключ и значение в русский файл
2. Добавить ключ в TypeScript типы (если используются)
3. Добавить переводы в другие локали (если есть)

### Добавление новой локали

1. Создать папку `/shared/i18n/{locale}/`
2. Скопировать все файлы из `/ru/`
3. Перевести строки
4. Добавить локаль в конфигурацию

### CI проверки

```yaml
# .github/workflows/i18n.yml
jobs:
  check-i18n:
    steps:
      - name: Check missing keys
        run: npm run i18n:check

      - name: Check unused keys
        run: npm run i18n:unused
```

### Скрипты

```json
// package.json
{
  "scripts": {
    "i18n:check": "i18n-check --source ru --targets en",
    "i18n:unused": "i18n-unused --source src --translations shared/i18n",
    "i18n:sort": "i18n-sort shared/i18n/**/*.json"
  }
}
```

## Примеры

### Пример 1: Базовое использование (React)

```tsx
import { useTranslation } from 'react-i18next';

function LoginForm() {
  const { t } = useTranslation();

  return (
    <form>
      <h1>{t('auth.login.title')}</h1>

      <input
        type="email"
        placeholder={t('auth.form.email')}
      />

      <input
        type="password"
        placeholder={t('auth.form.password')}
      />

      <button type="submit">
        {t('auth.login.button')}
      </button>

      <a href="/forgot">
        {t('auth.login.forgot_password')}
      </a>
    </form>
  );
}
```

### Пример 2: Плюрализация

```tsx
function ItemsCount({ count }) {
  const { t } = useTranslation();

  return (
    <span>
      {t('items.count', { count })}
    </span>
  );
}

// <ItemsCount count={1} />  → "1 элемент"
// <ItemsCount count={3} />  → "3 элемента"
// <ItemsCount count={5} />  → "5 элементов"
```

### Пример 3: Форматирование с переменными

```tsx
function WelcomeMessage({ user, lastVisit }) {
  const { t } = useTranslation();

  const formattedDate = new Intl.DateTimeFormat('ru-RU', {
    dateStyle: 'long',
    timeStyle: 'short'
  }).format(lastVisit);

  return (
    <p>
      {t('welcome_back', {
        name: user.name,
        date: formattedDate
      })}
    </p>
  );
}
```

### Пример 4: Ошибки валидации

```tsx
function ValidationError({ field, error }) {
  const { t } = useTranslation();

  const messages = {
    required: t('validation.required'),
    minLength: t('validation.min_length', { min: error.min }),
    maxLength: t('validation.max_length', { max: error.max }),
    email: t('validation.email.invalid')
  };

  return (
    <span className="error">
      {messages[error.type] || t('errors.generic.unknown')}
    </span>
  );
}
```

### Пример 5: Конфигурация i18next

```typescript
// /shared/i18n/index.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';

// Импорт переводов
import ruCommon from './ru/common.json';
import ruErrors from './ru/errors.json';
import ruValidation from './ru/validation.json';
import ruAuth from './ru/auth.json';

i18n
  .use(initReactI18next)
  .init({
    resources: {
      ru: {
        translation: {
          ...ruCommon,
          ...ruErrors,
          ...ruValidation,
          ...ruAuth
        }
      }
    },
    lng: 'ru',
    fallbackLng: 'ru',
    interpolation: {
      escapeValue: false
    }
  });

export default i18n;
```
