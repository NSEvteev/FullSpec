## Назначение

Папка `/.claude/state/` предназначена для:
- Временных файлов, создаваемых скиллами
- Промежуточных результатов работы агентов
- Кэша между этапами выполнения
- Логов отладки (debug mode)

**Ключевой принцип:** Файлы в state/ могут быть удалены в любой момент без потери функциональности.

---

## Структура

```
/.claude/state/
  {skill-name}/           # папка для скилла
    {session-id}.json     # данные сессии
    {timestamp}.log       # логи выполнения
  {agent-name}/           # папка для агента
    context.json          # контекст агента
    cache/                # кэш агента
```

### Именование файлов

| Паттерн | Назначение | Пример |
|---------|------------|--------|
| `{skill-name}/{id}.json` | Данные сессии скилла | `test-execute/2025-01-21-001.json` |
| `{skill-name}/{id}.log` | Логи выполнения | `skill-create/debug.log` |
| `{agent-name}/context.json` | Контекст агента | `explorer/context.json` |

## Правила

### Правило 1: Изоляция по скиллу/агенту

Каждый скилл или агент использует **свою подпапку**.

```
✅ Правильно:  /.claude/state/test-execute/result.json
❌ Неправильно: /.claude/state/result.json
```

### Правило 2: Автоочистка

Скилл/агент **отвечает за очистку** своих временных файлов:
- При успешном завершении — удалить временные файлы
- При ошибке — оставить для отладки (с меткой времени)

### Правило 3: Отсутствие зависимости

Функциональность **не должна зависеть** от наличия файлов в state/.

```
✅ Правильно:  Проверить наличие файла, если нет — создать заново
❌ Неправильно: Падать с ошибкой, если файл не найден
```

### Правило 4: Формат JSON

Предпочтительный формат для данных — **JSON**.

```json
{
  "created": "2025-01-21T12:00:00Z",
  "skill": "test-execute",
  "data": { ... }
}
```

### Правило 5: Gitignore

Папка `/.claude/state/` добавлена в `.gitignore`. Файлы не попадают в репозиторий.

## Примеры использования

### Пример 1: Сохранение промежуточного результата

```bash
# Скилл test-execute сохраняет результаты тестов
mkdir -p .claude/state/test-execute
echo '{"passed": 10, "failed": 2}' > .claude/state/test-execute/results.json
```

### Пример 2: Логирование отладки

```bash
# При --verbose или ошибке — сохранить лог
echo "[2025-01-21 12:00:00] Step 1: Reading files..." >> .claude/state/skill-create/debug.log
```

### Пример 3: Контекст агента между сессиями

```json
// .claude/state/explorer/context.json
{
  "last_search": "authentication",
  "visited_files": ["/src/auth/handler.ts"],
  "timestamp": "2025-01-21T12:00:00Z"
}
```
