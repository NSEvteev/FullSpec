## Стандартная структура

### Обязательные шаги

| Шаг | Название | Описание |
|-----|----------|----------|
| 1 | Получить входные данные | Параметры, аргументы, контекст |
| 2-N | Основные шаги | Логика скилла |
| N+1 | Проверка по чек-листу | Верификация выполнения |
| N+2 | Результат | Итоговый отчёт |

### Опциональные шаги

| Шаг | Когда использовать |
|-----|-------------------|
| Валидация | Если есть входные ограничения |
| Подтверждение | Если изменения критичны |
| Ревью | Если нужна проверка качества |
| Синхронизация | Если есть связанные сущности |
| Откат | Если возможны ошибки |

## Шаблон шага

```markdown
### Шаг N: {Название шага}

{Описание что делает шаг — 1-2 предложения}

1. {Действие 1}
2. {Действие 2}
3. {Действие 3}

**Команда/Код:**
\`\`\`bash
{команда или код}
\`\`\`

**Результат:**
\`\`\`
{ожидаемый вывод}
\`\`\`
```

## Стандартные паттерны

### Паттерн: Получение входных данных

```markdown
### Шаг 1: Получить {объект}

1. Из аргумента: `/{skill-name} {arg}`
2. Или спросить: "{вопрос}?"
3. Валидация:
   - {условие 1}
   - {условие 2}
```

### Паттерн: Проверка существования

```markdown
### Шаг N: Проверить существование

1. Проверить файл/ресурс через `Glob` / `gh` / etc
2. Если существует:
   \`\`\`
   ⚠️ {Объект} уже существует: {путь}

   Варианты:
   [1] Перезаписать
   [2] Редактировать
   [3] Отменить
   \`\`\`
```

### Паттерн: Подтверждение действия

```markdown
### Шаг N: Показать предпросмотр

\`\`\`
📋 {Действие} {объекта}

{Детали}

{Действие}? [Y/n]
\`\`\`
```

### Паттерн: Ревью

```markdown
### Шаг N: Ревью

**Формат вывода:**
\`\`\`
📋 Ревью {объекта}

✅ Сильные стороны:
- {что хорошо}

💡 Предложения по улучшению:
1. {улучшение 1}
2. {улучшение 2}

Применить? [Y/n/выборочно]
\`\`\`
```

### Паттерн: Проверка по чек-листу

```markdown
### Шаг N: Проверка по чек-листу

\`\`\`
✅ Шаг 1: {описание}
✅ Шаг 2: {описание}
⚠️ Шаг 3: {пропущено} — выполняю...
✅ Шаг 4: {описание}
\`\`\`
```

### Паттерн: Результат

```markdown
### Шаг N: Результат

\`\`\`
✅ {Действие} завершено

{Секция 1}:
- {пункт 1}
- {пункт 2}

Следующие шаги:
- {рекомендация 1}
- {рекомендация 2}
\`\`\`
```

## Примеры наборов шагов

### Минимальный (простой скилл)

```
Шаг 1: Получить входные данные
Шаг 2: Выполнить действие
Шаг 3: Проверка по чек-листу
Шаг 4: Результат
```

### Стандартный (*-create скиллы)

```
Шаг 1: Получить параметры
Шаг 2: Проверить существование
Шаг 3: Сгенерировать содержимое
Шаг 4: Создать файлы
Шаг 5: Ревью
Шаг 6: Синхронизация ссылок
Шаг 7: Проверка по чек-листу
Шаг 8: Результат
```

### Расширенный (issue-execute)

```
Шаг 1: Получить номер Issue
Шаг 2: Показать Issue
Шаг 3: Назначить исполнителя
Шаг 4: Добавить метку
Шаг 5: Создать ветку
Шаг 6: Добавить комментарий
Шаг 6.5: Сохранить состояние
Шаг 7: Выполнить задачу
Шаг 8: Проверка по чек-листу
Шаг 9: Результат
Шаг 10: Ревью и завершение
```

## Режим --dry-run

При флаге `--dry-run`:

1. Выполнить все шаги до изменения файлов
2. Вывести план изменений:
   ```
   📋 Предварительный просмотр (--dry-run)

   Будет создано:
   - {список файлов}

   Будет изменено:
   - {список файлов}

   Будет удалено:
   - {список файлов}
   ```
3. НЕ вносить изменения
4. Вывести: `ℹ️ Изменения НЕ применены (--dry-run)`
