## Статусы тестов

| Статус | Символ | Описание | Когда использовать |
|--------|--------|----------|-------------------|
| `passed` | ✅ | Тест пройден | Все проверки успешны |
| `failed` | ❌ | Тест провален | Хотя бы одна проверка не прошла |
| `skipped` | ⏭️ | Тест пропущен | Не применим или заблокирован |
| `not_run` | ⬜ | Не выполнен | Начальное состояние |
| `flaky` | ⚠️ | Нестабильный | 3+ переключений за 7 дней |

### Диагностика flaky vs реальная ошибка

| Критерий | Flaky | Реальная ошибка |
|----------|:-----:|:---------------:|
| Воспроизводится? | Не всегда | Всегда |
| Зависит от внешних факторов? | Да | Нет |
| 3+ переключений за 7 дней | Да | Нет |
| Исправляется retry? | Да | Нет |

**При подозрении на flaky:**
1. Запустить тест 3 раза подряд
2. 3 failed → реальная ошибка
3. 1 failed → flaky → пометить, исследовать

## Типы тестов

> **Scope detection:** см. [shared/scope.md](../shared/scope.md)

### По scope

| Scope | Типы тестов |
|-------|-------------|
| `claude` | smoke, functional, integration |
| `project` | unit, integration, e2e, smoke, load |

### Описание типов

| Тип | Что проверяет | Когда запускать |
|-----|---------------|-----------------|
| `smoke` | Базовая работоспособность | После создания |
| `functional` | Корректность выполнения | После изменения |
| `integration` | Взаимодействие компонентов | После изменения воркфлоу |
| `regression` | Нет регрессий | Перед релизом |
| `unit` | Изолированные функции | При разработке |
| `e2e` | Сквозные сценарии | Перед деплоем |

## Шаблоны тестов

### Smoke test

```
📋 Smoke test: {name}

Команда: /{name}
Триггер: "{фраза}"

Ожидание:
- Скилл/код запускается
- Выводит ожидаемый результат
- Не падает с ошибкой

Результат: ⬜ Не выполнен
```

### Functional test

```
📋 Functional test: {name}

Сценарий: {описание}

Входные данные:
- param1: value1

Шаги:
1. {шаг 1}
2. {шаг 2}

Ожидаемый результат:
- {результат}

Фактический результат:
- {что получилось}

Статус: ⬜ Не выполнен
```

### Integration test

```
📋 Integration test: {название цепочки}

Сценарий: {описание взаимодействия}

Компоненты:
- {компонент 1}
- {компонент 2}

Шаги:
1. {шаг 1}
2. {шаг 2}

Проверки:
- [ ] {проверка 1}
- [ ] {проверка 2}

Статус: ⬜ Не выполнен
```

### Regression test

```
📋 Regression test: {название изменения}

Что изменилось:
- {описание}

Проверяемые сценарии:
1. {сценарий 1}
   - [ ] {проверка 1.1}
   - [ ] {проверка 1.2}

Статус: ⬜ Не выполнен
```

## Чек-листы

### Чек-лист smoke test

- [ ] Вызывается по команде `/{name}`
- [ ] Вызывается по русской фразе
- [ ] Вызывается по английской фразе
- [ ] Выводит описание при вызове без аргументов
- [ ] Не падает с ошибкой

### Чек-лист functional test

- [ ] **Позитивный сценарий:** основной use case работает
- [ ] **Негативный сценарий:** некорректные данные обрабатываются
- [ ] **Граничные случаи:** пустые значения, максимальные длины
- [ ] **Идемпотентность:** повторный вызов не ломает систему
- [ ] **Откат:** при ошибке состояние восстанавливается
