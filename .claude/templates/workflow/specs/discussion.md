## Контекст

**Проблема:** Текущая система использует только access tokens с коротким сроком жизни, что заставляет пользователей часто перелогиниваться.

**Область обсуждения:**

## Варианты решения

| # | Вариант | Плюсы | Минусы |

| A | Refresh tokens | Безопасно, можно отозвать | Сложнее реализация |
| B | Долгоживущий access token | Просто | Нельзя отозвать |

### Вариант A: Refresh tokens

**Описание:** Access token (1 час) + Refresh token (7 дней)

**Плюсы:**
- Можно отозвать сессию
- Баланс безопасности и UX

**Минусы:**
- Требуется Redis

### Вариант B: Долгоживущий access token

**Описание:** Увеличить время жизни access token до 7 дней

**Плюсы:**
- Простая реализация

**Минусы:**
- Нельзя отозвать до истечения
- Высокий риск при утечке

| UX | Высокий | Частота перелогинов |
