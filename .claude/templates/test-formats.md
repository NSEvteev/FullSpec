## Статусы тестов

| Статус | Символ | Описание | Когда использовать |
|--------|--------|----------|-------------------|
| `passed` | ✅ | Тест пройден | Все проверки успешны |
| `failed` | ❌ | Тест провален | Хотя бы одна проверка не прошла |
| `skipped` | ⏭️ | Тест пропущен | Тест не применим или заблокирован |
| `not_run` | ⬜ | Не выполнен | Начальное состояние |
| `flaky` | ⚠️ | Нестабильный | Проходит не всегда (3+ переключений за 7 дней) |

## Типы тестов

> **SSOT:** Полное описание типов по scope см. в [scope-detection.md](/.claude/templates/scope-detection.md#формат-тестов-по-scope).

**Общие типы тестов:**

| Тип | Что проверяет | Когда запускать |
|-----|---------------|-----------------|
| `smoke` | Базовая работоспособность | После создания |
| `functional` | Корректность выполнения | После изменения |
| `integration` | Взаимодействие компонентов | После изменения воркфлоу |
| `regression` | Нет регрессий | Перед релизом |

**По scope:**
- `claude`: smoke, functional, integration
- `project`: unit, integration, e2e, smoke, load

## Шаблон: Smoke test

```
📋 Smoke test: {name}

Команда: /{name}
Триггер: "{фраза}"

Ожидание:
- Скилл/код запускается
- Выводит ожидаемый результат
- Не падает с ошибкой

Результат: ⬜ Не выполнен
```

**Пример заполненного:**
```
📋 Smoke test: issue-create

Команда: /issue-create
Триггер: "создай задачу"

Ожидание:
- Скилл запускается
- Выводит список сервисов для выбора
- Запрашивает заголовок задачи

Результат: ✅ Passed (2026-01-20)
```

## Шаблон: Functional test

```
📋 Functional test: {name}

Сценарий: {описание сценария}

Входные данные:
- param1: value1
- param2: value2

Шаги:
1. {шаг 1}
2. {шаг 2}
3. {шаг 3}

Ожидаемый результат:
- {результат 1}
- {результат 2}

Фактический результат:
- {что получилось}

Статус: ⬜ Не выполнен
```

## Шаблон: Integration test

```
📋 Integration test: {название цепочки}

Сценарий: {описание взаимодействия}

Компоненты:
- {компонент 1}
- {компонент 2}

Шаги:
1. {шаг 1}
2. {шаг 2}

Проверки:
- [ ] {проверка 1}
- [ ] {проверка 2}

Статус: ⬜ Не выполнен
```

## Шаблон: Regression test

```
📋 Regression test: {название изменения}

Что изменилось:
- {описание изменения}

Проверяемые сценарии:

1. {сценарий 1}
   - [ ] {проверка 1.1}
   - [ ] {проверка 1.2}

2. {сценарий 2}
   - [ ] {проверка 2.1}
   - [ ] {проверка 2.2}

3. {сценарий 3}
   - [ ] {проверка 3.1}

Статус: ⬜ Не выполнен
```

## Шаблон: История запусков

```markdown
### История запусков

| Дата | Статус | Комментарий |
|------|--------|-------------|
| 2026-01-20 | ✅ Passed | — |
| 2026-01-19 | ❌ Failed | Ошибка в шаге 3 |
| 2026-01-18 | ✅ Passed | — |
```

```
└─────────────────────────────────────────────────────────────┘
```

## Чек-листы

### Чек-лист smoke test

- [ ] Вызывается по команде `/{name}`
- [ ] Вызывается по русской фразе
- [ ] Вызывается по английской фразе
- [ ] Выводит описание при вызове без аргументов
- [ ] Не падает с ошибкой при базовом вызове

### Чек-лист functional test

- [ ] **Позитивный сценарий:** основной use case работает
- [ ] **Негативный сценарий:** некорректные данные обрабатываются
- [ ] **Граничные случаи:** пустые значения, максимальные длины
- [ ] **Идемпотентность:** повторный вызов не ломает систему
- [ ] **Откат:** при ошибке состояние восстанавливается
