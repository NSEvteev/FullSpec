# SSOT: Правила документации (doc-*)

> **Тип:** SSOT (Single Source of Truth)
> **Используется в:** doc-create, doc-update, doc-delete
> **Связан с:** [scope-detection.md](./scope-detection.md)

---

## Назначение

Этот файл содержит общие правила для всех doc-* скиллов:
- [doc-create](/.claude/skills/doc-create/SKILL.md)
- [doc-update](/.claude/skills/doc-update/SKILL.md)
- [doc-delete](/.claude/skills/doc-delete/SKILL.md)

**Принцип SSOT:** Изменения вносятся ТОЛЬКО в этот файл. doc-* скиллы ССЫЛАЮТСЯ на него.

## Маппинг путей

### Правило маппинга

```
/{any-path}/{file}.{ext} → /doc/{any-path}/{file}.md
```

**Принцип:** Документация зеркалирует структуру проекта в папке `/doc/`.

**Примеры:**

| Исходный файл | Документация |
|---------------|--------------|
| `/src/auth/backend/handlers.ts` | `/doc/src/auth/backend/handlers.md` |
| `/config/settings.yaml` | `/doc/config/settings.md` |
| `/platform/docker/compose.yml` | `/doc/platform/docker/compose.md` |
| `/shared/utils/helpers.ts` | `/doc/shared/utils/helpers.md` |
| `/scripts/build.sh` | `/doc/scripts/build.md` |
| `/scripts/deploy.sh` | `/doc/scripts/deploy.md` |
| `/Makefile` | `/doc/Makefile.md` |

### Алгоритм преобразования

```python
def get_doc_path(file_path: str) -> str:
    """
    Преобразует любой путь в путь документации.

    /src/auth/handlers.ts → /doc/src/auth/handlers.md
    /config/settings.yaml → /doc/config/settings.md
    /Makefile → /doc/Makefile.md
    """
    # Проверка исключений
    if is_excluded_path(file_path):
        raise ValueError(f"Путь {file_path} исключён из документирования")

    if is_self_documenting(file_path):
        raise ValueError(f"Файл {file_path} самоописывающийся")

    # Убираем расширение (если есть), добавляем /doc/ в начало
    if '.' in file_path.split('/')[-1]:
        base_path = file_path.rsplit('.', 1)[0]
    else:
        base_path = file_path

    doc_path = '/doc' + base_path + '.md'

    return doc_path
```

### Исключённые пути

| Путь | Причина |
|------|---------|
| `/doc/**` | Целевая папка документации |
| `/.claude/**` | Уже содержит документацию (SKILL.md, инструкции) |
| `/.git/**` | Служебные файлы Git |
| `/tests/**` | Тесты (описываются в `/.claude/instructions/tests/`) |
| `/node_modules/**` | Внешние зависимости |
| `/.venv/**`, `/venv/**` | Виртуальные окружения Python |

### Самоописывающиеся файлы (исключены)

| Расширение | Тип файла |
|------------|-----------|
| `.md` | Markdown — уже документация |
| `.rst` | reStructuredText — уже документация |
| `.txt` | Текстовые файлы |
| `.adoc` | AsciiDoc — уже документация |

### Особые случаи

| Случай | Поведение |
|--------|-----------|
| Файл без расширения | Добавить `.md` к полному пути |
| Папка | Создать `README.md` внутри папки |
| Файл в корне проекта | `/doc/{filename}.md` |

## Валидация путей

### Обязательные проверки (блокирующие)

| Проверка | Ошибка |
|----------|--------|
| Путь НЕ в `/doc/**` | "Нельзя документировать документацию" |
| Путь НЕ в `/.claude/**` | "/.claude/ уже содержит документацию" |
| Путь НЕ в `/.git/**` | "Служебные файлы Git не документируются" |
| Файл НЕ самоописывающийся (.md, .rst, .txt) | "Файл уже является документацией" |
| Путь не является директорией | "Укажите файл, не папку (или используйте для README)" |
| Файл существует (для doc-create, doc-update) | "Файл {путь} не найден" |

### Рекомендуемые проверки (предупреждения)

| Проверка | Предупреждение |
|----------|----------------|
| Файл уже задокументирован | "Документация существует: {путь}" |
| Расширение поддерживается | "Неизвестное расширение, будет создан минимальный шаблон" |
| Путь в `/node_modules/` | "Документирование внешних зависимостей не рекомендуется" |
| Путь в `/venv/`, `/.venv/` | "Документирование виртуальных окружений не рекомендуется" |

## Поддерживаемые языки

### Таблица расширений

#### Языки программирования

| Расширение | Язык | Как извлекать API |
|------------|------|-------------------|
| `.ts`, `.tsx` | TypeScript | `export function`, `export class`, `export const` |
| `.js`, `.jsx` | JavaScript | `export function`, `export class`, `export const`, `module.exports` |
| `.py` | Python | `def`, `class` без `_` префикса |
| `.go` | Go | Функции и типы с заглавной буквы |
| `.sql` | SQL | `CREATE TABLE`, `CREATE FUNCTION`, `CREATE VIEW` |
| `.rs` | Rust | `pub fn`, `pub struct`, `pub enum` |
| `.java` | Java | `public class`, `public interface`, `public` методы |
| `.sh`, `.bash` | Shell | Функции, основные переменные |
| `.ps1` | PowerShell | `function`, `param` блоки |

#### Конфигурационные файлы

| Расширение | Тип | Что документировать |
|------------|-----|---------------------|
| `.yaml`, `.yml` | YAML | Структура, ключевые поля, примеры значений |
| `.json` | JSON | Схема, обязательные поля, примеры |
| `.toml` | TOML | Секции, параметры |
| `.ini`, `.cfg` | INI | Секции, параметры |
| `.env`, `.env.*` | Environment | Переменные окружения (без значений!) |
| `Dockerfile` | Docker | Этапы сборки, порты, volumes |
| `docker-compose.yml` | Docker Compose | Сервисы, сети, volumes |
| `Makefile` | Make | Targets, переменные |

### Формат комментариев по языку

| Язык | Формат ссылки на документацию |
|------|-------------------------------|
| TypeScript/JavaScript | `/** ... * Документация: /doc/src/... */` |
| Python | `""" ... Документация: /doc/src/... """` |
| Go | `// ... // Документация: /doc/src/...` |
| SQL | `-- ... -- Документация: /doc/src/...` |
| Rust | `/// ... /// Документация: /doc/src/...` |
| Java | `/** ... * Документация: /doc/src/... */` |

## Шаблон документации

### Базовый шаблон

```markdown
# {Название модуля}

> Исходный код: [{filename}](/{path-to-source})

{Краткое описание — заполнить вручную}

## API

### {FunctionName}

{Описание — заполнить вручную}

**Сигнатура:**
\`\`\`{language}
{signature}
\`\`\`

**Параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| ... | ... | ... |

**Возвращает:** {тип} — {описание}

---

## Зависимости

- [{dependency}](/{path}) — {description}

---

## Примеры

\`\`\`{language}
// Пример использования
\`\`\`
```

### Минимальный шаблон (для файлов без API)

```markdown
# {Название модуля}

> Исходный код: [{filename}](/{path-to-source})

{Краткое описание — заполнить вручную}

## Содержимое

{Описание содержимого файла}

---

## Связанные файлы

- {ссылки на связанные файлы}
```

## Формат пометки (doc-delete)

### Стандартная пометка

```markdown
> ⚠️ **ТРЕБУЕТ РЕВЬЮ:** Исходный файл `{путь}` был удалён.
>
> - [ ] Удалить этот файл документации
> - [ ] Или обновить ссылку на новый файл
>
> См. Issue: #{issue-number}
```

### Пометка без Issue

```markdown
> ⚠️ **ТРЕБУЕТ РЕВЬЮ:** Исходный файл `{путь}` был удалён.
>
> - [ ] Удалить этот файл документации
> - [ ] Или обновить ссылку на новый файл
```

## Связанные скиллы

### Цепочки вызовов

```
doc-create
    └── links-update (Шаг 7)

doc-update
    └── links-update (Шаг 7)

doc-delete
    ├── issue-create (Шаг 5)
    └── links-delete (Шаг 6)
```

### Когда какой скилл

| Событие | Скилл |
|---------|-------|
| Создан новый файл (любой, кроме исключённых) | `/doc-create` |
| Изменён файл (любой, кроме исключённых) | `/doc-update` |
| Удалён файл (любой, кроме исключённых) | `/doc-delete` |
| Перемещён файл | `/doc-delete` + `/doc-create` |

**Исключённые пути:** `/doc/`, `/.claude/`, `/.git/`, самоописывающиеся файлы (.md, .rst, .txt)

---

## Интеграция со scope-detection

Для определения scope документации используй [scope-detection.md](./scope-detection.md), раздел "Scope для документации".

**Правило:**
- Документация в `/doc/src/` имеет scope `project`
- Документация в `/.claude/` имеет scope `claude`

## Как использовать этот SSOT

### В doc-* скиллах

```markdown
## Правила

### Маппинг путей

**SSOT:** [doc-rules.md](/.claude/templates/doc-rules.md#маппинг-путей)

{Краткое описание + ссылка на SSOT}
```

### При обновлении правил

1. Изменить ТОЛЬКО этот файл
2. Не дублировать правила в скиллах
3. Проверить, что все скиллы ссылаются на SSOT
