**Приоритет скиллов:** Перед использованием низкоуровневых инструментов (Bash, Edit, Write, rm, mkdir) проверить: есть ли в списке скиллов подходящий скилл для этой задачи? Если да — использовать скилл.

**SSOT:** Если в документе есть ссылка `**SSOT:** [файл](путь)` — прочитать файл перед выполнением. Нельзя работать по памяти или предположениям.

**Шаблоны:** При создании файлов найти шаблон/пример в SSOT-инструкции и скопировать его структуру. Запрещено придумывать свой формат.

**Блокирующее подтверждение:** Если в воркфлоу скилла есть шаг с "подтверждением" — остановиться и ждать ответа пользователя.

**Принципы кода:** При написании кода следовать [standard-principles.md](/.instructions/standard-principles.md).

**Структура проекта:** При создании/изменении/удалении папок или README.md — СТОП. ЗАПРЕЩЕНО: mkdir, rm -rf, Write для README.md напрямую. Использовать скиллы:
  - `/structure-create` - создание папки (включая .instructions/)
  - `/structure-modify` - изменение папки
  - `/structure-validate` - валидация структуры

**Черновики:** После создания или изменения черновика в `/.claude/drafts/` — запустить `/draft-validate` для проверки соответствия стандарту.

**State (для агентов с Edit/Write):** Перед записью — проверить блокировку в `/.claude/state/locks.json`. Если занят — 6 проверок × 5 сек (30 сек), затем ошибка. Добавить свою блокировку, выполнить операцию, снять блокировку сразу после (ВСЕГДА, даже при ошибке!). Вести лог в `agent-{name}-operation.json`. См. [standard-state.md](/.claude/.instructions/state/standard-state.md).

**Миграция стандартов:** При выполнении `/migration-create` — ОБЯЗАТЕЛЬНО выполнить ВСЕ 9 шагов из SSOT:
  1. Определить стандарт
  2. Проверить версию (`bump-standard-version.py --check`)
  3. Найти зависимые файлы (`check-version-drift.py`) — ⚠️ проверяет только ВЕРСИИ, не содержание!
  4. Обновить Level 1:
     - `grep "^## " <стандарт>` — извлечь секции
     - Заполнить таблицу "секция стандарта → шаг в Level 1"
     - Добавить недостающие шаги, обновить frontmatter
  5. Обновить Level 2 — содержание по новым требованиям
  6. **ОБЯЗАТЕЛЬНО** обновить скрипты:
     - Найти скрипты в Level 1 документах
     - Заполнить таблицу "секция стандарта → код ошибки в скрипте"
     - Добавить недостающие проверки через `/script-modify`
  7. Обновить README области
  8. Валидация (0 расхождений версий + содержание проверено вручную)
  9. Отчёт

  **ЗАПРЕЩЕНО:**
  - Обновлять только `standard-version` без изменения содержания
  - Пропускать шаг 6 (скрипты) — даже если "версии синхронизированы"
  - Полагаться на `check-version-drift.py` как на проверку содержания
  - Считать миграцию завершённой без таблиц соответствия

  **SSOT:** [create-migration.md](/.instructions/migration/create-migration.md)
