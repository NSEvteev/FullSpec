**Приоритет скиллов:** Перед использованием низкоуровневых инструментов (Bash, Edit, Write, rm, mkdir) проверить: есть ли в списке скиллов подходящий скилл для этой задачи? Если да — использовать скилл.

**SSOT:** Если в документе есть ссылка `**SSOT:** [файл](путь)` — прочитать файл перед выполнением. Нельзя работать по памяти или предположениям.

**Шаблоны:** При создании файлов найти шаблон/пример в SSOT-инструкции и скопировать его структуру. Запрещено придумывать свой формат.

**Блокирующее подтверждение:** Если в воркфлоу скилла есть шаг с "подтверждением" — остановиться и ждать ответа пользователя.

**Принципы кода:** При написании кода следовать [standard-principles.md](/.instructions/standard-principles.md).

**Структура проекта:** При создании/изменении/удалении папок или README.md — СТОП. ЗАПРЕЩЕНО: mkdir, rm -rf, Write для README.md напрямую. Использовать скиллы:
  - `/structure-create` - создание папки (включая .instructions/)
  - `/structure-modify` - изменение папки
  - `/structure-validate` - валидация структуры

**Черновики:** После создания или изменения черновика в `/.claude/drafts/` — запустить `/draft-validate` для проверки соответствия стандарту.

**State (для агентов с Edit/Write):** Перед записью — проверить блокировку в `/.claude/state/locks.json`, добавить свою, снять сразу после операции. Вести лог в `agent-{name}-operation.json`. См. [standard-state.md](/.claude/.instructions/state/standard-state.md).
