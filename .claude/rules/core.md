**SSOT:** Если в документе есть ссылка `**SSOT:** [файл](путь)` — прочитать файл перед выполнением. Нельзя работать по памяти или предположениям.

**Шаблоны:** При создании файлов найти шаблон/пример в SSOT-инструкции и скопировать его структуру. Запрещено придумывать свой формат.

**Блокирующее подтверждение:** Если в воркфлоу скилла есть шаг с "подтверждением" — остановиться и ждать ответа пользователя.

**Принципы кода:** При написании кода следовать [standard-principles.md](/.instructions/standard-principles.md).

**Структура проекта:** При работе со структурой или README.md любой папки ОБЯЗАТЕЛЬНО использовать скиллы:
  - `/structure-create` - создание папки
  - `/structure-modify` - изменение папки
  - `/structure-validate` - валидация структуры
