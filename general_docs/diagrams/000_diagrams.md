## Правила обновления индекса

**КРИТИЧЕСКИ ВАЖНО для LLM:**

При создании/изменении/удалении диаграмм:

1. **После создания новой диаграммы:**
   - Добавить строку в таблицу "Индекс диаграмм"
   - Обновить статистику (увеличить счетчик для соответствующего статуса)
   - Обновить дату последнего обновления
   - Добавить ссылку на диаграмму в соответствующий архитектурный документ

2. **При изменении статуса:**
   - Обновить статус в таблице
   - Пересчитать статистику
   - Обновить дату последнего обновления

3. **При удалении диаграммы:**
   - Удалить строку из таблицы
   - Обновить статистику
   - Обновить дату последнего обновления
   - Удалить ссылку из архитектурного документа

**Автоматизация:** Использовать `/doc-index diagrams` для автоматического обновления этого файла.

## Автоматическое обновление

Этот файл автоматически обновляется скиллом `/doc-index`.

**Команды:**
```bash
# Обновить индекс диаграмм
/doc-index diagrams

# Проверить актуальность индекса
/doc-index diagrams --check

# Обновить все индексы в general_docs
/doc-index
```

**Проверка здоровья:**
```bash
# Проверить документацию и индексы
make docs-health

# Проверить только индексы
make docs-check-indexes
```

## Структура хранения диаграмм

**Именование папок:** `[ID]_[название_архитектурного_документа]/`

**Пример:**
```
diagrams/
├── 001_system_overview/
│   ├── 001-context.drawio
│   ├── 001-container.drawio
│   └── 001-component.drawio
│
└── 002_auth_module/
    ├── 002-auth-flow.drawio
    └── 002-data-model.drawio
```

**Правило:** ID папки должен совпадать с ID архитектурного документа в `architecture/`.

## Создание новой диаграммы

**Процесс:**

1. **Определить архитектурный документ:**
   - Найти ID документа (например, 001)
   - Проверить, есть ли папка `diagrams/[ID]_название/`

2. **Создать папку (если нет):**
   - Имя: `[ID]_[название_архитектурного_документа]/`
   - Пример: `001_system_overview/`

3. **Создать файл диаграммы:**
   - Формат: `.drawio`
   - Имя: `[ID]-[описание].drawio`
   - Пример: `001-context.drawio`

4. **Обновить этот индекс:**
   - Добавить запись в таблицу
   - Обновить статистику

5. **Добавить ссылку в архитектуру:**
   - Открыть `architecture/[ID]_название.md`
   - Добавить ссылку на диаграмму
