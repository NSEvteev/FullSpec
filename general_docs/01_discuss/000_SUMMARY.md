## Правила обновления

При переходе дискуссии в статус `🟢 approved`:

1. **Определить область** решения (UI, Backend, Auth, Database, Infra, Architecture)
2. **Добавить строку** в таблицу "Принятые решения" с указанием области
3. **Добавить ссылку** в соответствующую секцию "Решения по областям"
4. **Обновить счётчик** в таблице "Быстрый обзор по областям"
5. **Добавить блок** в секцию "Агрегация знаний по сервисам"

**Скилл `/summary-doc`** автоматически выполняет эти шаги.

**Workflow:**
```
🟣 review → 🟢 approved → /summary-doc (обновляет этот файл) → /architect → ⚪ final
```
