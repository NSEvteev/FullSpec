## Исходный запрос пользователя

> Любому проекту нужна страница входа (UI). Мы сейчас не обсуждаем хранение пользователей и методы АПИ для аутентификации. Давай обсудим UI и что нам нужно, чтобы у нас был UI стайлгайд, выбраны стили и т.д.

## Анализ цели и контекст

**Проблема:** Проекту нужна единая система дизайна для консистентного UI. Страница входа — первый экран пользователя, и она должна задать визуальный стандарт для всего приложения.

**Область обсуждения:**
- Выбор UI-библиотеки/фреймворка
- Система дизайн-токенов (цвета, типографика, отступы)
- Темизация (светлая/тёмная тема)
- Компоненты форм (input, button, validation)

**Вне области:** Логика аутентификации, хранение данных пользователей, API endpoints

## Текстовое сравнение вариантов

**Mantine vs shadcn/ui:**
- Mantine — готовая библиотека "из коробки", shadcn — копируемые компоненты
- Mantine лучше для быстрого старта, shadcn — для максимального контроля над кодом
- Оба имеют отличную TypeScript поддержку

**Mantine vs Radix UI:**
- Mantine — полноценная UI библиотека с готовыми стилями
- Radix — headless примитивы, требуют написания стилей с нуля
- Radix даёт максимальную гибкость, но требует больше времени на разработку

**Mantine vs Chakra UI:**
- Оба отличны в кастомизации и DX
- Mantine имеет больше готовых компонентов (100+ vs ~50)
- Mantine лучше с TypeScript и SSR

**Почему не MUI и Ant Design:**
- MUI: Большой бандл (200+ KB), сложная кастомизация, "Material Design look"
- Ant Design: Специфичный enterprise-стиль, тяжело уйти от дефолтного вида

**Headless UI vs Radix UI:**
- Оба headless, но Headless UI от команды Tailwind — лучшая интеграция
- Radix имеет больше примитивов, Headless UI — минимальный набор
- Headless UI проще для старта, Radix — для сложных UI

**Общие наблюдения:**
- Лидеров объединяет гибкая кастомизация и хороший TypeScript
- Решающие критерии: DX и компоненты форм (страница входа = формы)
- Chakra выделяется лучшей accessibility из коробки

## Принятое решение

**Выбран вариант:** Mantine

**Обоснование:**
Mantine — лучший выбор для проекта без дизайнера, где нужны готовые красивые компоненты из коробки. Отличная TypeScript поддержка и DX ускорят разработку. 100+ компонентов закрывают все потребности для страницы входа и будущих экранов.

**Аргументы за:**
- **DX (9/9):** Отличная документация, интуитивный API, хуки для форм
- **Компоненты форм (9/9):** TextInput, PasswordInput, Checkbox, валидация — всё есть
- **TypeScript (9/9):** TypeScript-first, отличное автодополнение
- **Темизация (6/6):** Встроенная поддержка dark/light mode

**Почему не другие варианты:**
- **shadcn/ui:** Требует ручной сборки компонентов, нет готовых стилей — не подходит без дизайнера
- **Chakra UI:** Меньше компонентов форм, слабее TypeScript поддержка
- **Radix UI:** Headless — нужно писать все стили самому

**Риски и как их минимизировать:**
- **Меньшее сообщество чем MUI:** Документация Mantine отличная, проблем с поддержкой не ожидается
- **Зависимость от библиотеки:** Mantine активно развивается (v7), риск заброшенности низкий

## Ревью решения

### Что вы СМОЖЕТЕ сделать

1. **Быстро создавать формы** — готовые TextInput, PasswordInput, Select, валидация через `@mantine/form` (Компоненты форм ⭐⭐⭐)
2. **Кастомизировать дизайн** — гибкая система стилей, CSS-in-JS, sx prop (Кастомизация ⭐⭐⭐)
3. **Поддерживать dark/light mode** — встроенный ColorSchemeProvider (Темизация ⭐⭐⭐)
4. **Использовать с Next.js** — SSR из коробки, App Router совместимость (SSR ⭐⭐⭐)
5. **Получить типобезопасность** — TypeScript-first, отличное автодополнение (TypeScript ⭐⭐⭐)
6. **Использовать 100+ компонентов** — модальные окна, уведомления, навигация, таблицы

### Что вы НЕ СМОЖЕТЕ сделать (или будет сложно)

1. **Минимизировать bundle до ~20KB** — Mantine ~80-100KB gzipped (Размер бандла ⭐⭐)
2. **Соответствовать WCAG AAA** — accessibility хорошая, но не идеальная (Accessibility ⭐⭐)
3. **Найти миллион примеров в интернете** — сообщество меньше чем MUI/Chakra
4. **Использовать с Vue/Svelte** — только React

### Риски и mitigation

| Риск | Вероятность | Влияние | Как минимизировать |
|------|-------------|---------|-------------------|
| Bundle size больше чем у shadcn | Средняя | Низкое | Tree-shaking, dynamic imports |
| Меньше примеров в сети | Низкая | Низкое | Официальная документация отличная |
| Breaking changes в v8 | Низкая | Среднее | Следить за roadmap |

### Сравнение с альтернативами

**Что вы теряете (по сравнению с shadcn/ui):**
- Полный контроль над кодом компонентов
- Минимальный bundle size (~20KB vs ~80KB)

**Что вы получаете (по сравнению с shadcn/ui):**
- Готовые стили из коробки — не нужен дизайнер
- Меньше работы по настройке
- Больше компонентов (100+ vs ~40)

### Вывод

**Рекомендуется.** Mantine — оптимальный выбор для проекта без дизайнера на React/Next.js. Ограничения некритичны.
