## Исходный запрос пользователя

> Любому проекту нужна страница входа (UI). Мы сейчас не обсуждаем хранение пользователей и методы АПИ для аутентификации. Давай обсудим UI и что нам нужно, чтобы у нас был UI стайлгайд, выбраны стили и т.д.

## Анализ цели и контекст

**Проблема:** Проекту нужна единая система дизайна для консистентного UI. Страница входа — первый экран пользователя.

**Область обсуждения:**
- Выбор UI-библиотеки/фреймворка
- Система дизайн-токенов (цвета, типографика)
- Темизация (светлая/тёмная тема)

**Вне области:** Логика аутентификации, хранение данных, API

## Текстовое сравнение вариантов

**Mantine vs shadcn/ui:**
- Mantine — готовая библиотека "из коробки", shadcn — копируемые компоненты
- Mantine лучше для быстрого старта, shadcn — для максимального контроля
- Оба имеют отличную TypeScript поддержку

**Mantine vs Radix UI:**
- Mantine — полноценная UI библиотека с готовыми стилями
- Radix — headless примитивы, требуют написания стилей
- Radix даёт максимальную гибкость, но больше работы

**Mantine vs Chakra UI:**
- Оба отличны в кастомизации и DX
- Mantine имеет больше готовых компонентов (100+ vs ~50)
- Mantine лучше с TypeScript и SSR

**Ant Design — особый случай:**
- Enterprise-фокус, специфичный визуальный стиль
- Сложно кастомизировать под уникальный дизайн
- Большой бандл, но огромная экосистема

**Общие наблюдения:**
- Лидеров объединяет гибкая кастомизация и хороший TypeScript
- MUI и Ant Design отстают из-за размера бандла и сложности кастомизации
- shadcn/ui и Radix требуют больше усилий, но дают контроль
