## Исходный запрос пользователя

> Любому проекту нужна страница входа (UI). Мы сейчас не обсуждаем хранение пользователей и методы АПИ для аутентификации. Давай обсудим UI и что нам нужно, чтобы у нас был UI стайлгайд, выбраны стили и т.д.

## Анализ цели и контекст

**Проблема:** Проекту необходим единый UI-стайлгайд для консистентности интерфейса. Первая точка применения — страница входа.

**Область обсуждения:**
- Выбор UI-фреймворка/библиотеки компонентов
- Определение дизайн-системы (цвета, типографика, отступы)
- Стандарты компонентов (кнопки, инпуты, формы)
- Подход к стилизации (CSS-in-JS, Tailwind, CSS Modules)

**Вне области:** Backend API, хранение пользователей, логика авторизации.

## Текстовое сравнение вариантов

**Chakra UI vs Mantine:**
- Оба имеют одинаковый балл (27/30), отличная документация и DX
- Chakra более популярен, Mantine — более современный с лучшей темизацией из коробки
- Mantine имеет более полный набор хуков и утилит

**shadcn/ui vs Chakra UI:**
- shadcn/ui даёт больше контроля через копируемые компоненты (код в проекте)
- Chakra UI быстрее в разработке, но менее гибкий в кастомизации стилей
- shadcn/ui требует знания Tailwind, Chakra — своей системы стилей

**MUI vs Radix UI:**
- MUI — полноценная дизайн-система с готовыми стилями Material Design
- Radix UI — только примитивы без стилей, максимальная гибкость
- MUI тяжелее (bundle size), Radix легче но требует больше работы

**Общие наблюдения:**
- Лидеры (Chakra UI, Mantine, shadcn/ui) объединены отличной поддержкой TypeScript и темизации
- Решающим критерием стал баланс между скоростью разработки и кастомизацией
- Все headless решения (Headless UI, Radix) проигрывают в скорости разработки
