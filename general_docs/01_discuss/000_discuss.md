## Правила обновления индекса

**КРИТИЧЕСКИ ВАЖНО для LLM:**

При создании/изменении/удалении дискуссий:

1. **После создания новой дискуссии:**
   - Добавить строку в таблицу "Индекс документов"
   - Обновить статистику (увеличить счетчик для соответствующего статуса)
   - Обновить дату последнего обновления

2. **При изменении статуса:**
   - Обновить статус в таблице
   - Пересчитать статистику (уменьшить старый статус, увеличить новый)
   - Обновить дату последнего обновления

3. **При удалении дискуссии:**
   - Удалить строку из таблицы
   - Обновить статистику
   - Обновить дату последнего обновления

**Автоматизация:** Использовать `/doc-index discuss` для автоматического обновления этого файла.

## Автоматическое обновление

Этот файл автоматически обновляется скиллом `/doc-index`.

**Команды:**
```bash
# Обновить индекс дискуссий
/doc-index discuss

# Проверить актуальность индекса
/doc-index discuss --check

# Обновить все индексы в general_docs
/doc-index
```

**Проверка здоровья:**
```bash
# Проверить документацию и индексы
make docs-health

# Проверить только индексы
make docs-check-indexes
```

**Workflow:**
1. Создать файл со статусом `draft`

4. Следовать [📖 workflow статусов](../glossary.md#workflow-статусов)
5. При переходе в `approved` → создать архитектуру
