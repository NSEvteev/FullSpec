# Индекс дискуссий

## О файлах в этой папке

**Этот файл (000_discuss.md) — ЧТО:**
- **Динамический индекс** всех дискуссий в папке
- Содержит таблицу с метаданными всех документов (ID, название, статус, даты)
- **Для LLM:** Быстрый обзор всех дискуссий и их состояния

**Файл [000_SUMMARY.md](000_SUMMARY.md) — ЗАЧЕМ:**
- **Агрегация решений** для передачи контекста на следующий этап (архитектура)
- Содержит краткое описание всех одобренных решений по областям
- **Для LLM:** Контекст для создания архитектурных документов
- Обновляется скиллом `/summary-doc` при переходе дискуссии в `approved`

**Файл README.md — КАК:**
- **Статичные правила** работы с дискуссиями
- Описывает, как создавать новые дискуссии
- Объясняет workflow статусов
- Содержит требования к структуре документов
- **Для LLM:** Инструкции по созданию и поддержке дискуссий

## Правила обновления индекса

При создании/изменении/удалении дискуссий:

1. **После создания** — добавить строку в таблицу
2. **При изменении статуса** — обновить статус в таблице
3. **При удалении** — удалить строку из таблицы

## Шаблон для новой дискуссии

**Имя файла:** `[ID]_краткое_описание.md`

**Пример:** `001_user_authentication.md`

**Шаблон:** См. [templates/discuss.md](../../llm_instructions/templates/discuss.md)

**Workflow:**
1. Создать файл со статусом `draft`
2. Добавить в этот индекс
3. Заполнить все обязательные разделы
4. Следовать [📖 workflow статусов](../glossary.md#workflow-статусов)
5. При переходе в `approved` → создать архитектуру
