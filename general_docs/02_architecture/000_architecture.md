## Быстрый поиск

### По статусу
- **draft** (0) — планирование архитектуры
- **in_progress** (0) — активная работа над архитектурой
- **feedback** (0) — обратная связь от ресурсов
- **review** (0) — ревью архитектуры
- **approved** (0) — одобренные, готовые к созданию ресурсов
- **final** (0) — завершенные архитектурные документы

### По дате
- **Последние обновления:** Нет документов
- **Недавно созданные:** Нет документов

## Правила обновления индекса

**КРИТИЧЕСКИ ВАЖНО для LLM:**

При создании/изменении/удалении архитектурных документов:

1. **После создания нового документа:**
   - Добавить строку в таблицу "Индекс документов"
   - Обновить статистику (увеличить счетчик для соответствующего статуса)
   - Обновить дату последнего обновления

2. **При изменении статуса:**
   - Обновить статус в таблице
   - Пересчитать статистику (уменьшить старый статус, увеличить новый)
   - Обновить дату последнего обновления

3. **При удалении документа:**
   - Удалить строку из таблицы
   - Обновить статистику
   - Обновить дату последнего обновления

**Автоматизация:** Использовать `/doc-index architecture` для автоматического обновления этого файла.

## Автоматическое обновление

Этот файл автоматически обновляется скиллом `/doc-index`.

**Команды:**
```bash
# Обновить индекс архитектуры
/doc-index architecture

# Проверить актуальность индекса
/doc-index architecture --check

# Обновить все индексы в general_docs
/doc-index
```

**Проверка здоровья:**
```bash
# Проверить документацию и индексы
make docs-health

# Проверить только индексы
make docs-check-indexes
```

## Шаблон для нового архитектурного документа

**Имя файла:** `[ID]_краткое_описание.md`

**Пример:** `001_system_overview.md`

**Шаблон:** См. [templates/architecture.md](../../llm_instructions/templates/architecture.md)

**Workflow:**
1. Создать файл на основе одобренной дискуссии
2. Статус `draft` — планирование архитектуры
3. Добавить в этот индекс
4. Заполнить все обязательные разделы
5. Следовать [📖 workflow статусов](../glossary.md#workflow-статусов)
6. При переходе в `approved` → создать ресурсы и план реализации
