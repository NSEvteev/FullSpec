- **Расположение:** general_docs/glossary.md

## Требования к глоссарию:
- Глоссарий должен содержать все термины, которые вводятся в проекте.
- Термины должны быть оформлены в виде ссылок, на которые должны ссылаться все *.md файлы (например, 02_architecture/01_system_overview.md и все *_doc.md).

## Бизнес-концепции

Термины, описывающие бизнес-уровень организации проекта.

<!-- Термины в категории добавляются в алфавитном порядке -->

### IT-сервис

- **Определение:** Средство предоставления ценности клиенту за счёт содействия достижению результатов, которых он хочет, без необходимости владеть определёнными затратами и рисками. Сервис определяется через результаты (outcomes) для бизнеса, а не через технические компоненты.
- **Контекст:** IT-сервис — это бизнес-концепция, которая стоит **НАД** [цепочкой зависимостей](#цепочка-зависимостей) документации. Вся техническая документация (Дискуссия → Архитектура → ADR → Ресурсы → План) служит для реализации сервиса.
- **Ключевые характеристики:**
  - Ориентирован на ценность для клиента/бизнеса
  - Определён через результаты, а не компоненты
  - Имеет SLA, стоимость, процессы поддержки
  - Включает технологии, людей, процессы, партнёров
  - Имеет жизненный цикл (проектирование → внедрение → поддержка → улучшение)
- **Пример:**
  - ❌ Не сервис: "Почтовый сервер Exchange" (компонент)
  - ✅ Сервис: "Сервис корпоративной электронной почты с 99.9% доступности и поддержкой 24/7" (ценность)
- **Синонимы:** Service, сервис, услуга.
- **Антонимы:** Компонент, продукт (технический).
- **Связанные термины:** [Цепочка зависимостей](#цепочка-зависимостей), [Ресурс](#ресурс), [План реализации](#план-реализации)
- **Источник:** ITIL v4, определение пользователя

```
            ┌─────────────────────────────────────┐
            │          IT-СЕРВИС                  │
            │   (бизнес-ценность для клиента)     │
            └─────────────────┬───────────────────┘
                              │
    ┌─────────────────────────┴─────────────────────────┐
    │           ТЕХНИЧЕСКАЯ РЕАЛИЗАЦИЯ                  │
    │                                                   │
    │  Дискуссия → Архитектура → ADR → Ресурсы → План  │
    └───────────────────────────────────────────────────┘
```
