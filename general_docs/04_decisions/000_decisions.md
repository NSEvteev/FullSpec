## Правила обновления индекса

При создании/изменении/удалении архитектурных решений:

1. **После создания** — добавить строку в таблицу
2. **При изменении статуса** — обновить статус в таблице
3. **При устаревании** — изменить статус, переместить файл в `archive/`

## Архив решений

Папка [archive/](archive/) содержит:
- Устаревшие версии решений (статус `deprecated`)
- Отклонённые решения
- Замещённые решения (статус `superseded`)

**Формат имени в archive/:**
```
DEC-XXX_название_v1.md

```

## Автоматическое обновление

Этот файл автоматически обновляется скиллом `/doc-index`.

**Команды:**
```bash
# Обновить индекс решений
/doc-index decisions

# Проверить актуальность индекса
/doc-index decisions --check

# Обновить все индексы в general_docs
/doc-index
```

**Проверка здоровья:**
```bash
# Проверить документацию и индексы
make docs-health

# Проверить только индексы
make docs-check-indexes
```

## Шаблон для нового архитектурного решения

**Имя файла:** `DEC-[XXX]_краткое_описание.md`

**Пример:** `DEC-001_auth_jwt.md`

**Шаблон:** См. [templates/decision_adr.md](../../llm_instructions/templates/decision_adr.md)

**Workflow:**
1. Создать файл на основе одобренной дискуссии и архитектурного документа
2. Статус `draft` — планирование решения
3. Добавить в этот индекс
4. Заполнить все обязательные разделы (Решение, Обоснование, Альтернативы, Последствия)
5. Следовать [📖 workflow статусов](../glossary.md#workflow-статусов)
6. При переходе в `final` → создать ресурсы и план реализации
