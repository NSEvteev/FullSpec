# IT-сервисы

## Назначение

Папка `00_services/` содержит описания [📖 IT-сервисов](../glossary.md#it-сервис) проекта.

**IT-сервис** — средство предоставления ценности клиенту, определяемое через результаты (outcomes), а не технические компоненты.

## Концепция

```
            ┌─────────────────────────────────────┐
            │          IT-СЕРВИС                  │
            │   (бизнес-ценность для клиента)     │
            │         00_services/                │
            └─────────────────┬───────────────────┘
                              │
    ┌─────────────────────────┴─────────────────────────┐
    │           ТЕХНИЧЕСКАЯ РЕАЛИЗАЦИЯ                  │
    │                                                   │
    │  01_discuss → 02_architecture → 04_decisions →    │
    │  → 05_resources → 06_imp_plans                    │
    └───────────────────────────────────────────────────┘
```

**Сервис НАД цепочкой:** Вся техническая документация служит для реализации сервиса.

## Структура папки

```
00_services/
├── README.md              # Этот файл (правила)
├── 000_services.md        # Индекс всех сервисов
└── [service-name]/        # Папка конкретного сервиса
    ├── README.md          # Описание сервиса (бизнес-ценность)
    └── links.md           # Ссылки на всю документацию сервиса
```

## Описание сервиса

Каждый сервис описывается через:

| Аспект | Описание |
|--------|----------|
| **Ценность** | Какую ценность получает клиент/бизнес? |
| **Результаты (Outcomes)** | Что клиент может делать благодаря сервису? |
| **SLA** | Гарантии доступности, времени реакции |
| **Владелец** | Кто отвечает за сервис |
| **Пользователи** | Кто использует сервис |

## Workflow

### Создание нового сервиса

1. **Идентифицировать сервис** — определить бизнес-ценность
2. **Создать папку** `00_services/[service-name]/`
3. **Создать README.md** — описание сервиса
4. **Обновить индекс** `000_services.md`
5. **Связать документацию** — по мере создания дискуссий, архитектуры и т.д.

### Когда создавать сервис?

- При начале новой дискуссии, если она описывает новую бизнес-функцию
- Сервис создаётся ДО или ВМЕСТЕ с первой дискуссией

## Примеры сервисов

| Не сервис (компонент) | IT-сервис (бизнес-ценность) |
|-----------------------|----------------------------|
| Redis Queue + Email Worker | **Сервис уведомлений** — надёжная доставка email о событиях |
| JWT + Sessions таблица | **Сервис аутентификации** — безопасный вход в систему |
| PostgreSQL + миграции | **Сервис хранения данных** — надёжное хранение с бэкапами |
