## О папке Decisions

Эта папка содержит **Architecture Decision Records (ADR)** — краткие записи о принятых архитектурных решениях.

### Назначение

**[📖 Decision (ADR)](../glossary.md#decision-adr)** — документ, фиксирующий архитектурное решение:
- **Что решили** — краткая суть решения
- **Почему решили** — обоснование выбора
- **Что рассмотрели** — альтернативы и почему отклонены
- **Последствия** — что это влечёт за собой

### Цепочка зависимостей

```
Дискуссия → Архитектура → Decision (ADR) → Ресурсы + Планы
```

**Decision (ADR)** создаётся на основе:
- [📖 Дискуссии](../glossary.md#дискуссия) — где обсуждались варианты
- Архитектурного документа — где детализировано решение

**Decision (ADR)** используется в:
- [📖 Ресурсах](../glossary.md#ресурс) — что нужно реализовать
- [📖 Планах реализации](../glossary.md#план-реализации) — как реализовать

## Формат Decision (ADR)

**Шаблон:** [decision_adr.md](../../llm_instructions/templates/decision_adr.md)

**Структура:**
```markdown
# DEC-XXX: Название решения

**Статус:** final
**Дата:** YYYY-MM-DD
**Связанные документы:** Дискуссия, Архитектура

## Решение
Что решили (1-2 предложения)

## Обоснование
Почему приняли это решение

## Рассмотренные альтернативы
1. Альтернатива 1 (отклонено) - причина
2. Альтернатива 2 (отклонено) - причина

## Последствия
Что влечёт за собой это решение
```

## Статусы Decision (ADR)

| Статус | Описание |
|--------|----------|
| `draft` | Черновик, решение ещё не принято |
| `proposed` | Решение предложено, ждёт обсуждения |
| `approved` | Решение одобрено, готово к реализации |
| `final` | Решение зафиксировано и используется |
| `deprecated` | Решение устарело, см. новую версию |
| `superseded` | Решение заменено новым (ссылка на DEC-XXX) |

## Архив решений

Папка `archive/` содержит устаревшие версии решений:
- Старые версии документов (DEC-XXX_название_v1.md)
- Отклонённые решения со статусом `deprecated`
- Замещённые решения со статусом `superseded`

**Правило:** При создании новой версии решения — старую перенести в `archive/`

## Процесс создания Decision (ADR)

1. **Предпосылки:**
   - Завершена [📖 дискуссия](../glossary.md#дискуссия) (статус `approved` или `final`)
   - Создан архитектурный документ (статус `approved` или `final`)

2. **Создание:**
   - Использовать шаблон [decision_adr_template.md](../../llm_instructions/templates/decision_adr_template.md)
   - Присвоить уникальный ID: `DEC-001`, `DEC-002`, ...
   - Заполнить все секции (Решение, Обоснование, Альтернативы, Последствия)

3. **Связывание:**
   - Указать ссылки на дискуссию и архитектуру
   - Добавить в индекс (этот файл)
   - Обновить связанные ресурсы и планы

4. **Финализация:**
   - Статус `draft` → `proposed` → `approved` → `final`
   - После статуса `final` — решение не меняется (только через новую версию)

---

## Правила работы с archive/

1. **Когда переносить в archive/:**
   - Решение устарело (статус `deprecated`)
   - Создана новая версия (статус `superseded`)
   - Решение отклонено после обсуждения

2. **Формат имени в archive/:**
   ```
   DEC-XXX_название_v1.md
   DEC-XXX_название_v2.md
   ```

3. **Обязательно:**
   - В устаревшем документе указать ссылку на новую версию
   - В новом документе указать, что заменяет старый
   - Обновить индекс
