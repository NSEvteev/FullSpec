## Назначение

Декомпозиция и отслеживание выполнения работ на основе архитектуры.

**[📖 План реализации](../glossary.md#план-реализации)** — документ, который:
- Разбивает архитектуру на задачи
- Определяет зависимости между задачами
- Отслеживает прогресс выполнения
- Служит основой для разработки

---

## Правила создания

### Именование
- Формат: `[ID]_plan_[название].md`
- Примеры: `001_plan_system_overview.md`
- ID совпадает с ID архитектуры

### Обязательные разделы
1. Метаданные (статус, даты, версия)
2. Ссылка на архитектуру
3. Декомпозиция задач
4. Зависимости
5. Прогресс выполнения

### Статусы задач
- `task_planned` — запланирована
- `task_in_progress` — в работе
- `task_testing` — на тестировании
- `task_done` — выполнена

---

## Workflow

```
draft → in_progress → review → test → approved → final
```

- **draft:** Планирование задач
- **in_progress:** Реализация задач
- **review:** Ревью выполнения
- **test:** Тестирование
- **approved:** Всё выполнено и протестировано
- **final:** План завершён

## Обратная связь

При изменении архитектуры:
1. Обновить задачи в плане
2. Пересчитать зависимости
3. Обновить прогресс

Чек-лист:
- [ ] Обновлен индекс
- [ ] Проверены зависимости
- [ ] Обновлен прогресс
- [ ] Статус корректный
