## 1. Когда валидировать

- После написания кода, работающего с Redis
- При code review
- Перед коммитом (автоматически через rule)

---

## 2. Коды ошибок

| Код | Описание | Severity |
|-----|----------|----------|
| RDS001 | Ключ не соответствует формату `{service}:{entity}:{id}` | error |
| RDS002 | Ключ конструируется inline (строковая конкатенация в бизнес-логике) | error |
| RDS003 | SET без TTL (утечка памяти) | error |
| RDS004 | Использование KEYS вместо SCAN | error |
| RDS005 | Хэш-поля не в snake_case | warning |
| RDS006 | Большое значение в одном ключе (JSON-массив вместо списка) | warning |
| RDS007 | Pub/Sub используется для гарантированной доставки | error |
| RDS008 | Отсутствует сервис-префикс в ключе | error |

---

## 3. Чек-лист

- [ ] Все ключи соответствуют формату `{service}:{entity}:{id}`
- [ ] Сервис-префикс первым сегментом во всех ключах
- [ ] Шаблоны ключей вынесены в модуль `keys` (не inline)
- [ ] Все SET-операции имеют TTL
- [ ] Используется SCAN вместо KEYS для поиска по паттерну
- [ ] Хэш-поля именованы в snake_case
- [ ] Каналы Pub/Sub в формате `{service}:{event}`
- [ ] Большие коллекции разбиты на отдельные ключи (не один JSON)
- [ ] Нет типичных ошибок из § 5 стандарта
