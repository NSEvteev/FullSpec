# 0001: Task Dashboard

## Проблема / Контекст

**Текущее состояние (AS IS):**
В проекте отсутствует система управления задачами. Задачи отслеживаются вручную или через внешние инструменты без интеграции с основной платформой.

**Проблема:**
Необходим встроенный дашборд задач, который позволит пользователям создавать, редактировать, назначать и отслеживать задачи в рамках платформы.

**Вне scope (v0.1.0):**
- Уведомления (email, push, in-app)
- Комментарии к задачам
- Мобильный интерфейс
- Подзадачи и зависимости между задачами
- Интеграция с внешними системами (Jira, GitHub Issues)

## Фичи

| ID | Название | Описание |
|----|----------|----------|
| F-1 | Создание задачи | Пользователь создаёт задачу с заголовком, описанием, приоритетом и исполнителем |
| F-2 | Канбан-доска | Визуальное представление задач по колонкам статусов (To Do, In Progress, Done) с drag-and-drop |
| F-3 | Фильтрация и поиск | Фильтрация задач по статусу, приоритету, исполнителю; полнотекстовый поиск по заголовку и описанию |
| F-4 | Детальный просмотр задачи | Просмотр всех полей задачи и история изменений |
| F-5 | Редактирование задачи | Изменение полей существующей задачи (заголовок, описание, приоритет, исполнитель, статус) |
| F-6 | Удаление задачи | Удаление задачи с подтверждением действия |

## User Stories

| ID | Роль | Действие | Цель |
|----|------|----------|------|
| US-1 | Пользователь | Создать новую задачу через форму на дашборде | Зафиксировать работу, которую нужно выполнить |
| US-2 | Пользователь | Перетащить задачу между колонками канбан-доски | Быстро обновить статус задачи |
| US-3 | Пользователь | Отфильтровать задачи по приоритету и исполнителю | Найти нужные задачи среди большого количества |
| US-4 | Пользователь | Открыть детальную карточку задачи | Просмотреть полную информацию и историю изменений |
| US-5 | Пользователь | Отредактировать поля существующей задачи через форму | Зафиксировать изменения в работе |
| US-6 | Пользователь | Назначить задачу другому пользователю | Делегировать работу коллеге |

## Требования

| ID | Связь | Требование |
|----|-------|------------|
| REQ-1 | F-1, US-1 | Пользователь заполняет форму создания задачи (заголовок, описание, приоритет, исполнитель) и нажимает "Создать" — задача появляется на канбан-доске в колонке "To Do" |
| REQ-2 | F-2, US-2 | Пользователь перетаскивает карточку задачи из одной колонки в другую — статус задачи обновляется, карточка остаётся в новой колонке после перезагрузки страницы |
| REQ-3 | F-3, US-3 | Пользователь выбирает фильтр по приоритету "High" — на доске отображаются только задачи с приоритетом "High", остальные скрыты |
| REQ-4 | F-4, US-4 | Пользователь кликает на карточку задачи — открывается детальная карточка со всеми полями и историей изменений |
| REQ-5 | F-1 | API возвращает ответ на создание задачи за время < 500ms (p95) |
| REQ-6 | F-1, US-1 | Пользователь отправляет форму создания задачи с пустым заголовком — форма не сохраняется, отображается ошибка валидации; заголовок обязателен, остальные поля опциональны |
| REQ-7 | F-5, US-5 | Пользователь изменяет поле задачи в детальной карточке и сохраняет — изменения персистируются и отображаются на канбан-доске |
| REQ-8 | F-6 | Пользователь нажимает "Удалить" на карточке задачи — отображается диалог подтверждения; при подтверждении задача удаляется и исчезает с канбан-доски |
| REQ-9 | — | Все пользователи аутентифицированы; каждый пользователь видит все задачи и может назначать исполнителем любого пользователя системы |
| REQ-10 | — | При истечении JWT frontend перенаправляет пользователя на страницу логина (upward feedback из Design: PROP-12) |

## Критерии успеха

- Все CRUD-операции с задачами работают end-to-end (frontend → API → backend → DB → ответ)
- Время ответа API < 500ms (p95) для всех эндпоинтов
- Канбан-доска корректно отображает задачи и поддерживает drag-and-drop
- Drag-and-drop: изменение статуса через перетаскивание применяется и визуально отображается пользователю < 1000ms
- Фильтрация возвращает корректные результаты для всех комбинаций фильтров
- 100% покрытие acceptance-тестами (REQ-1 — REQ-10)
