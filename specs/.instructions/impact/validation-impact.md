## Когда валидировать

После любого изменения в документе импакт-анализа:

- Создание → [create-impact.md](./create-impact.md)
- Изменение → [modify-impact.md](./modify-impact.md)
- Перед переводом DRAFT → WAITING (включая финал create-воркфлоу) — проверить: 0 маркеров, 0 Dependency Barrier
- При code review

## Шаги

### Шаг 0: Автоматическая валидация

```bash
python specs/.instructions/.scripts/validate-impact.py {путь}
```

Скрипт проверяет все правила I001-I036, включая: milestone cross-check с parent Discussion (I034), соответствие NNNN Impact = NNNN parent (I036), содержательность Резюме (I033), непустоту колонки "Тип" в зависимостях (I035). Если валидация пройдена — **готово**, шаги 1-9 не нужны.

**Если скрипт недоступен** — выполнить шаги 1-9 вручную.

### Шаг 1: Именование файла

**SSOT:** [standard-impact.md § 2](./standard-impact.md#2-расположение-и-именование)

- Файл расположен в `specs/impact/`
- Имя соответствует regex: `^impact-(\d{4})-(.+)\.md$`
- `NNNN` — четырёхзначный номер, **равен NNNN parent Discussion** (disc-0002 → impact-0002)
- `topic` — kebab-case, латиница

### Шаг 2: Frontmatter

**SSOT:** [standard-impact.md § 3](./standard-impact.md#3-frontmatter)

**Проверить обязательные поля:**

| Поле | Проверка |
|------|----------|
| `description` | Присутствует, до 1024 символов |
| `standard` | = `specs/.instructions/impact/standard-impact.md` |
| `standard-version` | Присутствует, формат `vX.Y` |
| `index` | = `specs/impact/README.md` |
| `parent` | Присутствует, путь к Discussion-документу (`discussion/disc-NNNN-*.md`) |
| `children` | Список путей или пустой список `[]` |
| `status` | Валидный статус: DRAFT, WAITING, RUNNING, DONE, CONFLICT, ROLLING_BACK, REJECTED |
| `milestone` | Присутствует, совпадает с milestone parent Discussion |

**Проверить parent:**

- `parent` указывает на существующий файл Discussion
- Формат пути: `discussion/disc-NNNN-topic.md`

### Шаг 3: Заголовок документа

**SSOT:** [standard-impact.md § 2](./standard-impact.md#2-расположение-и-именование)

- Формат: `# impact-NNNN: {Тема}`
- `NNNN` в заголовке совпадает с `NNNN` в имени файла

### Шаг 4: Обязательные разделы

**SSOT:** [standard-impact.md § 5](./standard-impact.md#5-разделы-документа)

Документ организован по сервисам. Проверить наличие обязательных секций:

| Раздел | Уровень | Заголовок | Проверка |
|--------|---------|-----------|----------|
| Резюме | h2 | `## 📋 Резюме` | Присутствует, заполнено контентом |
| Секции SVC | h2 | `## SVC-N: {Сервис}` | Минимум 1 секция |
| ├── Компоненты | h3 | `### 📦 Компоненты` | В каждой SVC-секции |
| ├── Данные и хранение | h3 | `### 💾 Данные и хранение` | В каждой SVC-секции |
| └── API | h3 | `### 🔌 API` | В каждой SVC-секции |
| Зависимости | h2 | `## 🔗 Зависимости` | Присутствует |
| Риски | h2 | `## ⚠️ Риски` | Присутствует |

### Шаг 5: Нумерация элементов

**SSOT:** [standard-impact.md § 5](./standard-impact.md#5-разделы-документа)

| Тип нумерации | Раздел | Проверка |
|---------------|--------|----------|
| `SVC-N` | Заголовки секций `## SVC-N:` | Нет дублей, сквозная |
| `CMP-N` | Подсекции Компоненты (внутри SVC) | Нет дублей, сквозная |
| `DATA-N` | Подсекции Данные и хранение (внутри SVC) | Нет дублей, сквозная |
| `API-N` | Подсекции API (внутри SVC) | Нет дублей, сквозная |
| `DEP-N` | Зависимости | Нет дублей |
| `RISK-N` | Риски | Нет дублей |

Пропуски в нумерации допустимы (элементы могли быть удалены).

### Шаг 6: Минимальное содержание

**SSOT:** [standard-impact.md § 5](./standard-impact.md#5-разделы-документа)

| Раздел | Требование |
|--------|------------|
| Резюме | Контент обязателен (не пуст, не placeholder). Должно содержать: scope влияния, характер изменений, ключевые risk areas |
| SVC-N секции | Минимум 1 секция. Каждая имеет метаданные (Тип влияния, Уверенность) и описание флоу (1–2 абзаца: триггер, поток, результат, связи) |
| ├── Компоненты | Контент или заглушка: _Компоненты не идентифицированы._ |
| ├── Данные и хранение | Контент или заглушка: _Изменений в данных нет._ |
| └── API | Контент или заглушка: _Изменений в API нет._ |
| Зависимости | Контент или заглушка: _Зависимостей нет._ |
| Риски | Минимум 1 элемент (RISK-N) |

**Заглушки:** Если подсекция содержит заглушку (курсив), контент-проверки не применяются.

### Шаг 7: Зона ответственности

**SSOT:** [standard-impact.md § 1](./standard-impact.md#1-назначение)

Документ **НЕ должен** содержать:
- Окончательное распределение ответственностей между сервисами ("auth **отвечает** за токены, gateway — за rate limiting")
- Полные request/response контракты API (→ Design)
- Архитектурные решения (→ ADR)
- Тестовые сценарии (→ План тестов)
- Задачи на реализацию (→ План разработки)

**Допустимо:**
- "auth-сервис **предположительно** затронут — нужна JWT-валидация" (предложение варианта)
- "Новый эндпоинт POST /auth/token" (высокоуровневое описание)
- "Компонент может быть shared между auth и gateway" (маркировка shared)
- "Нужен новый notification-service" (план создания)

### Шаг 8: Маркеры и статус

**SSOT:** [standard-impact.md § 4](./standard-impact.md#4-переходы-статусов), [standard-impact.md § 6](./standard-impact.md#6-clarify)

Если `status` > DRAFT (WAITING, RUNNING, DONE, CONFLICT, ROLLING_BACK, REJECTED):
- Нет неразрешённых `[ТРЕБУЕТ УТОЧНЕНИЯ]` маркеров
- Нет `⛔ DEPENDENCY BARRIER`

### Шаг 9: Регистрация в README

**SSOT:** [standard-impact.md § 2](./standard-impact.md#2-расположение-и-именование)

- Запись существует в `specs/impact/README.md`
- Колонки: `ID`, `Документ`, `Статус`, `Parent Discussion`, `Design`, `Milestone`, `Описание`
- Статус в README совпадает со статусом в frontmatter документа

## Чек-лист

### Именование
- [ ] Файл в `specs/impact/`
- [ ] Имя: `impact-NNNN-topic.md`
- [ ] Заголовок: `# impact-NNNN: {Тема}`
- [ ] NNNN в имени = NNNN в заголовке
- [ ] NNNN Impact = NNNN parent Discussion (disc-0002 → impact-0002)

### Frontmatter
- [ ] `description` — до 1024 символов
- [ ] `standard` = `specs/.instructions/impact/standard-impact.md`
- [ ] `standard-version` — формат `vX.Y`
- [ ] `index` = `specs/impact/README.md`
- [ ] `parent` — путь к Discussion-документу (обязателен, формат `discussion/disc-NNNN-topic.md`)
- [ ] `children` — список или `[]`
- [ ] `status` — валидный статус
- [ ] `milestone` — совпадает с milestone parent Discussion (автоматическая проверка I034)

### Содержание
- [ ] Секция "📋 Резюме" заполнена контентом (scope, характер, risk areas — не placeholder)
- [ ] Минимум 1 секция SVC-N присутствует
- [ ] Каждая секция SVC-N имеет метаданные (Тип влияния, Уверенность) и описание флоу (1–2 абзаца)
- [ ] Каждая секция SVC-N содержит 3 подсекции (Компоненты, Данные и хранение, API)
- [ ] Подсекции заполнены контентом или заглушкой
- [ ] Новые сервисы помечены "Новый (план создания)"
- [ ] Shared-компоненты маркированы `shared ({сервисы})` в колонке "Scope"
- [ ] Секция "🔗 Зависимости" — контент или заглушка
- [ ] Секция "⚠️ Риски" — минимум 1 элемент (RISK-N)
- [ ] Нумерация сквозная, без дублей (SVC-N, CMP-N, DATA-N, API-N, DEP-N, RISK-N)

### Зона ответственности
- [ ] Нет распределения ответственностей между сервисами (→ Design)
- [ ] Нет полных request/response контрактов API (→ Design)
- [ ] Нет архитектурных решений (→ ADR)
- [ ] Нет тестовых сценариев (→ План тестов)
- [ ] Нет задач на реализацию (→ План разработки)

### Маркеры (если статус > DRAFT)
- [ ] Нет `[ТРЕБУЕТ УТОЧНЕНИЯ]`
- [ ] Нет `⛔ DEPENDENCY BARRIER`

### README
- [ ] Запись в `specs/impact/README.md`
- [ ] Статус в README = статус в frontmatter

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Неверный формат имени файла | I001 | Имя не соответствует `impact-NNNN-topic.md` | Переименовать файл |
| Отсутствует `description` | I002 | Пустой или отсутствующий frontmatter | Добавить description |
| Неверный `standard` | I003 | Путь не указывает на standard-impact.md | Исправить путь |
| Невалидный `status` | I004 | Статус не из допустимого списка | Использовать валидный статус |
| Отсутствует `parent` | I005 | Impact обязан иметь parent Discussion | Добавить поле parent |
| Parent не существует | I006 | Файл Discussion не найден по пути из parent | Исправить путь или создать Discussion |
| Отсутствует "Резюме" | I007 | Обязательный раздел пропущен | Добавить раздел `## 📋 Резюме` |
| Нет секций SVC-N | I008 | Отсутствуют секции `## SVC-N:` | Добавить минимум 1 секцию `## SVC-N: {Сервис}` |
| Подсекция "Компоненты" отсутствует | I009 | SVC-секция без подсекции Компоненты | Добавить `### 📦 Компоненты` в секцию SVC |
| Подсекция "Данные и хранение" отсутствует | I010 | SVC-секция без подсекции Данные | Добавить `### 💾 Данные и хранение` в секцию SVC |
| Подсекция "API" отсутствует | I011 | SVC-секция без подсекции API | Добавить `### 🔌 API` в секцию SVC |
| Отсутствует "Зависимости" | I012 | Обязательный раздел пропущен | Добавить раздел `## 🔗 Зависимости` |
| Отсутствует "Риски" | I013 | Обязательный раздел пропущен | Добавить раздел `## ⚠️ Риски` |
| Дублирование SVC-N | I014 | Два элемента с одинаковым номером | Исправить нумерацию |
| Дублирование CMP-N | I015 | Два элемента с одинаковым номером | Исправить нумерацию |
| Дублирование DATA-N | I016 | Два элемента с одинаковым номером | Исправить нумерацию |
| Дублирование API-N | I017 | Два элемента с одинаковым номером | Исправить нумерацию |
| Дублирование DEP-N | I018 | Два элемента с одинаковым номером | Исправить нумерацию |
| Дублирование RISK-N | I019 | Два элемента с одинаковым номером | Исправить нумерацию |
| Маркер при статусе > DRAFT | I020 | Неразрешённый `[ТРЕБУЕТ УТОЧНЕНИЯ]` | Разрешить маркер или вернуть в DRAFT |
| Dependency Barrier при статусе > DRAFT | I021 | Неразрешённый барьер | Разрешить барьер или вернуть в DRAFT |
| Распределение ответственностей | I022 | Окончательное распределение по сервисам (→ Design) | Заменить на предложение с "предположительно" |
| Нет записи в README | I023 | Документ не зарегистрирован | Добавить запись в README |
| Рассинхрон статуса | I024 | Статус в README ≠ статус в frontmatter | Синхронизировать |
| NNNN не совпадает | I025 | Номер в имени файла ≠ номер в заголовке | Исправить заголовок или имя файла |
| Резюме пустое/placeholder | I033 | Резюме не заполнено или содержит шаблонный текст | Заполнить Резюме контентом |
| Milestone не совпадает | I034 | Milestone Impact ≠ milestone parent Discussion | Синхронизировать milestone с parent |
| Тип зависимости пуст | I035 | Колонка "Тип" в DEP-N пустая | Заполнить тип зависимости |
| NNNN ≠ parent NNNN | I036 | Номер Impact не равен номеру parent Discussion | Переименовать файл: disc-0002 → impact-0002 |
