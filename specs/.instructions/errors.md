## spec-create

### Таблица ошибок

| Ошибка | Причина | Действие |
|--------|---------|----------|
| Неверный тип документа | Тип не discussion/impact/adr/plan | Показать список допустимых типов |
| Родитель не найден | Указанный ID не существует | Показать существующие документы для выбора |
| Родитель в неправильном статусе | Статус не позволяет создание дочернего | Показать текущий статус и требуемый |
| Сервис не существует | Сервис не найден в /specs/services/ | Предложить `--new` для создания |
| Документ с таким номером существует | Коллизия номеров | Пересчитать номер (следующий свободный) |
| Шаблон не найден | Файл шаблона отсутствует | Сообщить об ошибке, указать путь к шаблону |
| Ошибка записи файла | Нет прав или диск полон | Сообщить, предложить проверить права |

### Требования к статусу родителя

| Тип документа | Родитель | Требуемый статус родителя |
|---------------|----------|---------------------------|
| `impact` | Discussion | `APPROVED` или выше |
| `adr` | Impact | `REVIEW` или выше |
| `plan` | ADR | `APPROVED` |

## spec-status

### Таблица ошибок

| Ошибка | Причина | Действие |
|--------|---------|----------|
| Документ не найден | Путь не разрешается в файл | Показать существующие документы |
| Недопустимый переход | Нарушение схемы переходов | Показать допустимые переходы из текущего статуса |
| Чек-лист не пройден | Не все пункты выполнены | Показать невыполненные пункты |
| Конфликт статусов | Статусы родителя/дочерних несогласованы | Предложить `/specs-sync` |
| Нет причины для REJECTED | Причина отклонения не указана | Запросить причину |
| Нет ссылки для SUPERSEDED | Не указан заменяющий документ | Запросить ID заменяющего документа |

### Недопустимые переходы

| Переход | Причина |
|---------|---------|
| `DONE → *` | Финальный статус (кроме SUPERSEDED) |
| `REJECTED → *` | Финальный статус (кроме переоткрытия) |
| `DRAFT → APPROVED` | Пропуск статуса REVIEW |
| `DRAFT → RUNNING` | Пропуск статусов REVIEW, APPROVED |
| `REVIEW → RUNNING` | Пропуск статуса APPROVED |

## Откат при ошибке

### Когда откатывать

Откат выполняется, если ошибка произошла **после** начала изменений:
- Файл документа создан, но README не обновлён
- README обновлён, но backlink не добавлен
- Изменения частично применены

### Алгоритм отката для spec-create

```bash
# 1. Удалить созданный файл (если создан)
rm /specs/{path}/{file}.md

# 2. Откатить изменения в README.md (если обновлён)
git checkout -- /specs/{path}/README.md

# 3. Откатить изменения в родителе (если обновлён)
git checkout -- {parent-path}
```

### Алгоритм отката для spec-status

```bash
# 1. Откатить изменения в документе
git checkout -- /specs/{path}/{file}.md

# 2. Откатить изменения в README.md
git checkout -- /specs/{path}/README.md

# 3. Откатить каскадные изменения (если были)
git checkout -- {parent-path}
git checkout -- {child-paths}
```

### Формат сообщения об откате

```
❌ Ошибка: {описание ошибки}

Выполнен откат:
- Удалён: /specs/discussions/003-test.md
- Восстановлен: /specs/discussions/README.md

Исправьте ошибку и повторите операцию.
```
