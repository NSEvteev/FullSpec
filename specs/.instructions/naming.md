## Формат названия

### Правила

| Правило | Описание |
|---------|----------|
| Только латиница | a-z (нижний регистр) |
| Цифры | 0-9 |
| Дефис | `-` для разделения слов |
| Нет пробелов | Пробелы запрещены |
| Нет подчёркиваний | `_` запрещён |

### Формат имени файла

```
{NNN}-{topic}.md
```

| Компонент | Описание | Пример |
|-----------|----------|--------|
| `{NNN}` | Трёхзначный номер | `001`, `002`, `042` |
| `{topic}` | Тема латиницей с дефисами | `auth-strategy`, `jwt-tokens` |

### Примеры

| Правильно | Неправильно |
|-----------|-------------|
| `001-auth-strategy.md` | `001-Auth-Strategy.md` |
| `002-jwt-tokens.md` | `002_jwt_tokens.md` |
| `003-api-v2.md` | `003 api v2.md` |

## Нумерация

### Автоинкремент

Номер документа определяется автоматически:

```
Следующий номер = max(существующие номера в папке) + 1
```

### Алгоритм определения номера

```bash
# Найти последний номер в папке
ls /specs/{type}/ | grep -E "^[0-9]{3}-" | sort | tail -1

# Если папка пуста → начать с 001
# Если последний 005 → следующий 006
```

### Формат номера

| Номер | Формат | Пример файла |
|-------|--------|--------------|
| 1 | `001` | `001-auth-strategy.md` |
| 10 | `010` | `010-caching.md` |
| 100 | `100` | `100-migration.md` |

**Важно:** Всегда 3 цифры с ведущими нулями.

## Сокращённые пути

Скиллы `/spec-status`, `/spec-update` принимают сокращённые пути для удобства.

### Таблица преобразования

| Сокращённый путь | Полный путь |
|------------------|-------------|
| `discussions/001` | `/specs/discussions/001-*.md` |
| `impact/001` | `/specs/impact/001-*.md` |
| `auth/adr/001` | `/specs/services/auth/adr/001-*.md` |
| `auth/plans/jwt` | `/specs/services/auth/plans/*jwt*.md` |

### Алгоритм разрешения

```
Вход: discussions/001
    │
    ├── Разбить по `/`: ["discussions", "001"]
    │
    ├── Первый элемент = тип или сервис?
    │   ├── discussions, impact → /specs/{type}/
    │   └── другое → /specs/services/{service}/
    │
    ├── Последний элемент = ID или тема?
    │   ├── Цифры → 001-*.md
    │   └── Текст → *{topic}*.md
    │
    └── Выход: /specs/discussions/001-*.md
```

### Примеры использования

```bash
/spec-status discussions/001 review
# → /specs/discussions/001-auth-strategy.md

/spec-update auth/adr/002
# → /specs/services/auth/adr/002-jwt-tokens.md

/spec-status gateway/plans/rate-limit done
# → /specs/services/gateway/plans/rate-limit-plan.md
```

## Валидация

### Проверки при создании

| Проверка | Ошибка | Действие |
|----------|--------|----------|
| Пустая тема | "Тема не указана" | Запросить тему |
| Пробелы в теме | "Пробелы запрещены" | Заменить на дефисы |
| Верхний регистр | "Только нижний регистр" | Привести к нижнему |
| Спецсимволы | "Недопустимые символы" | Удалить или заменить |

### Regex валидации

```regex
^[0-9]{3}-[a-z0-9]+(-[a-z0-9]+)*\.md$
```
