## Когда валидировать

После любого изменения в документе проектирования:

- Создание → [create-design.md](./create-design.md)
- Изменение → [modify-design.md](./modify-design.md)
- Перед переводом DRAFT → WAITING — проверить: 0 маркеров, 0 Dependency Barrier
- При code review

## Шаги

### Шаг 0: Автоматическая валидация

```bash
python specs/.instructions/.scripts/validate-design.py {путь}
```

Скрипт проверяет все правила D001-D040. Если валидация пройдена — **готово**, шаги 1-9 не нужны.

**Если скрипт недоступен** — выполнить шаги 1-9 вручную.

### Шаг 1: Именование файла

**SSOT:** [standard-design.md § 2](./standard-design.md#2-расположение-и-именование)

- Файл расположен в `specs/design/`
- Имя соответствует regex: `^design-(\d{4})-(.+)\.md$`
- `NNNN` — четырёхзначный номер, **равен NNNN parent Discussion** (disc-0002 → design-0002)
- `topic` — kebab-case, латиница

### Шаг 2: Frontmatter

**SSOT:** [standard-design.md § 3](./standard-design.md#3-frontmatter)

**Проверить обязательные поля:**

| Поле | Проверка |
|------|----------|
| `description` | Присутствует, до 1024 символов |
| `standard` | = `specs/.instructions/design/standard-design.md` |
| `standard-version` | Присутствует, формат `vX.Y` |
| `index` | = `specs/design/README.md` |
| `parent` | Присутствует, путь к Impact-документу (`impact/impact-NNNN-*.md`) |
| `children` | Список путей к ADR или пустой список `[]` |
| `status` | Валидный статус: DRAFT, WAITING, RUNNING, DONE, CONFLICT, ROLLING_BACK, REJECTED |
| `milestone` | Присутствует, совпадает с milestone parent Discussion (через Impact) |

**Проверить parent:**

- `parent` указывает на существующий файл Impact
- Формат пути: `impact/impact-NNNN-topic.md`
- NNNN в parent Impact = NNNN в имени Design

### Шаг 3: Заголовок документа

**SSOT:** [standard-design.md § 2](./standard-design.md#2-расположение-и-именование)

- Формат: `# design-NNNN: {Тема}`
- `NNNN` в заголовке совпадает с `NNNN` в имени файла

### Шаг 4: Обязательные разделы

**SSOT:** [standard-design.md § 5](./standard-design.md#5-разделы-документа)

Документ содержит три типа секций. Проверить наличие:

| Раздел | Уровень | Заголовок | Проверка |
|--------|---------|-----------|----------|
| Резюме | h2 | `## 📋 Резюме` | Присутствует, заполнено контентом |
| Секции SVC | h2 | `## SVC-N: {Сервис}` | Минимум 1 секция |
| ├── Ответственность | h3 | `### 📋 Ответственность` | В каждой SVC-секции, контент обязателен |
| ├── Компоненты | h3 | `### 📦 Компоненты` | В каждой SVC-секции |
| └── Зависимости | h3 | `### 🔗 Зависимости` | В каждой SVC-секции |
| Блоки INT | h2 | `## INT-N: {Взаимодействие}` | Минимум 1 (если > 1 сервиса) |
| ├── Контракт | h3 | `### Контракт` | В каждом INT-блоке, контент обязателен |
| └── Sequence | h3 | `### Sequence` | В каждом INT-блоке, контент обязателен |
| Тест-сценарии | h2 | `## 🧪 Системные тест-сценарии` | Присутствует |

### Шаг 5: Нумерация элементов

**SSOT:** [standard-design.md § 5](./standard-design.md#5-разделы-документа)

| Тип нумерации | Раздел | Проверка |
|---------------|--------|----------|
| `SVC-N` | Заголовки секций `## SVC-N:` | Нет дублей, сквозная |
| `CMP-N` | Подсекции Компоненты (таблица) | Нет дублей, сквозная |
| `INT-N` | Заголовки блоков `## INT-N:` | Нет дублей, сквозная |
| `STS-N` | Таблица тест-сценариев | Нет дублей, сквозная |

Пропуски в нумерации допустимы (элементы могли быть удалены).

### Шаг 6: Минимальное содержание

**SSOT:** [standard-design.md § 5](./standard-design.md#5-разделы-документа)

**Резюме:**

| Проверка | Критерий |
|----------|----------|
| Контент обязателен | Не пуст, не placeholder |
| Scope | Количество сервисов подтверждённых/изменённых/добавленных/отклонённых |
| Ключевые решения | Распределение ответственностей, shared-компоненты |
| Отклонённые сервисы | Если Impact предлагал сервисы, которые Design отклонил — упомянуты с обоснованием |

**Секции SVC-N:**

| Проверка | Критерий |
|----------|----------|
| Метаданные | `**Impact:** SVC-N ({Тип}, {Уверенность}) \| **Решение Design:** {Решение}` |
| Описание ответственности | 1–2 абзаца (нарратив флоу), не дублирует подсекцию "Ответственность" |
| Подсекция "Ответственность" | Bullet list конкретных действий (контент обязателен) |
| Подсекция "Компоненты" | Таблица (ID, Компонент, Scope, Решение) или заглушка: _Компоненты наследуются из Impact без изменений._ |
| Подсекция "Зависимости" | Bullet list со ссылками на INT-N или заглушка: _Внешних зависимостей нет._ |
| Порядок секций SVC | Первичный: тип (Основной → Вторичный → Новый). Вторичный: решение (Подтверждённые → Изменённые → Добавленные) |

**Колонка "Решение" в Компонентах:**

| Формат | Когда |
|--------|-------|
| `Подтверждён из Impact (CMP-N)` | Имя, scope и назначение без изменений |
| `Изменён: Impact (CMP-N, ...) → ...` | Любое изменение (scope, структура, объединение) |
| `Добавлен Design — описание` | Новый компонент, не упомянутый в Impact |

**Блоки INT-N:**

| Проверка | Критерий |
|----------|----------|
| Метаданные | Участники, Паттерн, Источник Impact (DEP-N или —) |
| Подсекция "Контракт" | Спецификация межсервисного API или формата события (контент обязателен) |
| Подсекция "Sequence" | Mermaid sequenceDiagram (контент обязателен) |
| Количество | Минимум 1 блок при > 1 сервиса. При 1 сервисе — 0 допустимо |

**Тест-сценарии (STS-N):**

| Проверка | Критерий |
|----------|----------|
| Формат | Таблица (ID, Сценарий, Участники, Тип, Источник) или заглушка: _Системных тест-сценариев нет._ |
| Уровень | Краткое описание флоу (без Given/When/Then) |
| Источник | Ссылки на INT-N блоки |

### Шаг 7: Зона ответственности

**SSOT:** [standard-design.md § 1](./standard-design.md#1-назначение)

Документ **НЕ должен** содержать:
- Бизнес-обоснования и требования ("пользователи жалуются", "бизнес хочет") — → Discussion
- Предположительные оценки без решения ("предположительно затронут") — → Impact ПРЕДЛАГАЕТ
- Детали реализации — алгоритмы, пакеты, классы, библиотеки, конфигурации ("RS256 вместо HS256", "class TokenGenerator", "pool size = 20") — → ADR
- Тестовые сценарии конкретного сервиса (unit, integration внутри сервиса) — → План тестов
- Задачи на реализацию — → План разработки

**Допустимо:**
- "auth отвечает за генерацию и валидацию JWT" — распределение ответственности (РЕШЕНИЕ)
- "POST /auth/token → {access_token, refresh_token, expires_in}" — межсервисный контракт API
- "auth публикует UserCreatedEvent → notifications, billing потребляют" — блок взаимодействия
- "Shared validator размещается в /shared/auth/" — решение о shared-компоненте

### Шаг 8: Маркеры и статус

**SSOT:** [standard-design.md § 4](./standard-design.md#4-переходы-статусов), [standard-design.md § 6](./standard-design.md#6-clarify)

Если `status` > DRAFT (WAITING, RUNNING, DONE, CONFLICT, ROLLING_BACK, REJECTED):
- Нет неразрешённых `[ТРЕБУЕТ УТОЧНЕНИЯ]` маркеров
- Нет `⛔ DEPENDENCY BARRIER`

### Шаг 9: Регистрация в README

**SSOT:** [standard-design.md § 2](./standard-design.md#2-расположение-и-именование)

- Запись существует в `specs/design/README.md`
- Колонки: `ID`, `Документ`, `Статус`, `Parent Impact`, `ADR`, `Milestone`, `Описание`
- Статус в README совпадает со статусом в frontmatter документа

## Чек-лист

### Именование
- [ ] Файл в `specs/design/`
- [ ] Имя: `design-NNNN-topic.md`
- [ ] Заголовок: `# design-NNNN: {Тема}`
- [ ] NNNN в имени = NNNN в заголовке
- [ ] NNNN Design = NNNN parent Discussion (disc-0002 → design-0002)

### Frontmatter
- [ ] `description` — до 1024 символов
- [ ] `standard` = `specs/.instructions/design/standard-design.md`
- [ ] `standard-version` — формат `vX.Y`
- [ ] `index` = `specs/design/README.md`
- [ ] `parent` — путь к Impact-документу (обязателен, формат `impact/impact-NNNN-topic.md`)
- [ ] `children` — список путей к ADR или `[]`
- [ ] `status` — валидный статус
- [ ] `milestone` — совпадает с milestone parent Discussion (через Impact)

### Содержание — Резюме
- [ ] Секция "📋 Резюме" заполнена контентом (scope, решения, отклонённые — не placeholder)

### Содержание — SVC
- [ ] Минимум 1 секция SVC-N присутствует
- [ ] Каждая SVC-N имеет метаданные (Impact → Design решение) и описание ответственности (1–2 абзаца нарратив)
- [ ] Каждая SVC-N содержит 3 подсекции (Ответственность, Компоненты, Зависимости)
- [ ] Подсекция "Ответственность" заполнена контентом (bullet list)
- [ ] Подсекции "Компоненты" и "Зависимости" — контент или заглушка
- [ ] Порядок SVC: Основной → Вторичный → Новый, внутри типа: Подтверждённые → Изменённые → Добавленные
- [ ] Shared-компоненты имеют решение о размещении (scope = shared ({путь}))
- [ ] Отклонённые сервисы Impact упомянуты в Резюме с обоснованием (без секции SVC)
- [ ] Косвенные сервисы Impact не имеют секции SVC (только в Резюме или INT)

### Содержание — INT
- [ ] Минимум 1 блок INT-N (если > 1 сервиса)
- [ ] Каждый INT-N имеет метаданные (участники, паттерн, источник Impact)
- [ ] Каждый INT-N содержит 2 подсекции (Контракт, Sequence) с контентом
- [ ] Контракт описывает **межсервисный** API/событие (не внутрисервисные детали)
- [ ] Sequence содержит mermaid sequenceDiagram
- [ ] Все SVC ссылаются на INT через подсекцию "Зависимости"

### Содержание — STS
- [ ] Секция "🧪 Системные тест-сценарии" присутствует (контент или заглушка)
- [ ] Формат: таблица (ID, Сценарий, Участники, Тип, Источник) — краткое описание без Given/When/Then
- [ ] Колонка "Источник" ссылается на INT-N

### Нумерация
- [ ] Нумерация SVC-N, CMP-N, INT-N, STS-N сквозная по документу
- [ ] Нет дублей нумерации
- [ ] Пропуски допустимы

### Зона ответственности
- [ ] Нет бизнес-обоснований и требований (→ Discussion)
- [ ] Нет предположительных оценок без решения (→ Impact)
- [ ] Нет деталей реализации — алгоритмы, пакеты, классы, библиотеки (→ ADR)
- [ ] Нет тестовых сценариев конкретного сервиса (→ План тестов)
- [ ] Нет задач на реализацию (→ План разработки)

### Маркеры (если статус > DRAFT)
- [ ] Нет `[ТРЕБУЕТ УТОЧНЕНИЯ]`
- [ ] Нет `⛔ DEPENDENCY BARRIER`

### README
- [ ] Запись в `specs/design/README.md`
- [ ] Статус в README = статус в frontmatter

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Неверный формат имени файла | D001 | Имя не соответствует `design-NNNN-topic.md` | Переименовать файл |
| Отсутствует `description` | D002 | Пустой или отсутствующий frontmatter | Добавить description |
| Неверный `standard` | D003 | Путь не указывает на standard-design.md | Исправить путь |
| Невалидный `status` | D004 | Статус не из допустимого списка | Использовать валидный статус |
| Отсутствует `parent` | D005 | Design обязан иметь parent Impact | Добавить поле parent |
| Parent не существует | D006 | Файл Impact не найден по пути из parent | Исправить путь или создать Impact |
| Отсутствует "Резюме" | D007 | Обязательный раздел пропущен | Добавить раздел `## 📋 Резюме` |
| Нет секций SVC-N | D008 | Отсутствуют секции `## SVC-N:` | Добавить минимум 1 секцию |
| Подсекция "Ответственность" отсутствует | D009 | SVC-секция без подсекции Ответственность | Добавить `### 📋 Ответственность` |
| Подсекция "Компоненты" отсутствует | D010 | SVC-секция без подсекции Компоненты | Добавить `### 📦 Компоненты` |
| Подсекция "Зависимости" отсутствует | D011 | SVC-секция без подсекции Зависимости | Добавить `### 🔗 Зависимости` |
| Нет блоков INT-N (при > 1 сервиса) | D012 | Обязательный раздел пропущен | Добавить минимум 1 блок `## INT-N:` |
| Подсекция "Контракт" отсутствует | D013 | INT-блок без подсекции Контракт | Добавить `### Контракт` |
| Подсекция "Sequence" отсутствует | D014 | INT-блок без подсекции Sequence | Добавить `### Sequence` |
| Отсутствует "Системные тест-сценарии" | D015 | Обязательный раздел пропущен | Добавить `## 🧪 Системные тест-сценарии` |
| Дублирование SVC-N | D016 | Два элемента с одинаковым номером | Исправить нумерацию |
| Дублирование CMP-N | D017 | Два элемента с одинаковым номером | Исправить нумерацию |
| Дублирование INT-N | D018 | Два элемента с одинаковым номером | Исправить нумерацию |
| Дублирование STS-N | D019 | Два элемента с одинаковым номером | Исправить нумерацию |
| Маркер при статусе > DRAFT | D020 | Неразрешённый `[ТРЕБУЕТ УТОЧНЕНИЯ]` | Разрешить маркер или вернуть в DRAFT |
| Dependency Barrier при статусе > DRAFT | D021 | Неразрешённый барьер | Разрешить барьер или вернуть в DRAFT |
| Детали реализации | D022 | Алгоритмы, пакеты, классы (→ ADR) | Вынести в ADR |
| Нет записи в README | D023 | Документ не зарегистрирован | Добавить запись в README |
| Рассинхрон статуса | D024 | Статус в README ≠ статус в frontmatter | Синхронизировать |
| NNNN не совпадает | D025 | Номер в имени файла ≠ номер в заголовке | Исправить заголовок или имя файла |
| Отсутствуют метаданные SVC | D026 | Секция SVC без строки `**Impact:** ... \| **Решение Design:**` | Добавить метаданные |
| Нет описания ответственности | D027 | SVC без абзаца после метаданных | Добавить нарратив 1–2 абзаца |
| Ответственность пуста | D028 | Подсекция "Ответственность" без bullet list | Заполнить bullet list |
| Контракт пуст | D029 | Подсекция "Контракт" без спецификации | Заполнить спецификацию API/события |
| Sequence пуст | D030 | Подсекция "Sequence" без mermaid-диаграммы | Добавить sequenceDiagram |
| Резюме пустое/placeholder | D031 | Резюме не заполнено или шаблонный текст | Заполнить контентом |
| Milestone не совпадает | D032 | Milestone Design ≠ milestone parent Discussion | Синхронизировать milestone |
| NNNN ≠ parent NNNN | D033 | Номер Design не равен номеру parent Discussion | Переименовать: disc-0002 → design-0002 |
| Нарушен порядок SVC | D034 | Секции SVC не в порядке Основной → Вторичный → Новый | Переупорядочить секции |
| SVC не ссылается на INT | D035 | Подсекция "Зависимости" не ссылается на INT-N при наличии блоков | Добавить ссылки на INT |
| Косвенный сервис имеет SVC | D036 | Сервис с типом "Косвенный" получил секцию SVC | Убрать секцию, упомянуть в Резюме/INT |
| Отклонённый без обоснования | D037 | Отклонённый сервис в Резюме без причины | Добавить обоснование отклонения |
| Внутрисервисные детали в контракте | D038 | Контракт содержит retry, timeout, headers (→ ADR) | Вынести внутрисервисные детали в ADR |
| Given/When/Then в STS | D039 | Тест-сценарий содержит детальные шаги (→ Plan тестов) | Сократить до краткого описания флоу |
| Метаданные INT отсутствуют | D040 | INT-блок без участников, паттерна или источника Impact | Добавить метаданные |
