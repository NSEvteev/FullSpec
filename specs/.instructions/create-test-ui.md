## Принципы

> **Playwright — только smoke.** Если тест можно написать через httpx — он идёт в `/test` (шаг 5.2). Playwright используется исключительно для проверки UI в браузере.

> **СТОП при падении smoke.** Если сценарий упал — остановиться, чинить UI, не переходить к следующему шагу.

> **MCP постоянно включён.** Playwright MCP не включается/выключается между скиллами. Tool Search автоматически откладывает загрузку инструментов вне `/test-ui`.

> **Номера сценариев — идентификаторы.** При удалении сценария не перенумеровывать оставшиеся.

## Шаги

### Шаг 1: Проверить предусловия

```bash
# Все сервисы healthy?
docker ps --format "table {{.Names}}\t{{.Status}}"
```

Убедиться, что все сервисы из [Текущие сценарии](#текущие-сценарии) доступны.

**При failure:** устранить проблему согласно [create-docker-env.md](/specs/.instructions/create-docker-env.md).

### Шаг 2: Выполнить сценарии

Для каждого сценария из раздела [Текущие сценарии](#текущие-сценарии):

1. Открыть URL через `mcp__playwright__navigate`
2. Выполнить проверки сценария
3. Сделать скриншот через `mcp__playwright__screenshot`
4. Зафиксировать результат: PASS / FAIL

**При FAIL:** СТОП. Зафиксировать симптом, не переходить к следующему сценарию.

### Шаг 3: Отчёт

```
## UI Smoke-тесты — {дата}

| Сценарий | Сервис | Результат | Примечание |
|----------|--------|-----------|------------|
| SMOKE-001 | {svc} | PASS/FAIL | ... |

**Вердикт:** PASS (все сценарии) / FAIL ({список упавших})
```

При PASS — переходить к шагу 5.4 (Ревью ветки).
При FAIL — чинить UI, повторить `/test-ui`.

## Сценарии SMOKE

### Формат сценария

```markdown
### SMOKE-{NNN}: {название}

**URL:** http://localhost:{port}/{path}
**Сервис:** {service-name}
**Проверки:**
1. Страница загрузилась (HTTP 200, заголовок присутствует)
2. Ключевые элементы видны ({список элементов})
3. Скриншот сохранён

**Предусловия:** {что должно быть поднято/настроено}
```

### Правила добавления

1. Определить: это UI-проверка или API-проверка? Если API — идёт в `/test`, не сюда
2. Добавить сценарий `SMOKE-{NNN}` в этот файл по формату выше
3. Указать сервис и порт (берутся из `docker-compose.yml`)
4. Перечислить минимальные проверки (загрузка, ключевые элементы, скриншот)
5. Указать предусловия (какие сервисы должны быть healthy)

### Правила удаления

1. Удалить `SMOKE-{NNN}` из этого файла
2. **НЕ перенумеровывать** оставшиеся (номера — идентификаторы, не порядок)

### Правила изменения

1. Обновить проверки/URL/элементы в существующем `SMOKE-{NNN}`
2. Если сервис сменил порт — обновить URL (источник: `docker-compose.yml`)

## Настройка Playwright MCP

Конфигурация в `.mcp.json` (корень проекта):

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

**Важно:** Playwright MCP постоянно включён — runtime-toggle не поддерживается без перезапуска Claude Code. Tool Search автоматически откладывает загрузку инструментов вне `/test-ui`.

После изменения конфигурации: перезапустить Claude Code, проверить наличие `mcp__playwright__*` через `/mcp`.
