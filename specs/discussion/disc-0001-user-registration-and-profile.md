# disc-0001: Регистрация пользователей и личный кабинет

## 📋 Проблема / Контекст

**Текущее состояние (AS IS):**
Новый проект. Системы регистрации и авторизации нет. Пользователи не могут создать аккаунт и использовать платформу.

**Проблема:**
Платформа ориентирована на разработчиков. Необходима регистрация через OAuth-провайдеры (Google, GitHub) — без email/password. После регистрации пользователю нужен личный кабинет с базовым профилем (имя, email из OAuth, дата регистрации).

**Вне scope:**
- Ролевая модель (RBAC) — отдельная дискуссия
- Двухфакторная аутентификация (2FA) — отдельная дискуссия
- Регистрация через email/password — отдельная дискуссия
- Настройки интерфейса (язык, тема) — отдельная дискуссия
- Загрузка аватара — отдельная дискуссия

## 🎯 Фичи

| ID | Название | Описание |
|----|----------|----------|
| F-1 | OAuth2 регистрация и авторизация | Регистрация и вход через OAuth2-провайдеры: Google и GitHub. При первом входе создаётся аккаунт, при последующих — авторизация |
| F-2 | Профиль пользователя | Личный кабинет с отображением имени, email (из OAuth), даты регистрации. Возможность редактирования отображаемого имени |
| F-3 | Управление сессией | Выход из системы (logout), автоматическое обновление токена (refresh). Сессия сохраняется между визитами |

## 👤 User Stories

| ID | Роль | Действие | Цель |
|----|------|----------|------|
| US-1 | Разработчик | Зарегистрироваться через Google или GitHub | Начать использовать платформу без создания отдельного пароля |
| US-2 | Зарегистрированный пользователь | Просматривать и редактировать свой профиль | Управлять отображаемой информацией о себе |
| US-3 | Зарегистрированный пользователь | Войти через ранее использованный OAuth-провайдер | Быстро получить доступ к платформе без повторной регистрации |
| US-4 | Авторизованный пользователь | Просматривать дату регистрации в профиле | Знать, когда был создан аккаунт |
| US-5 | Авторизованный пользователь | Автоматически продлить сессию при активности | Не прерывать работу повторной авторизацией |

## 📐 Требования

| ID | Связь | Требование |
|----|-------|------------|
| REQ-1 | F-1, US-1 | Незарегистрированный пользователь выбирает "Войти через Google" и проходит OAuth-флоу — создаётся аккаунт, пользователь авторизован и перенаправлен в личный кабинет |
| REQ-2 | F-1, US-1 | Незарегистрированный пользователь выбирает "Войти через GitHub" и проходит OAuth-флоу — создаётся аккаунт, пользователь авторизован и перенаправлен в личный кабинет |
| REQ-3 | F-1, US-3 | Зарегистрированный пользователь входит через ранее использованный OAuth-провайдер — авторизован и перенаправлен в личный кабинет без создания дубликата аккаунта |
| REQ-4 | F-2, US-2 | Авторизованный пользователь на странице профиля изменяет отображаемое имя и сохраняет — имя обновлено, отображается новое значение |
| REQ-5 | F-3 | Авторизованный пользователь нажимает "Выйти" — сессия завершена, перенаправлен на страницу входа |
| REQ-6 | F-1, US-1 | Пользователь отклоняет OAuth-доступ — отображается ошибка "Доступ отклонён", пользователь возвращён на страницу входа |
| REQ-7 | F-1 | OAuth-провайдер недоступен или таймаут — отображается ошибка "Сервис временно недоступен", предлагается использовать альтернативный провайдер |
| REQ-8 | F-1 | OAuth-доступ отозван провайдером при повторном входе — пользователь информирован об отзыве доступа, требуется повторное подтверждение разрешений |
| REQ-9 | F-2 | OAuth-ответ не содержит email — система запрашивает email дополнительно перед завершением регистрации |
| REQ-10 | F-2, US-2 | Валидация отображаемого имени: минимум 2 символа, максимум 64, допустимы буквы, цифры, пробел, дефис, подчёркивание |
| REQ-11 | F-3 | Access token действителен 15 минут, refresh token — 30 дней |
| REQ-12 | F-3, US-5 | Refresh token истёк — пользователь разлогинен, перенаправлен на страницу входа с сообщением "Сессия истекла" |
| REQ-13 | F-1 | OAuth callback валидирует state parameter для защиты от CSRF — запрос без валидного state отклоняется |
| REQ-14 | F-1 | Не более 10 попыток авторизации за 5 минут с одного IP — ответ 429 Too Many Requests |

## ✅ Критерии успеха

- Время регистрации (OAuth-флоу + создание аккаунта) < 3 секунд
- Время авторизации (повторный вход) < 2 секунд
- Время отображения страницы профиля < 500ms
- Все критические события (регистрация, вход, выход, изменение профиля) логируются с timestamp, user_id, IP, user-agent
- 100% покрытие acceptance-тестами критических путей (REQ-1 — REQ-14)
