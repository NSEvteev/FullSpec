## 1. Контуры документации

| Контур | Папка | Назначение | Аудитория |
|--------|-------|-----------|-----------|
| **Аналитика** | `analysis/` | Процесс принятия решений: дискуссии, проектирование, анализ | Пользователь + LLM при планировании |
| **Документация** | `docs/` | Рабочий контекст: описание сервисов, архитектура, конвенции, стандарты технологий | LLM-разработчик при написании кода |

## 2. Подпапки

### [analysis/](./analysis/)

**Аналитическая документация (контур 1).**

Документы процесса принятия решений. Группировка по изменению: `analysis/0001-{slug}/`. Состав TBD — [открытые вопросы](/.claude/drafts/2026-02-19-sdd-structure.md#открытые-вопросы).

### [docs/](./docs/)

**Документация для поставки (контур 2).**

Рабочий контекст LLM-разработчика: per-service документы (`{svc}.md`), системные документы (`.system/`), стандарты технологий (`.technologies/`). Пока пусто — будет создаваться из [драфтов](/.claude/drafts/2026-02-19-sdd-orchestrator.md).

### [.instructions/](./_.instructions/)

**Стандарты и воркфлоу.**

Инструкции для создания, валидации и обновления документов specs/. Будут создаваться из объектных драфтов.

### [_old/](./_old/)

**Архив старой модели.**

Полная копия предыдущей структуры specs/ (6-уровневая модель SDD). Ссылки внутри _old/ могут быть битыми — это архив, не рабочие документы.

---

## 3. Архив

Предыдущая структура specs/ перемещена в `_old/` целиком. Содержит:
- `discussion/`, `impact/`, `design/` — документы старой цепочки
- `architecture/` — system overview, services, domains
- `technologies/` — per-tech стандарты (postgresql, redis)
- `.instructions/` — старые стандарты и скрипты
- `services/`, `tests/`, `glossary/` — прочие артефакты
