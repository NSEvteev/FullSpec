## Назначение

Система управления задачами для работы LLM с проектом через структурированные файлы в `llm_tasks/`.
**Важно:** Все задачи проекта ведутся через `llm_tasks/`, а не через временные файлы.

---
### Формат ID задач

Задачи имеют уникальные ID формата: `CATEGORY-NNNNN`

**Категории:**
- `FEAT-00001` — новая функциональность
- `FIX-00001` — исправление бага
- `REFACTOR-00001` — рефакторинг кода
- `DOCS-00001` — документация
- `TEST-00001` — тесты
- `INFRA-00001` — инфраструктура
- `ID-00001` — общие задачи (без категории)
- `AMY-00001` — задачи для Amy Santiago

**Счётчики:** Каждая категория имеет отдельный счётчик в `.task_counter`

### Команды задач

```bash
# Makefile (рекомендуется)
make task-new                              # Интерактивное создание
make task-new-feat TITLE="..." PRIORITY="high"
make task-complete ID=FEAT-00001           # Завершить (вызывает Amy)
make task-move-current ID=FEAT-00001       # future → current
make tasks-current                         # Показать текущие

# Python скрипты (альтернатива)
python scripts/task_new.py -i
python scripts/task_complete.py FEAT-00001
python scripts/task_move.py FEAT-00001 current
```

### Автоматическое документирование

При завершении задачи (`task_complete.py`) автоматически вызывается **Amy Santiago** со скиллом `task-documentation` для обновления связанной документации:

- Обновляет дискуссии (добавляет ссылку на задачу)
- Обновляет архитектуру (добавляет в историю изменений)
- Обновляет планы реализации (отмечает выполненные подзадачи)
- Обновляет индексы документации (000_*_index.md в general_docs/)
- **Обновляет индекс завершённых задач** `completed/YYYY-MM/{assignee}/0_task_index.md`

**Колонка "Документация обновлена":**
- `Success: [Doc1](путь), [Doc2](путь)` — все связанные документы обновлены
- `Need` — требуется обновление (Amy не смогла автоматически обновить)
- `N/A` — нет связанных документов

**Индексы с группировкой:**

Current/future индексы (`0_task_index.md`) группируются по исполнителям:
- Навигация: [LLM-main](#llm-main) | [Amy Santiago](#amy-santiago) | [Другие](#другие)
- Секции с таблицами задач для каждого исполнителя
- Единая точка входа для всех задач

**Подробнее:** См. [tasks.md](llm_instructions/tasks.md) — полная документация системы управления задачами


## Структура llm_tasks/

```
llm_tasks/
├── .task_counter              # Счётчики ID для каждой категории
│
├── current/                   # Текущие задачи (всех исполнителей)
│   ├── 0_task_index.md        # Индекс, группировка по исполнителям
│   ├── FEAT-00001.md          # Задача LLM (assignee: llm-main)
│   ├── AMY-00002.md           # Задача Amy (assignee: amy-santiago)
│   └── ID-00003.md            # Общая задача
│
├── future/                    # Бэклог (всех исполнителей)
│   ├── 0_task_index.md        # Индекс, группировка по исполнителям
│   ├── FEAT-00100.md
│   └── FIX-00101.md
│
├── completed/                 # Архив выполненных задач
│   └── 2026-01/               # Месяц (YYYY-MM)
│       ├── llm-main/          # Исполнитель: основной LLM
│       │   ├── 0_task_index.md # Индекс завершённых задач
│       │   ├── FEAT-00001.md
│       │   └── ID-00005.md
│       └── amy-santiago/      # Исполнитель: Amy Santiago
│           ├── 0_task_index.md
│           └── AMY-00002.md
│
└── temp/                      # Временные файлы
    └── amy-santiago/          # Временные файлы Amy (отчёты, логи)
```

**Принципы:**
- **Единая папка current/future** — задачи всех исполнителей в одном месте
- **Группировка в индексе** — по исполнителям (LLM-main, Amy Santiago, Другие)
- **Архив по месяцам** — `completed/YYYY-MM/{assignee}/`
- **Индексы** — `0_task_index.md` во всех папках
- **Навигация** — ссылки на разделы по исполнителям в индексах current/future

## Система именования задач

### Формат ID

**Формат:** `[PREFIX]-[5 цифр]`

**Примеры:** `FEAT-00001`, `FIX-00023`, `AMY-00005`

### Префиксы по категориям

| Префикс | Категория | Описание | Пример |
|---------|-----------|----------|--------|
| `FEAT-XXXXX` | Новая функциональность | Добавление нового функционала | `FEAT-00001` |
| `FIX-XXXXX` | Исправление бага | Багфиксы | `FIX-00002` |
| `REFACTOR-XXXXX` | Рефакторинг | Рефакторинг кода | `REFACTOR-00003` |
| `DOCS-XXXXX` | Документация | Работа с документацией | `DOCS-00004` |
| `TEST-XXXXX` | Тесты | Добавление/исправление тестов | `TEST-00005` |
| `INFRA-XXXXX` | Инфраструктура | DevOps, CI/CD, конфигурация | `INFRA-00006` |

### Префиксы без категории

| Префикс | Назначение | Пример |
|---------|------------|--------|
| `ID-XXXXX` | Общие задачи основного LLM | `ID-00001` |
| `AMY-XXXXX` | Задачи агента Amy Santiago | `AMY-00001` |

### Счётчик задач (.task_counter)

**Файл:** `llm_tasks/.task_counter`

```json
{
  "FEAT": 0,
  "FIX": 0,
  "REFACTOR": 0,
  "DOCS": 0,
  "TEST": 0,
  "INFRA": 0,
  "ID": 5,
  "AMY": 0,
  "description": "Счётчики для каждой категории задач. Инкрементируются независимо."
}
```

**Правила:**
- Каждая категория имеет независимый счётчик
- При создании задачи счётчик инкрементируется автоматически (через `task_new.py`)
- Счётчик гарантирует уникальность ID

## Статусы задач

| Статус | Описание | Применение |
|--------|----------|------------|
| `pending` | Ожидает начала | Задача создана, но не начата |
| `in_progress` | В работе | Задача активно выполняется |
| `blocked` | Заблокирована | Есть блокеры (указать в заметках) |
| `completed` | Завершена | Задача выполнена и перемещена в completed/ |
| `cancelled` | Отменена | Задача отменена (указать причину) |

## Приоритеты

| Приоритет | Описание |
|-----------|----------|
| `high` | Критичная задача, требует немедленного внимания |
| `medium` | Важная задача, выполнить в ближайшее время |
| `low` | Некритичная задача, можно отложить |

## Workflow работы с задачами

### 1. Создание новой задачи

**Скрипт:** `scripts/task_new.py`

#### В текущие задачи (current/)

```bash
# Интерактивный режим (рекомендуется)
python scripts/task_new.py -i
make task-new

# Прямое создание в current/
python scripts/task_new.py -t "Добавить тесты" -p high -c test
make task-new-feat TITLE="Новая функция" PRIORITY="high"
```

#### В бэклог (future/)

```bash
# Интерактивный режим
python scripts/task_new.py -i
# Выбрать: 2. future (бэклог)

# Прямое создание в future/
python scripts/task_new.py -t "Рефакторинг" -c refactor -f future
make backlog-new-feat TITLE="Будущая функция" PRIORITY="medium"
make backlog-new-fix TITLE="Багфикс" PRIORITY="low"
make backlog-new-docs TITLE="Обновить доку" PRIORITY="low"
```

**Что происходит:**
1. Скрипт читает `.task_counter`
2. Инкрементирует счётчик для выбранной категории
3. Создаёт файл `{folder}/{PREFIX}-{ID}.md` из шаблона (folder = current или future)
4. Обновляет `.task_counter`

**Далее:**
- Отредактировать файл задачи вручную (заполнить описание, контекст, подзадачи)
- Обновить `{folder}/0_task_index.md` (добавить в таблицу соответствующего исполнителя)

### 2. Работа над задачей

**Во время выполнения:**
- Открыть файл задачи
- Отмечать выполненные подзадачи: `- [ ]` → `- [x]`
- Добавлять заметки в секцию "Заметки"
- Обновлять поле `updated` в frontmatter

**Использование TodoWrite:**
- Создавать TodoWrite на основе подзадач из файла задачи
- Синхронизировать прогресс TodoWrite → файл задачи

### 3. Завершение задачи

**Скрипт:** `scripts/task_complete.py`

```bash
python scripts/task_complete.py FEAT-00001
# или
make task-complete ID=FEAT-00001
```

**Что происходит:**
1. Скрипт определяет `assignee` из frontmatter задачи
2. Скрипт обновляет задачу:
   - Меняет `status: in_progress` → `completed`
   - Добавляет `completed: YYYY-MM-DD`
   - Добавляет секцию "Результат" после заголовка
3. Перемещает файл: `current/FEAT-00001.md` → `completed/YYYY-MM/{assignee}/FEAT-00001.md`
4. **Автоматически вызывает Amy Santiago** (скилл `task-documentation`)

**Amy Santiago обновляет:**
- Связанные документы (дискуссии, архитектура, планы)
- Индексы документации (`000_*_index.md` в general_docs/)
- **Индекс завершённых задач** `completed/YYYY-MM/{assignee}/0_task_index.md` с колонкой "Документация обновлена"
- Добавляет отметку в задачу о завершении документирования

**Далее (вручную):**
- Обновить `current/0_task_index.md` (удалить задачу из секции соответствующего исполнителя)

### 4. Перемещение задачи (current ↔ future)

**Скрипт:** `scripts/task_move.py`

```bash
# Из future в current (взять в работу из бэклога)
python scripts/task_move.py FEAT-00100 current
# или
make task-move-current ID=FEAT-00100

# Из current в future (отложить)
python scripts/task_move.py FEAT-00001 future
# или
make task-move-future ID=FEAT-00001
```

**Далее (вручную):**
- Обновить индексы в обеих папках

## Задачи агентов

### Структура для Amy Santiago

Задачи Amy хранятся в общих папках `current/`, `future/`, `completed/` с полем `assignee: amy-santiago`.

**Временные файлы:**
```
llm_tasks/temp/amy-santiago/
├── task_doc_FEAT-00001_2026-01-15.md
├── audit_report_2026-01-15.md
└── broken_links.json
```

### Правила для агентов

**При запуске агента:**
1. Прочитать `current/0_task_index.md`
2. Показать пользователю текущие задачи
3. Предложить варианты (продолжить, новые, бэклог)

**При начале новой задачи:**
1. Очистить папку `temp/` (удалить старые временные файлы)
2. Обновить статус задачи: `pending` → `in_progress`

**Во время работы:**
1. Обновлять статусы подзадач в файле задачи
2. Сохранять временные файлы в `temp/`
3. Использовать имена с датой: `report_2026-01-15.md`

**При завершении задачи:**
1. Использовать `task_complete.py AMY-00001`
2. Задача переместится в `completed/YYYY/MM-month/`
3. Временные файлы остаются в `temp/` (для истории)

## Интеграция с документацией

### Связь задач с дискуссиями

Когда задача требует архитектурных решений:
1. Создать дискуссию в `general_docs/01_discuss/`
2. Указать ссылку в задаче (секция "Связанные документы")
3. После утверждения дискуссии — обновить задачу

**Пример в задаче:**
```markdown
## Связанные документы

- **Дискуссия:** [005_auth_system.md](../../general_docs/01_discuss/005_auth_system.md)
```

### Связь задач с планами реализации

Когда начинается реализация:
1. Создать план в `general_docs/06_imp_plans/`
2. Указать ссылку в задаче
3. Отслеживать прогресс в плане

**Пример:**
```markdown
## Связанные документы

- **План реализации:** [003_plan_auth_service.md](../../general_docs/06_imp_plans/003_plan_auth_service.md)
```

### Автоматическое обновление документации

При завершении задачи Amy Santiago (скилл `task-documentation`):
1. Читает секцию "Связанные документы"
2. Обновляет дискуссии → добавляет ссылку на задачу в раздел "Связанные задачи"
3. Обновляет архитектуру → добавляет запись в "История изменений"
4. Обновляет планы → отмечает выполненные подзадачи
5. Обновляет индексы документации
6. Отмечает в задаче: "Документация обновлена ✓"

## Команды Makefile

```bash
# Создание задач
make task-new                # Интерактивное создание
make task-new-feat TITLE="..." PRIORITY="high"
make task-new-fix TITLE="..." PRIORITY="medium"
make task-new-docs TITLE="..." PRIORITY="low"

# Управление задачами
make task-complete ID=FEAT-00001
make task-move-current ID=FEAT-00100  # future → current
make task-move-future ID=FEAT-00001   # current → future

# Просмотр
make tasks-current      # Показать текущие задачи
make tasks-future       # Показать бэклог
make tasks-completed    # Показать выполненные
```

## Шаблон задачи

**Файл:** `templates/task.md`

Используется скриптом `task_new.py` для создания новых задач.

Плейсхолдеры:
- `{{ID}}` → ID задачи (FEAT-00001)
- `{{TITLE}}` → Название
- `{{STATUS}}` → pending
- `{{PRIORITY}}` → high/medium/low
- `{{CATEGORY}}` → feat/fix/etc
- `{{CREATED_DATE}}` → YYYY-MM-DD
- `{{UPDATED_DATE}}` → YYYY-MM-DD
- `{{ASSIGNEE}}` → llm-main/amy-santiago
- `{{DESCRIPTION}}` → Описание
- `{{CONTEXT}}` → Контекст
