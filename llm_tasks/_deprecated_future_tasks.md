## Назначение

Файл для хранения идей и задач, которые планируется взять в работу в будущих [📖 LLM-сессиях](../general_docs/glossary.md#llm-сессия).

## Приоритеты

| Приоритет | Описание |
|-----------|----------|
| `P0` | Критично — блокирует работу |
| `P1` | Высокий — важно для проекта |
| `P2` | Средний — улучшение |
| `P3` | Низкий — nice to have |

## Категории

| Категория | Описание |
|-----------|----------|
| `arch` | Архитектура |
| `feat` | Новая функциональность |
| `fix` | Исправление |
| `docs` | Документация |
| `infra` | Инфраструктура |

## Формат записи

```markdown
- [приоритет] [категория] **Название** — Описание
  - Контекст: почему это нужно
  - Ссылка: [документ](путь/к/документу.md)
```

## Правила работы

1. **Добавление:** Записывать идеи сразу при возникновении
2. **Приоритизация:** Указывать приоритет и категорию
3. **Перенос в работу:** При старте [📖 LLM-сессии](../general_docs/glossary.md#llm-сессия) переносить задачи в [current_tasks.md](current_tasks.md)
4. **Очистка:** Периодически ревьюить и удалять неактуальные

   - Решение: Массовая замена через sed/find (~10 минут)

   - Причина: В `llm_instructions/general_docs.md` отсутствуют разделы для типов документации

     - `#дискуссии-discuss`

     - `#диаграммы-diagrams`
     - `#ресурсы-resources`
     - `#план-реализации-imp_plans`
   - Затронуты: general_docs/*/000_*.md, README.md, templates/*.md
   - Решение: Добавить недостающие разделы (~30 минут)

3. **Форматирование Markdown (1025 проблем) - КОСМЕТИКА:**
   - Проблема: Отсутствие пробела после маркера списка
   - Неправильно: `-Элемент`
   - Правильно: `- Элемент`
   - Затронуты: почти все .md файлы в templates/
   - Решение: Автоматическая замена (~5 минут)
