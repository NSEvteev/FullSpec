### [ID-005] Улучшить check_doc_health.py для программного использования

**Статус:** pending
**Приоритет:** средний
**Создано:** 2026-01-15

**Контекст:**
Amy Santiago создала временный wrapper `run_check.py` для программного запуска `check_doc_health.py` с сохранением результатов в JSON. Wrapper использует хакерский подход с `exec()` и заменой строк в коде, что делает решение хрупким.

**Проблема:**
Текущий `check_doc_health.py`:
- Только CLI интерфейс (выводит результаты в консоль)
- Нельзя импортировать как модуль
- Нет возможности получить результаты в JSON формате
- Нет опции сохранения результатов в файл

**Что нужно сделать:**
1. Рефакторинг структуры:
   - Разделить CLI логику и core функциональность
   - Сделать возможность импорта как модуля
   - Добавить функцию `run_checks()` для программного использования

2. Добавить флаги:
   - `--json` - вывод результатов в JSON формате
   - `--output FILE` - сохранение результатов в файл
   - `--format {text|json}` - формат вывода

3. Обратная совместимость:
   - CLI интерфейс должен остаться прежним
   - Существующие флаги и команды работают как раньше

**Требование проверки:**
Добавить требование использовать `check_doc_health.py` в автоматических проверках документации (скиллы, CI/CD).

**Пример использования после улучшения:**

Bash:
```bash
# CLI (существующий)
python scripts/check_doc_health.py --verbose

# Новый JSON вывод
python scripts/check_doc_health.py --json --output results.json

# Вывод в файл (текст)
python scripts/check_doc_health.py --output report.txt
```

Python:
```python
from scripts.check_doc_health import run_checks

results = run_checks(verbose=False, format='json')
# returns: {
#   'success': bool,
#   'total_issues': int,
#   'issues_by_category': {...}
# }
```

**Временный wrapper:**
- Файл: `llm_tasks/agents/amy-santiago/temp/run_check.py`
- После улучшения можно удалить

**Связанные файлы:**

- `scripts/check_doc_links.py` - импортируемые функции
