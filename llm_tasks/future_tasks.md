# Будущие задачи

## Назначение

Файл для хранения идей и задач, которые планируется взять в работу в будущих сессиях.

**Связанный файл:** [current_tasks.md](current_tasks.md) — текущие задачи сессии

## Приоритеты

| Приоритет | Описание |
|-----------|----------|
| `P0` | Критично — блокирует работу |
| `P1` | Высокий — важно для проекта |
| `P2` | Средний — улучшение |
| `P3` | Низкий — nice to have |

## Категории

| Категория | Описание |
|-----------|----------|
| `arch` | Архитектура |
| `feat` | Новая функциональность |
| `fix` | Исправление |
| `docs` | Документация |
| `infra` | Инфраструктура |

## Формат записи

```markdown
- [приоритет] [категория] **Название** — Описание
  - Контекст: почему это нужно
  - Ссылка: [документ](путь/к/документу.md)
```

## Правила работы

1. **Добавление:** Записывать идеи сразу при возникновении
2. **Приоритизация:** Указывать приоритет и категорию
3. **Перенос в работу:** При старте сессии переносить задачи в [current_tasks.md](current_tasks.md)
4. **Очистка:** Периодически ревьюить и удалять неактуальные
