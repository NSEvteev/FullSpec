## Запланированные задачи

### Приоритет 1: Высокий

**Задача: Повторный аудит документации с использованием скиллов**

**Приоритет:** high
**Описание:** Провести полный аудит документации проекта, используя все доступные скиллы для автоматизации.

**Цель:** Проверить, что новая система автоматизации через скиллы работает корректно и даёт полный охват всех файлов проекта.

**Последовательность выполнения:**
1. **Очистить** `temp/` перед началом
2. **Запустить скиллы в последовательности:**
   - `/doc-health` — комплексная проверка документации
   - `/doc-claude` — проверка актуальности CLAUDE.md и llm_instructions.md
   - `/glossary-candidates` — поиск новых терминов во всех .md файлах
   - `/glossary-link` — автоматическое добавление ссылок на глоссарий
3. **Собрать результаты** всех скиллов в `temp/`
4. **Сформировать сводный отчёт** на основе результатов скиллов
5. **Сравнить** с первым аудитом (ручным) — проверить полноту покрытия
6. **Обновить** `current_tasks.md` и `completed_tasks.md`

**Ожидаемый результат:**
- Отчёты от всех 4 скиллов в `temp/`
- Сводный отчёт с автоматически собранными данными
- Подтверждение, что скиллы охватывают все файлы проекта
- Список найденных проблем (если есть)
- Рекомендации по улучшению

**Критерий успеха:**
- Все скиллы запущены и выполнены
- Результаты сохранены в `temp/`
- Сводный отчёт сформирован
- Не осталось незакрытых вопросов
