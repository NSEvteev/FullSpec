# Воркфлоу рефакторинга скиллов (SSOT)

**Цель:** Вынести общие концепции из скиллов в инструкции, устранить дублирование.

**Трекер прогресса:** [refactoting-claude-and-instructions.md](./refactoting-claude-and-instructions.md)

> Этот документ описывает переиспользуемый воркфлоу. Пример применения: рефакторинг links-* (2025-01-22).

---

## Правила рефакторинга (ОБЯЗАТЕЛЬНО)

> ⚠️ **КРИТИЧНО:** Эти правила ОБЯЗАТЕЛЬНЫ для всех рефакторингов.

### 0. Подтверждение после каждой фазы

**БЛОКИРУЮЩЕЕ ПРАВИЛО:** После завершения КАЖДОЙ фазы (PHASE 1-5):

1. **Подвести итог фазы** — что сделано, какие файлы затронуты, ключевые решения
2. **Показать пользователю** — вывести сводку результатов
3. **Запросить подтверждение** — "Продолжить к фазе N?" [Y/n]
4. **ЖДАТЬ ОТВЕТА** — НЕ переходить к следующей фазе без явного подтверждения

**Нарушение этого правила = критическая ошибка.**

### 1. Создание инструкций

При создании/изменении инструкций **ОБЯЗАТЕЛЬНО**:
1. Прочитать [/.claude/instructions/instructions/](/.claude/instructions/instructions/) — SSOT для инструкций
2. Использовать [/instruction-create](/.claude/skills/instruction-create/SKILL.md) — скилл создания
3. Использовать шаблон [/.claude/templates/instructions/instruction.md](/.claude/templates/instructions/instruction.md)

### 2. Создание скиллов

При создании/изменении скиллов **ОБЯЗАТЕЛЬНО**:
1. Прочитать [/.claude/instructions/skills/](/.claude/instructions/skills/) — SSOT для скиллов
2. Использовать [/skill-create](/.claude/skills/skill-create/SKILL.md) — скилл создания
3. Использовать шаблон [/.claude/templates/skills/skill.md](/.claude/templates/skills/skill.md)

### 3. Структура скиллов после рефакторинга

**Запрещено:**
- Дублировать информацию из SSOT-инструкций в скиллах
- Добавлять детальные описания шагов (детали — в SSOT)
- Добавлять алгоритмы и примеры в шаги (всё — в SSOT)

**Обязательно:**
- ШАГ 0 с блокирующим чтением SSOT
- Каждый шаг содержит ТОЛЬКО название и ссылку на SSOT
- Ссылка формата: `> **SSOT:** [file.md](path#якорь)`

### 4. Шаблон шага в скилле

```markdown
### Шаг N: {Название}

> **SSOT:** [{file}.md](/.claude/instructions/{область}/{file}.md#{якорь})
```

**БЕЗ дополнительных комментариев, алгоритмов, примеров.**

### 5. README.md папки инструкций

**ОБЯЗАТЕЛЬНО** создавать по шаблону [/.claude/templates/instructions/readme.md](/.claude/templates/instructions/readme.md):
- Заголовок: `# Инструкции /{folder}/`
- Нумерованные секции (`# N. {Тема}`)
- Таблица Оглавление (Секция | Инструкция | Описание)
- Дерево файлов

### 6. Переформатирование существующего README

**ЕСЛИ README.md в папке переработки уже существует (до рефакторинга):**

1. **ОБЯЗАТЕЛЬНО переформатировать** полностью по шаблону [/.claude/templates/instructions/readme.md](/.claude/templates/instructions/readme.md)
2. **Перенести уникальную информацию** из README в соответствующие инструкции:
   - Таблицы → в тематическую инструкцию
   - Примеры кода → в examples.md
   - Диаграммы/схемы → в тематическую инструкцию
   - Принятые решения → в rules.md или отдельный decisions.md
3. **README должен содержать ТОЛЬКО:**
   - Краткое описание секции (1-2 предложения)
   - **Оглавление:** со ссылками на якоря инструкции
   - **Инструкция:** ссылка на файл
4. **Секции без отдельного файла** (например, "Структура папки") — оставить в README

**Алгоритм:**
```
README содержит информацию X?
    │
    ├── X — структура папки → оставить в README
    │
    ├── X — уникальная инфа по теме Y → перенести в Y.md
    │   └── если Y.md не существует → создать
    │
    └── X — общая инфа (правила, решения) → перенести в rules.md
```

**Пример переноса (specs/README.md):**

| Было в README | Перенесено в |
|---------------|--------------|
| Секция "Принятые решения" (таблица) | rules.md#принятые-решения |
| Секция "Связи между документами" | relations.md (новый файл) |
| Секция "Связь /specs/ ↔ /doc/" | relations.md#связь-specs--doc |

## Воркфлоу рефакторинга (для повторного использования)

### PHASE 1: Сбор информации

**Цель:** Понять, какие концепции дублируются в скиллах.

**Шаги:**

1. **Определить скиллы для рефакторинга**
   ```
   Glob: .claude/skills/{prefix}-*/SKILL.md
   ```
   Для links-*: links-create, links-update, links-delete, links-validate

2. **Прочитать все скиллы**
   - Записать количество строк каждого
   - Выделить общие концепции
   - Найти дублирование

3. **Составить таблицу источников**
   ```markdown
   | Скилл | Строк | Назначение |
   |-------|-------|------------|
   | links-create | 359 | Создание ссылок |
   | links-update | 734 | Обновление ссылок |
   | links-delete | 611 | Пометка битых |
   | links-validate | 242 | Валидация |
   ```

4. **Выделить общие концепции**
   - Форматы (что повторяется в нескольких скиллах)
   - Паттерны (regex, алгоритмы поиска)
   - Правила (исключения, edge cases)
   - Жизненный цикл (фазы работы)

---

### PHASE 2: Проектирование структуры инструкций

**Цель:** Определить, какие инструкции создать.

**Принципы:**
- Одна инструкция = одна концепция
- Не дробить слишком мелко
- Связывать через `related` в frontmatter

**Шаги:**

1. **Определить список инструкций**
   ```
   /.claude/instructions/{область}/
   ├── README.md           # Индекс (обязательно)
   ├── {concept1}.md       # Концепция 1
   ├── {concept2}.md       # Концепция 2
   └── ...
   ```

2. **Для links-* получилось:**
   ```
   /.claude/instructions/links/
   ├── README.md           # Индекс
   ├── format.md           # Форматы ссылок
   ├── patterns.md         # Паттерны поиска
   ├── workflow.md         # Жизненный цикл
   ├── validation.md       # Правила валидации
   └── edge-cases.md       # Граничные случаи
   ```

3. **Составить таблицу соответствия**
   ```markdown
   | Концепция в скиллах | Целевая инструкция |
   |---------------------|-------------------|
   | Формат [text](path) | format.md |
   | Формат marked links | format.md |
   | Regex поиска | patterns.md |
   | Исключения (код, bash) | patterns.md |
   | CREATE/UPDATE/DELETE | workflow.md |
   | Типы ссылок | validation.md |
   | Edge cases | edge-cases.md |
   ```

---

### PHASE 3: Создание инструкций

**Цель:** Создать SSOT-инструкции.

**Шаги:**

1. **Создать папку**
   ```bash
   mkdir -p .claude/instructions/{область}/
   ```

2. **Использовать шаблоны**
   - [instruction.md](/.claude/templates/instructions/instruction.md)
   - [readme.md](/.claude/templates/instructions/readme.md)

3. **Для каждой инструкции:**
   - Frontmatter: type, description, governed-by, related
   - Навигационные ссылки: Индекс | Папка
   - Оглавление
   - Содержимое (из скиллов)
   - Секция "Скиллы" — какие скиллы используют
   - Секция "Связанные инструкции"

   **ВАЖНО: governed-by**
   ```yaml
   governed-by: {folder}/README.md  # НЕ instructions/README.md!
   ```
   Каждая инструкция должна указывать на README своей папки, а не главный README.

4. **README.md папки (по шаблону [readme.md](/.claude/templates/instructions/readme.md)):**
   - Заголовок: `# Инструкции /{folder}/`
   - Краткое описание и **Содержание:**
   - Оглавление (таблица с колонками: Секция | Инструкция | Описание)
   - Дерево файлов (в блоке кода)
   - Нумерованные секции (`# N. {Тема}`) с:
     - Краткое описание
     - **Оглавление:** (список якорей)
     - **Инструкция:** (ссылка)
   - Шаблоны (# N. Шаблоны) или "отсутствуют"
   - Скиллы (# N. Скиллы)
   - Скрипты (# N. Скрипты) или "отсутствуют"
   - Связанные инструкции

5. **Обновить главный README.md**
   - Добавить в оглавление
   - Добавить в дерево
   - Добавить таблицу с инструкциями
   - Обновить счётчик

### PHASE 4: Рефакторинг скиллов

**Цель:** Заменить дублирование на ссылки к SSOT.

**КРИТИЧЕСКОЕ ПРАВИЛО: Блокирующее чтение SSOT**

> ⚠️ Скилл ДОЛЖЕН содержать ШАГ 0 с ОБЯЗАТЕЛЬНЫМ чтением SSOT-инструкций.
> Без этого шага LLM будет игнорировать SSOT и идти по кратким описаниям.

**Структура обновлённого скилла (по шаблону [skill.md](/.claude/templates/skills/skill.md)):**

```markdown
---
name: {skill-name}
description: {описание}
...
---

# {Название}

{Краткое описание}

**Связанные скиллы:**
- [skill-1](path) — описание

## Оглавление

- [Формат вызова](#формат-вызова)
- [Воркфлоу](#воркфлоу)
- [Чек-лист](#чек-лист)
- [Примеры](#примеры)

- [ ] {Шаг 1}
- [ ] {Шаг N-1}

> **SSOT:** [examples.md](/.claude/instructions/{область}/examples.md#{skill-name})
```

**Что ВЫНОСИТЬ в инструкции:**
- Форматы данных
- Regex-паттерны
- Правила и исключения
- Edge cases
- Таблицы типов/вариантов
- Детальные описания шагов

**Что ОСТАВЛЯТЬ в скилле:**
- Frontmatter
- Формат вызова и параметры
- ШАГ 0 с блокирующим чтением SSOT
- Названия шагов (БЕЗ детальных описаний)
- Чек-лист (первый пункт: "Прочитал SSOT")

**Что УДАЛЯТЬ из скилла:**
- Детальные описания шагов (теперь в SSOT)
- Примеры (достаточно в SSOT)
- FAQ/Troubleshooting (переносить в SSOT)
- Edge cases (переносить в SSOT)

### PHASE 4.1: Валидация скиллов (ОБЯЗАТЕЛЬНО)

**Цель:** Проверить, что все изменённые скиллы соответствуют шаблону и правилам.

> ⚠️ **КРИТИЧНО:** Этот шаг ОБЯЗАТЕЛЕН после рефакторинга скиллов. Без него нельзя переходить к PHASE 5.

**Шаги:**

1. **Прочитать SSOT для валидации:**
   - [/.claude/templates/skills/skill.md](/.claude/templates/skills/skill.md) — шаблон скилла
   - [/.claude/skills/skill-create/SKILL.md](/.claude/skills/skill-create/SKILL.md) — скилл создания (требования)
   - [/.claude/instructions/skills/validation.md](/.claude/instructions/skills/validation.md) — правила валидации

2. **Для каждого изменённого скилла проверить:**

   **Frontmatter (обязательно):**
   - [ ] `name` — имя скилла
   - [ ] `description` — описание (1 строка)
   - [ ] `allowed-tools` — список инструментов
   - [ ] `category` — категория из rules.md
   - [ ] `triggers.commands` — команды для вызова
   - [ ] `triggers.phrases` — фразы (ru/en)

   **Структура (обязательно):**
   - [ ] Заголовок `# {Название}`
   - [ ] Связанные скиллы (после описания)
   - [ ] `## Оглавление` с 4 ссылками: `#формат-вызова`, `#воркфлоу`, `#чек-лист`, `#примеры`
   - [ ] `## Формат вызова`
   - [ ] `## Воркфлоу` с ШАГ 0
   - [ ] `## Чек-лист`
   - [ ] `## Примеры` со ссылкой на SSOT

   **Воркфлоу (обязательно):**
   - [ ] ШАГ 0 с блокирующим чтением SSOT
   - [ ] Каждый шаг имеет `> **SSOT:**` ссылку
   - [ ] `### Шаг N-1: Проверка по чек-листу` со ссылкой `См. [Чек-лист](#чек-лист) ниже.`
   - [ ] `### Шаг N: Результат`

3. **При нарушениях — исправить** немедленно.

4. **Формат отчёта валидации:**

   ```
   📋 Валидация скиллов после рефакторинга

   Проверено: 6 скиллов

   ✅ spec-create: OK
   ✅ spec-status: OK
   ⚠️ spec-update: Исправлено (добавлен #примеры в Оглавление)
   ✅ specs-health: OK
   ✅ specs-sync: OK
   ✅ specs-index: OK

   Итого: 6/6 соответствуют шаблону
   ```

### PHASE 5: Верификация

**Цель:** Убедиться, что всё работает.

**Шаги:**

1. **Проверить созданные файлы**
   ```bash
   ls -la .claude/instructions/{область}/
   ```

2. **Проверить размеры скиллов**
   ```bash
   wc -l .claude/skills/{prefix}-*/SKILL.md
   ```

3. **Ожидаемое сокращение: 50-70%**

4. **Проверить ссылки**
   ```
   /links-validate .claude/instructions/{область}/
   /links-validate .claude/skills/{prefix}-*/
   ```

5. **Проверить соответствие правилам инструкций**

   Сверить с [/.claude/instructions/instructions/validation.md](/.claude/instructions/instructions/validation.md):
   - [ ] Frontmatter (type, description, governed-by, related)
   - [ ] Заголовок (# Название)
   - [ ] Навигационные ссылки
   - [ ] Оглавление (## Оглавление)
   - [ ] Скиллы (## Скиллы)
   - [ ] Связанные инструкции
   - [ ] governed-by указывает на README папки (НЕ instructions/README.md)

6. **Проверить README.md папки по шаблону**

   Сверить с [/.claude/templates/instructions/readme.md](/.claude/templates/instructions/readme.md):
   - [ ] Заголовок: `# Инструкции /{folder}/`
   - [ ] Краткое описание и **Содержание:**
   - [ ] Оглавление (таблица: Секция | Инструкция | Описание)
   - [ ] Дерево файлов (в блоке кода)
   - [ ] Нумерованные секции (`# N. {Тема}`)
   - [ ] Каждая секция имеет **Оглавление:** и **Инструкция:**
   - [ ] Секции: Шаблоны, Скиллы, Скрипты
   - [ ] Связанные инструкции

7. **Проверить скиллы по шаблону**

   Сверить с [/.claude/templates/skills/skill.md](/.claude/templates/skills/skill.md):
   - [ ] `## Оглавление` с ссылками на секции (включая `#чек-лист`)
   - [ ] `## Воркфлоу` с ШАГ 0 (блокирующее чтение SSOT)
   - [ ] Каждый шаг имеет `> **SSOT:**` ссылку (два формата):
     - Шаг с инструкцией: `> **SSOT:** [file.md](/.claude/instructions/.../file.md#якорь)`
     - Шаг с вызовом скилла (`→ /{skill}`): `> **SSOT:** [SKILL.md](/.claude/skills/{skill}/SKILL.md)`
   - [ ] `### Шаг N-1: Проверка по чек-листу` со ссылкой `См. [Чек-лист](#чек-лист) ниже.`
   - [ ] `## Чек-лист` — внутренняя секция (НЕ ссылка на внешний SSOT)
   - [ ] `## Примеры` со ссылкой на SSOT: `> **SSOT:** [examples.md](...)`
   - [ ] Нет секции `## Обработка ошибок` (удалена, заменена на `## Чек-лист`)

8. **Обновить трекер рефакторинга**

   Внести итоги в [refactoting-claude-and-instructions.md](./refactoting-claude-and-instructions.md):
   - Обновить статусы в таблицах (instructions/, skills/)
   - Добавить сводку в раздел "После каждой переработки"
   - Добавить запись в "Историю изменений"

   **Формат сводки:** см. существующие сводки в файле (instructions/, links/).
