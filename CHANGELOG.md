## Правила ведения Changelog

1. **Формат дат:** ГГГГ-ММ-ДД (ISO 8601)
2. **Группировка:** По типам изменений (Added, Changed, Fixed, etc.)
3. **Язык:** Русский, кратко и ясно
4. **Ссылки:** Использовать ссылки на issues/PR при необходимости
5. **Версионирование:**
   - MAJOR: breaking changes (несовместимые изменения API)
   - MINOR: новая функциональность (обратно совместимая)
   - PATCH: исправления багов (обратно совместимые)
